## 题目名称：毛线小精灵的羁绊

### 题目背景

在《毛线小精灵2》的世界里，红蓝两只小精灵“雅尼”（Yarny）被一根长度有限的魔法毛线连接在一起。为了跨越危险的悬崖，他们必须像登山者一样，沿着不断向前延伸的锚点相互配合前进。如果两人之间的距离超过了毛线的极限，羁绊就会断裂，游戏结束。

### 题目描述

红色小精灵前方有 `n` 个锚点，位置序列为数组 `a`；蓝色小精灵前方有 `m` 个锚点，位置序列为数组 `b`。**保证这两个数组都是严格升序的**，即锚点一直向前方延伸。连接两人的魔法毛线最大拉伸长度为 `k`。

初始时，两个小精灵各自位于自己的第 1 个锚点（即红色位于 `a[0]`，蓝色位于 `b[0]`）。游戏机制保证了起始锚点之间的距离绝对值不超过 `k`。

为了安全前进，他们的移动策略如下：

- **一方行动时，另一方必须在原地（某个锚点上）等待**，以固定毛线。
- 行动的一方可以沿着自己的锚点不断向前跨越，但前提是：**目标锚点与等待方所在的锚点距离的绝对值不能超过 `k`**。

双方可以随时切换行动权。如果在某个时刻，双方都无法再向前迈出哪怕一个锚点，说明他们被彻底困住了。

请你编写程序判断，这两只小精灵能否凭借这种移动方式，最终双双到达各自的最后一个锚点（即第 `n` 个和第 `m` 个锚点）。

如果能，请输出 `WIIIIIN?`；如果中途被困住，请输出 `LOOOOSE!`。

### 输入格式

第一行包含三个正整数 `n`、`m`、`k`，分别代表红蓝小精灵的锚点数量，以及毛线的最大拉伸长度。

第二行包含 `n` 个严格升序的整数，表示红色小精灵的锚点位置数组 `a`。

第三行包含 `m` 个严格升序的整数，表示蓝色小精灵的锚点位置数组 `b`。

### 输出格式

输出一行字符串。如果能成功到达，输出 `WIIIIIN?`；否则输出 `LOOOOSE!`。

### 输入输出样例

**输入样例 1：**

Plaintext

```
3 3 3
1 5 10
3 8 12
```

**输出样例 1：**

Plaintext

```
WIIIIIN?
```

> **样例 1 解释：**
>
> - 初始状态：红在锚点 1，蓝在锚点 3。两人距离为 2，不超过毛线长度 3。
>
> - 蓝等待，红行动：红尝试前往锚点 5，距离蓝（锚点 3）为 2，成功。红尝试前往锚点 10，距离蓝超过 3，失败。红停留在锚点 5。
>
> - 红等待，蓝行动：蓝尝试前往锚点 8，距离红（锚点 5）为 3，成功。蓝尝试前往锚点 12，距离红超过 3，失败。蓝停留在锚点 8。
>
> - 蓝等待，红行动：红前往锚点 10，距离蓝（锚点 8）为 2，成功。红到达终点！
>
> - 红等待，蓝行动：蓝前往锚点 12，距离红（锚点 10）为 2，成功。蓝到达终点！
>
>   双方均成功通关。

**输入样例 2：**

Plaintext

```
2 2 5
1 20
3 4
```

**输出样例 2：**

Plaintext

```
LOOOOSE!
```

> **样例 2 解释：**
>
> 初始红在锚点 1，蓝在锚点 3。
>
> 蓝可以轻松走到自己的第 2 个锚点 4（距离红的锚点 1 为 3，不超过 5）。此时蓝到达终点。
>
> 轮到红行动时，红需要跨越到锚点 20，但距离蓝所在的锚点 4 高达 16，远超毛线长度 5。红无法前进，两人彻底卡死。

### 数据范围

- `1 <= n, m <= 10^5`
- `1 <= k <= 10^9`
- `1 <= a[i], b[i] <= 10^9`
- 保证序列 `a` 和 `b` 严格单调递增。
- 保证 `|a[0] - b[0]| <= k`。
