## 题目名称：Sans的最终审判

### 题目背景

“真糟糕啊，你感觉你要度过一个不愉快的经历了。”

在审判大厅的尽头，你直面了骷髅 Sans。作为最后的防线，Sans 已经不再对你抱有任何仁慈。他召唤出了无数的“龙骨炮”（Gaster Blaster），准备用密集的激光弹幕将你的“决心”彻底粉碎。

为了在弹幕中存活下来，你需要快速计算出审判大厅中绝对安全的区域。

### 题目描述

审判大厅可以看作一个拥有 $n$ 行和 $m$ 列的巨大网格，网格的行从 $1$ 到 $n$ 编号，列从 $1$ 到 $m$ 编号。

在 Sans 的这一回合攻击中，他一共召唤了 $k$ 发龙骨炮。每发龙骨炮的攻击模式如下：

- **行攻击**：射出一道横向的激光，瞬间摧毁第 $x$ 行的所有网格。
- **列攻击**：射出一道纵向的激光，瞬间摧毁第 $x$ 列的所有网格。

有些网格可能会被多发激光重复击中（Sans 并不介意对同一个地方多开几炮）。在所有龙骨炮发射完毕后，请你计算出整个审判大厅中**未被任何激光击中**的安全网格数量。

### 输入格式

第一行包含三个正整数 $n$、$m$、$k$，分别表示大厅的行数、列数以及龙骨炮的攻击次数。

接下来 $k$ 行，每行包含两个整数 $op$ 和 $x$，表示一次龙骨炮的攻击指令：

- 当 $op = 0$ 时，表示一发**行攻击**，激光覆盖了第 $x$ 行。
- 当 $op = 1$ 时，表示一发**列攻击**，激光覆盖了第 $x$ 列。

### 输出格式

输出一个整数，表示未被任何攻击覆盖的绝对安全格子数。

------

### 输入输出样例

**输入样例 1：**

Plaintext

```
3 3 2
0 2
1 2
```

**输出样例 1：**

Plaintext

```
4
```

> **样例 1 解释：**
>
> 审判大厅是一个 $3 \times 3$ 的网格。
>
> 第一发龙骨炮摧毁了第 2 行，第二发摧毁了第 2 列。
>
> 剩下的安全格子位于左上、左下、右上、右下四个角，共计 4 个。

**输入样例 2：**

Plaintext

```
4 5 3
0 2
0 2
1 1
```

**输出样例 2：**

Plaintext

```
12
```

> **样例 2 解释：**
>
> 行数 $n=4$，列数 $m=5$，共 20 个格子。
>
> Sans 对第 2 行进行了两次重复攻击，对第 1 列进行了一次攻击。
>
> 实际被占据的是 1 行和 1 列。根据公式，安全格子数为 $20 - (1 \times 5 + 1 \times 4 - 1 \times 1) = 12$。

------

### 数据范围

- $1 \le n, m \le 10^9$
- $1 \le k \le 10^5$
- $op \in \{0, 1\}$
- 当 $op = 0$ 时，$1 \le x \le n$；当 $op = 1$ 时，$1 \le x \le m$
