> | —— 并集，往大了扩充置 1
>
> & —— 交集、往小了缩小置 0
>
> ^ —— 并集 - 交集，不进位加法，异位和





## 数位问题

数位问题（Digit DP / Bitwise Algorithms）中，最让人头疼的确实是**“0-based索引”**（从0开始计数）与我们日常**“1-based思维”**（从1开始计数）之间的打架。

为了彻底解决这个问题，我建议你不再死记硬背，而是建立一个**“分界线”**的几何直觉。

以下是为你整理的**二进制数位核心记忆手册**：

------

### 一、 核心概念：三个“不同的 k”

假设我们要处理 **$k$ 位** 二进制数（例如 $k=8$），你需要区分以下三个概念：

#### 1. 容量 (Capacity)：$2^k$

- **含义**：$k$ 个坑位能填满多少种情况。
- **记忆**：包括 `0` 在内。
- **例子**：8位二进制，一共能表示 $2^8 = 256$ 个数。

#### 2. 上界 (Max Value)：$2^k - 1$

- **含义**：因为从 0 开始数，所以最大值要减 1。
- **二进制形态**：全都是 1。
- **例子**：8位最大值是 $255$。二进制是 `1111 1111`。
- **口诀**：**$2^k$ 是最小的 $k+1$ 位数**（1后面跟 $k$ 个 0），所以 $2^k - 1$ 是最大的 $k$ 位数。

#### 3. 最高位下标 (MSB Index)：$k - 1$

- **含义**：计算机里数位是从第 0 位开始数的（右边是第 0 位）。
- **权重**：第 $i$ 位的权重是 $2^i$。
- **例子**：8位二进制，最高位（最左边）的下标是 $7$。它的权重是 $2^7 = 128$。

> 注意最高位下标与容量和上界的幂次差异

------

### 二、 视觉模型：二分法切割 (The Great Divider)

这是做数位题（尤其是递归/分治）最关键的视角。不要把二进制看成一串数字，要看成**对区间的一刀切**。

假设我们现在看第 $k$ 位（权重为 $2^k$），这一位把整个数字世界一分为二：

- **当前位下标**：$k$
- **当前位权重**：$2^k$
- **全集范围**：$[0, 2^{k+1} - 1]$ （长度为 $2^{k+1}$）

这一刀切下去：

1. **左半区（该位为 0）**：
   - 数值范围：$[0, 2^k - 1]$
   - 大小：$2^k$
   - 特征：这一位是 `0`，后面 $k$ 位随意变化。
2. **右半区（该位为 1）**：
   - 数值范围：$[2^k, 2^{k+1} - 1]$
   - 大小：$2^k$
   - 特征：这一位是 `1`，后面 $k$ 位随意变化。
   - **转换视角**：这里的任意数 $X$，都可以看作 $2^k + Y$（其中 $Y$ 是左半区的一个数）。

**记忆案例（以 k=3 为例，总共3位，范围 0-7）：**

- 最高位是第 2 位（权重 $2^2=4$）。
- **0~3** (000, 001, 010, 011)：最高位是 0。
- **4~7** (100, 101, 110, 111)：最高位是 1。
- 看见 4 ($2^2$)，就知道它是**后半区的起点**。

> 从低到高，从右往左

------

### 三、 常见误区纠正表

在写代码时，遇到以下情况请查表：

| **你的想法 (自然语言)**                | **代码中的写法 (C++/Java)** | **为什么？**                                         |
| -------------------------------------- | --------------------------- | ---------------------------------------------------- |
| 第 $k$ 位                              | `(x >> k) & 1`              | 要习惯于有第0位                                      |
| 权重 $2^k$                             | `1LL << k`                  | `1` 左移 $k$ 位就是 $2^k$。**注意用 1LL 防止溢出！** |
| 第 $k$ 位之后全为 1                    | `(1LL << k) - 1`            | $2^k$ 是 `100...0`，减 1 就变成了 `011...1`。        |
| 清除高位，只保留第 $k$ 位后的低位1     | `x & ((1LL << k) - 1)`      | 类似于对 $2^k$ 取模。                                |
| 清除低位，只保留第 $k$ 位及之前的高位1 | x >> k << k                 | 把低 `0~k-1` 位掩盖掉。                              |

------

### 四、 实战检查点 (Checklist)

做题时，脑子里过一遍这三个问题，基本就不会错：

1. **位数 vs 下标**：题目说的“第 $k$ 位”是从 0 开始数的还是从 1 开始数的？
   - *算法惯例*：通常默认下标 $0$ 是最低位（权重 $2^0=1$）。
   - *你的例子*：8位二进制，下标范围是 $0 \sim 7$。
2. **区间开闭**：$2^k$ 到底在左边还是右边？
   - $2^k$ 是 `100...0`，它是**右半区（最高位为1）的第一个数**。
   - 所以左区间是 $[0, 2^k - 1]$，右区间是 $[2^k, 2^{k+1} - 1]$。
3. **数量 vs 偏移**：
   - 如果只看最高位为1的右半区，把这最高位去掉后，它就变成了 $[0, 2^k - 1]$，和左半区范围一致
   - 这在递归中非常重要：**右半区的子问题结构与左半区完全一样，只是每个数都大了 $2^k$**。

### 五、 终极记忆图：尺子

把它想象成一把尺子：

Plaintext

```
Index(下标):     k      k-1    ...    2      1      0
Weight(权重):   2^k    2^{k-1} ...    4      2      1
                |      |              |      |      |
Border(边界): 2^k是分界线
                |
    [0, 2^k-1]  |  [2^k, 2^{k+1}-1]
   (最高位是0)  |   (最高位是1)
```

**你的例子修正：**

- **8位数** (Bits $0 \sim 7$)。
- **最高位**：下标 $7$，权重 $2^7 = 128$。
- **分界线**：$128$。
- **区间 0 (最高位0)**：$[0, 127]$ （即 $[0, 2^7-1]$）。
- **区间 1 (最高位1)**：$[128, 255]$ （即 $[2^7, 2^8-1]$）。

## 位运算计数函数

在 C++ 编程中，特别是涉及到**算法竞赛（Competitive Programming）或高性能系统编程**时，GCC 和 Clang 编译器提供了一系列 `__builtin` 开头的内建函数。

这些函数通常直接对应 CPU 的硬件指令（如 x86 的 `POPCNT`, `LZCNT`, `TZCNT` 等），因此它们的速度极快，通常被视为 $O(1)$ 的操作。

以下是处理位运算时最常用的 `__builtin` 函数详解。

------

### 1. 核心函数概览

这些函数默认处理的是 `unsigned int` (通常是 32 位) 类型。

#### 1.1 `__builtin_popcount(x)`

- **含义：** Population Count（人口计数）。
- **功能：** 计算 $x$ 的二进制表示中 **1 的个数**。
- **示例：** `__builtin_popcount(13)` (二进制 `1101`) 返回 `3`。

#### 1.2 `__builtin_clz(x)`

- **含义：** Count Leading Zeros（统计前导零）。
- **功能：** 返回 $x$ 的二进制表示中，从最高位（左边）开始连续 **0 的个数**。
- **注意：** 如果 $x=0$，该行为通常是**未定义**的（Undefined Behavior）。务必特判 $0$。
- **示例 (32位 int)：** `__builtin_clz(16)` (二进制 `...00010000`)。16 是 $2^4$，前面有 $32 - 1 - 4 = 27$ 个零。返回 `27`。

#### 1.3 `__builtin_ctz(x)`

- **含义：** Count Trailing Zeros（统计尾随零）。
- **功能：** 返回 $x$ 的二进制表示中，从最低位（右边）开始连续 **0 的个数**。
- **注意：** 如果 $x=0$，该行为也是**未定义**的。
- **示例：** `__builtin_ctz(8)` (二进制 `1000`) 返回 `3`。

------

### 2. 数据类型扩展（重要！）

默认的函数（如 `__builtin_popcount`）处理的是 `int` 类型。如果你使用的是 `long` 或 `long long`，**必须**使用对应的后缀版本，否则会导致计算错误（高位被截断）。

| **功能**        | **int (通常 32 位)** | **long**              | **long long (64 位)**  |
| --------------- | -------------------- | --------------------- | ---------------------- |
| **数 1 的个数** | `__builtin_popcount` | `__builtin_popcountl` | `__builtin_popcountll` |
| **前导零**      | `__builtin_clz`      | `__builtin_clzl`      | `__builtin_clzll`      |
| **尾随零**      | `__builtin_ctz`      | `__builtin_ctzl`      | `__builtin_ctzll`      |
| **奇偶性**      | `__builtin_parity`   | `__builtin_parityl`   | `__builtin_parityll`   |

> **警告：** 在 64 位系统上刷题或开发时，为了保险起见，如果变量是 `long long`，请务必习惯性加上 `ll` 后缀。

------

### 3. 常见应用场景

#### A. 计算 $\log_2(n)$ (向下取整)

在算法（如 ST 表、倍增法）中经常需要计算 $\lfloor \log_2(n) \rfloor$。

利用 `clz` 可以快速计算：

$$\lfloor \log_2(n) \rfloor = 31 - \text{\_\_builtin\_clz}(n)$$

当前数的最高位是多少位（0index）：$$31 - \text{\_\_builtin\_clz}(n)$$
当前数的最高位往右共有多少位：$$32 - \text{\_\_builtin\_clz}(n)$$

*(注意：前提是 n 为 32 位正整数)*

C++

```
// 64位版本的 log2
int log2_val = 63 - __builtin_clzll(n); 
```

#### B. 获取最低位的 1 (Lowbit)

虽然可以用 `x & -x` 获取 lowbit 的数值，但如果想知道它在第几位（索引）：

C++

```
int index = __builtin_ctz(n); 
// 例如 n = 12 (1100), ctz 返回 2，表示第 2 位（从 0 开始）是 1
```

#### C. 遍历集合的所有子集 (状态压缩 DP)

虽然通常用循环，但结合 `ctz` 可以快速跳跃遍历所有为 1 的位。

## 位运算符

位运算不仅是计算机底层的语言，也是**高性能代码**和**算法竞赛**中的“瑞士军刀”。这些操作符的特点是：**极快**（单指令周期）且**省空间**。

下面我按功能分类，为你拆解 `&`, `|`, `^`, `>>`, `<<` 最常见、最实用的应用场景。

------

### 1. 移位运算符：构建与数学

#### `<<` 左移 (Left Shift)

- **数学本质：** $x \ll n$ 等价于 $x \times 2^n$。

- **核心应用 1：构造掩码 (Mask)**

  这是最常用的操作。想要操作第 $n$ 位，首先需要把 1 移动到那个位置。

  C++

  ```
  int mask = 1 << n; // 构造一个只有第 n 位是 1 的数
  ```

#### `>>` 右移 (Right Shift)

- **数学本质：** $x \gg n$ 等价于 $x / 2^n$ (向下取整)。

- **核心应用 1：快速除法**

  二分查找中常用的 `mid = (left + right) >> 1`，比除法指令略快。

- **核心应用 2：取出第 n 位的值**

  配合 `& 1` 使用。

  ```
  int bit_val = (x >> n) & 1; // 获取 x 第 n 位的数值 (0 或 1)
  ```

------

### 2. 逻辑运算符：控制与修改

我们将一个整数看作一排开关（0关，1开）。

#### `|` 按位或 (OR) —— “扩充设置” (Set)

- **口诀：** 有 1 则 1。

- **应用：将某一位设为 1 (开启开关)**

  不管原来那一 位是 0 还是 1，运算后一定会变成 1。

  ```
  x |= (1 << n);
  ```

- 判断当前值 $x$ 是否存在模板 $raw$ 以外的位 1

  ```
  if(x | raw == raw)
  ```

#### `&` 按位与 (AND) —— “提取”与“清零”

- **口诀：** 全 1 则 1，有 0 则 0。

- **应用 1：判断奇偶性**

  判断最低位是 0 还是 1。比 `% 2` 更快。

  C++

  ```
  if (x & 1) { /* 是奇数 */ }
  else       { /* 是偶数 */ }
  ```

- **应用 2：检查某一位是否为 1**

  ```
  if (x & (1 << n)) { /* 第 n 位是 1 */ }
  ```

- **应用 3：将某一位清零 (关闭开关)**

  一般配合取反操作符 `~`。

  ```
  x &= ~(1 << n); 	// 将 x 的第 n 位强制设为 0
  x &= ((1LL << k) - 1)	//将k-的高位抹掉
  x &= (~((1LL << k) - 1))	//将0-k-1的低位抹掉
  x &= raw	//将模板值raw所含有的1以外的1抹掉
  ```

- **应用 4：快速取模 (仅限对 2 的幂取模)**

  如果 $M = 2^k$，则 `x % M` 等价于 `x & (M - 1)`。

  ```
  // 比如 hashMap 中计算索引，长度 cap 通常是 16, 32...
  index = hash_val & (cap - 1); 
  ```

- **应用 5：Lowbit**

  ```
  x & -x 	//提取出 x 二进制中最右侧的 1 以及后面的 0。常用于树状数组。例如 `12 (1100)` -> `4 (100)`。
  ```

#### `^` 按位异或 (XOR) —— “翻转”与“找不同”

- **口诀：** 相同为 0，不同为 1。（也可以理解为：**不进位加法**）。

- **应用 1：翻转某一位 (Toggle)**

  0 变 1，1 变 0。

  ```
  x ^= (1 << n); // 翻转第 n 位
  ```

- **应用 2 找到不同位的个数**

  ```
  __builtin_popcount(x ^ raw)		// 得到x与模板值不同位的个数
  ```

- **应用 3：数值交换 (不使用临时变量)**

  虽然现代编译器优化后 `swap(a,b)` 更好，但这依然是经典技巧。

  ```
  a ^= b;
  b ^= a;
  a ^= b;
  ```

- **应用 4：简单的对称加密**

  `A ^ B = C`，则 `C ^ B = A`。异或两次回到原值。

> 注意：==与!=的优先级高于&、|、^，联合使用时注意加括号

------

### 总结速查表

假设我们要操作变量 `x` 的**第 `n` 位**：

1. **检查 (Check):** `(x >> n) & 1` 或 `x & (1 << n)`
2. **设置 (Set):** `x | (1 << n)`
3. **清零 (Clear):** `x & ~(1 << n)`
4. **翻转 (Toggle):** `x ^ (1 << n)`



## 位运算符相关公式

### 1. 算术运算与位运算的转换（最核心部分）

这部分与你给出的例子 $A + B = (A | B) + (A \& B)$ 属于同一类，描述了“数值加法”与“位逻辑”之间的联系。

- **加法分解定理（容斥原理）：**

  $$A + B = (A | B) + (A \& B)$$

  *加法 = 并集 + 交集*

- **异或与加法的关系（进位公式）：**

  $$A + B = (A \oplus B) + 2 \cdot (A \& B)$$

  *加法 = 不进位加法 + 进位*

- **减法与异或：**

  $$A - B = (A \oplus B) - 2 \cdot ((\sim A) \& B)$$

  *减法 = 不进位加法 - 借位*

- **三者关系：**

  $$(A | B) = (A \oplus B) + (A \& B)$$

  *并集 = 异或 + 交集*

### 2. XOR（异或）的特殊性质

异或运算在算法中极其重要，因为它构成了阿贝尔群（Abel Group）。

- **自反性（消消乐）：**

  $$A \oplus A = 0$$

- **结合律与交换律：**

  $$A \oplus B \oplus C = A \oplus (B \oplus C)$$

- **归零律：**

  $$A \oplus 0 = A$$

- **逆运算（解方程用）：**

  若 $A \oplus B = C$，则 $A \oplus C = B$ 且 $B \oplus C = A$。

- **构造性恒等式：**

  $$A \oplus B = (A | B) - (A \& B)$$

### 3. 集合论性质（类比逻辑代数）

把二进制位看作集合，位运算完全符合集合运算定律。

- **德摩根定律（De Morgan's Laws）：**

  $$\sim (A \& B) = (\sim A) | (\sim B)$$

  $$\sim (A | B) = (\sim A) \& (\sim B)$$

- **分配律（Distributive Laws）：**

  $$A \& (B | C) = (A \& B) | (A \& C)$$

  $$A | (B \& C) = (A | B) \& (A | C)$$

- **吸收律：**

  $$A \& (A | B) = A$$

  $$A | (A \& B) = A$$

### 4. 常见不等式（用于估算与剪枝）

在算法优化中，这些不等式常用于确定边界。

- **基本大小关系：**

  $$A \& B \le \min(A, B) \le \max(A, B) \le A | B$$

  $$| A - B | \le A \oplus B \le A + B$$

这些虽不是纯代数等式，但构成了位运算算法的基础（如树状数组、状态压缩）。

### 1. 观察“修改的代价” (The Cost of Change)

在位运算题目中，如果我们被迫修改第 $i$ 位（比如 $x$ 的第 $i$ 位是 1，但为了配合 $y$，必须变成 0），这对数值的影响是巨大的（$\pm 2^i$）。

**关键直觉：**

为了弥补修改第 $i$ 位带来的巨大误差，我们在 $i$ 位以下（$0$ 到 $i-1$ 位）通常只有两种最优选择：

1. **全变为 0**：试图让数字变小，接近原来的值（对应截断）。
2. **全变为 1**：试图让数字变大，接近原来的值（对应借位/进位后的补偿）。



> 题目的被迫修改第i位不是只有1变成0的情况吗，那为什么不只考虑全变成1的选择？

这是一个非常深刻的问题！你的直觉在很大程度上是对的：**解决冲突最直接的办法确实是把 $1$ 变成 $0$。**

但是，**只考虑“变小”（全变成1）是不够的，因为有时候“变大”（进位）反而离原数更近。**

我们来详细拆解一下为什么我们需要 **2种** 候选策略，而不仅仅是你提到的那一种。

------

#### 1.1 核心误区：不仅仅是“解决冲突”，更是“找最近的数”

我们的目标是 $Cost = |x - P|$ 最小。

当 $x$ 的某一位 $k$ 是 $1$，而我们因为冲突被迫把它变成 $0$ 时，我们确实损失了 $2^k$ 的数值。

这时我们有两个补救方向：

1. **向下补救（你的思路）**：把 $k$ 位以下的 $0$ 都变成 $1$。试图用 $2^{k-1} + 2^{k-2} + \dots$ 来弥补损失。
   - 结果：$P < x$。
2. **向上补救（进位思路）**：把 $k$ 位以上的某一位（原本是 $0$）变成 $1$。
   - 结果：$P > x$。

#### 1.2 举个极端的例子（反例）

假设 $x = 15$（二进制 `01111`），并且假设这就这 4 个位置和 $y$ 都有冲突，必须改。

- **你的策略（向下变 0，低位全 1）**：
  - 被迫把这些 $1$ 都去掉。
  - 结果 $P = 0$（二进制 `00000`）。
  - **代价**：$|15 - 0| = 15$。
- **进位策略（向上变 1，低位全 0）**：
  - 我们观察到 $15$ 其实离 $16$（二进制 `10000`）非常近！
  - 如果我们把第 4 位的 `0` 变成 `1`，然后把低位清零。
  - 结果 $P = 16$（二进制 `10000`）。
  - **代价**：$|15 - 16| = 1$。

**结论：** 在这个例子里，把 `0` 变成 `1`（进位）比把你提到的“全变成 1”优秀得多。

------

#### 1.3 为什么会有三种情况？（本题实际不需要第一种）

代码中的三种情况分别对应了数字在数轴上的三种逼近方式：

假设我们在第 $i$ 位截断/修改：

1. **直接截断 (`a`) -> “舍”（可删除）**

> 这种情况在本题实际上只处理了到第0位了，q和y也完全对照的情况，相当于在第0位选择向下填充，这种情景被第二种情况在i==0时包含

- **操作**：保留高位，第 $i$ 位及以下全变 $0$。
- **适用场景**：$x$ 的低位本来就比较小，丢掉也不心疼。
- **例子**：$x = 10001$ (`10001`)。冲突在最后一位。
  - 变 $10000$，代价很小，没必要大动干戈去进位或填满1。

2. **向下取整并填充 (`a + (1<<i) - 1`) -> “地板”**

- **操作**：保留高位，第 $i$ 位（原为1）变为 $0$，**低位全变 $1$**。
- **这是你提到的情况**。
- **适用场景**：$x$ 在第 $i$ 位是 $1$，且丢掉这个 $1$ 损失太大，需要低位全变成 $1$ 来止损。
- **例子**：$x = 11000$ (`11000`)，强制把第 3 位变成 0。
  - 与其变成 $10000$（损失 1000），不如变成 $10111$（损失 1）。

3. **向上进位 (`a + (1<<i)`) -> “天花板”**

- **操作**：保留高位，第 $i$ 位（原为0）变为 $1$，低位全变 $0$。
- **适用场景**：$x$ 的低位全是 $1$（或者很大），像 $x=01111$ 这种情况。
- **逻辑**：虽然第 $i$ 位原本是 $0$，没冲突，但为了让数值更接近 $x$，我们主动把它变成 $1$（进位），这样就可以把低位那些**有冲突的 $1$ 全部心安理得地变成 $0$**。
- **例子**：$x = 01111$，冲突在低位。
  - 变成 $10000$ 是最优解。

#### 1.4 总结

你说得对，**“被迫修改”**通常指的是 $1 \to 0$。

但是，**“为了让结果更优而主动做的修改”** 可能包含了 $0 \to 1$。

我们在暴力枚举的时候，不仅是在处理“被迫”，也是在寻找“机会”。有时候，**主动进位（$0 \to 1$）一次，能消除低位所有的冲突，而且代价还很小**。这就是为什么必须包含 `a + (1 << i)` 这个选项的原因。

### 2. 从“如何构造”转向“长什么样” (Structure over Construction)

你的贪心思路是：“我正在第 $i$ 位，我该选 0 还是 1？”这是**构造性**思维。

通用的暴力思路是：“最终的最优解 $P$，一定长什么样子？”这是**结构性**思维。

对于大多数这类题目，最优解 $P$ 通常由两部分组成：

- **高位部分（前缀）**：完全等于原数 $x$ 的高位（或者进位后的高位）。
- **低位部分（后缀）**：在某一位 $i$ 发生“突变”后，低位要么全是 0，要么全是 1。

因此，我们不需要一位一位地去推导，只需要**枚举这个“突变点” $i$ 在哪里**。

**疑问**：为什么总是让 $Q$ 让步？这样不会漏掉最优解吗？

- **解答**：不会。因为我们是双重循环枚举 $i$ 和 $j$。在某一次枚举中，我们优先保留 $P$ 的形状；但在另一次枚举（当 $P$ 的候选形态本身在那一位就是 $0$ 时），自然就相当于 $P$ 让步了。
- 而且，这 $2 \times 2$ 的组合实际上是在暴力尝试“$x$ 在位 $i$ 发生结构突变”和“$y$ 在位 $j$ 发生结构突变”的所有可能组合。

### 3. 利用数据范围“降维打击”

这类题目通常涉及 `int` 或 `long long`。

- 二进制只有 30~60 位。
- $O(30^2)$ 甚至 $O(30^3)$ 的复杂度对于计算机来说几乎是 $O(1)$ 的。
- **不要试图用 $O(1)$ 的贪心去解决 $O(30^2)$ 就能过的题。** 贪心往往需要极其严密的证明，而枚举只需要覆盖所有情况。

------

### 具体实战：下次如何通过“枚举”解题

当你发现题目满足：

1. 涉及位运算（与、或、异或）。
2. 涉及加减法带来的数值距离（$|A-B|$）。
3. 你需要构造一个数。

**请直接尝试以下“套路”：**

1. **定义候选集生成器**：

   写一个小的 lambda 函数或辅助逻辑。给定原数 $x$ 和一个分割点 $i$，生成 3 个由 $x$ 衍生出的“候选数”：

   - **Keep & Cut**: 保留 $i$ 位以上，$i$ 位及以下清零。（对应直接舍弃）
   - **Floor Fill**: 保留 $i$ 位以上，但第 $i$ 位变为 0（如果原来是 1），$i-1$ 位以下全变 1。（对应向下取整的最大值）
   - **Ceil Clear**: 保留 $i$ 位以上，但第 $i$ 位进位（+1），$i$ 位及以下清零。（对应向上取整）
   - *注：上面的 `Solve` 代码中用 `a`, `a+(1<<i)`, `a+(1<<i)-1` 完美覆盖了这些情况。*

2. **暴力枚举分割点**：

   由于有两个数 $x, y$，且它们的变化可能是独立的。

   - 枚举 $x$ 在第 $i$ 位发生突变。
   - 枚举 $y$ 在第 $j$ 位发生突变。

3. **计算与打擂台**：

   - 双重循环 $i, j$ (0 到 30)。
   - 取出 $x$ 的 3 个候选形态，$y$ 的 3 个候选形态。
   - 计算代价，更新最小值。

### 你的代码 vs 通用解法

- **你的代码 (贪心)**：
  - **优点**：跑得快（线性遍历位数），不仅 AC 了而且效率很高。
  - **缺点**：**思维负担重**。你需要时刻考虑 `sum` 是多少，`ok` 状态是什么，如果冲突了是让 $x$ 变还是让 $y$ 变（你的代码通过 `get(x,y)` 和 `get(y,x)` 规避了部分双变量同时变的情况，这其实有风险，除非题目性质保证了只需单侧调整）。
  - **风险**：一旦题目条件稍微变复杂（比如 $p, q$ 还有额外限制），贪心逻辑就要重写。
- **通用解法 (枚举)**：
  - **优点**：**无脑**。不需要去想“当前位该怎么办”，只需要想“所有可能的最终形态都在我的枚举范围内吗？”。只要候选集覆盖了最优解的结构，就一定对。
  - **缺点**：常数略大（$30 \times 30 \times 9$ 次计算），但在 1秒/2秒 的时限下完全可以忽略。

### 总结

下次看到这种题，先告诉自己：**“只有 30 位，我可以暴力枚举‘断点’。”**

不需要在循环里写复杂的 `if (bit == 1) ... else ...`，而是直接：

```cpp
// 伪代码思路
for (i from 0 to 30) { // x 的断点
    for (j from 0 to 30) { // y 的断点
        vector cX = {x的断点i处的几种形态};
        vector cY = {y的断点j处的几种形态};
        for (auto p : cX) 
            for (auto q : cY) 
                if (check(p, q)) ans = min(ans, cost(p, q));
    }
}
```

这就是通用的“枚举暴力解法”的核心心法。
