### 1. 题目归类：给思维贴标签

为了在未来遇到类似题目时能秒速提取策略，我们将这道题归类为：

**核心标签：**

- **贡献法 (Contribution Technique)**：不直接枚举结果（子序列），而是枚举产生结果的“关键点”（即前半段的最后一个字符），计算该点对总答案的贡献。
- **范德蒙德卷积 (Vandermonde's Identity)**：组合数学中的经典恒等式变形，用于处理 $\sum \binom{n}{k} \binom{m}{c-k}$ 形式的求和。

**辅助标签：**

- **线性扫描 (Linear Scan)**：利用前缀/后缀和思想，在 $O(N)$ 时间内维护计数。
- **组合计数 (Combinatorial Counting)**。

本质描述：

这是一道**“带约束的子序列计数”问题。通常这类问题如果 $N$ 较小 ($N \le 2000$) 会考虑 DP；但当 $N$ 很大 ($10^5+$) 时，往往需要寻找具有唯一性的特征点**（Pivot），将问题转化为组合数学公式求和。

> 从特征点开始一左一右，或是两个特征点左左右右，总之特征点的方向不能有重合，不然难以计数

### 2. 思维复盘：从零到一的推导过程

假设我们在比赛场上，看到这道题，大脑应该如何一步步运作：

#### **阶段一：直觉与试错（分析约束）**

1. **读题**：求满足“1122模式”的子序列个数。
2. **看数据范围**：$|S| \le 10^6$。
   - *条件反射*：$O(N^2)$ 的 DP 绝对不行。必须是 $O(N)$ 或者 $O(N \log N)$。
   - *结论*：只能扫一遍或者两遍字符串，不能做复杂的区间 DP。

#### **阶段二：寻找切入点（寻找 Pivot）**

1. **思考结构**：“1122字符串”的结构非常特殊。它由两部分组成：前半部分全是 $d$，后半部分全是 $d+1$，且长度相等。
2. **寻找“不变量”或“分割点”**：
   - 如果我们枚举整个子序列，太复杂。
   - 有没有哪个位置是“决定性”的？
   - *灵感*：所有的“1122字符串”都有一个**交界处**，即从 $d$ 变成 $d+1$ 的那个地方。
3. **确定枚举对象**：
   - 尝试枚举交界线？但在子序列中，交界线是虚拟的，不好定位。
   - 尝试枚举**前半段的最后一个字符**？
   - 假设 $S[i]$ 是某个合法子序列前半段的**最后一个** $d$。那么：
     - 这个子序列的前 $k-1$ 个字符必须从 $S[0 \dots i-1]$ 里的 $d$ 中选。
     - 这个子序列的后 $k$ 个字符必须从 $S[i+1 \dots N]$ 里的 $d+1$ 中选。
   - *验证*：这个定义是否唯一？是的。对于任何一个合法的“1122子序列”，它前半段的最后一个字符在原字符串 $S$ 中的位置是唯一的。**不会重复计数**。

#### **阶段三：数学建模与化简（核心难点突破）**

1. **列式子**：
   - 设左边有 $p$ 个 $d$，右边有 $q$ 个 $d+1$。
   - 对于固定的 $S[i]$，我们需要枚举所有可能的半长度 $k$。
   - 贡献 = $\sum_{k \ge 1} \binom{p}{k-1} \times \binom{q}{k}$。
2. **遭遇瓶颈**：这个求和公式如果每次用 $O(N)$ 计算，总复杂度还是回到了 $O(N^2)$。我们需要 $O(1)$ 的公式。
3. **模式匹配（ICPC金牌点）**：
   - 看到两个组合数相乘并求和 $\sum \binom{A}{...} \binom{B}{...}$，立刻联想**范德蒙德卷积**（或其变体）。
   - *直观理解*：
     - 左边取 $k-1$ 个，右边取 $k$ 个。总数 $2k-1$ 是变的，不好搞。
     - **利用对称性**：$\binom{q}{k} = \binom{q}{q-k}$（选 $k$ 个保留 = 选 $q-k$ 个丢弃）。
     - 代入：左边取 $k-1$，右边取 $q-k$。
     - 总数：$(k-1) + (q-k) = q-1$。**总数恒定！**
   - *物理意义*：这就等价于从 $p+q$ 个物品中，一共选 $q-1$ 个的方法数。
   - *最终公式*：$\binom{p+q}{q-1}$ 或 $\binom{p+q}{p+1}$。

#### **阶段四：算法成型**

- 预处理组合数阶乘。
- 预处理每种数字的总数（为了快速算右边的 $q$）。
- 遍历 $S$，维护左边的 $p$，计算公式，累加答案。

### 3. 通用解题方法论：子序列计数问题的“三板斧”

以后遇到“求特定模式的子序列个数”且 $N$ 很大时，请按此流程思考：

步骤一：锚定关键点 (Anchor the Pivot)

不要试图构建整个子序列。寻找该模式中最特殊的一个元素，通常是：

- 两个不同字符的交界处（如本题）。

- 某种峰值（如求山峰形状的子序列）。

- 区间的中间点。

- 区间的开头、结尾。

  原则：选定这个点后，能否把问题清晰地切割成“左边”和“右边”两个独立的问题？

步骤二：独立计算左右状态 (Isolate Sides)

一旦确定了 Pivot（例如 index $i$），问题就变成了：

- 左边有多少种合法选法？（记为 $L_i$）
- 右边有多少种合法选法？（记为 $R_i$）
- 如果是简单的拼接，答案通常是 $L_i \times R_i$。
- 如果是联动长度（如本题，左右长度需一致），则进入步骤三。

步骤三：数学降维 (Mathematical Reduction)

如果式子涉及 $\sum \binom{n}{i} \dots$，去检查：

1. **范德蒙德卷积**：能否通过 $\binom{n}{r} = \binom{n}{n-r}$ 凑出恒定的下标和？
2. **杨辉三角性质**：$\sum_{i=0}^n \binom{i}{k} = \binom{n+1}{k+1}$ （曲棍球棒恒等式）。
3. **二项式定理**：$\sum \binom{n}{i} x^i = (1+x)^n$。

**针对本题的Meta总结**：

> 只要看到“左右两侧选取的数量这就相关（如相等）”的子序列计数，**90% 是范德蒙德卷积的变体**。

### 4. 顶尖选手的编码流描述

ICPC金牌选手在编码时，脑中的状态机不是“写一行想一行”，而是**“模板块组装”**。以下是针对此题的流畅编码过程：

**阶段 0：环境与宏定义 (The Setup)**

- *Mindset*: "这题要取模，要组合数，先写好模数运算类或预处理函数，别等用到再写。"
- *Action*: 敲入 `MOD = 998244353`，快速写好 `fact` (阶乘) 和 `inv` (逆元) 的预处理代码。这是肌肉记忆。

**阶段 1：全局数据预处理 (Global Analysis)**

- *Mindset*: "我需要知道右边剩下多少个 $d+1$，与其每次算，不如先统计所有数字的总出现次数。"

- *Action*:

  Python

  ```
  cnt_total = [0] * 10
  for c in S: cnt_total[int(c)] += 1
  ```

**阶段 2：状态机扫描 (The Sweep)**

- *Mindset*: "开始遍历 $i$。我需要维护两个状态：`cnt_left` (目前的 $p$) 和通过 `cnt_total` 推导出的 `cnt_right` (目前的 $q$)。"
- *Action*:
  - 定义 `cnt_now = [0] * 10`。
  - 写主循环 `for char in S:`。
  - **关键点**：在更新 `cnt_now` 之前还是之后计算？
    - *思考*：题目定义 $S[i]$ 是前半段最后一个，所以 $k-1$ 个 $d$ 必须在 $i$ 之前。所以先由 `cnt_now` (代表 $i$ 之前的) 计算贡献，再 update `cnt_now`。

**阶段 3：核心逻辑注入 (Core Logic)**

- *Mindset*: "如果是 9 就跳过。否则，套公式 $\binom{p+q}{p+1}$。"

- *Action*:

  Python

  ```
  d = int(char)
  if d != 9:
      p = cnt_now[d]  # 左边 d 的数量
      target = d + 1
      q = cnt_total[target] - cnt_now[target] # 右边 d+1 的数量
      ans = (ans + nCr(p + q, p + 1)) % MOD
  cnt_now[d] += 1 # 别忘了更新状态！
  ```

**阶段 4：边界检查 (Sanity Check)**

- *Mindset*: "有没有可能越界？$p+q$ 是否足够大？组合数函数处理了 $n < m$ 返回 0 吗？"
- *Action*: 确认 `nCr` 函数第一行有 `if r < 0 or r > n: return 0`。OK，提交。
