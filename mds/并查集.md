这是一个非常经典且工整的并查集（Disjoint Set Union, DSU）模板。作为冲刺 ICPC 金牌的选手，你不仅要拥有这个模板，更要将它**内化**为一种本能。

为了让你在考场上能够“大脑放空”般流畅地写出逻辑，我们需要从**形象理解**、**核心逻辑拆解**、**场景条件反射**三个维度来重构你对它的认知。

> 元素是否具备连通性，联通的元素是否是等价的？

------

### 一、 核心概念：它到底是什么？（形象化记忆）

并查集的本质是在维护**森林（Forest）**，也就是多棵树的集合。但在脑海中，我建议你用**“江湖帮派”**的模型来理解，这在中文算法圈非常传神：

1. **初始化 (`init`)**：
   - **每个人初始都是自立门户**。你是你自己的老大。
   - 映射代码：`p[i] = i` （我归我管）。
   - `siz[i] = 1` （我的帮派只有我自己）。
2. **查找 (`find`) —— “找掌门”**：
   - 你想知道 `x` 属于哪个帮派？不仅要问他的上级，还要顺藤摸瓜一直问到**掌门人**（根节点）。
   - **路径压缩（核心优化）**：这是并查集的灵魂。在找掌门的过程中，把沿途所有的小弟都**直接挂在掌门名下**。下次再问，一步就能找到掌门，不需要层层汇报。
   - 映射代码：`p[x] = find(p[x])`。
3. **合并 (`merge`) —— “结盟”**：
   - 两个小弟 `x` 和 `y` 想结盟？
   - **先找各自的掌门**：如果掌门是同一个人，说明早就是一家人了，不用操作。
   - **吞并**：如果不是一家人，那就让其中一个掌门 `y_root` 认另一个掌门 `x_root` 做大哥。
   - **资产转移**：把 `y` 帮派的人数全部加到 `x` 帮派账上。

------

### 二、 代码逻辑拆解：考场默写指南

在考场紧张时，不要死记硬背 `if` 和 `return`，而是按照**动作流**来写：

#### 1. 结构体与变量

- **本能反应**：由于涉及到大量的 `p` 数组访问，封装在 `struct` 里是最好的习惯（避免全局变量污染）。
- **必要数据**：
  - `p` (parent)：存上级。
  - `siz` (size)：存子树大小（金牌选手必须习惯维护这个，因为很多题目需要查集合大小，或者用到启发式合并）。

#### 2. `find` 函数（最容易写错，也最关键）

逻辑链条：

1. **是不是掌门？** 如果 `x == p[x]`，说明到底了，返回 `x`。
2. **不是掌门？** 递归找掌门，**顺手**把我的上级改成掌门。
3. **返回**掌门。

*默写口诀：* **“如果我不是老大，我就让我的老大变成（递归找到的）总老大，然后返回我的老大。”**

```
int find(int x) {
    if (x != p[x]) p[x] = find(p[x]); // 路径压缩：一步到位
    return p[x];
}
```

#### 3. `merge` 函数（业务逻辑核心）

逻辑链条：

1. **找头目**：先别急着连线，先把 `x` 和 `y` 都变成各自的根节点 (`find`)。
2. **判重**：如果 `x == y`，已经在同一个集合，返回 `false`（或者直接 return）。
3. **连边**：`p[y] = x` （y 认 x 做大哥）。
4. **维护信息**：`siz[x] += siz[y]` （x 的势力变大了）。**注意：一定要先加大小，再连边，或者逻辑要对齐。你模板里的顺序是没问题的，因为此时 x 和 y 都是根。**

------

### 三、 场景条件反射：什么时候用？

看到以下关键词，你的脑海中必须立刻浮现 `DSU` 的结构体：

1. **连通性判断**：

   - 题目给定的元素具备连通性，且同一联通块内的各个元素在题目意义下是**等价的**

   [C-炮火轰炸_2026牛客寒假算法基础集训营2](https://ac.nowcoder.com/acm/contest/120562/C)

   - 题目给定的元素具备连通性，且贡献的最小单位为联通块而非元素

   [F-Bracket Coloring_牛客周赛 Round 131](https://ac.nowcoder.com/acm/contest/127703/F)

2. **传递性——“朋友的朋友是朋友”**：

   - 传递性关系的维护。例如：A 和 B 等价，B 和 C 等价 -> A 和 C 等价。

3. **时光倒流（反向操作）**：

   - 题目说：“在这个图中**删掉**一条边，问连通性”。
   - **警觉**：并查集**不支持**删边（或者很难）。
   - **对策**：把“删边”转化为从最终状态开始**倒着加边**。这是 ICPC 经典套路。

4. **带权并查集 / 扩展域并查集**：

   - 题目不仅问连通，还问关系（比如“敌人的敌人是朋友”——食物链问题，或者二分图染色）。
   - 这时候需要在 `find` 函数里额外维护一个 `dist` 数组。

5. **链式并查集**

   - 题目约束让每个点只能被它的第一次访问
   - 当前点被访问后就将它合并给下一次理应访问的点（有序，即需要保证合并后的根节点正是下一个节点）

   [F-Permutation Counting_2026牛客寒假算法基础集训营1](https://ac.nowcoder.com/acm/contest/120561/F)

### 总结

你现在的这个模板非常标准，是一个很好的基底。

- **Init** 是建帮派。
- **Find** 是压缩层级找掌门。
- **Merge** 是帮派吞并资产重组。

## 数据分块与并查集

[C-炮火轰炸_2026牛客寒假算法基础集训营2](https://ac.nowcoder.com/acm/contest/120562/C)

这是一个非常棒的复盘请求。针对 ICPC 金牌的目标，仅仅通过题目是不够的，必须建立**模型化思维**和**标准化的解题流**。

> 按行对炮火点隔开的段做分块，按列对相交的段（暂未被围起来的段）做并查集
>
> 对于多段等价的信息，只需要保留其中一个作为特征信息存储起来即可

------

### 一、 题目归类：这是什么本质？

**标签：** `网格图连通性` + `坐标离散化` + `扫描线思想` + `并查集`

**本质描述：**

这道题的本质是**在稀疏障碍物的无限网格中，求解空白区域的连通分量**。

它属于一类经典模型：**“有限障碍物切割无限空间”**。

- **低维投影**：如果是一维数轴上有几个点把数轴切断，问两点是否连通？（直接看区间）
- **本题形态**：二维平面，障碍物很少，坐标很大。
- **核心矛盾**：庞大的坐标系 vs 稀疏的信息量。
- **转化方向**：将“点阵图”转化为“拓扑图”。不关心具体的坐标 $100$ 和 $1000$ 之间有多少个空点，只关心它们是否属于同一个“空白块”。

------

### 二、 思维复盘：从零推导的“上帝视角”

假设我没有看过题解，面对这道题，我的大脑应该如何运作？

#### Step 1: 朴素分析与瓶颈确认

- **直觉**：判断一个点是否被包围，就是看它能不能跑出边界（走到无穷远）。
- **动作**：如果图很小（例如 $1000 \times 1000$），我直接跑 BFS/DFS 染色。
- **阻碍**：坐标 $10^9$，内存炸了，时间炸了。
- **结论**：不能真实地存图，必须“压缩”。

#### Step 2: 捕捉关键特征（稀疏性）

- **观察**：障碍物只有 $10^5$ 个。
- **思考**：在 $x=10$ 有个障碍，在 $x=10000$ 有个障碍。那 $x \in [11, 9999]$ 这一大片区域其实是**完全等价**的。它们在拓扑结构上就是一个巨大的矩形块。
- **决策**：**坐标离散化**。只保留有意义的坐标。

#### Step 3: 构建离散化策略（最关键的转折）

- **初级离散化**：只存 $x$。
  - *自我反驳*：如果有两个障碍物分别在 $x=2$ 和 $x=4$，中间 $x=3$ 其实是一条路。如果我只存 $\{2, 4\}$，它们在数组里就相邻了，路也没了。
- **修正策略**：为了保留“路”，对于每个障碍物 $x$，我必须保留 $x$ 和 $x+1$。
  - $x$ 代表障碍物所在的列。
  - $x+1$ 代表障碍物右边的空白列（如果下一个障碍物很远，这个 $x+1$ 就代表了中间那一片无限宽的区域）。

#### Step 4: 降维打击（扫描线思想）

- **思考**：现在的地图已经被我压缩成了 $2N$ 个竖条（Column）。
- **局部视角**：在每一个竖条内部，障碍物把这一列切成了好几段**线段（Interval）**。
- **整体视角**：如果把每个“空白线段”看作一个**节点**。那么，相邻两列的线段如果在此处 $y$ 轴上有重叠，就代表这两个节点之间有一条**边**。
- **模型成型**：
  1. 提取出所有空白线段，给它们编号 $1, 2, \dots, K$。
  2. 扫描相邻列，有重叠就连边（并查集 `merge`）。
  3. 最后问起点所在的线段和“无穷远线段”是否在同一个集合。

#### Step 5: 算法特异化（针对本题的微调）

- **模板选择**：离散化 + DSU。
- **细节处理**：
  - 无穷远怎么表示？我们可以人为规定边界或者把所有触达边界的线段合并到一个特殊的 `dummy node`（比如 0 号或者是 1 号节点）。
  - $y$ 轴需要离散化吗？本题中 $y$ 只需要在每一列内部比较，不需要全局离散化。直接在每一列内部排序即可（这比全局二维离散化省空间且好写）。
  - **查询处理**：对于查询点 $(qx, qy)$，先通过 `upper_bound` 找到它属于哪个离散化后的列，再 `lower_bound` 找到属于哪条线段。

------

### 三、 你的思路卡在哪里？如何修正？

**你的初始思路**：“注意到炮火点很少，利用这点解题，但没想到好办法。”

**卡点分析**：

你卡在了 **“稀疏性” 到 “具体算法” 的映射** 上。你知道 $N$ 小是有用的，但你脑子里可能在想几何计算（比如判断点在多边形内），或者还在想怎么在稀疏图上跑 BFS。

**思维修正路径（下次该怎么想）**：

当你发现 **“坐标很大” 但 “关键点很少”** 时，脑中应立刻弹出两个选项：

1. **动态开点线段树**（用于维护区间信息）。
2. **离散化**（用于将无限网格变为有限网格）。

**破局的关键提问**：

> “如果我把所有没有障碍物的行和列都抽掉，只保留有障碍物的行和列，图的连通性会变吗？”
>
> -> “会变，因为把路抽没了。”
>
> -> “那如果我保留有障碍物的行/列，同时在它们旁边各保留一行/列空白呢？”
>
> -> “连通性不变！而且图变小了！” -> **解法出现。**

------

### 四、 通用解题方法论（针对陌生难题）

当你在 ICPC 赛场上遇到一道题，脑中一片空白时，请按这个流程执行：

1. **看数据范围（第一直觉来源）**
   - $N \le 20$：状压 DP / 暴搜。
   - $N \le 10^5$：$O(N \log N)$，分治 / 线段树 / 贪心 / 离散化。
   - $N \le 10^7$：$O(N)$，线性筛 / 单调队列 / 线性 DP。
   - 坐标 $10^9$：必须离散化 或 动态开点。
2. **简化与类比（降维打击）**
   - **如果是 1D 怎么办？**（本题 1D 就是区间合并）。
   - **如果是 树 怎么办？**
   - **如果没有某个限制怎么办？**
   - 通过简化问题，找到核心逻辑，然后再把限制加回去。
3. **逆向思维（补集思想）**
   - 正难则反。
   - 求“被包围”很难 $\to$ 求“能逃出去”很容易。
   - 求“至少由一个...” $\to$ 总数 - “一个都没有...”。
4. **图论建模（万物皆图）**
   - 状态是点，转移是边。
   - 网格图往往隐含着连通性问题。
   - 利用并查集维护“集合”属性，利用最短路维护“代价”属性。

------

### 五、 顶尖选手的编码流（Coding Flow）

即使思路有了，写代码卡顿通常是因为**数据结构设计不清晰**。顶尖选手在敲键盘前，脑子里已经有了下面这个“骨架”：

**本题的编码流剧本：**

1. **全局定义与结构体 (The Setup)**

   - 先写 `DSU` 结构体（或者复制板子）。这是独立模块，不用动脑。
   - 定义 `const int INF`。
   - 思考存储结构：我们需要存离散化后的列。
   - `vector<int> col_x;` (存离散化的X坐标)
   - `struct Segment { int y1, y2, id; };` (显式定义线段结构，比 `array` 更清晰)
   - `vector<vector<Segment>> grid;`

2. **离散化预处理 (Pre-calc)**

   - 读入所有点。
   - 把 `x` 和 `x+1` 塞入 `col_x`。
   - `sort` & `unique`。
   - **关键点**：此时要确认 `col_x` 的大小 $M$。

3. **构建线段 (Build Graph)**

   - 这是最容易写错的地方。
   - 遍历 $M$ 列。对于每一列，拿出属于这列的所有原始障碍点 $y$。
   - 排序 $y$。
   - **扫描生成**：遍历 $y$，如果 `curr_y > last_y`，说明中间有缝隙，`push_back` 一个 `Segment`，并分配 `ID`。
   - *Self-Check*：边界处理好了吗？`0` 到 `first_y`，`last_y` 到 `INF`。

4. **连边合并 (Merge)**

   - 双指针遍历相邻两列的 `Segment` 列表。

   - 逻辑：`seg[i]` 和 `seg[i+1]`。

   - 如果是区间重叠问题，直接套双指针模板：

     C++

     ```
     while(j < s1.size() && k < s2.size()) {
        if (intersect(s1[j], s2[k])) dsu.merge(...);
        if (s1[j].y2 < s2[k].y2) j++; else k++;
     }
     ```

5. **处理询问 (Query)**

   - 读入 `(qx, qy)`。
   - 二分找列 `idx`。
   - 二分找线段 `seg_idx`。
   - `dsu.find(seg_id) == dsu.find(outside_id)` ? "NO" : "YES".

**在这个流程中，最重要的是第 1 步。** 不要急着写 `main`，先定义好 `struct` 和 `vector` 的含义。数据结构定义清楚了，逻辑自然就顺了。
