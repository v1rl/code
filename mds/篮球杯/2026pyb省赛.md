## **单调栈贡献 + 梯形分段等差求和**

[P16267 蓝桥杯 2026 省 Python B 组 位数求和 - 洛谷](https://www.luogu.com.cn/problem/P16267)

给定长度为 n 的数组 a，求：所有子区间的 **区间最大值** 乘以 **区间长度在十进制下的位数** 的总和，结果对 998244353 取模。(n <= 500000)

------

**思路 Trick 总结**

这道题的推导是极其经典的，包含了三个层次的思维跳跃：

- **Trick 1：视角反转 (算贡献模型)**

  不要“枚举区间去求最大值和长度”，而是“**枚举每个元素 $a_i$ 作为最大值，去反推它能辐射到的区间集合**”。配合非对称单调栈（左边找严格大于，右边找大于等于）划定边界，完美去重。

- **Trick 2：维度投影 (二维组合降为一维分段)**

  已知左端点有 x 种选择，右端点有 y 种选择，求长度为 k 的区间个数。

  正常思维是解二维不等式，但本题将其投影到一维长度轴上：发现随着 k 增大，方案数 $g(k)$ 呈现出先递增、再平稳、后递减的**梯形分段函数**分布。这个跳跃直接把 $O(n^2)$ 的枚举降维成了 $O(1)$ 的数学公式。

- **Trick 3：变量打包 (将非线性权重分块)**

  题目要求乘上“长度的位数”，这是一个非线性的阶跃函数。处理手段是将长度域按位数划分成常数个区块：[1, 9]、[10, 99]、[100, 999]... 在每一个区块内，位权是个常数，问题就变成了求区间内 $g(k)$ 的和。

------

**编码 Trick 总结 (降低认知负荷的优雅实现)**

你的这版 AC 代码在工程实现上已经非常成熟，其中包含了几个极为实用的编码范式，强烈建议固化到你的肌肉记忆里：

- **编码 Trick 1：强行对称化 (消灭多余分支)**

  在计算 $g(k)$ 时，左右端点 x 和 y 的地位在数学上是对称的。代码中直接一句 `if (x > y) swap(x, y);` 强行认定左侧总是较短边。这一手直接砍掉了平稳期到底是 $[x+1, y]$ 还是 $[y+1, x]$ 的分类讨论，让后续代码逻辑减半，极大幅度降低了赛场上写挂的概率。

- **编码 Trick 2：区间“无脑”取交集**

  在处理目标长度区间 $[a, b]$ 与梯形的三个阶段求交集时，没有写一堆恶心的嵌套 if 来判断谁包含谁、谁截断谁。而是直接利用数学上的交集定义：`max(阶段左界, a)` 和 `min(阶段右界, b)`。只要算出来 `L > R` 就说明无交集直接返回 0。这种写法鲁棒性极高。

- **编码 Trick 3：Lambda 闭包内聚**

  将求 $g(k)$、等差数列求和 $get(l, r)$ 全部以 Lambda 的形式封装在 $calc$ 内部。这不仅仅是“看起来高级”，更重要的是它实现了**变量隔离**。内部函数直接捕获上下文的 x 和 y，不需要传来传去，也不会污染外层的命名空间，让主干逻辑（单调栈 + 分块求和）清爽见底。
