### 一、 本质概念：它到底是干什么的？

#### 1. 核心定义

普通的 GCD 算法（欧几里得算法）只能求出两个数 $a$ 和 $b$ 的最大公约数 $\gcd(a, b)$。

而 **exgcd** 在求出 $\gcd(a, b)$ 的同时，还能帮我们找到一组整数解 $(x, y)$，满足著名的**裴蜀定理（Bézout's Lemma）**方程：

$$ax + by = \gcd(a, b)$$

#### 2. 脑中应浮现的画面

当你看到 "exgcd" 时，脑海中不要只出现代码，而应该浮现出这样一个逻辑链条：

- **输入**：两个整数 $a, b$。
- **黑盒内部**：递归地碾转相除。
- **输出**：三个值 —— $d$ (即 gcd)、$x$ (系数1)、$y$ (系数2)。
- **物理意义**：虽然 $a$ 和 $b$ 可能很大，但我总能通过这两个数的**线性组合**（加减乘除）凑出它们的最大公约数。

#### 3. 它的主要用途

- **求解线性同余方程**：$ax \equiv b \pmod m$。

```cpp
// 求解 ax ≡ b (mod m)
// 返回所有解（模 m 意义下），若无解返回空 vector
vector<int64> linear_congruence(int64 a, int64 b, int64 m) {
    // 统一到 [0, m-1]
    a %= m; if (a < 0) a += m;
    b %= m; if (b < 0) b += m;

    int64 x, y;
    int64 g = exgcd(a, m, x, y); // ax + my = g

    // 无解：g 不整除 b
    if (b % g != 0) return {};

    // 先求一个特解
    // a/g * x0 ≡ b/g (mod m/g)
    int64 a1 = a / g, b1 = b / g, m1 = m / g;

    // x 是 a 在模 m 下的一个“逆元系数”，特解为 x * (b/g)
    __int128 t = (__int128)x * b1; // 防溢出
    int64 x0 = (int64)(t % m1);
    if (x0 < 0) x0 += m1;

    // 一共有 g 个解：x = x0 + k*m1 (mod m), k = 0..g-1
    vector<int64> res;
    res.reserve(g);
    for (int64 k = 0; k < g; k++) {
        res.push_back(x0 + k * m1);
    }
    return res;
}

ll solve_linear_congruence(ll a, ll b, ll m) {
    ll x, y;
    ll g = exgcd(a, m, x, y);
    if (b % g != 0) return -1;
    
    // 方程化为: (a/g) * x = (b/g) (mod m/g)
    // x = (b/g) * inv(a/g, m/g)
    
    // a0​x ≡ b0(mod m0)
    // x ≡ b0​⋅a0−1(mod m0)
    ll m0 = m / g;
    ll a0 = a / g;
    ll b0 = b / g;
    
    // 求 a0 关于 m0 的逆元
    // 由于 gcd(a0, m0) = 1，逆元存在
    ll inv_a0, temp_y;
    exgcd(a0, m0, inv_a0, temp_y);
    inv_a0 = (inv_a0 % m0 + m0) % m0;
    
    return (__int128)b0 * inv_a0 % m0;
}
```

- **求模逆元**：当 $ax \equiv 1 \pmod m$ 时，求 $x$。这是组合数学计算（如求组合数取模）中的前置技能。

```cpp
i64 inva, _;

// or
pair<i64, i64> invGcd(i64 a, i64 b) {
    a = safeMod(a, b);
    if(a == 0) {
        return {b, 0};
    }

    i64 s = b, t = a;
    i64 m0 = 0, m1 = 1;

    while (t) {
        i64 u = s / t;
        s -= t * u;
        m0 -= m1 * u;

        std::swap(s, t);
        std::swap(m0, m1);
    }

    if(m0 < 0) {
        m0 += b / s;
    }
    return {s, m0};
}
```

### 二、 核心逻辑推导：为什么代码长那样？

这是你在考场上默写模板时**不会卡住的关键**。不要死记代码，要记这个**推导过程**（只需要三行公式）。

#### 1. 递归的思维

exgcd 是基于普通 gcd 的递归逻辑的。

- **当前层**：我们要解决 $ax + by = \gcd(a, b)$。

- **下一层**：根据欧几里得算法，$\gcd(a, b) = \gcd(b, a \% b)$。

  所以下一层的问题变成了：求 $bx' + (a \% b)y' = \gcd(b, a \% b)$。

#### 2. 状态的回溯（也就是代码中 update 的逻辑）

假设我们已经通过递归，算出了下一层的解 $x'$ 和 $y'$，即满足：

$$b x' + (a \% b) y' = \gcd(a, b)$$

我们现在的任务是：**利用 $x', y'$ 推导出当前层的 $x, y$。**

**推导步骤（考场卡住时，在草稿纸上划两下）：**

1. 把**模运算展开**：$a \% b = a - \lfloor \frac{a}{b} \rfloor \cdot b$

2. 代入下一层的方程：

   $$b x' + (a - \lfloor \frac{a}{b} \rfloor \cdot b) y' = \gcd(a, b)$$

3. 按 $a$ 和 $b$ 整理系数（把含 $a$ 的放在一起，含 $b$ 的放在一起）：

   $$a(y') + b(x' - \lfloor \frac{a}{b} \rfloor \cdot y') = \gcd(a, b)$$

4. 对比我们要解的目标方程 $ax + by = \gcd(a, b)$，系数对应相等：

   - **$x = y'$**
   - **$y = x' - \lfloor \frac{a}{b} \rfloor \cdot y'$**

这就是代码中递归返回后，更新 $x$ 和 $y$ 的核心逻辑！

### 三、 算法模板与实现细节

根据上面的推导，我们可以写出标准模板。

#### 1. 标准 C++ 模板

C++

```cpp
// 函数返回值为 gcd(a, b)
// x 和 y 通过引用传回，即求出的特解
int exgcd(int a, int b, int &x, int &y) {
    // 【Base Case】: 当 b = 0 时
    if (b == 0) {
        x = 1;
        y = 0;
        return a; // 此时 gcd(a, 0) = a
    }
    
    // 【Recursive Step】: 递归求解下一层
    // 注意：这里的 d 是最大公约数，它在递归过程中是不变的
    int d = exgcd(b, a % b, x, y);
    
    // 【Update Logic】: 利用下一层的 x, y 更新当前层的 x, y
    // 此时的 x 是上一层的 x'，此时的 y 是上一层的 y'
    // 根据公式：x_new = y_old, y_new = x_old - (a/b) * y_old
    // 因为是引用传递，此时变量 x 存储的是 x'，变量 y 存储的是 y'
    
    int z = x; // 暂存 x'
    x = y;     // 当前 x 变成 y'
    y = z - (a / b) * y; // 当前 y 变成 x' - (a/b) * y'
    
    return d;
}
```

#### 2. 考场简写版（常用 Trick）

在实际竞赛中，为了写得更快，我们通常利用引用的特性来缩短代码。

**观察：** 这里的 `y` 实际上就是下一层的 `x` 更新过来的，逻辑可以简化。

如果我们调用 `exgcd(b, a % b, y, x)` （注意这里我交换了 parameter 中的 x 和 y），那么递归回来后：

- 变量 `y` 存着下一层的 $x'$
- 变量 `x` 存着下一层的 $y'$

代码就会变得非常短：

C++

```cpp
int exgcd(int a, int b, int &x, int &y) {
    if (!b) {
        x = 1; y = 0; // 边界：a*1 + 0*0 = a
        return a;
    }
    int d = exgcd(b, a % b, y, x); // ！！注意这里 x 和 y 换了位置
    // 此时 x 存的是 y', y 存的是 x'
    // 根据推导：当前 x 应该是 y' (已经满足，不需要动)
    // 当前 y 应该是 x' - (a/b)*y' -> 即 y -= (a/b) * x
    y -= (a / b) * x; 
    return d;
}
```

*注：如果不习惯交换参数的写法，建议背诵第一种标准写法，更符合直觉，不容易错。*

### 四、 考场记忆与防坑指南

为了防止你在考场上默写时卡住，请记住以下几个关键点：

1. **边界条件（Base Case）**：

   - **何时停止？** 当 `b == 0` 时。
   - **此时解是多少？** 此时方程是 $a \cdot x + 0 \cdot y = a$。显然 $x=1, y=0$。这是递归的起点。

2. **递归后的更新（Update）**：

   - 如果你忘记了公式，只需要在草稿纸上写下 $bx' + (a\%b)y' = ax + by$，然后把 $a\%b$ 展开即可。这是**最保险**的方法。

3. **通解与最小正整数解（非常重要）**：

   exgcd 求出的 $x, y$ 只是**一组特解**。

   - 通解公式：

     $$X = x + k \cdot \frac{b}{\gcd(a, b)}$$

     $$Y = y - k \cdot \frac{a}{\gcd(a, b)}$$

   - **考点**：题目通常要求输出 **$x$ 的最小正整数解**。

     模板代码求出的 $x$ 可能是负数。处理方法如下：

     ```cpp
     i64 d = exgcd(a, b, x, y);
     
     i64 mx = b / d, my = a / d;
     x = (x % mod + mod) % mod;
     y = (d - x * a) / b;
     ```

### 总结：如何建立 exgcd 的心理表征？

当你下次看到 **exgcd** 时，你的思维路径应该是：

1. **目的**：解方程 $ax+by=g$。
2. **手段**：递归碾转相除，到底部 ($b=0$) 后，回溯时顺便把 $x, y$ 算出来。
3. **核心变换**：$x$ 继承 $y'$， $y$ 变成 $x' - \lfloor a/b \rfloor y'$。
4. **坑点**：算出来的是特解，可能为负，记得根据题目调整为最小正整数。

### 2. “上帝视角”之外的思维复盘 (Zero-to-One Derivation)

假如我是第一次看到这道题，我的思维链路应该是这样的：

**阶段一：模型抽象（从文字到数学）**

1. **捕获关键条件**：

   - 总长度 $x$（定值）。
   - 早苗 $a$（系数1），灵梦 $s$（系数2）。
   - “恰好处于同一坐标” -> 两人的位移之和等于总长 $x$（注意方向是相对的）。
   - 时间最短 -> 两人是并行的，时间取决于移动次数多的人。即求 $\min(\max(c_1, c_2))$。

2. **建立方程**：

   $$a \cdot c_1 + s \cdot c_2 = x$$

   约束：$c_1, c_2 \ge 0, c_1, c_2 \in \mathbb{Z}$。

**阶段二：解的存在性与通解（工具选择）**

1. 看到 $Ax + By = C$，条件反射想到 **Bézout's identity（裴蜀定理）** 和 **exgcd**。

2. **判无解**：如果 $x \% \gcd(a, s) \neq 0$，直接输出 No。

3. **求特解**：用 `exgcd(a, b, x, y)` 求出一组特解 $(x, y)$，但这组解可能是负数，且不唯一。

4. **构造通解**：

   令 $g = \gcd(a, b)$，通解形式为：

   $$c_1 = x_0 \cdot \frac{x}{g} + k \cdot \frac{s}{g}$$

   $$c_2 = y_0 \cdot \frac{x}{g} - k \cdot \frac{a}{g}$$

   为了方便，设 $dx = b/g, dy = a/g$。

   则 $c_1(k) = X_0 + k \cdot dx$，$c_2(k) = Y_0 - k \cdot dy$。

**阶段三：最值优化（核心难点突破）**

1. **观察变量趋势**：随着 $k$ 的增加，$c_1$ 单调递增，$c_2$ 单调递减。
2. **确定目标**：我们要让 $\max(c_1, c_2)$ 最小。
   - 若 $c_1$ 很大，$c_2$ 很小，$\max$ 取决于 $c_1$。
   - 若 $c_1$ 很小，$c_2$ 很大，$\max$ 取决于 $c_2$。
   - **直觉告诉我们**：当 $c_1$ 和 $c_2$ **最接近**的时候，$\max(c_1, c_2)$ 往往取到最小值。这就像两条直线交叉，交点处的纵坐标最小（对于 $\max$ 函数而言）。
3. **确定 $k$ 的搜索范围**：
   - 首先必须保证 $c_1 \ge 0$ 且 $c_2 \ge 0$。这确定了一个 $k$ 的合法区间 $[k_{min}, k_{max}]$。
   - 在这个区间内，我们要找一个 $k$，使得 $c_1 \approx c_2$。
   - 令 $X_0 + k \cdot dx = Y_0 - k \cdot dy \Rightarrow k \cdot (dx + dy) = Y_0 - X_0 \Rightarrow k \approx \frac{Y_0 - X_0}{dx + dy}$。

**阶段四：特异化修改（算法落地）**

- **问题**：直接解不等式求范围太繁琐，容易写错边界。
- **策略调整（这也是你代码中的策略）**：
  1. 先找到 $c_1$ 的**最小非负值**。这是 exgcd 的标准操作：`c1 = (c1 % dx + dx) % dx`。此时 $k$ 被固定，$c_1$ 最小，对应的 $c_2$ 是最大可能值。
  2. 如果此时 $c_2 < 0$，说明连 $c_1$ 最小的时候 $c_2$ 都不够减，无解（或者说合法区间为空）。
  3. 此时我们处于 $c_1$ 极小、$c_2$ 极大的状态。
  4. 如果 $c_1 \ge c_2$，说明 $c_1$ 已经是主导因素且无法更小了，直接输出。
  5. 如果 $c_1 < c_2$，说明我们需要增大 $c_1$（同时减小 $c_2$）来让它们接近。我们要移动 $k$ 步，使得差距缩小。
  6. 这就是你代码中 `t = (y - x) / (mx + my)` 的由来！你不需要二分，因为这是线性的，直接算出大概还需要走几步，然后微调（检查 $t$ 和 $t+1$）即可。

### 3. 通用解题方法论 (Methodology for Unknown Problems)

针对这类“构造/数论+最值”题目，建议建立以下思考流程：

1. **变量分离与方程化**：

   不要被题目背景（早苗、灵梦、移动、停止）干扰。

   *动作* $\rightarrow$ *变量*。

   *限制* $\rightarrow$ *方程/不等式*。

   *目标* $\rightarrow$ *函数表达式*。

2. **解空间的几何形态分析**：

   对于 $Ax + By = C$：

   - 解都在一条直线上。
   - 非负约束 ($x \ge 0, y \ge 0$) 截取了直线的一段线段。
   - 目标函数 $\max(x, y)$ 的等高线是 L 型（或正方形边界）。
   - **思考核心**：我们在合法的线段上，找一个离直线 $y=x$ 最近的点。

3. **归一化起手式**：

   处理不定方程最值问题，不要试图一下子求出最优解。

   **First valid, then optimal.**

   - 先求出一个特解。
   - 利用取模技巧，将其中一个变量（通常是 $x$）归一化到最小非负整数状态。这相当于找到了解集的“左端点”。
   - 然后从左端点向右（增加 $x$，减小 $y$）单向调整。这样可以将复杂的双向搜索变为单向判定。

4. **防溢出意识（金牌选手素质）**：

   - 中间计算过程（如 `x * p` 或 `d - a * x`）极易爆 `long long`。
   - **条件反射**：涉及 $10^{18}$ 的乘法，必须使用 `__int128`。你的代码中用得很好，这是区分普通选手和高阶选手的细节。

### 4. 顶尖选手的编码流 (Coding Flow)

当你的脑海中已经有了上述思路，编写代码时应该遵循这种**肌肉记忆**般的流程，以防卡壳：

1. **模板区 (Library)**：
   - 不假思索地写下 `exgcd`。如果是手写，确保 `x, y` 的更新逻辑正确。
2. **预处理与特判 (Pre-check)**：
   - 读入 $d, a, b$。
   - 计算 `g = gcd(a, b)`。
   - `if (d % g)` -> `No`。
   - 注意：此时顺手把 $a, b, d$ 都除以 $g$，简化系数，防止后续计算溢出风险，也省去了后面反复除 $g$ 的麻烦。（你的代码是在后面除的，也可以，但在前面除更清爽）。
3. **基准解构造 (Canonicalization)**：
   - 调用 `exgcd` 得到 $x_0, y_0$。
   - **放大**：`x *= (d/g)`。此时 `x` 是原方程的解。
   - **取模标准化**：计算步长 `mx = b/g`, `my = a/g`。
   - **关键行**：`x = (x % mx + mx) % mx;` -> 得到最小非负 $x$。
   - **回代求y**：`y = (d - a*x) / b;` -> 算出对应的 $y$。**注意不要用 exgcd 的 y 去推，直接用原方程回代最稳。**
4. **逻辑分支 (Decision Logic)**：
   - `if (y < 0)` -> 无解（因为 $x$ 已经是最小了，$y$ 还是负的，没救了）。
   - `if (x >= y)` -> 最优解就是当前（因为 $x$ 增加 $y$ 只会更小，差距更大）。
   - `else` ($y > x$) -> 需要平衡。
     - 计算步数：`step = (y - x) / (mx + my)`。
     - 应用步数：`x += step * mx; y -= step * my;`
     - **邻域检查**：因为整数除法向下取整，可能再多走一步更优，或者不走更优。简单的方法是：算完后，比较当前状态和“再走一步”的状态，取优者。或者像你代码里那样写判断条件。

**自我问答 Check List (编码中途)**：

- Q: `exgcd` 求出的 $x, y$ 对应的是 $ax+by=g$ 还是 $ax+by=d$？
  - A: 是 $g$！记得乘倍数。
- Q: 取模后 $x$ 变为正数，方程还成立吗？
  - A: $x$ 变了，$y$ 必须重算。不要偷懒直接用原来的 $y$ 做变换，容易错。
- Q: `__int128` 用了吗？
  - A: 涉及乘法的地方都套上。
