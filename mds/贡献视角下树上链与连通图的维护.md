## 联通子树维护与换根

[D-数字积木_2026牛客寒假算法基础集训营2](https://ac.nowcoder.com/acm/contest/120562/D)

> 与常见的双端点树链结构的计数不同，联通子树可以存在多个端点，这也代表着在计算当前节点为根的子树中有多少联通子树时，当前根节点的每个子树带给计数的贡献都是独立的，只需要利用乘法原理，所以$dp[u] = \prod dp[v]+1$（最后的加一代表空集），相应地，要删除某个子树的贡献时，则将它除掉。

### 1. 题目归类与本质透视

- **一级标签**：**树上计数 (Tree Counting)**、**组合数学 (Combinatorics)**
- **二级标签**：**贡献法 (Contribution Technique)**、**MEX性质应用**
- **核心模型**：**斯坦纳树 (Steiner Tree) 上的挂载统计**

**本质解析**：

这道题的本质不是求“MEX”，而是求“包含集合 $S$ 的最小连通块（骨架）及其扩展方案数”。

MEX 只是一个幌子，用来引导你把问题转化为**单调递增的集合包含问题**。所有的连通子图计数问题，最终都会归结为：确定一个“骨架”（必须选的点），然后计算骨架周围有多少个“挂件”（可选的子树）可以自由组合。

### 2. 上帝视角移除：从零开始的推导复盘

假设我现在没看过题解，只看到了题目：求 $\sum \text{MEX}(S)$。

#### 第一步：转化目标（破题口）

- **直觉**：直接枚举子图 $S$ 算 $\text{MEX}$ 是不可能的。$\text{MEX}$ 这个函数很烦，它依赖于“缺失的最小值”。

- **数学直觉**：在计算 $\sum \text{MEX}$ 或 $\sum \text{Expected Value}$ 时，标准的转化公式是：

  $$\sum \text{MEX}(S) = \sum_{k=1}^{n} P(\text{MEX}(S) \ge k)$$

  其中 $P(\text{condition})$ 是满足条件的子图数量。

- **条件翻译**：$\text{MEX}(S) \ge k$ 意味着什么？意味着子图 $S$ **必须包含** $\{0, 1, 2, \dots, k-1\}$ 这 $k$ 个点。

- **新目标**：算出包含 $\{0, \dots, k-1\}$ 的连通子图有多少个，记为 $Cnt_k$，答案就是 $\sum Cnt_k$。

#### 第二步：几何直觉（寻找不变性）

- **思考**：在树上，包含点集 $\{0, \dots, k-1\}$ 的最小连通子图是什么？
- **结论**：是这些点在树上的**斯坦纳树**（即连接这些点的最小路径并集）。我们把它称为**“骨架”**。
- **推论**：任何一个合法的子图，必须完全包含这个“骨架”。
  - 骨架上的点：**必选**。
  - 骨架旁边的点（直接相邻但不在骨架上）：**可选**（即作为该分支子树的根，选了它就要继续算它的子树，不选它则该分支截止）。

#### 第三步：算法设计（从静态到动态）

- **静态思考**：如果只问 $k=1$（只包含0号点），答案是多少？
  - 这是一个经典的树形DP。$dp[u] = \prod (dp[v] + 1)$（这里的+1代表空集）。
  - 既然0号点必选，那答案就是以0为根，所有子树方案的乘积。
- **动态思考**：从 $k$ 变到 $k+1$，我们需要加入点 $k$。
  - 如果点 $k$ 已经在当前的骨架里了，骨架不变，$Cnt_{k+1} = Cnt_k$。
  - 如果点 $k$ 不在骨架里，我们需要把从 $k$ 到当前骨架的路径打通，加入骨架。
- **贡献维护**：
  - 当一个点 $u$ 从“非骨架”变成“骨架”时，发生了什么？
  - **变化前**：它作为父节点的一个分支，贡献是 $dp[u]$（包含“不选u”这一选项）。
  - **变化后**：它必须被选中。此时它不再提供“不选”的选项，它的贡献变成了 $\prod dp[v]$（其中 $v$ 是它不在骨架路径上的子节点）。
  - **数学关系**：注意到 $dp[u] = \prod dp[v] + 1$，所以 $\prod dp[v] = dp[u] - 1$。
  - **操作**：把原来的贡献 $dp[u]$ 除掉，乘上新的贡献 $dp[u]-1$。

#### 第四步：成型

- 我们需要一个游标，维护当前的乘积 $Ans$。
- 每次加入点 $x$，沿父节点向上跳，直到遇到已访问过的点。
- 沿途所有点：`Ans = Ans / dp[curr] * (dp[curr] - 1)`。
- 加上处理模数0的技巧。

### 4. 通用解题方法论（ICPC 金牌向）

遇到陌生难题，请遵循 **STR (Simplify, Transform, Reduce)** 流程：

#### 3. Reduce Complexity (降维/增量)

## 树链维护与LCA或换根

[F - Sum of Mex](https://atcoder.jp/contests/abc438/tasks/abc438_f)

这是一个非常棒的题目，你的代码写得很扎实，逻辑也很清晰（利用 BFS 预处理 LCA，利用两端点维护链，利用容斥或者方向判断计算子树大小）。

针对你的需求，我将从题目归类、思维推导、通用方法论、编码流以及换根思考这几个维度进行深度的剖析。

> **当题目要求的连通块/路径/集合 永远包含某个固定点 $R$ 时：**
>
> 1. **无脑定根**：直接以 $R$ 为整棵树的根。
> 2. **分支视角**：不要再把连通块看作“路径”，而要看作**“从 $R$ 射出的若干条射线”**。
> 3. **染色法代LCA**：利用 `vis` 数组和向父节点回溯的方式，来代替 LCA 和路径判断。因为所有路径最终都要汇聚到 $R$，回溯必然会相交。

------

### 1. 题目归类与本质透视

- **一级标签**：**树上计数 (Tree Counting)**、**树形 DP / 构造**
- **二级标签**：**贡献法 (Contribution Technique)**、**MEX 性质**、**链的动态维护 (Dynamic Chain Maintenance)**
- **核心模型**：**$\sum \text{MEX}$ 的转化模型**

**本质解析**：

这道题披着 MEX 的外衣，实际上考查的是**树上点集的拓扑结构**。

核心公式转化：

$$\sum \text{MEX}(S) = \sum_{k \ge 1} \text{Count}(\text{MEX}(S) \ge k)$$

$$\text{MEX}(S) \ge k \iff \{0, 1, \dots, k-1\} \subseteq S$$

在树上，如果一组点 $\{0, 1, \dots, k-1\}$ 能够同时被某条简单路径包含，那么这组点本身在树上必须构成一条**链（Chain）**。

因此，题目本质是：**随着 $k$ 的增大，维护包含 $\{0, \dots, k-1\}$ 的最小链的两个端点 $(u, v)$，并统计有多少对 $(s, t)$ 的路径覆盖了这条链。**

------

### 2. 脱离上帝视角：从零开始的思维复盘

假设我在考场上，没有看过题解，我会如何思考：

**Step 1: 捕捉痛点与转化 (Simplify)**

- **痛点**：直接枚举所有点对 $(i, j)$ 需要 $O(N^2)$，再求 MEX 又是 $O(N)$，总复杂度 $O(N^3)$，即使优化也是 $O(N^2)$，显然不行。我们需要 $O(N \log N)$ 或 $O(N)$。
- **联想**：遇到 $\sum \text{MEX}$ 或者 $\sum \text{Expected Value}$，第一反应必须是**拆解贡献**。
- **转化**：
  - 与其问“这个路径的 MEX 是多少”，不如问“MEX 至少为 1 的路径有多少？至少为 2 的有多少？... 至少为 $k$ 的有多少？”
  - 令 $cnt[k]$ 为“路径包含节点 $0, 1, \dots, k-1$ 的点对数量”。
  - 答案 $= \sum_{k=1}^{N} cnt[k]$。

**Step 2: 几何直觉与结构特征 (Visualize)**

- **思考**：什么样的路径会包含 $\{0, 1, \dots, k-1\}$？
- **观察**：
  - $k=1$: 必须包含 $\{0\}$。
  - $k=2$: 必须包含 $\{0, 1\}$。
  - $k=3$: 必须包含 $\{0, 1, 2\}$。
- **关键结论**：如果点集 $\{0, \dots, k-1\}$ 能被一条路径覆盖，那么这 $k$ 个点在树上的分布必须也是一条“线型”的（即它们构成的最小连通块是一条链）。如果它们分叉了（比如 0 在中间，1 和 2 分别在两个分支），那么就不存在一条简单路径能同时穿过它们。

**Step 3: 动态维护与算法设计 (Incremental)**

- **状态设计**：既然这 $k$ 个点必须构成一条链，我不需要记录所有点，只需要记录这条**核心链的两个端点** $(u, v)$。
- **状态转移**：
  - 假设我知道了包含 $\{0, \dots, k-1\}$ 的核心链端点是 $(u, v)$。
  - 现在加入点 $k$。
  - **Case A**: $k$ 已经在路径 $(u, v)$ 上 $\rightarrow$ 端点不变，核心链仍是 $(u, v)$。
  - **Case B**: $k$ 不在路径上，但可以通过延伸 $u$ 或 $v$ 到达 $k$（且不经过其他叉路） $\rightarrow$ 更新端点为 $(k, v)$ 或 $(u, k)$。
  - **Case C**: $k$ 在某个分叉上，使得这 $k+1$ 个点无法共线 $\rightarrow$ 游戏结束，后续所有 $cnt$ 均为 0，直接 Break。

**Step 4: 计数计算 (Counting)**

- **问题**：给定核心链 $(u, v)$，有多少对外部点 $(S, E)$ 的路径经过这段链？
- **解法**：$S$ 必须在 $u$ 的“外侧”子树中，$E$ 必须在 $v$ 的“外侧”子树中。
- **公式**：$Size(u\_part) \times Size(v\_part)$。这里需要利用 LCA 和子树大小进行容斥计算。

### 3. 通用解题方法论 (Gold Medal Mindset)

针对此类“树上计数/路径统计”问题，建立以下 CheckList：

1. **贡献视角转化**：
   - 看到 $\sum f(path)$，先想能不能转化为 $\sum P(\text{condition})$。
   - 常见的 Condition 包括：路径经过某点、路径经过某边、路径包含集合 $S$。
2. **拓扑结构分析**：
   - 涉及“点集”和“路径”的关系时，思考**最小覆盖结构**。
   - 两点 $\to$ 路径。
   - 三点 $\to$ 一个“Y”字形中心（或者共线）。
   - 多点共线 $\to$ 维护端点。
3. **增量维护 (Incremental Approach)**：
   - 不要试图一次性算出 $f(i, j)$。
   - 从小到大加入元素（本题是节点编号），看结构如何变化。
   - 这种方法通常能把 $O(N^2)$ 降维到 $O(N)$。
4. **树上计算工具箱**：
   - 判断点 $z$ 是否在 $x-y$ 路径上：`dist(x, z) + dist(z, y) == dist(x, y)`。
   - 计算路径覆盖方案数：`size[u] * size[v]`（注意处理父子关系，如果是直系祖先，由于 `size` 数组通常只存子树大小，另一侧的大小需要用 $N - size[child]$ 计算）。

我之前的思维定势在于通用的“树上链维护”，即链可以在树的任意位置。

但你抓住了一个本题的致命弱点：**MEX 集合 $\{0, 1, \dots, k\}$ 永远包含 0。**

- **通用视角**：链是 $u \leftrightarrow v$，0 只是链上一点。
- **你的新视角**：既然 0 是根且 0 必在链上，那么这条链只可能是以下两种形态之一：
  1. **直链**：$0 \leftrightarrow u$（0 是端点）。
  2. **折线**：$u \leftrightarrow 0 \leftrightarrow v$（0 是拐点，链由 0 的两个不同子树中的分支拼接而成）。

这使得我们只需要维护两个“触手”端点 `l` 和 `r`，且这两个触手必然是从 0 向下延伸的。

#### 5.2. 代码逻辑解析（为什么它是线性的？）

这份代码最精彩的地方在于去掉了所有的 LCA 和倍增数组，仅靠 `vis` 数组和 `parent` 指针就完成了维护。

- **平摊复杂度分析 ($O(N)$)**：

  代码中有一个 `while(x >= 1 && !vis[x])` 循环。

  看起来是两重循环，但注意 `vis[x] = true`。每条边、每个节点只会被“向上跳”并标记一次。一旦标记过，下次遇到就停止。

  **这正是并查集或 Tarjan LCA 中常用的着色思想。** 这种平摊分析使得整个过程是严格线性的。

- **省去 LCA 与 Jump**：

  - **判断是否在链上**：通用做法需要 `dist` 公式。你的做法：直接往上跳，跳到了 `vis` 过的点。如果这个点是 `l` 或 `r`（或者 0），说明接上了；否则说明分叉了。
  - **计算 Size**：
    - 通用做法：需要 `jump` 找次级祖先来做容斥。
    - 你的做法：在向上跳的过程中，特判了 `if(p[x] == 0) v0 = x;`。因为是从 0 向下延伸的，直接记录了 $x$ 是 0 的哪个儿子的后代。这直接锁定了需要的子树大小。

#### 5.3. 重写结论：以0为根的降维打击

基于你的代码，关于“以0为根”的结论修正如下：

#### **结论：对于“包含根节点”的连通块增长问题，以根为根是降维级优化。**

1. **数据结构的降维 ($O(N \log N) \to O(N)$)**：
   - **旧结论**：还需要 LCA，只是少一点判断。
   - **新结论**：**完全不需要 LCA 和倍增表**。因为链一定经过根，所以任意一点到链的距离判定、连接判定，都可以通过简单的“向上爬树 + 染色”解决。所有的拓扑关系都退化成了“是否在同一棵子树”或“父子关系”。
2. **状态维护的简化**：
   - **旧结论**：维护端点 $(u, v)$，需要分类讨论 LCA 是 $u$ 还是 $v$ 还是其他。
   - **新结论**：维护的不再是抽象的端点，而是**“左臂”和“右臂”**。
     - 状态仅需：`l`（左臂末端），`r`（右臂末端）。
     - 核心判断简化为：新节点 $x$ 是接在 `l` 下面？接在 `r` 下面？还是在第三个分支（非法）？
     - 这种**分支归属判断**比**路径距离判断**要快得多。
3. **计算公式的统一**：
   - 利用 `v0`（0 的直系子节点）极其巧妙地解决了容斥问题。
   - 当 $l=0$ 时（单链），方案数是 `(siz[0] - siz[v0]) * siz[r]`。这里的 `siz[0]` 就是 $N$。逻辑极其直观：除了 $r$ 所在的那一枝（`v0`），其他所有点都可以作为路径的一端（包括 0 的其他子树、0 本身）。

#### 5.4. 这种解法的适用性与“方法论”更新

你的这个解法给我的通用方法论增加了一条非常重要的**“特殊点定根法则”**：

**更新后的方法论条目：**

> **当题目要求的连通块/路径/集合 永远包含某个固定点 $R$ 时：**
>
> 1. **无脑定根**：直接以 $R$ 为整棵树的根。
> 2. **分支视角**：不要再把连通块看作“路径”，而要看作**“从 $R$ 射出的若干条射线”**。
> 3. **染色法代LCA**：利用 `vis` 数组和向父节点回溯的方式，来代替 LCA 和路径判断。因为所有路径最终都要汇聚到 $R$，回溯必然会相交。
