这是一个非常深刻且适合 ACM 竞赛进阶的话题，你肯定遇到过那种求和符号套了三四层、数据范围在 $10^7$ 甚至 $10^9$ 的数论题。这类题目如果硬推导，往往会发现自己在做“去重”的工作，这其实就是**容斥原理**。

而**莫比乌斯反演（Möbius Inversion）**，本质上就是把容斥原理中“奇加偶减”的系数（+1, -1, 0）封装到了一个函数 $\mu(n)$ 中，使得我们可以通过**代数变换**（而不是画韦恩图）来“自动化”地处理复杂的去重逻辑。

下面我从**核心定义**、**与容斥的本质联系**以及**实战中的 GCD 求和**三个方面为你详细拆解。

------

### 1. 最核心本质

> **莫比乌斯反演 = 在“约数包含关系”的偏序结构上做“去前缀和”（反卷积）**
> 在整除偏序上把“约数和 / 倍数和”反过来还原原函数

换成更直观的话：

- 你有一个函数 **F(n)**
- 它不是原始信息，而是**把原始信息 f(d) 按 d|n 汇总起来之后的结果**
- 你现在想从这些“汇总后的值”里把“原始值 f(n)”还原出来
- **莫比乌斯反演就是这个还原公式**

这就是它存在的意义：
✅ **从“按约数累计/按倍数累计”的信息中恢复单点值。**

### 2. 它解决的典型问题长什么样？（看到就要条件反射）

你一看到下面这种式子，就要想到莫反：

#### 模板 1：约数求和型


按约数累加：$$F(n)=\sum_{d\mid n} f(d)$$

其反演为：$$f(n)=\sum_{d\mid n}\mu(d)F\left(\frac{n}{d}\right)$$

#### 模板 2：倍数求和型


按倍数累加：$$G(n)=\sum_{k\ge 1} f(kn)$$

其反演公式为：$$f(n)=\sum_{k\ge1}\mu(k)G(kn)$$



或者我们按照模板1的形式定义关系：$$F(n) = \sum_{n|d} f(d)$$

其反演公式为：$$f(n) = \sum_{n|d} \mu(\frac{d}{n}) F(d)$$

这里的 $F(n)$ 通常好算，表示“**是 $n$ 的倍数**的所有情况之和”。

这里的 $f(n)$ 通常难算，表示“**恰好等于 $n$** 的情况”。

#### 它是如何体现容斥的？

- **$借助莫比乌斯函数 \mu$ **：
  - $\mu(1)F(n)$：加上 $1 \times$ (是 $n$ 的倍数)。
  - $\mu(2)F(2n)$：减去 $1 \times$ (是 $2n$ 的倍数)。
  - $\mu(3)F(3n)$：减去 $1 \times$ (是 $3n$ 的倍数)。
  - ...
  - $\mu(6)F(6n)$：加上 $1 \times$ (是 $6n$ 的倍数)...（因为在减 2 和 3 的时候，6 被多减了，要补回来）。

这就是一个自动化的、代数化的容斥过程。

### 3. 为什么需要它？（为了拆掉“重复计数”）

> 统计容易得到的是宽松条件（累计）
> 想要严格条件（精确值）就必须反演剥离重复贡献

很多问题的本质是：

- 我们“容易统计”某类对象的 **宽松条件**
- 但我们真正想要的是 **严格条件**

宽松统计经常会出现：$$\text{宽松统计} = \sum_{d\mid n} \text{严格统计}(d)$$

也就是严格信息被“叠加”成了约数和，于是你就需要一个“反操作”把它剥离出来，而莫比乌斯反演就是这个“剥离器”。

### 4. 核心工具：莫比乌斯函数 $\mu(n)$

莫比乌斯反演的“灵魂”在于 $\mu(n)$ 这个函数。它的定义直接对应了容斥原理中的系数：

- **$\mu(1) = 1$**
- **$\mu(n) = (-1)^k$**：如果 $n$ 等于 $k$ 个**不同**质数的乘积（即 $n$ 是无平方因子数）。
  - 例如：$\mu(2) = -1$ （1个质因子），$\mu(6) = \mu(2\times3) = 1$ （2个质因子）。
- **$\mu(n) = 0$**：如果 $n$ 含有平方因子（例如 $4, 12, 18$ 等）。

#### 为什么这么定义？（关键性质）

因为它满足一个莫比乌斯反演成立的基石恒等式：

$$\sum_{d|n} \mu(d) = \begin{cases}  1, & n = 1 \\  0, & n > 1  \end{cases}$$

**直观理解：** μ 在约数求和意义下扮演单位的角色，这就是容斥原理的代数表达。

#### 证明：

根据算术基本定理，我们可以把 $n$ 分解质因数：

$$n = p_1^{a_1} p_2^{a_2} \dots p_k^{a_k}$$

其中 $p_i$ 是不同的质数，$k \ge 1$（因为 $n > 1$）。

我们要计算 $\sum_{d|n} \mu(d)$。

注意 $\mu(d)$ 的定义：**如果 $d$ 含有平方因子（即某个质数的指数 $\ge 2$），则 $\mu(d) = 0$。**

这意味着，在求和过程中，我们可以**直接忽略**那些含有平方因子的 $d$（比如 $p_1^2, p_1^2 p_2$ 等），因为它们对总和的贡献是 0。

**真正有贡献的 $d$ 只能由 $p_1, p_2, \dots, p_k$ 中的若干个**不同**质数相乘得到。**

我们可以按照 $d$ 包含的质因子的**个数**来进行分类讨论：

1. **选 0 个质因子：** $d = 1$，$\mu(1) = 1$。
   - 这样的 $d$ 有 $C_k^0$ 个（即 1 个）。
   - 贡献：$C_k^0 \times 1$。
2. **选 1 个质因子：** $d = p_i$。
   - 这样的 $d$ 有 $C_k^1$ 个。
   - 因为 $d$ 有 1 个质因子，$\mu(d) = (-1)^1 = -1$。
   - 贡献：$C_k^1 \times (-1)$。
3. **选 2 个质因子：** $d = p_i p_j$。
   - 这样的 $d$ 有 $C_k^2$ 个。
   - 因为 $d$ 有 2 个质因子，$\mu(d) = (-1)^2 = 1$。
   - 贡献：$C_k^2 \times 1$。
4. **选 $m$ 个质因子：**
   - 这样的 $d$ 有 $C_k^m$ 个。
   - $\mu(d) = (-1)^m$。
   - 贡献：$C_k^m \times (-1)^m$。

**把所有贡献加起来：**

$$\sum_{d|n} \mu(d) = C_k^0 - C_k^1 + C_k^2 - C_k^3 + \dots + (-1)^k C_k^k$$

你眼熟这个式子吗？

根据**二项式定理** $(x+y)^k = \sum_{i=0}^k C_k^i x^i y^{k-i}$，如果我们令 $x = -1, y = 1$，则有：

$$\sum_{i=0}^k C_k^i (-1)^i (1)^{k-i} = (-1 + 1)^k = 0^k = 0$$

对于 $n > 1$，它的所有因子的 $\mu$ 值加起来刚好抵消。这相当于：$\binom{k}{0} - \binom{k}{1} + \binom{k}{2} - \dots = 0$。

**得证：当 $n > 1$ 时，总和为 0。**

------

### 5 欧拉筛求莫比乌斯函数

下面按“线性筛（Euler 筛）+ Möbius 函数”来把你这段代码的**核心思路**捋清楚，并指出你代码里一个**关键 bug**（会导致很多 `mu` 没被正确赋值）。

------

#### 5.1 Möbius 函数 $$\mu(n)$$ 是什么（算法需要用到的性质）

定义（最常用版本）：

- $$\mu(1)=1$$
- 若 $$n$$ 含有平方因子（存在素数 $$p$$ 使得 $$p^2 \mid n$$），则 $$\mu(n)=0$$
- 若 $$n$$ 是由 $$k$$ 个**互不相同**素数相乘：$$n=p_1p_2\cdots p_k$$，则 $$\mu(n)=(-1)^k$$

所以你只要在筛的过程中知道两件事：

1. $$n$$ 是否出现重复素因子（平方因子） → 变成 0
2. 否则素因子个数奇偶性 → $$\pm 1$$

------

#### 5.2 线性筛的“只筛一次”思路

线性筛维护：

- `primes`：已经找到的素数表
- `minp[x]`：$$x$$ 的最小质因子（0 表示还没被标记过）
- `mu[x]`：$$\mu(x)$$

线性筛的经典框架是：

对于每个 $$i=2..n$$：

- 如果 `minp[i]==0`，说明 $$i$$ 是素数：
  - `minp[i]=i`
  - `mu[i]=-1`（素数只有 1 个素因子）
  - `primes.push_back(i)`
- 然后枚举素数 `p`，令 `x=i*p`：
  - 若 `x>n` 停止
  - 设 `minp[x]=p`
  - 分两种情况：
    1. **p 整除 i**：说明 $$x=i\cdot p$$ 里出现了 $$p^2$$（平方因子）
       $$\Rightarrow \mu(x)=0$$，并且要 `break`（保持“线性”关键点）
    2. **p 不整除 i**：说明 $$x$$ 只是多乘了一个新素数
       $$\Rightarrow \mu(x)=-\mu(i)$$（素因子个数奇偶性翻转）

为什么要 `break`：
当 `p` 是 `i` 的最小质因子时，`i*p` 的最小质因子一定还是 `p`，并且这一对 `(i,p)` 是生成 `i*p` 的**唯一**线性筛路径；继续用更大的素数会让同一个数被重复生成，破坏线性复杂度。

------

#### 5.3 你这段代码对应的逻辑在做什么

你代码的主体：

```cpp
vector<int> minp, primes, mu;
void sieve(int n) {
    minp.assign(n + 1, 0);
    mu.assign(n + 1, 0);

    mu[1] = 1;
    for(int i = 2; i <= n; i ++) {
        if(!minp[i]) {
            mu[i] = -1;
            minp[i] = i;
            primes.emplace_back(i);
        }
        for(auto p : primes) {
            if(p * i > n) break;
            minp[p * i] = p;
            if(i % p == 0) break;
            mu[i * p] = -mu[i];
        }
    }

    for(int i = 1; i <= n; i ++) {
        mu[i] += mu[i - 1];
    }
}
```

- `if(!minp[i])`：判断 $$i$$ 是否为素数 → 是则 `mu[i]=-1`
- 内层枚举 `p`：尝试构造 `i*p`
- `if(i%p==0) break;`：走到了“平方因子”分支并停止（这一点是对的）
- `mu[i*p] = -mu[i]`：走到了“新添一个不同素因子”的分支（也是对的）

------

#### 5.4 末尾前缀和在干什么

```cpp
for(int i = 1; i <= n; i ++) {
    mu[i] += mu[i - 1];
}
```

这一步把 `mu[i]` 从“点值 $$\mu(i)$$”变成了“前缀和”：

$$
\text{mu}[i] = \sum_{k=1}^{i} \mu(k)
$$

很多题里会用到 **Mertens 函数** $$M(n)=\sum_{k\le n}\mu(k)$$，尤其是做整除分块/反演求和时，所以习惯直接把 `mu` 改成前缀形式。

⚠️ 注意：这会覆盖原本的 $$\mu(i)$$。
如果你后面还要用点值 $$\mu(i)$$，就需要另开数组保存，或者最后再做前缀和拷贝。

### 6 实战应用：GCD 求和问题

[P3455 [POI 2007\] ZAP-Queries - 洛谷](https://www.luogu.com.cn/problem/P3455)

这是莫比乌斯反演最经典的“杀手锏”应用。

**题目：** 给定$ (1\le a,b\le n)$，求满足$\gcd(a,b)=1$

（暴力做法是 $O(NM)$，显然无法通过 $10^5$ 以上的数据。）

#### 6.1 直接统计 gcd=1 很难，但统计 “能被 d 整除” 很容易

如果你要求 gcd=1，条件太“精确”，不好算。

但你如果问：有多少对 (a,b) **同时能被 d 整除**？

这就非常好算了，因为：

- (a) 必须是 (d) 的倍数
- (b) 也必须是 (d) 的倍数

在 (1..n) 里，(d) 的倍数个数是$ (\lfloor n/d\rfloor)$

所以：$\#{(a,b): d\mid a,, d\mid b} = \left\lfloor \frac{n}{d}\right\rfloor^2$

这个就是典型的 **“宽松统计”**：它不关心 gcd 是多少，只关心“至少有 d 这个公共因子”。

------

#### 6.2 定义两个量：严格的 & 宽松的（这是莫反的标准建模）

我们定义：

- 严格统计（你真正想要的）：$f(g)={(a,b): \gcd(a,b)=g}$
- 宽松统计（你容易算的）：$F(d)=(a,b): d\mid \gcd(a,b)$，也就是 gcd(a,b) 至少包含一个因子 d。

刚才我们已经算出：$F(d)=\left\lfloor \frac{n}{d}\right\rfloor^2$

------

#### 6.3 宽松统计和严格统计有什么关系？

如果 $(d\mid \gcd(a,b))$，等价于：$\gcd(a,b)$可能是 $d$ 的倍数，比如 $d,2d,3d,\dots$

所以：$F(d)=\sum_{k\ge1} f(kd)$

这就是第二类模板：**宽松统计 = 严格统计按“倍数”累加起来**

------

#### 6.4 莫比乌斯反演：用 μ 把“倍数累加”拆回去

现在你有：$F(d)=\sum_{k\ge1} f(kd)$

那么莫比乌斯反演告诉你：$f(d)=\sum_{k\ge1}\mu(k)F(kd)$

特别地，你要的答案是 ：$f(1)=\sum_{k\ge1}\mu(k)F(k)=\sum_{k=1}^{n}\mu(k)\left\lfloor \frac{n}{k}\right\rfloor^2$

------

#### 6.5. 这到底“去掉了什么重”？（直觉解释：μ 就是整除版容斥）

我们看一个具体对 $(a,b)$。

设 $\gcd(a,b)=g$

那么它在 $F(k)$ 里出现的条件是：只要 $k\mid g$，它就会被计入 $F(k)$

所以在和式$\sum_{k=1}^{n}\mu(k)F(k)$里，这个固定的 $(a,b)$ 的贡献是：$\sum_{k\mid g}\mu(k)$

而我们知道莫比乌斯函数最核心恒等式：$\sum_{k\mid n}\mu(k)=\begin{cases}  1, & n = 1 \\  0, & n > 1  \end{cases}$

所以：

- 如果这对数的 gcd=1，它最终贡献 1 
- 如果 gcd>1，它最终贡献 0 

这就解释了莫反的“魔法”：μ 的加权求和会把 gcd≠1 的全部抵消掉，只留下 gcd=1 的对数。

------

#### 6.6 竞赛里更常见的泛化：给你一个数组/集合，求互质对数

比如给你一个数组$A$，元素值 $\le V$，问有多少对 ((i,j)) 满足 $\gcd(A_i,A_j)=1$。

套路几乎一模一样，只是 $F(d)$ 换成：

- 令 `cnt[d]` = 数组里能被 d 整除的数的个数
  那么同时被 d 整除的 pair 数就是：
- 无序对：$\binom{cnt[d]}{2}$
- 有序对：$cnt[d]^2 - cnt[d]$

于是 gcd=1 的无序对数是：$\sum_{d=1}^{V} \mu(d)\binom{cnt[d]}{2}$

这是你以后做题最常遇到的模板之一。

------

#### 6.7 你应该怎么在脑中“自动播放”这套流程？

我给你一个“触发脚本”，你以后看到 gcd=1 题，脑内直接播放：

1. **目标很精确**：gcd=1（难）
2. 换成 **宽松条件**：能被 d 同时整除（好算）
3. 得到关系：$F(d)=\sum f(kd)$
4. 用 μ 反演：$f(1)=\sum \mu(d)F(d)$
5. 最后变成一个“枚举 d 的求和式”



[P1447 [NOI2010\] 能量采集 - 洛谷](https://www.luogu.com.cn/problem/P1447)

若给定 $N, M$，求 $\sum_{i=1}^N \sum_{j=1}^M \gcd(i, j)$。

#### 最终公式

把所有东西拼回去：

$$\text{Ans} = \sum_{k=1}^{\min(N, M)} k \left( \sum_{d=1}^{\min(\lfloor N/k \rfloor, \lfloor M/k \rfloor)} \mu(d) \lfloor \frac{N}{kd} \rfloor \lfloor \frac{M}{kd} \rfloor \right)$$

此时，我们可以通过**整除分块（数论分块）**技术，将复杂度从 $O(N)$ 降低到 $O(\sqrt{N})$。

------

### 7. 和容斥的关系（让你更“本质地记住”）

**容斥原理**是思想，**莫比乌斯反演**是这一思想在数论函数上的代数化工具。

容斥原理处理集合，莫比乌斯反演处理**整除关系**（偏序集）。

莫比乌斯反演其实就是一种**更抽象、更结构化的容斥**：

- 容斥是在集合包含关系上做反操作
- 莫反是在“约数整除关系”上做反操作

μ 的符号变化，就是“约数结构版的容斥系数”。

所以你要形成一句记忆：

> **莫比乌斯反演 = 对整除关系做容斥**



## 例题2

[P2522 [HAOI2011\] Problem b - 洛谷](https://www.luogu.com.cn/problem/P2522)

这道题是经典的 **莫比乌斯反演（Mobius Inversion）** 结合 **二维数论分块** 的入门题

------

### 1. 题目归类：给知识点打标签

为了建立索引，我们需要把这道题挂在知识树的正确位置：

- **主分类**：**数论 (Number Theory)**
- **核心算法**：**莫比乌斯反演 (Mobius Inversion)**
- **辅助技巧**：
  - **二维前缀和/容斥原理**（处理区间限制 $a \le x \le b$）
  - **数论分块 (整除分块)**（将 $O(n)$ 优化至 $O(\sqrt{n})$）
- **抽象模型**：求二维矩形区域内满足 $\gcd(x, y) = k$ 的点对数量。

**金牌选手视角**：看到 $\gcd(x, y) = k$ 或者 $\sum \gcd(x, y)$，第一反应必须是**反演**或者**欧拉函数**。看到区间 $[a, b]$，第一反应必须是**容斥**转化为 $[1, b] - [1, a-1]$。

------

### 2. “上帝视角”去除后的思维复盘：从零推导

假设我没看过题解，拿到题目：$a \le x \le b, c \le y \le d, \gcd(x, y) = k$。

**Step 1: 降维打击（处理区间限制）**

- **直觉**：上下界都有两个变量（$a, b$ 和 $c, d$），这太麻烦了。如果下界都是 1 就好了。

- **转换**：这是一个二维平面的计数问题。求矩形区域内的点数，且满足特定属性。根据二维前缀和（或容斥原理），任意矩形 $[a, b] \times [c, d]$ 的答案可以拆解为四个以 $(0,0)$ 为原点的矩形组合：

  $$Ans(b, d) - Ans(a-1, d) - Ans(b, c-1) + Ans(a-1, c-1)$$

- **新目标**：我现在只需要解决一个子问题：`solve(N, M, k)`，即求 $1 \le x \le N, 1 \le y \le M$ 且 $\gcd(x, y) = k$ 的对数。

**Step 2: 提取本质（去除 k）**

- **观察**：$\gcd(x, y) = k$ 这个条件很难直接用。
- **转换**：$\gcd(x, y) = k \iff \gcd(x/k, y/k) = 1$。
- **新目标**：令 $N' = \lfloor N/k \rfloor, M' = \lfloor M/k \rfloor$。问题转化为求 $1 \le i \le N', 1 \le j \le M'$ 且 $\gcd(i, j) = 1$ 的对数。

**Step 3: 莫比乌斯反演（核心推导）**

- **公式联想**：怎么求 $\gcd(i, j) = 1$？利用莫比乌斯函数的性质：$\sum_{d|\gcd(i,j)} \mu(d) = [\gcd(i, j) = 1]$。

  $$\sum_{d=1}^{\min(N', M')} \mu(d) \cdot \lfloor \frac{N'}{d} \rfloor \cdot \lfloor \frac{M'}{d} \rfloor$$

**Step 4: 复杂度分析与优化**

- **现状**：如果我们直接循环跑上面这个式子，单词查询复杂度是 $O(N)$。如果查询次数 $T=50000$，最大 $N=50000$，总复杂度 $2.5 \times 10^9$，会 TLE。
- **优化**：观察式子 $\lfloor \frac{N'}{d} \rfloor$，我们发现对于连续的一段 $d$，$\lfloor \frac{N'}{d} \rfloor$ 的值是不变的。这就是**数论分块**的切入点。
- **结论**：配合 $\mu$ 的前缀和预处理，单次查询复杂度降为 $O(\sqrt{N})$。总复杂度 $O(N + T\sqrt{N})$，可以通过。

**自我问答环节（特异化修改模板时思考的问题）：**

- **Q: 这个题的 $\mu$ 需要筛到多少？**
  - A: 题目中 $a, b, c, d$ 最大是 $5 \times 10^4$，所以筛到 $50000$ 即可。
- **Q: 只有一组询问吗？**
  - A: 不，有多组。所以 `sieve` 放外面只做一次，`work` 每次调用。
- **Q: 分块的时候 $N$ 和 $M$ 不一样怎么办？**
  - A: 分块的右边界 $r = \min(N/(N/l), M/(M/l))$，取两个块边界的较小值，这是二维数论分块的标准写法。

### 3. 你的初次想法错在哪？

**你的思路**：“用 b、d 的答案减去 a-1 和 b-1 的答案即可”。

**公式化表示**：你试图计算 $F(b, d) - F(a-1, b-1)$。这里 $F(n, m)$ 代表 $[1, n] \times [1, m]$ 的答案。

**错误原因**：你混淆了 **一维区间减法** 和 **二维区域减法**。

- **一维**：求 $[L, R]$ 的和，确实是 $S(R) - S(L-1)$。
- **二维**：求 $[a, b] \times [c, d]$ 这个矩形的和。
  - 想象一个坐标系。你的目标是求右上方的一个小矩形。
  - $F(b, d)$ 是从原点 $(0,0)$ 到 $(b, d)$ 的大矩形。
  - $F(a-1, b-1)$ 是从原点 $(0,0)$ 到 $(a-1, b-1)$ 的小矩形（大概在左下角）。
  - **问题来了**：直接相减，你只减去了左下角，**左上角** $[1, a-1] \times [c, d]$ 和 **右下角** $[a, b] \times [1, c-1]$ 的区域并没有被去掉！

**正确思路（容斥原理）**：

$$Ans = F(b, d) - F(a-1, d) - F(b, c-1) + F(a-1, c-1)$$

你需要减去“整个左边”和“整个下边”，但这样“左下角”被减了两次，所以要加回来一次。

### 4. 通用解题方法论（针对数论/计数类题目）

遇到陌生题目，不要慌，按这个 SOP (Standard Operating Procedure) 走：

1. **形式化 (Formalization)**：
   - 扔掉题目背景（什么公约数游戏之类的），只看数学式子。
   - 把题目翻译成数学语言：$\sum_{i=a}^b \sum_{j=c}^d [\text{condition}]$。
2. **前缀化 (Prefix Transformation)**：
   - 只要看到 $a \le i \le b$，立刻检查是否满足区间可减性。
   - 如果是，马上转化为 `calc(b) - calc(a-1)` 或二维容斥。这能极大简化推导难度。
