### 一、 算法归类与本质剖析

这道题的本质是**有限状态空间上的离散时间马尔可夫链（Markov Chain）模型**。

在算法竞赛中，这类问题通常被归类为 **概率 DP**。

- **状态的无后效性（核心本质）**：骑士在第 $k$ 步跳到某个格子的概率，**仅仅取决于**它在第 $k-1$ 步时在各个格子的概率分布，而与它前 $k-2$ 步是怎么走过来的完全无关。这就是马尔可夫性的完美体现，也是我们可以使用动态规划的根本原因。
- **转移的有向性**：题目是一个步数严格单调递增的过程（时间之矢从 $0$ 到 $k$），因此状态转移图是一个**有向无环图（DAG）**，可以用简单的循环递推解决，不需要借助高斯消元（如果是可以在棋盘上无限随机游走求期望步数，就需要高斯消元了）。

### 二、 “零视角”思维复盘：如何从条件推导到解法？

假设我们在赛场上第一次看到这道题，如何通过“自我问答”来锁定解法并定制模板？

**1. 捕获关键条件与抽象转化**

- **提取线索**：“每次有 8 个方向”、“等概率随机移动”、“求 $k$ 步后仍在棋盘上的概率”。
- **直觉排雷**：遇到树/图上的概率/期望问题，第一反应是 DFS/BFS 模拟。但一看数据范围，分支因子是 8，$k=100$，如果暴搜，时间复杂度是 $O(8^{100})$，必定 TLE。
- **转化视角**：既然不能追踪“一条条具体的路径”，我们就必须追踪“状态的宏观分布”。这就好比与其去算每一个水分子怎么流动（暴搜），不如去算各个区域的水位高度（DP）。

**2. 确立状态与目标**

- **问自己：决定当前局面的变量有哪些？** -> 剩余/已走步数、当前行、当前列。
- **定义状态**：设 $f(k, i, j)$ 表示骑士走了 $k$ 步之后，停在坐标 $(i, j)$ 的概率。
- **明确目标**：最终答案就是 $\sum f(m, i, j)$ 对所有合法的 $(i, j)$ 求和。

**3. 推导转移方程（Push vs Pull）**

此时你需要对 DP 模板进行特异化思考。概率 DP 有两种经典转移视角：

- **填表法 (Pull)**：“我当前在 $(i, j)$，我上一步可能在哪里？” -> $f(k, i, j) = \sum f(k-1, nx, ny) \times \frac{1}{8}$。
- **刷表法 (Push)**：“我当前在 $(i, j)$，我下一步能去哪里？” -> $f(k+1, nx, ny) \mathrel{+}= f(k, i, j) \times \frac{1}{8}$。
- **你的选择**：你的代码使用了刷表法，这在处理越界问题时非常直观（跳出去了就不把概率推演过去），是非常棒的实战选择。

**4. 边界与初始化的“自我灵魂拷问”**

- 最开始骑士在哪？在 $(x, y)$，所以只有 $f(0, x, y) = 1$，其余全是 $0$。
- 跳出棋盘的概率去哪了？跳出去就“死亡”了，不用管它，概率自然流失，留在棋盘上的概率总和自然会小于 $1$。

### 四、 ICPC 金牌视角的“降维打击”（进阶扩展）

为了达成你的金牌目标，这道题我们还可以挖掘更深的角度：

**1. 维度升级：如果 $k = 10^9$ 怎么办？**

如果题目把 $k$ 放大到 $10^9$，$O(k \cdot n^2)$ 的时间会立刻 TLE。这在区域赛中是非常经典的变形。

- **破局点**：状态转移是线性的（乘法和加法），且转移规则在每一步都是**固定的**。
- **降维打击武器**：**矩阵快速幂 (Matrix Exponentiation)**。
- **推导**：将 $n \times n$ 的二维状态压缩成一个长度为 $n^2$ 的一维向量 $V$。构建一个 $n^2 \times n^2$ 的状态转移矩阵 $M$。矩阵 $M_{u, v}$ 表示从一维状态 $u$ 跳到状态 $v$ 的概率（相邻则为 $0.125$，否则为 $0$）。
- **最终答案**：$V_{k} = V_{0} \times M^k$。计算矩阵的 $k$ 次方只需要 $O(\log k)$ 的时间。总复杂度变为 $O((n^2)^3 \log k)$。当 $n$ 比较小（比如 $n \le 10$）时，这是唯一的解法。

### 一、 算法归类与本质剖析

这道题的算法类型可以精准归类为 **余数状态机 DP（或者叫模意义下的限制型路径 DP）**。

**它的核心本质在于：利用模运算的同态性进行状态压缩。**

在标准的网格寻路中，如果我们要求“路径和恰好等于某个值”，状态必然是 $f[i][j][sum]$。由于矩阵元素可以很大，路径很长，$sum$ 的状态空间会爆炸式增长，导致时间与空间双重崩溃。

但这道题只关心“是否能被 $k$ 整除”。根据模运算的加法同态性：

$(A + B) \pmod K = (A \pmod K + B \pmod K) \pmod K$

这就意味着，对于到达格子 $(i, j)$ 的无数条路径，我们**不需要记住它们各自真实的路径和，只需要记住它们和的余数**。我们将原本无限延伸的连续状态 $sum$，完美地坍缩到了 $0$ 到 $k-1$ 这 $k$ 个**等价类（Equivalence Classes）**中。这就是状态机 DP 的灵魂所在。

### 二、 零视角思维复盘：如何从零推导？

假设你在赛场上第一次遇到这道题，如何快速切入？

1. **捕获特征，明确基准算法**
   - **特征**：“矩阵”、“只能向下/向右”、“求方案数”、“取模 $10^9+7$”。
   - **直觉**：如果不考虑“和被 $k$ 整除”，这就是最经典的 $O(nm)$ 基础二维 DP。
2. **叠加限制，寻找破局点（核心转换）**
   - 题目多了一个限制：“路径和被 $k$ 整除”。
   - **遇到附加条件时，最通用的手段就是“升维”**。我们将原本的二维状态 $f[i][j]$ 扩展为三维 $f[i][j][x]$，其中 $x$ 代表某种与和相关的信息。
   - **灵魂拷问**：$x$ 存什么？如果存“具体的和”，数组开不下。此时观察目标：“被 $k$ 整除”，即“余数为 $0$”。那我们就把 $x$ 定义为“路径和模 $k$ 的余数”。
3. **推导状态转移与无后效性（特异化修改模板）**
   - **问自己：上一步的余数是 $r$，走到当前格子加上 $val$，现在的余数是多少？**
   - **答**：是 $(r + val) \pmod k$。
   - **检查无后效性**：只要我知道前面的余数，加上当前的数字，就能唯一确定现在的余数。至于前面是怎么走的，和具体是多少，完全不影响现在的余数。无后效性成立！
   - **套用模板的思考**：
     - **方向 1（刷表法/Push）**：我在 $(i, j)$ 余数是 $r$，我能把这个状态推给 $(i+1, j)$ 和 $(i, j+1)$。这对应你注释掉的代码。
     - **方向 2（填表法/Pull）**：我在 $(i, j)$ 想要凑出余数 $r_{new}$，必须从 $(i-1, j)$ 和 $(i, j-1)$ 的某个余数 $r_{old}$ 转移过来。这对应你现在的代码。你巧妙地利用 `int nk = (k + g[i][j]) % p;` 正向推算目标索引，这比逆向求解 `(r_new - val + p) % p` 要优雅得多。

### 一、 算法归类与本质剖析

这道题在算法竞赛中通常被归类为：**带有下界截断的二维 0-1 背包问题**。

**它的核心本质是：状态空间的单向坍缩（State Space Collapse）。**

- **传统背包：** 我们要求“体积恰好为 $V$”或者“价值最大化”，状态转移是精准的点对点转移，状态的值域空间（比如总利润）是随着物品增加而无限膨胀的。
- **本题的本质：** 题目要求的不是“恰好达到某个利润”，而是“至少达到 $minProfit$”。这就意味着，在数学模型中，利润为 $minProfit$ 的状态，和利润为 $minProfit + 10000$ 的状态，对于最终答案的贡献是**完全等价**的。
- 因此，我们将原本应该无限延伸的利润维度，在 $minProfit$ 这个下界处一刀切断（坍缩）。所有溢出的数值，统统被吸入 $k = minProfit$ 这个“黑洞”状态中。这就是 `max(0, k - w[i])` 在代数结构上的物理意义。

### 三、 通用解题方法论：约束驱动的降维打击

面对具有复杂限制条件的陌生 DP 题，脑海空白通常是因为**试图一步到位想出完美方程**。顶尖选手通常使用**“约束驱动的降维打击”**框架：

1. **第一步：无视限制，写出裸模型 (Unconstrained Model)**
   - 先不管题目里乱七八糟的限制条件。这题去掉“利润限制”，就是一个单纯的“100个人选任务求方案数”问题，状态直接就是 $f[i][j]$。
2. **第二步：逐个叠加维度 (Dimension Expansion)**
   - 把去掉的限制加回来。题目加了利润要求，那就直接把利润作为一个新维度塞进 DP 数组，变成 $f[i][j][k]$。
3. **第三步：评估数据范围，寻找等价类压缩 (Equivalence Compression)**
   - 如果新加的维度数据范围爆炸（比如本题的总利润可能达到 10000），立刻去题干中找**“边界约束”**。
   - 一旦出现“至少为 X”、“被 Y 整除”、“大于等于 Z”这类字眼，这往往是明确的提示：**不需要记录真实值，只需要记录到达该状态所需的“欠款”或者“余数”**。将大值域映射到小值域，完成降维。

------

### 四、 顶尖选手的编码流：如何手不沾泥地写代码

在想出 $f[j][k] \leftarrow f[j-v][\max(0, k-w)]$ 之后，怎么写才能如丝般顺滑、毫无卡顿？

核心原则是：**解耦状态转移与边界防御**。

1. **语义声明（锚定心智）：**

   在写下 `vector f` 的瞬间，在脑子里（或者注释里）大声念一遍：“`f[j][k]` 代表**恰好**用 `j` 人，产生**保底** `k` 利润的方案数。” 记住这个语义，后面的每一行都不能违背它。

2. **初始化的绝对确信：**

   万物起源 $f[0][0] = 1$。语义翻译：“恰好用 0 个人，产生保底 0 利润的方案有 1 种（即什么都不做）。” 完美契合逻辑。

3. **骨架铺设（降维后的倒序遍历）：**

   看到 $i$ 转移到 $i+1$，立刻知道可以把第一维滚动掉。因为是 0-1 背包（每个物品只能选一次），必须**倒序遍历**体积（人数 $j$）和价值（利润 $k$）。这一步是肌肉记忆，完全不需要动脑。

   C++

   ```
   for(int i = 1; i <= m; i++)
       for(int j = n; j >= v[i]; j--) // 倒序，下界自然就是 v[i]，连 if (j >= v[i]) 都省了
           for(int k = tar; k >= 0; k--) // 倒序
   ```

4. **填装核心逻辑（拥抱数学，拒绝 if-else）：**

   新手在写这一步时，脑子里会想：“如果 $k < w[i]$，那我要从 $0$ 转移过来，否则从 $k-w[i]$ 转移过来”。然后写出一个丑陋的 `if-else`。

   顶尖选手的编码流是**数学化的**：所需的前置利润是 $k - w[i]$，但利润不能是负的，所以前置状态的利润指标下限就是 $\max(0, k - w[i])$。

   直接写出：`f[j][k] = (f[j][k] + f[j - v[i]][max(0, k - w[i])]) % mod;`。

   一行代码，没有任何分支预测，优雅且不易出错。

### 五、 值得一提的角度：状态定义的进阶转换

你的代码最终计算答案时，使用了一个循环：

```cpp
for(int j = 0; j <= n; j ++) {
    ans += f[j][tar];
}
```

这是因为你的状态 $f[j][tar]$ 定义的是**恰好**使用 $j$ 个人。

**进阶思考：能不能把最后的 `for` 循环也省掉？**

如果你将状态定义稍微扭转一下：让 $f[j][k]$ 表示 **最多** 使用 $j$ 个人，产生至少 $k$ 利润的方案数。

这种定义在初始化时，`f[j][0] = 1` 对于所有的 $0 \le j \le n$ 都成立（不管给你多少人的额度，什么都不干就是1种方案）。

在这种定义下，状态转移方程在处理边界时会有略微的不同，但最终的答案就是直截了当的 `f[n][tar]`，不需要再做一次累加。

**为什么你现在的写法更好？**

在实战中，**“恰好”往往比“最多”更容易推导状态转移方程**，因为“恰好”保证了状态之间的严格互斥，不会出现方案重复计算的问题。在最后一步做一次 $O(N)$ 的遍历求和，时间成本完全可以忽略不计。你潜意识里选择了“恰好”来定义状态，这正是算法直觉精准的体现。

### 1. 题目本质与算法归类

**算法归类：** **区间 DP (Interval DP)** / **状态空间搜索 (State Space Search)**。

**题目本质：** **二叉树同构判断 (Isomorphic Binary Trees)** 的变体。

这道题表面上是字符串操作，本质上是在问：“是否存在一种构建二叉树的方式，使得这两棵以字符串为叶子节点的二叉树，在允许同层左右子树交换的情况下，最终的叶子序列完全一致？”

一旦看透了这是“树的同构”和“区间分裂”，你就能迅速反应出此类问题的最优解法一定是**分治 (Divide and Conquer)**，而带有重叠子问题的分治，自然就导向了**记忆化搜索**或**区间 DP**。

### 2. 从零推导：思维复盘（剥离上帝视角）

如果在赛场上遇到这道题，我们该如何一步步推导出状态方程？

- **第一步：敏锐捕获关键条件（数据范围是最大的剧透）**
  - 看到字符串匹配，第一反应可能是 KMP、后缀自动机、或者贪心。但是！看一下这道题的数据范围：长度 $N \le 30$。
  - 这个极其反常的小数据范围直接排除了线性的字符串算法。$O(N^3)$ 或 $O(N^4)$ 的算法是被允许的。这就暗示了我们需要进行高维度的状态枚举。
- **第二步：操作拆解与不变量寻找**
  - **操作：** “切割”并“决定是否交换”。
  - **不变量：** 无论怎么切割和交换，子串的**长度**是不变的，子串内的**字符集合（词频）**是不变的。
  - 要判断 $S_1$ 能否变成 $S_2$，只要找到一个切割点 $k$，满足下面两种情况之一即可：
    1. **不交换：** $S_1$ 的左边匹配 $S_2$ 的左边，且 $S_1$ 的右边匹配 $S_2$ 的右边。
    2. **交换：** $S_1$ 的左边匹配 $S_2$ 的右边，且 $S_1$ 的右边匹配 $S_2$ 的左边。
- **第三步：形式化定义状态 (State Formalization)**
  - 如何描述上面的逻辑？我们需要比较两个子串。一个子串由起点和终点决定，也就是 $[l_1, r_1]$ 和 $[l_2, r_2]$。
  - **特异化修改（降维打击）：** 因为我们上面找到了“不变量”——两个能匹配的子串，长度必须相等。所以 $r_1 - l_1 = r_2 - l_2$。
  - 因此，我们不需要 4 个变量。用 `(起点1, 起点2, 长度)` 就可以完美定义一个状态。这就得出了你代码中的 `f[i][j][len]`。

### 3. ICPC 通用解题方法论：面对陌生题目的破局框架

当你在赛场上大脑一片空白时，强制自己走以下 SOP（标准作业程序）：

1. **算力逆推 (Reverse Engineering by Constraints)：**
   - 死盯数据范围。$N \le 30$ 往高维 DP、折半搜索 (Meet in the middle) 想；$N \le 10^5$ 往单调栈、线段树、贪心、双指针想。数据范围是出题人留下的最直接的后门。
2. **暴力起手，寻找重叠 (Start with Brute Force, Find Overlaps)：**
   - 不要一开始就憋完美的 Bottom-Up DP。先写一个最暴力的 DFS 伪代码。
   - 问自己：“这个 DFS 会重复计算相同的子问题吗？” 如果会，加一个 `memo` 数组，这就成了记忆化搜索。
3. **寻找拓扑序，转为递推 (Find Topological Order for DP)：**
   - 如果你想把记忆化搜索改成 `for` 循环的 DP（就像你做的那样），你需要问自己：“状态依赖的拓扑序是什么？”
   - 在这道题里，长区间的答案依赖于短区间的答案。所以**最外层循环必须是区间长度 `len`**。这是区间 DP 最核心的套路。

### 4. 值得一提的 ICPC 进阶视角 (Bonus)

**记忆化搜索 vs. 纯 DP 的胜负局**

在你的代码中，你把记忆化搜索注释掉，改写成了 `for` 循环的底层 DP。在大部分普通的 DP 题中，这是一种进步（省去了递归开销）。

**但是，对于这道题以及很多状态空间巨大的题目，记忆化搜索才是最优解，甚至能实现降维打击。**

为什么？因为底层的 `for` 循环 DP 是一种**“盲目铺场”**。它会把所有 $O(N^3)$ 个状态全部计算一遍，无论这些子串字符集合是否相同。 而记忆化搜索是**“按需计算”**（Lazy Evaluation）。

**ICPC 极限优化方案：加入剪枝 (Pruning)**

如果在记忆化搜索的开头，加入一个判断：如果 $S_1[i \dots i+len-1]$ 和 $S_2[j \dots j+len-1]$ 的**字符频率（哈希表/数组记录）不一致，直接返回 false**。

这个简单的剪枝，可以直接砍掉 90% 以上的无效搜索分支，在实际评测中，耗时会远低于 $O(N^4)$ 的纯 DP。这是竞技编程中极度推崇的“常数优化”与“无效状态剪枝”思想。
