## 二维线性动态规划

[1143. 最长公共子序列 - 力扣（LeetCode）](https://leetcode.cn/problems/longest-common-subsequence/description/)

这道题虽然在绝大多数平台上都被标记为基础或中等难度，但它是动态规划领域的一块“基石”。对于目标是金牌的竞技编程训练来说，重要的不仅仅是 AC，而是要把这道题背后的图纸拆解得明明白白。你能意识到单纯写完是不够的，这种对“题理”的极致追求，正是跨越算法瓶颈的关键。

我们来从你要求的几个维度，深度拆解这道题。

------

### 1. 题目归类与本质剖析

**算法归类：** **线性动态规划（Linear DP）中的双序列/双字符串模型**。

**题目本质：**

LCS 的本质是在两个**有序集合**之间寻找一种**保序的最大匹配结构**。

- **线性映射**：它是在处理多阶段决策问题，状态沿着序列的长度方向做线性递推。
- **双序列**：不同于最长递增子序列（LIS）只需要看自身，双序列模型的核心在于寻找两个序列元素之间的“对齐”或“匹配”关系。

**为何重要：**

这道题确立了处理两个字符串/序列匹配问题的**标准状态空间定义**。以后你遇到编辑距离（Edit Distance）、正则表达式匹配、甚至是复杂的树形结构比对，其最内核的思维框架（用二维坐标 $(i, j)$ 表示两个结构的前缀匹配状态）都脱胎于此。

------

### 2. 零基础思维复盘：从直觉到 DP 的推导

假设我们没有上帝视角，面对“最长公共子序列”，应该如何一步步逼近最终的解法？

**Step 1: 捕获关键条件与直觉试错**

- **条件捕获**：“子序列”意味着元素相对顺序不能变，但可以不连续；“公共”意味着要在 $s$ 和 $t$ 中同时存在；“最长”意味着这是一个最优化问题。
- **直觉试错（贪心法）**：我们能贪心吗？比如在 $s$ 中找到 $t$ 的第一个字符，然后再找第二个？
  - *证伪*：假设 $s = \text{"abcde"}$, $t = \text{"ace"}$，贪心可以。但如果 $s = \text{"abac"}$, $t = \text{"bac"}$，如果你贪心让 $s$ 的第一个 $\text{'a'}$ 去匹配 $t$ 的第一个 $\text{'a'}$，你会漏掉更长的 $\text{"bac"}$。这说明**前面的选择会影响后面的最优解（存在后效性），贪心局部最优不等于全局最优**。

**Step 2: 目标转换与降维（寻找重叠子问题）**

- 既然不能贪心，就必须穷举所有匹配可能。但暴力的时间复杂度是指数级的。
- **降维思考**：如果我们只看 $s$ 的最后一个字符 $s[n]$ 和 $t$ 的最后一个字符 $t[m]$，会发生什么？
  - **情况 A**：如果 $s[n] == t[m]$，那么这个字符**一定**在它们的最长公共子序列中。我们只需要求 $s[1 \dots n-1]$ 和 $t[1 \dots m-1]$ 的 LCS 即可。
  - **情况 B**：如果 $s[n] \neq t[m]$，那么它们不可能同时出现在同一个公共子序列的末尾。最优解要么不包含 $s[n]$，要么不包含 $t[m]$。我们需要分别求 $s[1 \dots n-1]$ 与 $t[1 \dots m]$ 的 LCS，以及 $s[1 \dots n]$ 与 $t[1 \dots m-1]$ 的 LCS，然后取最大值。
- 到这里，我们成功把原问题转化为了规模更小的同类问题，且存在大量重复计算（重叠子问题），**动态规划（DP）** 的大门正式打开。

**Step 3: 模板特异化（自我问答）**

- **Q: 状态怎么定义？**

  - A: 需要记录两个序列当前处理到的位置。定义 $f(i, j)$ 表示 $s$ 的前 $i$ 个字符和 $t$ 的前 $j$ 个字符的最长公共子序列长度。

- **Q: 状态转移方程是什么？**

  - A: 根据 Step 2 的推导：

    如果 $s[i] == t[j]$，则 $f(i, j) = f(i-1, j-1) + 1$

    如果 $s[i] \neq t[j]$，则 $f(i, j) = \max(f(i-1, j), f(i, j-1))$

- **Q: 边界条件怎么处理？越界怎么办？**

  - A: 当任一字符串长度为 0 时，LCS 长度为 0。即 $f(i, 0) = 0$ 且 $f(0, j) = 0$。为了避免在循环里疯狂写 `if (i == 0 || j == 0)`，最好的做法是将字符串下标向右平移一位（也就是你在代码里做的 `s = '#' + s`），这叫**增加虚拟哨兵**。

------

### 3. 陌题通用解题方法论

在赛场上遇到完全没有思路的题目，可以尝试以下标准的“破题”流程：

1. **数据范围逆推（Scale Check）**：
   - 首先看 $N, M$ 的量级。如果 $N \le 10^3$，暗示 $O(N^2)$ 算法（通常是二维 DP）。如果 $N \le 10^5$，暗示 $O(N \log N)$ 算法（贪心、分治、线段树或二分）。这一步能直接帮你排除掉一半以上的错误方向。
2. **寻找“最后一步”的决策（Decision Point）**：
   - 不要顺着想怎么一步步构建答案，要**逆着想**：如果我已经得到了最优解，这个最优解的最后一步操作是什么？（比如本题的最后一步就是看两个字符串的末尾字符是否匹配）。
3. **弱化问题与增量构造（Simplification）**：
   - 如果题目有三个限制条件，先拿掉两个，只看最简单的情况。
   - 比如本题，如果不求“子序列”而是求“连续子串”，你会怎么做？（连续子串的转移方程只需保留相等的情况，不等时直接归零）。通过对比弱化问题，能让你更敏锐地察觉到当前这道题“难”在哪里，从而针对性地设计状态。
4. **验证“无后效性”**：
   - 当你设计出一个状态后，问自己：如果我只知道当前状态的值，不知道我是怎么走到这一步的，我还能不能推导出下一步？如果能，状态成立；如果不能，说明你的状态维度不够，需要升维（比如增加一维表示某种限制条件的余额）。

------

### 4. 顶尖选手的编码流（思路到代码的无缝转换）

高水平的选手写代码不会“边写边想逻辑”，而是将编码过程模块化。针对 DP 题，流畅的编码流如下：

- **Phase 1: 基础设施建设（肌肉记忆）**

  迅速读入数据。你代码中的 `s = '#' + s, t = '#' + t;` 是极其成熟的做法（1-based 索引化）。这是一种“防御性编程”，直接消灭了后续所有的边界越界讨论。

- **Phase 2: 状态容器分配**

  `vector f(n + 1, vector(m + 1, 0));`。初始化为 0 完美契合了空串 LCS 为 0 的边界条件。这步写完，脑子里就不再去想边界了。

- **Phase 3: 机械化状态转移（Top-Down 思考，Bottom-Up 编写）**

  写外层循环 `i` 从 1 到 `n`，内层循环 `j` 从 1 到 `m`。

  **核心心法**：在写循环内部时，**绝对不去想这个 $f[i-1][j-1]$ 是怎么一路递推过来的**。你要把 $f[i-1][j-1]$ 当作一个绝对正确、已经求解完毕的黑盒。

  你只关心当前的局部逻辑：

  - 相等？我就把左上角的黑盒拿过来加一。
  - 不等？我就看看上方和左方的黑盒谁大。

- **Phase 4: 导出答案**

  循环结束，答案一定在 $f[n][m]$。

这种“信任状态定义”的编码方式，能让你在写长达几十行的复杂 DP 转移方程时，大脑不会因为堆栈溢出而卡住。

------

### 5. 值得一提的扩展角度（面向金牌的进阶视野）

针对 LCS，在更高阶的竞赛体系中，有两个重要的变形和优化：

**角度一：空间复杂度从 $O(NM)$ 降到 $O(\min(N, M))$ （滚动数组优化）**

仔细观察转移方程，计算第 $i$ 行的状态时，只依赖于第 $i-1$ 行（上一行）和第 $i$ 行当前计算出的部分。第 $i-2$ 行及之前的数据全成了垃圾。

在赛场上，如果遇到 $N, M = 5000$ 的数据规模，你的二维 `vector` 会直接 MLE（内存超限）。此时需要用 `vector<int> f(2, vector<int>(m + 1, 0))` 或一维数组进行滚动更新，这是金牌选手必须掌握的基本功。

**角度二：LCS 转化为 LIS（极其精妙的 $O(N \log N)$ 降维打击）**

如果题目给出了一个极其苛刻的条件：**序列 $s$ 和 $t$ 中的元素都不重复（例如它们都是 $1 \sim N$ 的排列）**，且 $N = 10^5$。此时 $O(N^2)$ 的 DP 直接 TLE。

**破局思路**：

1. 由于元素不重复，我们可以把序列 $s$ 的元素替换为其对应的索引号。例如 $s = [A, B, C]$ 变成 $[1, 2, 3]$。

2. 把序列 $t$ 中的元素也替换为在 $s$ 中的索引号。如果 $t$ 中有 $s$ 没有的元素，直接删掉。

3. 此时，求 $s$ 和 $t$ 的 LCS，就完美等价于求**转换后序列 $t$ 的最长递增子序列（LIS）**！

4. 利用 LIS 的二分 + 贪心算法，我们可以在 $O(N \log N)$ 时间内解决原本是 $O(N^2)$ 的问题。

   这个技巧在 ICPC 赛场上经常作为某道复杂题目的最后一步优化出现。



## 二维区间动态规划

[516. 最长回文子序列 - 力扣（LeetCode）](https://leetcode.cn/problems/longest-palindromic-subsequence/description/)

这道题你不仅做出来了，还在复盘自己的错误思路，并且寻求更本质的解题框架。这种死磕到底、“榨干一道题”的训练方式，正是通往 ICPC 金牌的必经之路。

我们来一点点把这道题的骨架拆解开。

------

### 1. 题目归类与本质剖析

**算法归类：** **区间动态规划（Interval DP）**。

**题目本质：**

这道题的本质是在一个一维序列中，寻找具有**镜像对称结构**的最优解。

普通的线性 DP（比如最长上升子序列 LIS）是单向推演的，从左到右形成一个链式结构。而**回文**的本质是“中心辐射”或“两端收缩”。这意味着你无法仅凭一个方向的历史信息来推导未来，你必须同时掌握**左右两个边界**的状态。区间 DP 就是专门用来处理这种“从两端向中间合并”或“从中间向两端扩展”的拓扑结构的。

------

### 2. 零基础思维复盘：上帝视角的逆推

假设毫无头绪，如何一步步逼近区间 DP？

**Step 1: 捕获条件，直觉映射**

- **关键词 1：“子序列”** $\to$ 允许不连续，排除双指针直接扫，极大概率是 DP。
- **关键词 2：“回文”** $\to$ 回文的判定条件永远是 $s[L] == s[R]$。这意味着问题的状态转移必然与左右两个端点息息相关。

**Step 2: 确立状态与目标转换**

既然要知道左右端点，那状态至少得是二维的。

设当前我们在看字符串的一个子串，从索引 $l$ 到 $r$。

- 如果 $s[l] == s[r]$，那么这两个字符一定可以作为最长回文子序列的首尾。我们只要知道内部 $s[l+1 \dots r-1]$ 的最长回文长度，再加上 2 即可。
- 如果 $s[l] \neq s[r]$，它们俩不可能同时存在于最长回文子序列中。你要么抛弃左边，去 $s[l+1 \dots r]$ 里找；要么抛弃右边，去 $s[l \dots r-1]$ 里找。两者取最大值。

**Step 3: 模板特异化（自我问答）**

- *Q: 这个转移方程 $f[l][r]$ 依赖于 $f[l+1][r-1]$、$f[l+1][r]$、$f[l][r-1]$。在二维矩阵上，这三个状态分别在当前状态的左下、正下、正左。我该怎么遍历才能保证用到它们时，它们已经被算出来了？*
- *A: 传统的 `for l`, `for r` 行不通，因为算 $f[l][r]$ 时 $f[l+1][r]$ 还没算。由于它总是依赖于**更短的区间**，所以我必须**按区间长度 `len` 作为最外层循环**。这就自然导出了区间 DP 的标准模板。

------

### 3. 你的初次思路诊断（为什么线性 DP 会失败？）

你提到一开始想定义一维状态：`f[i]` 表示以当前位置 `i` 为结尾的 `{最大回文串长度，回文串起始位置}`。

这个思路之所以行不通，根本原因在于它**破坏了动态规划的无后效性（Markov Property）**，并且**丢失了关键的维度信息**。

**具体原因：**

假设字符串是 `abacaba`。你算到了 `i = 4`（字符 `c`），你记录了以前面某个位置为起点的回文信息。

当你推导 `i = 5`（字符 `a`）时，你想把这个 `a` 接到前面的回文串上。但是，回文串要在尾部加一个 `a`，**头部也必须有一个 `a` 来呼应**。

一维的 `f[i]` 只记录了“以 `i` 结尾的最长回文”，它把内部的字符结构完全压缩成了一个长度数字，你根本不知道这个回文串的前面（左边界之外）到底还有没有多余的 `a` 可以用来匹配当前的 `a`。

**一句话总结你的误区：** 回文是一个二维约束（左右同时扩展），而你试图用一维状态去强行降维，导致在转移时“缺乏历史记忆”。

------

### 4. 陌题通用解题方法论

在赛场上遇到脑中一片空白的题，可以按照这套流程强行启动：

1. **数据范围逆推法 (Scale Inference)：**
   - 看题目 $N$ 的大小。本题 LeetCode 没有明说，但通常 $N \le 1000$。看到 1000，立刻锁定时间复杂度 $O(N^2)$。
   - $O(N^2)$ 的算法有哪些？二维 DP、图论的稠密图遍历、暴力枚举两个端点。这能帮你瞬间排除掉一堆错误方向。
2. **升维与降维测试：**
   - 先试着定义一维 DP $f[i]$，想转移方程。如果发现“需要用到前面丢弃的信息”，说明维度不够，立刻升维到 $f[i][j]$。
   - 考虑 $i$ 和 $j$ 的物理意义。是前缀和后缀？是区间的左右端点？是树的节点和容量？
3. **寻找“最后一步”决策 (Last Step Analysis)：**
   - 假设我已经得到了最优解，这个解的最后一步操作是什么？对于本题，最后一步就是看子序列的最外层两个字符是不是一样。倒推这个逻辑，状态转移方程就呼之欲出了。



## 二维树形动态规划

[二叉树](https://www.nowcoder.com/practice/aaefe5896cce4204b276e213e725f3ea)

这道题看似代码简短，但你没有停留在“能过就行”的阶段，而是主动去剖析背后的思维链路，这种“透视”题目的习惯是向 ICPC 金牌迈进的必备素质。

从你的代码可以看出，这道题的核心是**求给定节点数和最大深度的二叉树形态数量**。我们来逐一拆解这道题的价值。

------

### 1. 题目归类与本质剖析

**算法归类：** **树形结构计数 DP（记忆化搜索实现）**，属于组合数学中**卡特兰数（Catalan Number）的带约束变种**。

**题目本质：**

这道题的本质是**分治与乘法原理的结合**。

一棵复杂的二叉树，无论长什么样，都可以被**唯一且无遗漏**地拆解为三个不可分割的部分：**1 个根节点 + 一棵左子树 + 一棵右子树**。

因为左右子树是相互独立的子问题，所以总的组合数等于“左子树的形态数 $\times$ 右子树的形态数”。加上深度限制后，无非是在这个基础模型上增加了一个“高度维度的拦截器”。

------

### 2. 零基础思维复盘：从上帝视角退回第一性原理

假设我们在赛场上第一次遇到这道题，如何从零推导？

**Step 1: 捕获关键条件与直觉映射**

- **条件 1：“二叉树形态计数”** $\to$ 映射：涉及到形态枚举，必定与左右子树的节点分配有关。
- **条件 2：“最大深度（或高度）限制 $m$”** $\to$ 映射：普通的节点分配不够，还需要跟踪当前所处的层数。

**Step 2: 目标转换（寻找重叠子问题）**

- 要组装一棵有 $num$ 个节点的树，必须拿出 $1$ 个节点当根。剩下的 $num - 1$ 个节点怎么办？
- 只能分给左子树和右子树。假设分给左子树 $i$ 个，那么右子树必然分到 $num - 1 - i$ 个。
- **核心发问（不重不漏原则）**：我怎么保证枚举出来的树不重复？
  - **答案**：只要左子树的节点数量 $i$ 不同，或者在 $i$ 相同的情况下左右子树的内部形态不同，构成的整棵树就一定不同。因此，遍历 $i$ 从 $0$ 到 $num - 1$，并在内部相乘，就能完美覆盖所有情况。

**Step 3: 模板特异化修改（自我问答法）**

在套用普通的“拆分子树 DP”时，进行以下提问：

1. **Q: 状态该如何定义？**
   - *A:* 需要知道当前剩下多少个节点要分配（$num$），还需要知道当前正在构建哪一层（高度 $h$），以防超限。所以状态是 `dfs(h, num)`。
2. **Q: 递归的边界（Base Case）是什么？**
   - *A:* * **成功边界**：如果 `num == 0`，说明没有节点要分配了，空树也算一种形态（唯一形态），返回 $1$。
     - **失败边界**：如果当前层数 $h > m$，说明超过了深度限制，这种形态不合法，返回 $0$。
3. **Q: 状态转移方程怎么写？**
   - *A:* 当前状态的总方案数 $=$ $\sum$ (左子树方案数 $\times$ 右子树方案数)。即代码中的 `res += self(h + 1, i) * self(h + 1, num - i - 1)`。

  - 越界了？`if(h > m) return 0;`

### 5. 值得一提的进阶角度（赛场优化细节）

你的代码已经能完美 AC，但从更极致的竞技角度来看，有一个**状态定义的视角转换**值得探讨：

**“绝对高度”与“相对剩余高度”**

在你的代码中，状态 `dfs(h, num)` 里的 $h$ 是**当前向下生长的绝对高度**。这导致你的 `f` 数组依赖于全局的限制 $m$。

实际上，一棵 $3$ 个节点的树，无论是在第 $1$ 层还是第 $5$ 层开始向下长，只要它被允许生长的**剩余层数**是一样的，它的合法形态数就是一样的。

因此，可以把状态定义修改为 `dfs(rem_h, num)`，表示**允许的最大深度还剩 $rem\_h$ 层，要分配 $num$ 个节点**。

- 边界变为：`if(num == 0) return 1;` 和 `if(rem_h == 0) return 0;`
- 转移变为：`self(rem_h - 1, i) * self(rem_h - 1, num - i - 1)`
- 初始调用：`dfs(m, n)`

**这种改写的好处是什么？**

如果题目改为多组询问（比如给 $10^5$ 次询问，每次不同的 $n$ 和 $m$），使用“相对剩余高度”作为状态，你的 `f` 数组在不同组询问之间是**完全可以复用的**（不用每次清空），这将使得整体复杂度降维。这也是 ICPC 命题人非常喜欢卡的一个常数优化点。

### 1. 题目归类与本质剖析

**算法归类：** **记忆化搜索 (Memoized DFS)** / **有向无环图上的动态规划 (DP on DAG)**。

**题目本质：**

这道题的本质是**利用物理规则（严格递增）打破网格图的对称性，将带环无向图降维成有向无环图 (DAG)，进而在其上求最长路**。

- 在普通的二维网格中游走，图是无向且有环的（你可以向左走再向右走，无限循环）。
- 但题目中“严格递增”这个约束，相当于给每一条边赋予了单向箭头（只能从低海拔指向高海拔）。因为高度不可能形成 $A < B < C < A$ 的悖论，所以图中**绝对不可能存在环**。
- 一旦确认图是 DAG，求极值的问题就从“NP-Hard 的图最长路问题”瞬间降维成了“多项式时间的动态规划问题”。

### 2. 零基础思维复盘：从上帝视角退回第一性原理

假设我们在赛场上初见此题，如何一步步推导出记忆化搜索？

**Step 1: 直觉试错与痛点捕获**

- **直觉**：求路径问题，第一反应是深度优先搜索 (DFS)。
- **痛点**：普通 DFS 搜索网格图的时间复杂度是指数级的 $O(4^{NM})$，因为路径会产生巨大的分岔。比如从低谷走到巅峰，可能有成千上万条不同的迂回路线，普通 DFS 会把每一条路线都走一遍。

**Step 2: 寻找破局点（重复子问题）**

- **关键转换**：假设存在一个坐标 $(x, y)$，它的值是 `5`。无论你是从左边（值 `3`）爬上来的，还是从下面（值 `4`）爬上来的，**一旦你站到了 $(x, y)$ 这个位置，接下来你能走出的最长递增路径是固定不变的**。
- **目标条件的确立**：既然 $(x, y)$ 后续的命运只取决于它自身的坐标，而不取决于你是怎么走到 $(x, y)$ 的（即无后效性），我们就可以把“从 $(x, y)$ 出发的最长路径”当成一个独立的目标条件存起来。这就导出了状态数组 `f[x][y]`。

**Step 3: 模板特异化（自我问答）**

在敲定使用 DFS 的基础上，我们需要把它特异化为“记忆化搜索”。

1. **Q: 我需要像经典 DFS 一样使用 `vis` (visited) 数组来防止死循环吗？**
   - *A:* **不需要！** 这是一个极其关键的特异化。因为移动条件是 `g[nx][ny] > g[x][y]`（严格递增），水往低处流，人往高处走，你永远不可能走回曾经走过的（比当前矮的）格子。约束条件本身就是最好的去重机制。
2. **Q: 状态 `f[x][y]` 应该初始化为什么？**
   - *A:* 初始路径长度至少包含自身，所以是 $1$。由于要求最大值，未计算过的状态可以初始化为 $-1$（或 $0$），以便在 DFS 开头做拦截。
3. **Q: 遇到已经算过的状态怎么办？**
   - *A:* 直接 `return f[x][y]`。这把指数级的搜索树硬生生修剪成了 $O(N \times M)$ 的线性时间。

### 3. 陌题通用解题方法论

面对网格图或图论搜索题，如果你脑中一片空白，可以套用这套**“降维打法”**：

1. **寻找单向性（Monotonicity Check）：**
   - 拿到题目，第一时间去寻找有没有让状态“单向流动”的规则。
   - 比如：严格递增、时间只能向前、只能向右下移动、金币只能增加。
   - 只要找到单向性，这题就**一定没有环**，立刻放弃复杂的图论算法，转向 **DP 或记忆化搜索**。
2. **选择执行流（Top-down vs Bottom-up）：**
   - 确定了是 DP，该用 `for` 循环递推，还是用递归 DFS？
   - **判定标准**：看状态转移的拓扑序是否容易显式写出。
   - 比如经典的背包问题，体积从小到大，直接写 `for` 循环很容易。

3. **状态定义的三连问：**
   - 这个节点的状态是否依赖于过去的路径？（决定维度大小）

作为算法训练，如果我们将题目的约束稍微改动一点，这道题的难度会发生质变。这是 ICPC 赛场上经常出现的“变式套路”：

**魔改假设：如果题目把“严格递增（$>$）”改成了“非严格递增（$\ge$）”，该怎么做？**

- **毁灭性的打击**：一旦允许等于，高度相同的格子之间就会形成**环**（比如 $A$ 和 $B$ 相邻且高度同为 5，你可以 $A \to B \to A$ 无限刷步数，或者被卡死）。之前的 DAG 前提彻底粉碎，记忆化搜索会陷入死循环（Stack Overflow）。

- **高维解法 (Tarjan + DAG DP)**：

  面对有环图上的最长路，金牌选手的标准应对方案是：

  1. 把整个网格视为一般的有向图，同等高度相邻的格子连双向边，高度变高的连单向边。
  2. 运行 **Tarjan 强连通分量 (SCC) 算法**，把高度相同且连通的区块“缩点”（Condensation）。把它们压缩成一个权值为该区块格子总数的超级节点。
  3. 缩点后，图必定再次变回 **DAG**。
  4. 最后，在这个带有权值的全新 DAG 上，再次运行你现在写的这份记忆化搜索代码。

这道题完成得很漂亮。你在代码中使用了 `s = '#' + s` 这种 1-base 索引对齐的技巧，这在处理边界条件时能极大减少越界特判（比如 `i-1`、`j-1`），是非常成熟的竞赛选手习惯。

### 一、 题目归类与本质探讨

- **核心归类**：**双序列线性 DP（匹配型）**。
- **同类经典**：最长公共子序列 (LCS)、编辑距离 (Edit Distance)、正则表达式匹配。
- **算法本质**：这是一个典型的**“组合计数与选择模型”**。在双序列匹配问题中，其本质是在一个二维网格上寻找合法路径。目标串 $t$ 是约束条件，母串 $s$ 是决策空间。对于 $s$ 中的每一个字符，我们面临的核心决策始终是**“选与不选”**（即是否将其作为 $t$ 中对应字符的匹配项），而动态规划的作用就是将这些重叠的局部决策通过递推关系合并，从而避免指数级的暴力搜索。

### 二、 零视角思维复盘：从直觉到状态转移方程

假设我们完全没有上帝视角，面对“在字符串 $s$ 中寻找 $t$ 作为子序列的出现次数”，该如何推导？

**1. 捕获关键条件，确定大方向**

- **关键词**：“子序列”、“出现次数（方案数）”。
- **初步判断**：子序列意味着不需要连续，这排除了简单的滑动窗口或 KMP。求方案数且不要求输出所有具体方案，这是极其强烈的**动态规划（DP）或组合数学**信号。由于字符可能任意重复出现，组合数学的容斥原理会极其复杂，因此锁定 **DP**。

**2. 状态定义（从已知找维度）**

- 处理两个字符串 $s$ 和 $t$ 的匹配问题，最通用的状态定义切入点是**前缀匹配**。
- 我们自然而然地设出二维状态：令 $f[i][j]$ 表示 $s$ 的前 $i$ 个字符（即 $s[1..i]$）中，包含 $t$ 的前 $j$ 个字符（即 $t[1..j]$）作为子序列的方案数。

**3. 推导状态转移（最后一步分析法）**

- **聚焦最后一个元素**：站在 $f[i][j]$ 的角度，思考最后一步是怎么达到的。我们现在手头有 $s[i]$ 和 $t[j]$。

- **默认继承（不选 $s[i]$）**：无论 $s[i]$ 是什么，由于我们在 $s[1..i]$ 中找子序列，那么在 $s[1..i-1]$ 中已经合法匹配了 $t[1..j]$ 的所有方案，自然也完全包含在 $s[1..i]$ 中。

  - 得出第一部分：$f[i][j] = f[i-1][j]$。

- **条件激活（选 $s[i]$）**：如果我们想把 $s[i]$ 加入到匹配方案中，它必须能与 $t$ 的最后一个字符匹配，即必须有 $s[i] == t[j]$。

  - 如果相等，那么 $s[i]$ 消耗掉了 $t[j]$，前置条件就变成了：我们需要在 $s[1..i-1]$ 中找到 $t[1..j-1]$ 的方案数。
  - 得出第二部分：如果 $s[i] == t[j]$，则新增 $f[i-1][j-1]$ 种方案。

- **合并方程**：

  $f[i][j] = f[i-1][j] + (s[i] == t[j] ? f[i-1][j-1] : 0)$

**4. 寻找边界（初始条件）**

- 空串是任何字符串的子序列。当 $j=0$ 时，表示目标串为空，无论 $s$ 多长，都有一种方案（什么都不选）。因此 $f[i][0] = 1$。
- 当 $i=0$ 且 $j > 0$ 时，母串为空但目标串不为空，显然无法匹配。因此 $f[0][j] = 0$（默认初始化即可）。

### 三、 面对陌生题目的通用解题方法论

遇到脑中一片空白的题目，不要慌，按照以下“漏斗型”思考法逐层降维：

1. **数据范围逆推法（Data-Driven Guess）**：

- **算法本质：DAG 上的最短路径问题。** 跳出 DP 的框架，编辑距离本质上是一个有向无环图 (DAG)。每一个状态 $(i, j)$ 是图上的一个节点，代表 $s$ 的前 $i$ 个字符和 $t$ 的前 $j$ 个字符。图上的边代表三种操作（增、删、改）以及“匹配”。我们要求的是从起点 $(0, 0)$ 到终点 $(n, m)$ 的**最短路径**。理解了这一点，你以后遇到需要输出具体编辑方案（路径还原），或者存在某些操作代价为负/随位置变化时，就能立刻将其映射到最短路算法上。

### 二、 零视角思维复盘：如何从无到有推导

如果在考场上遇到一个披着复杂外衣的“编辑距离”变体，脑子一片空白，该如何破局？

**1. 捕获关键信号，明确方向**

- **关键词提取：** “两个字符串”、“最小操作数”、“只能进行增删改”。
- **直觉映射：** 涉及两个线性结构求最优解，绝大多数情况是**双序列动态规划**。贪心通常无法处理“修改当前字符会影响后续最优选择”的后效性问题。

**2. 降维打击与状态定义**

- **思考特例：** 如果 $s$ 和 $t$ 都只有一个字符，怎么变？如果 $s$ 为空串，怎么变成 $t$？
- **寻找不变量：** 无论怎么操作，字符串都是从左向右生长的。因此，以**“前缀”**作为子问题边界是最稳妥的。
- **确立状态：** 设 $f[i][j]$ 为将 $s$ 的前 $i$ 个字符（$s[1..i]$）转换成 $t$ 的前 $j$ 个字符（$t[1..j]$）所需的最少操作数。

**3. “最后一步”的完全解耦（MECE 原则）**

站在 $(i, j)$ 这个状态，强迫自己只看 $s[i]$ 和 $t[j]$，严格按照**“互斥且穷尽 (MECE)”**的原则分类讨论：

- **情况 A：$s[i] == t[j]$。** 最完美的局面，末尾已经匹配。操作数不需要增加，直接继承没有这两个字符时的状态：$f[i][j] = f[i-1][j-1]$。

- **情况 B：$s[i] \neq t[j]$。**

  必须通过操作来弥补差异，有且仅有三条路可以走：

  1. **替换（Replace）：** 把 $s[i]$ 变成 $t[j]$，这样末尾就一样了。代价是 $1$。前置状态是啥？是 $s[1..i-1]$ 变成 $t[1..j-1]$ 的代价。也就是 $f[i-1][j-1] + 1$。
  2. **删除（Delete）：** 我觉得 $s[i]$ 太碍事了，删掉它。代价是 $1$。删掉后，我们需要把剩下的 $s[1..i-1]$ 变成完整的 $t[1..j]$。也就是 $f[i-1][j] + 1$。
  3. **插入（Insert）：** $s[i]$ 我保留着，但我在它后面插一个 $t[j]$。代价是 $1$。既然已经硬生生造出了一个 $t[j]$，那目标串的 $t[j]$ 就被搞定了。前置状态变成了把完整的 $s[1..i]$ 变成 $t[1..j-1]$。也就是 $f[i][j-1] + 1$。

将以上逻辑组合，并套用 $\min$ 函数，状态转移方程就如水到渠成般推导出来了。

### 二、 零视角思维复盘：破局与推演

假设我们在赛场上第一次遇到这道题，如何从零推导？

- **关键词**：“交错组成”、“相对顺序不变”。

**2. 核心条件的数学转换（寻找不变量）**

- 题目要求 $s_1$ 和 $s_2$ 完整拼出 $s_3$。这里隐藏着一个最关键的物理守恒定律（也就是你的第二版代码优于第一版的根本原因）：

  **消耗 $s_1$ 的长度 $i$ + 消耗 $s_2$ 的长度 $j$ $\equiv$ 匹配 $s_3$ 的总长度 $k$**。

- 这个等式 $k = i + j$ 将原本需要三个维度的状态空间 $(i, j, k)$，直接降维成了二维空间 $(i, j)$。

**3. 状态转移的“最后一步分析法”**

- 站在 $f[i][j]$（能否用 $s_1$ 的前 $i$ 个和 $s_2$ 的前 $j$ 个拼出 $s_3$ 的前 $i+j$ 个）这个目标上，我们只看**最后一步是怎么走过来的**。

- **路径 A**：最后一步是 $s_1[i]$ 提供的。前置条件必须是 $s_1[i] == s_3[i+j]$，且在此之前 $f[i-1][j]$ 必须是通的。

- **路径 B**：最后一步是 $s_2[j]$ 提供的。前置条件必须是 $s_2[j] == s_3[i+j]$，且在此之前 $f[i][j-1]$ 必须是通的。

- 只要两条路有一条能通，当前状态就通。这就自然推导出了你的状态转移方程：

  $f[i][j] = (f[i-1][j] \text{ and } s_1[i] == s_3[i+j]) \text{ or } (f[i][j-1] \text{ and } s_2[j] == s_3[i+j])$

### 二、 零视角思维复盘：从直觉到状态转移方程

假设在赛场上，我们面临这道题，如何一步步剥丝抽茧推导出这个极为简洁的代码？

**1. 捕获关键条件，拆解约束**

- **动作**：“删除 $s_1$ 中的字符”。这意味着 $s_1$ 最后保留下来的是一个**子序列（Subsequence）**，可以不连续。
- **目标**：“成为 $s_2$ 的子串”。这意味着 $s_2$ 中被匹配的部分必须是**连续的（Substring）**。
- **目标代价**：最小删除次数 = $s_1$ 的长度 - 匹配成功的最大长度。

**2. 状态定义（寻找不变的锚点）**

- 处理两个序列，最稳妥的锚点始终是“前缀”。

- 设 $f[i][j]$ 表示 $s_1$ 的前 $i$ 个字符，匹配上 $s_2$ 的前 $j$ 个字符的**后缀**，所需的最小删除次数。

- *自我问答：为什么是匹配 $s_2$ 的“后缀”而不是“前缀”？*

  因为 $s_2$ 的目标是子串！子串就是某个前缀的后缀。我们在扫描 $s_2$ 的第 $j$ 个字符时，强制这部分匹配以 $s_2[j]$ 结尾，这样就能保证连续性。

**3. 推导转移方程（限制与自由的博弈）**

站在 $f[i][j]$，看最后一步怎么走（考虑 $s_1[i]$ 和 $s_2[j]$）：

- **情况 A：$s_1[i] == s_2[j]$。** 天作之合，匹配上了。

  既然匹配上了，$s_1[i]$ 不需要删除，前置状态自然是 $s_1[1..i-1]$ 匹配 $s_2$ 以 $j-1$ 结尾的子串。

  转移：$f[i][j] = f[i-1][j-1]$。

- **情况 B：$s_1[i] \neq s_2[j]$。** 出现分歧了，必须有人妥协。

  $s_1$ 可以妥协吗？可以，$s_1$ 允许删除，这意味着我抛弃 $s_1[i]$，用 $s_1[1..i-1]$ 继续去匹配 $s_2$ 以 $j$ 结尾的子串。代价 + 1。

  转移：$f[i][j] = f[i-1][j] + 1$。

  *$s_2$ 可以妥协吗？* **绝对不行！** 如果我们写出类似 $f[i][j-1]$ 的转移，就意味着我们在 $s_2$ 中跳过了一个字符，这就变成了匹配 $s_2$ 的“子序列”，破坏了题目要求的“子串（连续）”性质。**（这是本题最核心的特异化修改！）**

**4. 边界的魔术（如何处理“任意起点”和“任意终点”）**

- **任意起点**：子串可以从 $s_2$ 的任何位置开始。这意味着，当 $s_1$ 为空（$i=0$）时，它匹配 $s_2$ 的任何位置，删除代价都是 0。所以 `f[0][j] = 0`。
- **约束起点**：如果 $s_2$ 为空（$j=0$），但 $s_1$ 不为空，那 $s_1$ 只能全部删光。所以 `f[i][0] = i`。
- **任意终点**：匹配结束的位置可以是 $s_2$ 中的任意一个 $j$。因此，最终答案不是死板的 $f[n][m]$，而是 $\min_{j=0}^{m} f[n][j]$。

------

### 三、 通用解题方法论（应对陌生 DP 变体的“三步降维法”）

在 ICPC 赛场上，遇到“长得很像经典模型，但条件有点怪”的题目时，不要直接默写模板，按以下流程跑一遍：

1. **确定“自由度”（Constraints vs. Freedoms）**
   - 列出各维度的限制。例如本题：$s_1$ 自由度高（可跳跃，即删字符），$s_2$ 自由度低（必须连续，不可跳跃）。
   - **原则**：自由度高的允许在状态转移时存在继承（如 $f[i-1][j]$），自由度低的严禁在转移时跳过（如本题没有 $f[i][j-1]$ 的转移）。
2. **定位“起点/终点”的松弛操作（Relaxation）**
   - 如果题目要求“全局匹配”，则初始化仅 `f[0][0]=0`，其余全为 $\infty$；答案在 `f[n][m]`。
   - 如果题目允许“任意起点”，则初始状态某一边界全为 0（如本题 `f[0][j]=0`）。
   - 如果题目允许“任意终点”，则最终答案在一整行或一整列中取最值（如本题 `min(f[n][j])`）。
3. **极端数据走查（Dry Run Edge Cases）**
   - 用极其极端的例子脑内跑一遍。比如 $s_1 = \text{"a"}$, $s_2 = \text{"bbba"}$。当推导到 $i=1, j=4$ 时，由于前面 $j=1,2,3$ 时 `f[0][j]` 都是 0，匹配到 `a` 时无缝衔接，验证了逻辑的严密性。

### 四、 顶尖选手的编码流（Coding Flow）

即便是思路完全清晰，写代码卡顿也是大忌。以下是将思路转化为代码的黄金流水线：

1. **Blueprint（无码推演）**

   不在键盘上敲一个字，先在草稿纸上写下：

   - `dp[i][j] = (s1[i]==s2[j]) ? dp[i-1][j-1] : dp[i-1][j] + 1`
   - `dp[0][j] = 0, dp[i][0] = i`
   - `ans = min(dp[n][j])`

2. **Scaffolding（脚手架搭建）**

   迅速写好输入输出、获取长度。**必须立刻加上 1-base 对齐的占位符**（`s1 = '#' + s1`）。这是防止比赛高压下产生 $+1, -1$ 越界幻觉的神器。

3. **Memory Allocation & Init（内存分配与边界）**

   根据 Blueprint，利用 `vector` 的默认初始化特性，直接声明 `vector f(n+1, vector(m+1, 0))`。此时 `f[0][j] = 0` 已经免费完成了！只需要写一个 `for` 循环处理 `f[i][0] = i`。

4. **Core Engine（核心转移引擎）**

   两层 `for` 循环。在这个阶段，**关闭大脑的业务逻辑区，开启“无情的打字机”模式**。严格照抄草稿纸上的转移方程，不要在写这几行的时候突然又去想“哎？这里需不需要取个 min？”。

5. **Output Extraction（答案提取）**

   扫一遍 `f[n][j]` 取最小值，输出。
