- **第二步：定义状态（消除后效性）**

  - 我们需要从小规模问题推导到大规模问题。
  - **自我问答：** 当我站在第 $i$ 个房子面前时，我做决策（偷还是不偷）需要知道未来的信息吗？不需要。我只需要知道“前面几家偷到的最大金额”以及“上一家有没有被偷”。

- **第三步：推导状态转移方程（分类讨论）**

  - 站在第 $i$ 间房前，我只有两个互斥的动作：

### 2. 零视角推导复盘（剥离上帝视角）

如果在赛场上遇到这道题，我们该如何一步步推导出现在的解法？

- **第一步：降维打击，退化到最简模型。**
  - 去掉“环形”这个条件，这题就是最经典的“最大子数组和”（LeetCode 53）。
  - 立刻调取大脑中的武器库：Kadane 算法，或者叫一维 DP，状态转移方程为 $f[i] = \max(f[i-1] + nums[i], nums[i])$。
- **第二步：精准定位增量信息（变体带来了什么？）。**
  - 环形带来的唯一改变是：子数组可以从数组尾部跨越到头部。
  - 这意味着，最终的最大子数组只可能有两种形态：
    1. **形态 A（不跨界）：** 完全在原数组内部。这完全等价于退化后的最简模型。
    2. **形态 B（跨界）：** 由原数组的一段前缀和一段后缀拼凑而成。
- **第三步：目标条件转换（破局的核心）。**
  - 形态 A 已经解决了。怎么求形态 B 的最大值？
  - **关键转换：** 一段前缀加上一段后缀，等于**整个数组的总和**减去**中间剩下的一段连续子数组**。
  - 推导式：$\text{跨界最大和} = \text{总和} - \text{中间连续子数组的和}$。
  - 因为总和是常数，要让“跨界和”最大，就必须让“中间连续子数组的和”**最小**！
- **第四步：特异化修改模板。**
  - 现在目标明确了：我们需要同时求出一个线性数组的**最大子段和**与**最小子段和**。
  - 套用 Kadane 模板，仅仅是把 `max` 换成 `min` 跑两遍的事情，甚至可以像你的代码一样合在一个循环里。
  - **致命特判预警：** 当所有数都是负数时，最小子段和就是整个数组。此时 $\text{总和} - \text{最小子段和} = 0$，也就是选择了一个空集。但题目要求非空，所以此时形态 B 不合法，只能取形态 A（最大子段和，即最大的那个负数）。

如果在赛场上面对一个变形的环形 DP，如何推导出现在的解法？

- **第一步：捕获异常，识别破坏 DP 结构的元凶。**
  - 观察：如果这是一条直线，状态方程 $dp[i] = \max(dp[i-1], dp[i-2] + a[i])$ 完美运行。
  - 异常：变成了环。环意味着节点 $0$ 和节点 $N-1$ 产生了连边。

- **第二步：目标条件转换——如何强行恢复无后效性？**
  - 既然 $0$ 和 $N-1$ 互相掣肘，那我们只需问自己一个问题：“如果我强行固定其中一个节点的状态，另一个节点是不是就解脱了？”

  - **情况 B：** 假设我**绝对不偷**节点 $N-1$。此时节点 $0$ 没有任何顾虑。整个问题退化成了在区间 $[0, N-2]$ 上的标准线性打家劫舍。
  - *注：你的代码逻辑是“强制偷 0”和“不偷 0（即在 $[1, N-1]$ 选）”，这在逻辑上是等价且完备的，同样覆盖了所有合法空间。*
- **第三步：空间合并。**
  - “不偷 0”的解集，加上“不偷 N-1”的解集，是否覆盖了所有合法的偷窃方案？
  - 是的。因为唯一非法的方案是“既偷 0 又偷 N-1”，这两种情况完美避开了这个雷区。因此，取这两种情况的最大值即为全局最优。

### 4. 通用解题方法论：面对复杂拓扑结构的破局框架

当遇到陌生的、拓扑结构复杂的 DP 题时，可以采用这套**“降维与外挂”**方法论：

1. **退化模型寻找 (Degradation)：**

   将题目中那个最让你难受的条件（比如这里的“环”，或者某题里的“树状关系”、“额外的一条边”）直接抹除。看看退化后的基础模型自己会不会做。如果基础模型都不会，说明问题出在基础算法积累上。

2. **定位冲突边 (Conflict Isolation)：**

   把刚才抹除的条件加回来，观察它具体影响了哪两个/哪几个状态。这条边就是“冲突边”。

3. **枚举冲突点状态 (State Enumeration)：**

   不要试图设计一个极其复杂的 DP 状态去包容这个冲突。相反，**在 DP 外部手动枚举**冲突点所有可能的状态（选或不选、黑或白）。

### 1. 题目本质与算法归类

**算法归类：** **动态规划 (Kadane 算法的二维扩展)** / **降维打击 (Dimension Reduction)**。

**题目本质：** **通过枚举边界将高维问题退化为低维经典问题**。

这道题的本质并不是让你去发明一个新的二维动态规划状态转移方程，而是考察你是否能看透高维结构与低维结构的**同构性**。一个二维的子矩阵，如果你站在极远处看（忽略其内部的行数），它其实就是一根“比较粗的”一维线段。

### 2. 从零推导：思维复盘（剥离上帝视角）

在赛场上，如果没有题解，如何一步步锁定这种解法？

- **第一步：暴露计算瓶颈（算力逆推）**
  - 最暴力的想法是：枚举左上角 $(x_1, y_1)$ 和右下角 $(x_2, y_2)$，再求和。二维前缀和可以将求和优化到 $O(1)$，但枚举四个边界仍然需要 $O(N^2 M^2)$。
  - 观察数据范围（假设通常的矩阵大小在 $N, M \le 200$），$200^4 \approx 1.6 \times 10^9$，必然超时（TLE）。这意味着我们最多只能接受 $O(N^3)$ 级别的算法。
- **第二步：寻找经典低维替身（降阶思考）**
  - 问自己：“如果去掉二维这个限制，退化成一维，这题是什么？”
  - 答：就是经典的“最大子数组和”（LeetCode 53）。它的最优解是 Kadane 算法，时间复杂度 $O(N)$。
- **第三步：搭建降维桥梁（变量重组）**
  - 如何把 $O(N^2 M^2)$ 和 $O(M)$ 结合起来？我们需要**舍弃一半的枚举**。
  - 如果我们强行把矩阵按列“压扁”，也就是把第 $i$ 列到第 $j$ 行的所有数字加起来，变成一个单一的数字。那这一整块二维矩阵，就变成了一个一维数组！
  - **核心转换：** 只要我们用 $O(N^2)$ 的代价枚举了“上边界(Top)”和“下边界(Bottom)”，矩阵在这两根线之间的部分，就可以被“拍扁”成一个长度为 $M$ 的一维数组。接下来，只需要对这个一维数组跑一次 $O(M)$ 的 Kadane 算法即可。总复杂度完美降至 $O(N^2 M)$。
- **第四步：特异化修改模板（针对本题的坐标追踪）**
  - 传统的 Kadane 算法只求最大值 `ans = max(ans, pre)`。但本题要求返回四个坐标 `[r1, c1, r2, c2]`。
  - **自我问答：** “Kadane 算法中，一段新的连续子数组是从什么时候开始的？”
  - **推断：** 当之前的累加和 `pre < 0` 时，带着它只会拖累当前元素。所以，一旦 `pre < 0`，当前元素就是新子段的**左边界（起点）**。这个关键信息使得我们必须在 Kadane 模板中引入一个动态的左指针 `start_col` 来记录当前子段的真正起点。

### 3. 通用解题方法论：面对高维/复杂拓扑结构的破局框架

当你遇到毫无头绪的复杂题目（如图论上的区间问题、二维/三维网格问题）时，强制执行以下 **“降阶与升维” (Degradation & Lifting)** 标准作业程序：

1. **极度退化 (Extreme Degradation):**
   - 把题目中最碍眼的条件去掉。二维变一维，图变成树，树变成链，带有通配符的字符串变成普通字符串。

2. **定位信息丢失 (Identify Information Loss):**
   - 问自己：“退化版解法不能直接用的原因是什么？它丢失了原题的什么信息？”（例如本题丢失了矩阵的“上下宽度”）。
3. **枚举补齐 (Enumerative Compensation):**
   - 对于丢失的那个维度/信息，能否通过**外部暴力枚举**来强行固定它？

4. **状态复用/增量更新 (Incremental Update):**
   - 外部枚举往往会导致复杂度超标。此时观察：在外部枚举的循环中，内部状态是否可以“继承”？
   - （例如本题，下边界 $d$ 每扩展一行，不需要把从 $u$ 到 $d$ 的所有元素重加一遍，只需要把第 $d$ 行的值叠加到一维数组上即可。这是从 $O(N^3 M)$ 优化到 $O(N^2 M)$ 的绝杀）。

### 4. 值得一提的进阶视角 (Bonus)：矩阵转置

这道题隐含了一个绝妙的常数/复杂度优化点，常常在 ICPC 赛场的反套路数据中出现。

你目前的复杂度是 $O(N^2 M)$，也就是 $O(\text{行数}^2 \times \text{列数})$。

如果题目给定的矩阵面积 $N \times M \le 10^5$，但并没有保证它是方形矩阵。如果评测机给你一个极其极端的测试用例：

**矩阵只有 $1$ 列，但有 $10^5$ 行 ($N = 10^5, M = 1$)。**

此时，你的 $O(N^2 M)$ 算法会执行 $(10^5)^2 \times 1 = 10^{10}$ 次运算，直接 TLE 爆炸。

**顶级防御策略 (Dynamic Dimension Selection)：**

**永远去压缩较小的那个维度。**

预处理（矩阵转置）

```cpp
bool rev = false;
if (n > m) {
    vector<vector<int>> ng(m, vector<int>(n));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            ng[j][i] = g[i][j]; // 矩阵转置
        }
    }
    g = move(ng); // 使用 std::move 避免深拷贝，极速交接内存
    swap(n, m);             // 同步交换行列变量
    rev = true;   // 记录时空被翻转的标记
}
```

这在竞技编程中叫做“启发式维度选择”，能够在非均衡数据下保住你的防线。

遇到陌生的复杂约束题目，使用以下思考 SOP：

1. **原子化预处理 (Atomic Preprocessing):**

你提到写码时容易卡住。对比你的两份代码，解法一（前后缀）变量繁多，容易出现 Off-by-one 错误；解法二（DP）结构更好，但三维数组 `array<array<int, 2>, 4>` 显得有些沉重。

顶尖选手在写这种“带有路径记录的 DP”时，大脑有一条清晰的流水线：

- **Phase 1: 绝对干净的数据摄入与滑窗。** （你的代码这里做得很好，把 $S$ 数组提出来了）。

- **Phase 2: 纯粹的价值 DP（不管路径）。**

  不要在 DP 数组里存坐标！只存纯粹的 Value。这样 DP 转移极其干净：

  ```cpp

  vector<vector<int>> f(n + 1, vector<int>(4, 0));

      for(int j = 1; j <= 3; j++) {

  }
  ```

- **Phase 3: 独立的倒推回溯器 (The Tracer)。**

  因为题目要求**字典序最小**，如果你从 $1$ 到 $n$ 正向跑 DP，那么回溯时其实是从 $n$ 往前回溯。

  此时脑中只有一个逻辑：**“当前状 $f[i][j]$ 是怎么来的？”**

  - 如果 $f[i][j]$ 和 $f[i-1][j]$ 相等，说明第 $i$ 个点其实可以不选（不选意味着字典序更靠前），果断向左走：`i--`。

通过将“求极值”和“求路径”在代码物理层面上切分为两个没有任何嵌套关系的 `for/while` 循环，你的思维会极其丝滑，绝不会在中途宕机。

- **第一步：正难则反，定义补集 (The Pivot)**

  直接算“哪两段变成 0”很难，因为负数的大小和分布不可控。

  转换视角：总和 = 数组总和 - 被清零的段。我们要让总和最大，意味着我们要寻找**被保留的段**。舍弃 2 段 $\implies$ 保留 3 段。目标转换为：**选出 $\le 3$ 个不相交的连续子段，使其和最大**。

- **第二步：几何切割，降维打击 (The Split)**

  直接找 3 个子段需要 $O(N^3)$ 暴力或 $O(N \times 3)$ 的二维 DP。

  **破局思考：** 既然是 3 个子段，不管它们怎么分布，只要我们在中间那个子段（或者子段间的空隙）切一刀，整个数组必然被分为两半：

  - **左半边：** 包含最多 2 个子段，且最后一个子段**紧贴着切口**（或者为空）。
  - **右半边：** 包含最多 2 个子段，且第一个子段**紧贴着切口**（或者为空）。

  Plaintext

  ```
  [ 保留段 1 ] (舍弃) [ 保留段 2 左 | 保留段 2 右 ] (舍弃) [ 保留段 3 ]
  <------- fp[i] 负责左半边 --------><------- fs[i+1] 负责右半边 ------->
                                    ^
                                 在 i 处切开
  ```

  左右两边在切口处“无缝拼接”，就自动合并成了最多 3 个子段！

- **第三步：推导特异化状态方程 (State Formulation)**

  如何求左半边（即你的 `fp[i]`）？

  *目标：* 在 $0 \dots i$ 中找 $\le 2$ 个子段，且第 2 个子段必须以 $i$ 结尾（如果不以 $i$ 结尾，说明第 2 个子段为空，退化为 1 个子段）。

  - **情况 A（延伸）：** 继续把 $a[i]$ 接在第 2 个子段后面 $\implies$ `fp[i-1] + a[i]`。
  - **情况 B（新起/退化）：** 此时第 2 个子段刚刚开始（或者为空），那么前 $i-1$ 个元素中只能有 1 个子段。前 $i-1$ 个元素的最大 1 个子段和是多少？就是普通的 Kadane 算法求出的 `mx`！
  - *合并：* `fp[i] = max(fp[i-1] + a[i], mx)`。推导到这里，你代码中所有神奇的变量就全部逻辑闭环了。

### 3. 通用解题方法论 (SOP)

当你在 ICPC 赛场上遇到“脑中一片空白”的组合最优化题目时，强制执行以下框架：

1. **动作翻转 (Action Reversal)：** 题目让你“删、减、变成0”，立刻在草稿纸上写出它的反面：“留、加、保持原样”。寻找哪个维度的约束更清晰。
2. **寻找几何断点 (Meet in the Middle)：** 遇到“选 $K$ 个区间”的问题，如果 $K$ 很小（如 2 或 3），永远优先考虑**前后缀分解**。枚举一个断点 $i$，把 $K$ 个区间拆成左边的 $K_1$ 和右边的 $K_2$。
3. **状态宽容化 (State Relaxation)：** 永远记得考虑“空集”状态。你在代码里初始化 `mx = max(0, a[0])` 就是神来之笔，它宽容了“全负数时一个都不选”的情况，省去了无数的特判。

### 4. 顶尖选手的编码流：思路到代码的快速兑现

你的代码在局部逻辑上非常精炼，但在 `fp` 和 `fs` 的循环中，状态维护有些交织（比如在同一个循环里算 `fp`, `pre`, `mx`，顺序极其敏感，很容易写出 Bug）。

顶尖选手的编码流讲究**“绝对的阶段隔离”**。他们会在脑海里把代码分成三条不可交叉的流水线：

- **Phase 1: 基础底噪 (Kadane 预处理)**

  先用最无脑的 `for` 循环，把左向右、右向左的**全局最大 1 个子段和**跑出来，存入数组。不要和复杂的 DP 混在一起。

- **Phase 2: 锚点 DP (计算 fp 和 fs)**

  拿着 Phase 1 已经计算好的绝对安全的 `mx` 数组，去专心跑 `fp[i] = max(fp[i-1] + a[i], mx_left[i-1])`。此时大脑没有任何负担。

- **Phase 3: 拉链缝合 (The Zipper)**

  一个纯净的循环，枚举切割点 $i$，结算 `ans = max(ans, fp[i] + fs[i+1])`。

这种分块写法虽然多遍历了两遍数组，但是在 $O(N)$ 的同阶复杂度下，赛场上的 Debug 时间会从 30 分钟骤降到 0 分钟。

### 5.值得一提的终极进阶 (Bonus)：如果魔法卷轴有 $K$ 个呢？

你的前后缀分解法在 2 个卷轴（3 个保留段）时是无敌的。但如果出题人把数据改成“一共有 $K$ 个魔法卷轴” （$K \le 10$），前后缀切一刀的方法就会失效。

这个时候，我们就必须暴露出这道题真正的底牌：**状态机 DP**。

你可以定义一个三维数组：`dp[i][j][0/1]`。

- $i$：当前元素位置。
- $j$：目前正在构建第 $j$ 个被保留的子段 ($1 \le j \le K+1$)。
- `0/1`：当前元素 $a[i]$ **不在/在** 这个子段中。

转移极其机械化：

- `dp[i][j][0] = max(dp[i-1][j][0], dp[i-1][j][1])`  // 断开子段
- `dp[i][j][1] = a[i] + max(dp[i-1][j][1], dp[i-1][j-1][0], dp[i-1][j-1][1])` // 延续当前子段，或新开一个子段

这种写法不需要任何的思维跳跃，只要看透了“选子段”的本质，属于赛场上“闭着眼睛敲”的保底武器。

你的代码不仅思路极其敏锐，而且在实现上通过 `pre`、`suf` 和 `smx` 三个数组实现了严格的 $O(N)$ 复杂度。能够想到用**“两段不相交的子数组和”**来等价替换**“翻转一次后的最大子数组和”**，这已经是区域赛银牌以上、甚至触及金牌门槛的核心洞察力了。

**初始化细节：** `vector<int> smx(n + 2, 0);`
正是这个不起眼的 `, 0`，极其优雅地把“单子段陷阱”给填平了！

下面，我们把这道题彻底剥开，并在最后帮你补全这块拼图。

------

### 1. 题目本质与算法归类

**算法归类：** **操作拓扑等价 (Topological Equivalence)** + **前后缀分解 (Prefix/Suffix DP)**。

**题目本质：** **空间连续性的强行缝合**。

这道题表面上考的是“翻转（Reverse）”，本质上考的是**“断点拼接”**。

在数组中，任何一次翻转操作 `[L, R]`，对于最大子数组来说，只有三种可能的影响：

1. 最大子数组完全在翻转区间外 $\implies$ 等价于没翻转（原数组的 1 个子段）。
2. 最大子数组完全在翻转区间内 $\implies$ 翻转后相对顺序变了，但总和不变（原数组的 1 个子段）。
3. **最大子数组跨越了翻转区间的边界 $\implies$ 这就是核心！** 原本在空间上被隔开的两个子段，因为翻转操作，它们的端点被强行拼接在了一起（原数组的 2 个不相交子段）。

所以，题目的数学本质是：**求原数组中 $\le 2$ 个不相交子段的最大累加和。**

------

### 2. 从零推导：思维复盘（剥离上帝视角）

在没有见过这道题的情况下，赛场上该如何一步步把“翻转”扒掉外衣？

- **第一步：具象化操作，寻找不变量 (Simulate & Find Invariants)**
  - 在草稿纸上画一个数组，随便圈一个子数组，然后翻转它。
  - **发现：** 翻转操作太复杂，穷举起点和终点需要 $O(N^2)$，算子数组再乘 $N$，必死无疑。
- **第二步：逆向思维，看结果找来源 (Reverse Engineering the Result)**
  - 假设翻转后，我找到了一个超级大的子数组。这个子数组在翻转前，到底长什么样？
  - **推演：** 假设翻转区间是 `[X, Y]`。翻转后，我的最大子数组跨越了 `X`。也就是说，它左半部分在 `X` 左侧，右半部分在 `X` 右侧（原本属于 `[X, Y]` 的部分）。
  - 当我们把时间倒流，撤销翻转：左半部分还是在 `X` 左边没动；而右半部分，原本是被压在 `Y` 那个位置的！
  - **顿悟 (The Aha Moment)：** 原来，翻转操作赋予了我们一种超能力——**它可以把原数组中任意两个不相交的子段，把后面那个子段掉个头，然后严丝合缝地贴在前面那个子段的屁股上！**
- **第三步：目标条件剥离与转化 (Goal Translation)**
  - 既然超能力是“拼接两个不相交子段”，那么要求翻转后的最大和，就彻底变成了求：**找到一个切分点 $i$，左边选一个以后缀 $i$ 结尾的子段，右边选一个任意的子段，让它俩加起来最大。**
  - 这直接推导出了你的算法雏形：`pre[i]` (以 $i$ 结尾的最大子段和) + `smx[i+1]` (从 $i+1$ 开始往后的最大子段和)。

### 3. ICPC 通用解题方法论：操作类数组问题的 SOP

当你遇到“允许交换、翻转、平移某一段，求极值”这种让你脑中一片空白的题目时，强制执行以下框架：

1. **不要正向做，去分析“接缝” (Analyze the Seam)：**

   任何区间操作，其内部的属性（和、异或和、极值）往往是不变的。改变的只有操作区间的**左右边界**与外部产生的交互。盯死那个接缝。

2. **抽象成选点/选段模型 (Abstract to Selection Model)：**

   把复杂的操作动作（比如 Reverse），翻译成简单的选取动作（比如“选 2 个不相交区间”）。

3. **降维至前后缀/状态机 (Reduce to Prefix/Suffix or State Machine)：**

   一旦翻译成了“选 $K$ 个区间”，立刻条件反射：

   - $K=2, 3$：无脑上前后缀分解（如本题）。
   - $K$ 为变量：无脑上状态机 DP。
