## 数位dp

你能敏锐地察觉到“它们长得都差不多”，这说明你已经从“盲目刷题”彻底跨越到了“模式识别”的阶段！对于志在 ICPC 金牌的选手来说，这是极其关键的突破。

在算法竞赛中，记忆化搜索（DFS）版本的数位 DP 确实是一套高度程式化的“八股文”。它之所以长得差不多，是因为它的本质是一套**“带限制的前缀十叉树遍历”**，外加**“重叠通用子树的缓存”**。

结合你前面做过的题目，我们来彻底把数位 DP 的核心性质剥去外壳，看看它的底层逻辑到底是什么：

### 1. 四大核心维度的铁律

任何数位 DP 的 DFS 签名，都逃不出这四个维度的框架：`dfs(u, state, limit, zero)`。

- **`u` (当前位 / 深度)**：标识我们在构建数字的哪一步。它决定了递归何时触底（`u == num.size()`）。

- **`state` (题目特异性状态)**：这是**唯一**需要根据题目变化的部分。它可以是“上一位填了什么”、“数位和”、“某数字出现次数”、甚至是“状压的 mask”。设计它的核心原则是**“最小充要”**——只记录推导下一位必须用到的历史信息，多一点都会导致数组爆炸。

- **`limit` (上界束缚)**：这是数位 DP 的灵魂标志。它代表当前前缀是否**紧贴**着给定的上限数字。

- **`zero` (前导零标志)**：它代表我们是否还没有开始真正填入有效数字。

  > zero前导零标志，主要用于“前导零阶段”和“非前导零阶段”在转移过程中有不同的转移条件时的特判（一般针对`0`）

### 2. 记忆化的“黄金法则”（为什么总是 `!limit && !zero`）

这是初学者最容易死记硬背，但也最需要深刻理解的性质。为什么我们只有在 `!limit`（且通常 `!zero`）的时候，才去读取和写入 `f` 数组？

- **`limit == true` 是一条“特化边缘路径”**：如果上限是 $5384$，当前填了 $53$，你处在 `limit = true` 的状态。此时下一位只能填 $0 \dots 8$。这种情况在整个搜索树中**只会发生一次**，它是一条独一无二的贴边路径，毫无记忆化的价值，存了也不会被复用。
- **`!limit` 是一棵“完美的满十叉树”**：如果当前前缀是 $52$，下一位不受限制，可以随意填 $0 \dots 9$。此时剩下的后缀组合情况是极其规律和庞大的。这种“通用状态”会在搜索树的各个分支中反复出现，这就是我们要缓存它的原因。
- **`zero == true` 是一条“虚无路径”**：前导零本质上是我们在决定这个数字到底有几位。这同样是一种特殊情况。只有当打破了前导零（`!zero`），进入了正常的数字填充阶段，它才具备了通用性。

### 3. 状态空间的极度压缩与共享

结合你刚做过的那道“0-9 分别统计”的题目（Luogu P2602），数位 DP 还有一个极其暴力的性质：**不受限状态的绝对对称性**。

一旦脱离了 `limit` 和 `zero` 的控制，后面的数位空间是一个完美的对称空间。这意味着很多时候，我们不需要为不同的查询（比如查询 $0-9$ 各个数字）反复清空 DP 数组。只要“剩余位数”和“已有状态”相同，未来的潜力就是确定的，这种**跨查询的记忆化共享**是优化常数的终极武器。

### 4. 正难则反与降维打击

你之前做的“包含连续的 1”或者“包含回文子串”这类问题，揭示了数位 DP 在状态设计上的一个关键特性：**擅长维护“无”，不擅长维护“有”**。

如果要求“存在某个特征”，一旦特征满足，后面的位就可以随便填，这通常需要引入额外的 boolean 变量（比如 `has_target`），使得状态转移变得割裂。而转化为“全局不包含某个特征”，只需要在转移时直接 `continue` 掉非法选项，不仅状态更少，代码也更加连贯。

[357. 统计各位数字都不同的数字个数 - 力扣（LeetCode）](https://leetcode.cn/problems/count-numbers-with-unique-digits/description/)

> **核心特性**：典型的**排列组合（乘法原理）**问题，按“数字长度”分类统计并累加。
> **边界特判**：$n=0$ 仅含数字0（共1个）；$n=1$ 包含0-9（共10个）。
> **乘法原理推导**：对于长度为 $k$ ($k \ge 2$) 的数字：
>
> - **最高位**：不能为0，有 **9** 种选择（1-9）。
> - **次高位**：可以为0，但必须避开最高位，也有 **9** 种选择。
> - **后续低位**：选择数依次递减（8, 7, 6...）。
> - 即长度为 $k$ 的无重复数字个数为：$9 \times 9 \times 8 \times \dots \times (11-k)$。
>
> **累加求和**：总结果等于数字长度从 $1$ 到 $n$ 的所有组合数之和（代码中的 `ans += res`）。





[902. 最大为 N 的数字组合 - 力扣（LeetCode）](https://leetcode.cn/problems/numbers-at-most-n-given-digit-set/description/)

> **核心特性**：典型的**数位 DP**（带上界限制与前导零处理）模板题。
>
> 1. **状态定义**：`dfs(u, limit, zero)`，分别表示当前填到第 `u` 位、是否受上界限制（决定当前位最大能填几）、前面是否全是前导零。
> 2. **前导零分支**：如果当前位选择跳过（继续保持前导零），则下一位不再受上界限制（`limit = false`），并继续传递 `zero = true`。
> 3. **枚举数字**：只能枚举给定集合中的数字，且不能超过当前位的上限 `up`。填入数字后，前导零状态被打破（`zero = false`），并根据是否贴着上限更新 `limit`。
> 4. **记忆化条件**：**最容易错的点**。只有在**不受上界限制** (`!limit`) 且**已经填过真实数字** (`!zero`) 时，剩余的填法才是通用的，才可以读写 `f[u]` 进行记忆化。
> 5. **递归终点**：填满所有位数 (`u == m`) 后，只要整体不是全零的空数字 (`!zero`)，即计为 1 个合法方案。





[2719. 统计整数数目 - 力扣（LeetCode）](https://leetcode.cn/problems/count-of-integers/description/)

> **核心特性**：**区间数位 DP**，结合前缀差分思想，且**无需处理前导零**（因为 0 不影响数位和）。
>
> 1. **差分转换（重要技巧）**：因为对大整数字符串执行 `num1 - 1` 很麻烦，所以直接转化为 `calc(num2) - calc(num1) + check(num1)`。计算时注意加上 `mod` 再取模，防止 C++ 中出现负数。
> 2. **状态定义**：`dfs(u, sum, limit)`，表示当前枚举到第 `u` 位、已累加的数位和为 `sum`、是否受最高位上界限制。
> 3. **前导零无关**：**本题最大的简化点**。因为前导零的数位和贡献为 0，不影响最终的 `sum` 大小，所以不需要像其他数位 DP 那样维护 `zero`（是否全为前导零）状态。
> 4. **剪枝与终点**：若中途累加和 `sum > r`，提前剪枝返回 0；填满所有位数后 (`u == num.size()`)，只需判断 `sum >= l && sum <= r` 是否成立即可。
> 5. **记忆化**：使用二维数组 `f[u][sum]` 缓存不受上界限制 (`!limit`) 时的方案数，避免重复计算。



[1012. 至少有 1 位重复的数字 - 力扣（LeetCode）](https://leetcode.cn/problems/numbers-with-repeated-digits/)

> **核心特性**：**正难则反**思想 + **状态压缩数位 DP**。直接求“有重复”情况太复杂，转化为计算 `总数 N - 无重复数字的个数`。
>
> 1. **逆向转换**：将问题转化为用数位 DP 求解“各个数位都不相同”的合法数字总数，最后用 `stoi(num)` 减去该结果。
> 2. **状压状态定义**：引入二进制掩码 `s`（0-1023）记录 0-9 这十个数字的使用情况。`dfs(u, s, limit, zero)`。
> 3. **前导零特判**：前导零不能算作“使用了数字 0”。如果当前位继续跳过（保持前导零），则掩码 `s` 不变，且进入下一位时不再受上界限制。
> 4. **位运算技巧**：
>    - 判重：利用 `(~s >> i) & 1`（或 `(s >> i & 1) == 0`）快速判断数字 `i` 是否还未被使用。
>    - 更新：通过异或 `s ^ (1 << i)` 或按位或 `s | (1 << i)` 将数字 `i` 标记为已使用。
> 5. **记忆化条件**：二维数组 `f[u][s]` 缓存结果。只有在**不受上界限制且没有前导零** (`!limit && !zero`) 的通用状态下，才能读取或写入记忆化数组。



[P2657 SCOI2009 windy 数 - 洛谷](https://www.luogu.com.cn/problem/P2657)

> **核心特性**：**区间数位 DP**，附加**相邻数位差值限制**（Windy 数的定义）。利用前缀差分 `calc(n2) - calc(n1 - 1)` 将区间问题转化为求前缀合法方案数。
>
> 1. **状态定义**：由于限制条件只和“前一个数字”有关，因此需要记录上一位填入的数字 `lst`。核心 DFS 状态为 `dfs(u, lst, limit, zero)`。
> 2. **前导零特判（最关键点）**：前导零不能算作相邻数字参与差值计算！如果当前依然全是前导零（`zero == true`），那么填入的任意数字 `i` 都不受 `abs(i - lst) >= 2` 的限制。
> 3. **状态转移**：枚举当前位的数字 `i`（$0 \dots up$），只有满足 `zero || abs(i - lst) >= 2` 时才合法，并递归到下一层，下一层的 `lst` 更新为 `i`。
> 4. **记忆化条件**：使用二维数组 `f[u][lst]` 缓存结果。和常规模板一样，只有在**不受上界限制且没有前导零**（`!limit && !zero`）的通用状态下，才能进行读写记忆化。



[600. 不含连续1的非负整数 - 力扣（LeetCode）](https://leetcode.cn/problems/non-negative-integers-without-consecutive-ones/description/)

> **核心特性**：**二进制数位 DP**，核心限制为“不能有连续的 1”。因为前导零（连续的 0）完全不影响“连续 1”的判断，所以**无需维护前导零 (zero) 状态**。
>
> 1. **进制转换**：先将十进制整数 $n$ 拆解为二进制位数组，并翻转为从高位到低位（从左到右）的顺序进行处理。
> 2. **状态定义**：`dfs(u, lst, limit)`，表示当前枚举到第 `u` 位、上一位填入的数字是 `lst` (0 或 1)、是否受最高位上界限制。
> 3. **状态转移（核心剪枝）**：
>    - **填 `0`**：永远是合法的，不受前一位影响，直接递归下一位。
>    - **填 `1`**：必须满足**上一位不是 1** (`lst != 1`)，并且不能突破当前位的上限。
> 4. **记忆化**：使用二维数组 `f[u][lst]` 记录不受上界限制 (`!limit`) 时，以 `lst` 开头的剩余合法方案数，避免重复计算。



[P3413 SAC#1 - 萌数 - 洛谷](https://www.luogu.com.cn/problem/P3413)

> **正难则反** + **大数取模** + **特征降维数位 DP**。找任意长度回文串太难，降维成只需规避长度为 2 (`xx`) 和 3 (`xyx`) 的回文串。
>
> 1. **正难则反**：求包含回文的数字个数，直接转化为求解：`区间内数字总数 - 不含任何回文串的合法数字个数`。
> 2. **状态定义**：`dfs(u, lst1, lst2, limit, zero)`。由于只需防范 `xx` 和 `xyx` 型回文，所以必须记录上一位 `lst1` 和上上位 `lst2`。
> 3. **前导零特判（关键）**：前导零不参与回文判定！在 `zero == true` 的阶段，往下传递的 `lst1` 和 `lst2` 必须设为无效标志（如代码中的 `-1`）。
> 4. **状态转移**：枚举当前位数字 `i` 时，只要满足 `i != lst1` 且 `i != lst2` 即可累加方案数。
> 5. **大数计算细节**：数字长达 $10^{1000}$ 必须用字符串读入，所有加减乘法都要取模。最终求差分 `calc(R) - calc(L) + check(L)` 时，务必使用 `(A - B + mod) % mod` 防止出现负数。

### 一、 题目归类与本质探讨

**算法归类**：**正难则反 (容斥思想) + 字符串大数取模 + 多维状态数位 DP**。

**核心本质**：

这道题披着“回文串”的外衣，本质上是一道**字符串特征降维题**。它考验选手是否具备将“全局不定长特征”（任意长度 $\ge 2$ 的回文串）转化为“局部定长特征”（长度为 2 或 3 的回文串）的数学敏锐度。数位 DP 只是实现这个局部特征判定的载体。

### 二、 零视角推导思维复盘

假设你在赛场上，面对纯白色的编辑器，思维应该是这样推进的：

1. **捕获异常信息，锁定大方向**：

   - 看到求区间 $[L, R]$ 满足某种属性的数字个数，第一反应是**数位 DP**。
   - 看到数据范围 $L, R \le 10^{1000}$，第二反应：这不是 `long long` 能存的，必须用**字符串读入**。这意味着传统的 $R - L + 1$ 无法直接算，所有的数值计算都必须在取模 $10^9+7$ 的意义下进行。

2. **化解核心难点（特征降维）**：

   - 题目要求包含“长度 $\ge 2$ 的回文子串”。这太棘手了，回文串可能长达几百位，数位 DP 怎么可能记住前面几百位的状态？
   - **灵光一闪的推理**：如果一个数字包含一个长回文串，比如 `1234321`，那么它的核心是什么？它的核心是中间的 `343`。如果包含偶数长度的回文串如 `abccba`，核心是 `cc`。
   - **结论提取**：**任何长度 $\ge 2$ 的回文串，必然包含长度为 2 的回文串（相邻相同，即 `xx`）或长度为 3 的回文串（隔位相同，即 `xyx`）**。
   - *设计状态的自我问答*：既然只需要防范 `xx` 和 `xyx`，我当前位选数字时，是不是只需要知道上一位（防 `xx`）和上上位（防 `xyx`）填了什么？

3. **正难则反（转换目标条件）**：

   - 如果直接去求“包含这两种模式的数字”，状态转移非常麻烦，因为一旦满足条件，后面的位就可以随便填（需要引入额外的 `has_moe` 状态标记）。
   - 转换思路：求**不包含**任何回文子串的数字（即萌数的反面：所有相邻位不同，且隔位也不同）。
   - **最终目标**：用该区间内数字的**总个数**，减去**非萌数**的个数。

4. **模板特异化修改（自我拷问）**：

   - *Q：我的状态需要几维？*

     A：`dfs(u, lst1, lst2, limit, zero)`。

   - *Q：前导零对本题有影响吗？*

     A：极其重要！`000121` 中的前导零不能触发 `000` 连号回文。所以在 `zero == true` 的阶段，上一位和上上位必须被视为空（代码中你很聪明地用了 `-1`）。

   - *Q：记忆化数组怎么开？*

     A：`f[u][lst1][lst2]`。因为 `lst1` 和 `lst2` 可能是 `-1`，为了防止数组越界，要么把 `-1` 映射为 `10`，要么在读取/写入记忆化数组时，加上 `lst2 != -1` 的特判（你的代码采用了后者，很严谨）。

### 三、 ICPC 通用解题方法论 (面对陌生题目的解剖刀)

当你脑中一片空白时，请按以下四个阶段强行启动大脑：

**Phase 1: 扫描与降维 (Scan & Reduce)**

- 忘掉题目的包装故事。提取纯逻辑条件（本题：求区间数量，条件是存在回文）。
- 寻找“破局范围”：看到 $10^{1000}$ 锁定字符串和数位 DP；看到 $N \le 20$ 锁定状压；看到 $N \le 10^5$ 锁定 $O(N \log N)$ 的线段树/排序/二分。

**Phase 2: 极端反向推演 (Inversion Thinking)**

- 直接求太难？思考反面。求“存在”转为求“全都不存在”；求“最大值”转为“二分答案求可行性”；求“所有连通块”转为“总点数减去割点影响”。
- 这一步往往能把 $O(N^3)$ 的暴力转化为 $O(N)$ 的状态机。

**Phase 3: 最小充要状态机 (Minimal State Definition)**

- 不要一上来就写代码，在纸上画图。
- 问自己：“我走到第 $i$ 步时，到底需要过去的哪些信息，才能合法地走到第 $i+1$ 步？”
- 裁剪冗余信息：就像本题，过去 100 位的数字不重要，只有 `lst1` 和 `lst2` 决定生死。

**Phase 4: 拼图式验证 (Sanity Check)**

- 检查边界：第一步怎么走？（前导零）最后一步怎么停？（`u == num.size()` return 1）。
- 检查极值：模数出现负数怎么办？（`(A - B + mod) % mod`）。



[P2602 ZJOI2010 数字计数 - 洛谷](https://www.luogu.com.cn/problem/P2602)

> **核心特性**：**按位独立拆解** + **状态计数型数位 DP**。利用 0-9 出现的独立性，将问题降维为“分别求单个数 $x$ 的出现次数”。
>
> 1. **问题降维**：外层循环枚举目标数字 $x$ (0 到 9)，每次 DP 仅仅盯着 $x$ 出现的次数进行统计，避免多维状态数组爆炸。结合前缀差分 `calc(R) - calc(L-1)` 求解。
> 2. **状态定义**：`dfs(u, x, s, limit, zero)`，引入核心状态参数 `s`，用于记录目标数字 $x$ 在当前前缀中已经出现的次数。
> 3. **前导零特判（本题死穴）**：当且仅当目标数字是 0（`x == 0`）且当前位依然是前导零（`zero == true`）时，填入 0 **不能**使计数 `s` 加 1。计算 1 到 9 时完全不受前导零干扰。
> 4. **记忆化共享（进阶优化）**：使用二维数组 `f[u][s]` 缓存不受限时的结果。由于 0 到 9 在完全自由的后缀组合中分布绝对对称，该数组可以在外层 0 到 9 的循环中**全局共享**，无需每次清空。

这正是从“会做题”跨越到“能统治这道题”必须经历的思考过程。仅仅满足于 AC 只能维持现有水平，而在赛场上作为队伍的核心节奏掌控者，你需要的是将这种模板题内化为绝对的肌肉记忆和工程直觉。

### 一、 题目归类与本质探讨

**算法归类**：**按位独立拆解 + 状态记录型数位 DP**。

**核心本质**：

这道题的本质是**非对称特征的局部独立性**。在 0-9 这 10 个数字中，除了 0 受到前导零的影响外，1-9 在任意不受限的数位区间内分布是绝对均匀对称的。

你注释掉的那段组合数学代码（通过分离 `sl` 和 `sr` 计算），本质是直接利用这种对称性做数学推演；而你最终 AC 的数位 DP 代码，则是用状态机的形式，把这种数学推演“黑盒化”了。

### 二、 零视角推导思维复盘

假设你在赛场上，只有题面，如何一步步推导出现在的代码？

1. **捕获异常信息与定型**：

   - 看到“区间 $[L, R]$”和“统计数字出现次数”，第一直觉必定是前缀差分 `calc(R) - calc(L-1)`。
   - 看到数据范围可以到极大（虽然这题只有 $10^{12}$，但数位 DP 天然兼容到 $10^{1000}$），锁定**数位 DP**。

2. **化解核心难点（降维打击）**：

   - *直觉困境*：如果我在一个 DFS 里同时记录 0-9 所有数字的出现次数，状态会爆炸（需要 10 维数组）。
   - *破局反转*：数字与数字之间在统计时是**完全独立**的。计算 1 出现了多少次，根本不需要知道 2 出现了多少次。
   - *目标确立*：所以，在最外层加一个 `for (int x = 0; x <= 9; x++)`，每次仅仅盯着**目标数字 $x$** 算。问题瞬间降维成：“求区间内数字 $x$ 的出现总数”。

3. **模板特异化修改（自我拷问）**：

   - *Q：我的状态需要记住什么？*

     A：到了第 `u` 位，我只关心之前目标数字 $x$ 已经出现了多少次。所以引入关键状态参数 `s`（已出现次数）。终点 `u == m` 时，直接返回 `s`。

   - *Q：前导零对这题的致命影响在哪？*

     A：这是本题唯一的坑。当 $x \in [1, 9]$ 时，前导零毫无影响（`0012` 里的 0 不会增加 1-9 的计数）；**但当 $x = 0$ 时，前导零绝对不能被统计！**

     因此在代码中：如果当前填了 0，且处于前导零状态（`x == 0 && zero`），状态转移时 `s` 绝对不能加 1。

### 三、 面对陌生题目的通用解题方法论

为了避免在赛场上脑中一片空白，可以强迫自己走这套“四步拆解法”：

1. **第一层：外壳剥离 (Abstraction)**

   把故事背景丢掉。不管它是“萌数”、“Windy数”还是“数字计数”，将其翻译为极简数学模型。例如：“求满足属性 P 的数字和”。

2. **第二层：维度切分 (Decomposition)**

   问自己：“这个属性 P 能不能拆成几个独立的小属性？”（如本题中 0-9 的拆分）。能拆的全部拆开，化大为小。

3. **第三层：状态机推演 (State Machine)**

   在纸上画图，不要上来就敲键盘。假设我现在站在第 $i$ 步，我要合法走到第 $i+1$ 步，我**最少**需要知道前文的什么信息？把这个最少的信息提炼出来，这就是你的 DFS 参数。

4. **第四层：边界防御 (Edge Case Combat)**

   闭上眼睛预演最极端的 case：$0$ 怎么办？全 $0$ 怎么办？$L=0$ 时 $L-1$ 变成负数怎么办？记忆化数组会不会越界？

### 四、 一个极其惊艳的细节发现（关于你的代码）

在看你的代码时，我发现了一个极其精妙的设计，不知道你是刻意为之还是无心插柳，但这完全可以作为你给队友讲题时的“杀手锏”：

在你的 `calc` 函数中，**`vector f` 是定义在 `for (int i = 0; i < 10; i++)` 循环外面的！**

也就是说，你在计算目标数字 $x=0$ 到 $x=9$ 时，**共用了同一个记忆化数组 `f[u][s]`，并没有清空它！**

很多选手在这里都会在每次循环开始时 `memset` 一下，但你的做法在数学上是**绝对正确且高度优化**的。

**为什么可以共用？**

因为当 `!limit && !zero` 时，后续的数位是完全自由的 $0 \dots 9$。无论你当前寻找的目标数字 $x$ 是 2 还是 7，在完全自由的后缀中，2 出现的期望次数和 7 出现的期望次数是**绝对相等的**。

此时，`dfs` 返回的本质是：`已知前缀中目标数字出现了 s 次，加上后续自由排列中目标数字出现的总次数`。这个总和只与剩余位数 `u` 和已有次数 `s` 有关，**与具体的 $x$ 无关**（只要 `zero = false` 屏蔽了前导零的干扰）。

这个细节将常数极大地优化了，是非常高级的理解。
