## 01背包

[P1048 NOIP 2005 普及组 采药 - 洛谷](https://www.luogu.com.cn/problem/P1048)

聊起01背包，它是所有背包问题乃至诸多线性DP模型的基石。从动态规划的本质来看，01背包完美契合了DP的三大核心要素，并且在状态转移的推导和空间优化上非常经典。

我们可以从以下几个维度的DP特性来拆解：

### 1. 无后效性 (No Aftereffect)

这是能用DP解决的先决条件。在01背包中，当我们考虑“前 $i$ 件物品，且背包当前可用容量为 $j$”这一状态时，此时的最大价值 $dp[i][j]$ 一旦被计算出来并固定，就不会因为后续第 $i+1$ 件、第 $i+2$ 件物品的选择而改变。

它将复杂的历史决策（前面究竟选了哪几个具体物品）直接“浓缩”为了当前的**阶段**（考虑到了哪件物品）和**状态**（还剩多少容量）。未来的决策只依赖于当前的状态，而不关心这个状态的具体生成路径。

### 2. 最优子结构 (Optimal Substructure)

全局的最优解，是由局部子问题的最优解推导出来的。对于第 $i$ 件物品（重量为 $w[i]$，价值为 $v[i]$），我们面临的决策空间非常干净：**选** 或 **不选**。

- **不选**：当前的最优解直接继承自考虑前 $i-1$ 件物品、容量同样为 $j$ 的最优解。
- **选**：当前的最优解转移自考虑前 $i-1$ 件物品、且强行预留了 $w[i]$ 容量的最优解，再加上当前物品带来的价值 $v[i]$。

这两个子结构共同支撑起了经典的状态转移方程：

$$dp[i][j] = \max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])$$

### 3. 重叠子问题 (Overlapping Subproblems)

如果使用纯暴力回溯去搜索（枚举每个物品选或不选），搜索树的时间复杂度会是指数级的 $O(2^n)$。在这个过程中，系统会无数次计算类似“在还剩 $X$ 容量时，从前 $Y$ 个物品中挑选的最大价值”这样相同的子状态。

动态规划的核心就是利用数组（或者记忆化搜索）把这些重叠子问题的解缓存下来，用空间换时间，把指数级的时间复杂度降维打击到了伪多项式级别的 $O(nV)$。

### 4. 空间压缩与遍历方向的约束

在实际的算法实现中，二维状态转移通常会被压缩为一维的滚动数组，以将空间复杂度从 $O(nV)$ 优化到 $O(V)$。此时的状态转移方程退化为：

$$dp[j] = \max(dp[j], dp[j-w[i]] + v[i])$$

这里隐藏着01背包最迷人的一个DP实现特性——**内层循环必须逆序遍历**。因为一维数组在更新当前层的状态时，需要依赖上一层的旧状态。如果正序遍历，$dp[j-w[i]]$ 会被当前第 $i$ 件物品更新后的新状态覆盖，导致同一件物品被重复放入（这就变成了完全背包）。逆序遍历利用了无后效性，严格保证了“每件物品最多只取一次”的“01”特性。

### 2. 思维复盘：假如没有上帝视角，该如何推导？

如果你在赛场上第一次遇到这道题，如何从零拨开迷雾？下面是标准的心智推导流程：

**第一步：捕获关键条件并“黑盒化”（化繁为简）**

- **原始信息**：面对一个物品，我们手里有三个变量 $x, y, z$，以及一堆复杂的业务操作描述。
- **动作**：无论题目描述的业务逻辑是“先抵扣 $y$ 再返还 $x$”，还是“需要 $y$ 的门槛然后花费 $x$”，我们只关心一个核心指标——**资源的净变化量（Delta）**。
- **推导**：经过简单的数学代数化简，你发现选择这件物品后，最终对我们容量池的真实消耗是 $t = 2y - x$。此时，你就可以把原题描述的复杂业务扔进黑盒，将其抽象为一个极其干净的属性：**“这件物品的净代价是 $t$，带来的价值是 $z$”**。

**第二步：基于“净代价”的符号分类讨论（寻找破局点）**

- 当你算出 $t = 2y - x$ 时，金牌选手的直觉会立刻警觉：**$t$ 可能是非正数！** * **自我QA**：如果一件物品的代价 $t \le 0$ 意味着什么？
  - *意味着它不仅不消耗我的容量，反而还会扩充我的容量，并且白送我 $z$ 的价值！*
- **策略分化**：
  - 对于 $t \le 0$ 的“白嫖怪”物品：毫无疑问，**贪心全收**。不仅要拿，还要在做后续决策**之前**拿，因为它们能扩大我的初始本金 $m$，让我后续能买得起更多东西。
  - 对于 $t > 0$ 的物品：此时变成了“花费 $t$ 容量，获得 $z$ 价值”，并且受限于最终的总容量 $m$。这就是教科书般的 **01背包**。

**第三步：算法模板的特异化审查（排雷）**

- 在套用贪心+背包模板前，必须问自己一个关键问题：**贪心部分有顺序要求吗？** * *思考*：如果题目隐含了“进行操作的前提是当前容量必须 $\ge y$”，那么即使 $t \le 0$，也不能无脑拿，而是需要按照门槛 $y$ 升序排序再贪心（类似经典的打怪升级题）。
  - *确认*：从你的AC代码来看，显然题目没有这个硬性门槛（或者数据保证初始 $m$ 足够），因此直接 $O(N)$ 累加即可。

### 3. 通用解题方法论：面对陌生复合题目的剖析流

以后再遇到包含多维属性或复杂操作规则的题目时，可以套用这套方法论：

1. **算净值，找 Delta**：不要去模拟题目的过程，直接列方程计算 `操作后状态 - 操作前状态`。将复杂规则压缩成单一的代价权重。
2. **正负性隔离 (Isolation by Sign)**：
   - 只要代价或收益的公式有可能出现“正负反转”，立刻将物品分为两类（有利类 vs 消耗类）。
   - **有利类**：用贪心、拓扑排序或反悔贪心来吃干抹净。
   - **消耗类**：用 DP、背包或网络流来进行受限资源下的最优分配。
3. **分离“门槛条件”与“净消耗”**：时刻警惕物品是否有“Activation Threshold”。这是 ICPC 题目最爱设的坑。一件物品的属性应该被你看作 `(开启门槛, 净消耗, 价值)`，而不仅仅是 `(净消耗, 价值)`。



[494. 目标和 - 力扣（LeetCode）](https://leetcode.cn/problems/target-sum/description/)

> 面对“增加/减少”、“买入/卖出”、“左移/右移”这种对称操作，尝试设一个基准状态（比如全选左、全选负）。计算目标状态与基准状态的**差值 (Delta)**
>
> 假定全是负数，此时目标变成`m+sum`，且重新变成正数的变化变成`2`倍数

从省赛银牌向 ICPC 金牌跨越，并且要在日常训练中建立起清晰的算法体系来带领队伍，需要的正是这种把一道题“敲骨吸髓”的复盘精神。你的第二段 AC 代码非常亮眼，利用倍增将问题巧妙转化，避开了传统题解中常见的奇偶性判断。

我们来从更深层的竞技算法视角，把这道题彻底拆解。

### 1. 题目归类：算法的本质定位

这道题的本质是：**状态空间映射 (State Space Mapping) + 组合类 01 背包 (Combinatorial 01 Knapsack)**。

表面上，它是一个“决策树搜索”问题（每个数字选 `+` 或 `-`）。但从代数角度看，它是一个**集合划分问题**。题目的核心在于教我们如何将一个带有负数域、双向选择的非标模型，通过数学等价变换，映射到一个极其干净的、只有“选与不选”、且全部为非负整数的标准背包模型上。

### 2. 思维复盘：从零推导与上帝视角剥离

假设我们在赛场上第一次遇到这题，脑海中应该经历怎样的推演？

**第一步：确立最暴力的状态转移（定义起点）**

- **直觉**：每个数字有两个状态，构成了 $O(2^n)$ 的搜索树。
- **发现重叠子问题**：搜索过程中，我们只关心“前 $i$ 个数字组合出的『当前总和』是多少”，而不关心具体是哪些数字加减。
- **初步提炼**：状态定义呼之欲出——设 $dp[i][j]$ 为前 $i$ 个元素凑成和为 $j$ 的方案数。转移方程为 $dp[i][j] = dp[i-1][j-nums[i]] + dp[i-1][j+nums[i]]$。
- **遇到阻碍**：和 $j$ 可能是负数，而数组下标不能为负。
- **直观补救（你的第一版代码）**：引入偏移量（Offset）。既然总和的绝对值不超过所有元素之和 $S$，那就把零点平移到 $S$。这就是**偏移量 DP**的由来。

**第二步：数学视角的降维打击（寻找本质）**

- **自我发问**：偏移量 DP 还是太笨重了，不仅要维护两倍的空间，状态转移还容易越界。能不能把 `+` 和 `-` 的操作，变成标准的“选”或“不选”？

- **变量代换与逻辑转换**：

  假设我先把所有数字都当作负数，此时的总和是 $-S$。

  现在我要把其中一些数字从 `-` 变成 `+`，最终让总和达到 $target$。

  对于任何一个数字 $a_i$，把它从 `-` 翻转为 `+`，对总和的贡献（Delta）不是 $a_i$，而是 **$2a_i$**。

  我们要填补的差距是：$target - (-S) = target + S$。

- **终极等价条件生成**：

  原问题完美等价于：**在一个容量为 $target + S$ 的背包中，从 $n$ 个重量为 $2a_i$ 的物品中挑选，求装满背包的方案数。**

  这正是你第二版代码的核心逻辑！它比传统的求正数集 $P = (target + S)/2$ 更巧妙，因为你直接让物品重量翻倍，规避了除以 2 带来的非整除判定（奇偶性判断）前置。

### 3. 通用解题方法论：代数化简与模型靠拢

以后遇到看似陌生、充满复杂业务逻辑或双向操作的 DP 题，可以套用以下思考流：

1. **写出暴力的状态机表达**：不要管时间复杂度和数组越界，先用严格的数学符号把 $O(2^n)$ 或 $O(n!)$ 的转移方程写出来。
2. **寻找代数恒等变形**：面对“增加/减少”、“买入/卖出”、“左移/右移”这种对称操作，尝试设一个基准状态（比如全选左、全选负）。计算目标状态与基准状态的**差值 (Delta)**。
3. **单向化处理**：考察能否通过改变单个决策，以某种固定的 Delta 去弥补上述的差值。如果能，这个问题就从“双向决策”退化成了“单向挑选”（即 01 背包或子集和问题）。
4. **识别非法状态（快速剪枝）**：在代数变形的过程中，一定要顺手检查边界。比如算出来的目标容量小于 0，或者在某些变形下不满足整除条件，这些就是极其优雅的 $O(1)$ 剪枝点。



## 从序列决策降维成集合划分

[1049. 最后一块石头的重量 II - 力扣（LeetCode）](https://leetcode.cn/problems/last-stone-weight-ii/description/)

> 最后剩下的一块石头的重量，**一定可以表示为原数组中各个元素前添加 `+` 或 `-` 号后的代数和**
> 题目要求最后剩下的石头重量最小，也就是求 $\min |\sum(+) - \sum(-)|$
> 我们就必须让 $neg$ **尽可能接近 $\lfloor S / 2 \rfloor$**
> 跑一个体积和价值相等的背包，答案即为`sum - 2 * f[sum / 2]`

这道题的代码非常简洁，逻辑一气呵成。从这短短十几行代码中，可以看出你对 01 背包模板的掌握已经达到了“肌肉记忆”的程度。

在冲击 ICPC 金牌的路上，或者在日常带领 ACM 队伍做专题拔高训练时，最需要培养的正是这种“看透本质”的直觉。回想一下在 CCPC 或者省赛的赛场上，很多中高档题目其实就是把极其基础的模型，包上了一层极厚的“业务外衣”。

我们来像解剖标本一样，对这道题进行深度拆解。

### 1. 题目归类：算法的本质定位

这道题的本质是：**操作代数化 (Algebraic Transformation of Operations) + 集合划分模型 (Set Partition) + 01 背包**。

在表面上，它是一个带顺序的、动态的“消除游戏”。但它的底层灵魂和上一道题（Target Sum 目标和）是**完全同构的（Isomorphic）**。

Target Sum 是求“达到特定差值的方案数”，而本题是求“所有可能差值中的绝对值最小值”。它们的核心都在于：**将动态的多步操作，等价映射为静态的集合符号分配。**

### 2. 思维复盘：从零推导与上帝视角剥离

假如我们在赛场上第一次遇到这题，没有“标签”提示，该如何从零推导？

**第一步：警惕模拟，寻找不变量（代数展开）**

- **直觉陷阱**：看到“每次选两块石头粉碎”，很多人第一反应是贪心（每次选最大的两个）或者用优先队列模拟。但这很快会被反例打破（贪心通常不具备全局最优性）。

- **代数推演**：既然不能贪心，我们不妨在纸上写几步。假设有石头 $a, b, c, d$。

  粉碎 $a$ 和 $b$，剩下 $a-b$（假设 $a>b$）。

  拿 $a-b$ 和 $c$ 粉碎，剩下 $c - (a-b) = c - a + b$。

  拿 $c-a+b$ 和 $d$ 粉碎，剩下 $d - (c - a + b) = d - c + a - b$。

- **捕获关键**：无论你以何种顺序、何种配对方式粉碎石头，最后剩下的一块石头的重量，**一定可以表示为原数组中各个元素前添加 `+` 或 `-` 号后的代数和**！

**第二步：模型转换（向已知领域靠拢）**

- **提炼目标**：题目要求最后剩下的石头重量最小，也就是求 $\min |\sum(+) - \sum(-)|$。
- **设参化简**：设所有石头的总重量为 $S$，被分配到 `-` 号的石头集合总重量为 $neg$。那么被分配到 `+` 号的石头总重量就是 $S - neg$。
- **核心公式**：最后的重量差就是 $(S - neg) - neg = S - 2 \times neg$。
- **终极推论**：因为 $S$ 是固定的常量，要让 $S - 2 \times neg$ 尽可能小且大于等于 0，我们就必须让 $neg$ **尽可能接近 $\lfloor S / 2 \rfloor$**。

**第三步：算法定型**

- **自我问答**：如何在一个数组中挑出若干个数，让它们的和尽可能接近但不超过 $S/2$？
- **结论**：这就是教科书般的 **01 背包问题**。背包容量为 $\lfloor S / 2 \rfloor$，物品重量为 $a[i]$，价值也是 $a[i]$，求最大能装多满。

### 3. 通用解题方法论：操作类题目的降维打击

针对这种“给出一种复杂的操作规则，求最终状态的最优解”的题目，可以总结出一套通用的分析流：

1. **拒绝游戏树，寻找代数恒等式 (Algebraic Expansion)**：永远不要在大脑里模拟“第一步怎么走，第二步怎么走”。直接抽出 $3 \sim 4$ 个变量，写出经过操作后的代数多项式，观察各项系数的规律（比如本题发现系数只有 $\pm 1$）。
2. **提取守恒量与差值 (Invariants & Deltas)**：总和 $S$ 往往是最大的守恒量。寻找目标状态与 $S$ 之间的数学关系。
3. **消除顺序与维度 (Order Elimination)**：一旦证明了最终结果只与元素的“正负归属”有关，就彻底打破了题目原有的“操作顺序”限制。问题从“序列决策”降维成了“集合划分”。

### 4. 顶尖选手的编码流：心智隔离与快速实现

你是否发现，当你彻底想通了 $S - 2 \times neg$ 这个公式后，写代码其实不到一分钟？顶尖选手的流畅感来源于**严格的“心智阶段隔离”**。

- **Phase 1: 业务逻辑剥离 (Math Proof Only)**

  此时键盘是不动的。大脑只做代数推导，直到得出结论：“求容量为 `sum/2` 的 01 背包最大值”。

- **Phase 2: 预处理与边界设立 (Environment Setup)**

  敲下 `sum += a[i];` 和 `sum /= 2;`。**一旦这两行写完，你在心理上必须把“石头”、“粉碎”这些概念彻底拉黑。** 现在你的世界里只有一个纯净的数学问题。

- **Phase 3: 无脑调库 (Template Muscle Memory)**

  直接开启一维滚动数组的双层 `for` 循环。不需要再去思考“这个 `f[j]` 代表几块石头”，只需要确保 `f[j] = max(f[j], f[j-a[i]] + a[i])` 的模板默写准确无误。

- **Phase 4: 结果翻译 (Result Mapping)**

  最后输出答案时，再把纯数学结果翻译回业务需求：`ans - 2 * f[sum]`。

这种编码流的好处是：在写极其容易越界或出错的 DP 转移方程时，你的大脑是处于“空杯”状态的，没有业务逻辑的干扰，就能做到又快又准。

### 5. 值得一提的进阶视角：图论视角的降维

除了代数视角，这道题还有一个更抽象的**图论视角 (Graph Theory Perspective)**。

如果我们把每一块石头看作图中的一个节点，每次“粉碎”操作相当于在两个节点之间连一条边。最终所有的石头被粉碎成一块，等价于所有节点连成了一棵**树（严格来说是一棵二叉树）**。

在这个树结构中，根节点是最后剩下的石头。根据树的高度和左右子树的关系，每个叶子节点（初始石头）对根节点的贡献，本质上是由它到根节点的路径上经历了多少次“被减”来决定的。经过严格的图论奇偶性分析，最终依然会推导出：每个叶子节点的权重系数只可能是 $+1$ 或 $-1$。

这说明在高等算法中，**代数多项式和图论拓扑结构往往是同一种本质的两种不同投影**。掌握这种多角度互相印证的能力，对于在更高级别的区域赛中破解那些无从下手的构造题，会有极大的帮助。



## 背包维度和属性的特异化定义

[P2918 USACO08NOV Buying Hay S - 洛谷](https://www.luogu.com.cn/problem/P2918)

> “至少”约束下的最小代价模型
> 动态规划建立为`f[买了至少 i 榜干草]的最少花销`

这版代码的核心亮点在于内层循环的那个 `max(0, j - v[i])`。这是一个极其优雅的“越界坍缩”技巧，不仅免去了额外开大数组的麻烦，还完美贴合了这道题的物理意义。

我们来对这道题进行深度剖析，看看这一行神仙代码背后的推导逻辑。

### 1. 题目归类：算法的本质定位

这道题的本质是：**完全背包的变体 ——“至少”约束下的最小代价模型 (Unbounded Knapsack with "At Least" Capacity Constraint)**。

标准背包探讨的是在“不超过”给定容量下的价值最大化，它对应的是资源的**受限分配**。

而本题探讨的是在“至少达到”给定容量下的代价最小化，它对应的是目标的**超额覆盖**。这种模型在很多涉及“任务进度达成”、“最低采购需求”的场景中非常常见。

### 2. 思维复盘：那行 `max(0, ...)` 是怎么推出来的？

假设在赛场上，我们遇到了“至少买 $M$ 磅”的条件，大脑的推演流应该是这样的：

**第一步：识别基础模型与极性翻转**

- **捕获信息**：供应商有无限多包干草 $\rightarrow$ 完全背包，意味着 DP 遍历方向必须是**正序**的。
- **目标转换**：求最小花费 $\rightarrow$ 这是一个 Min 问题。DP 数组的初始化不再是 0，而是正无穷大 `INF`，且基础状态 `f[0] = 0`（需求为 0 磅时，花费为 0）。

**第二步：处理核心冲突（上帝视角的剥离）**

- **冲突点**：标准背包中，如果我们当前背包容量还剩 $j$，放进一个体积为 $v$ 的物品后，如果 $j - v < 0$，说明放不下了，状态非法。但在这道题里，如果还需要买 $j$ 磅，买了一包重 $v$ 磅的干草，且 $j - v < 0$，这叫“超额完成目标”，是**完全合法**的！
- **常规解法（笨办法）**：既然可能买超，那就把 DP 数组的容量开到 $M + \max(v_i)$。在求答案的时候，去遍历 $f[M]$ 到 $f[M + \max(v_i)]$ 这个区间，找一个最小值。这种做法不仅空间消耗大，还要额外找最大物品体积，略显繁琐。

**第三步：目标状态的本质转换（顿悟时刻）**

- **自我发问**：当我还差 $j$ 磅需求，但我强行塞了一包重 $v$ 磅（$v > j$）的干草时，我的**剩余需求**变成了什么？
- **逻辑坍缩**：剩余需求变成了负数。但在业务逻辑中，“还需要买负数磅”和“还需要买 0 磅”是绝对等价的！因为只要达到 0，任务就结束了，再往下负多少没有任何意义。
- **目标条件生成**：既然任何小于 0 的剩余需求都等价于 0 需求，那么在状态转移时，一旦 $j - v < 0$，我们就强行让它从 $f[0]$ 转移过来。这就诞生了这道题的灵魂代码：`f[max(0, j - v[i])]`。

### 3. 通用解题方法论：状态边界坍缩法 (State Boundary Collapse)

通过这道题，我们可以提炼出一个应对“至少达到某目标”类 DP 的通用杀手锏：

1. **明确目标驱动 (Target-Driven)**：将 DP 状态 $f[j]$ 的定义从“目前已经拥有 $j$ 容量”改为“目前**还差** $j$ 容量才能完成目标”。
2. **设定坍缩零点 (Zero-Floor)**：当一个动作造成的收益使得“剩余容量/任务量”变为负数时，不要去扩展数组的负数下标，也不要去开大数组的上限。直接使用 `max(0, next_state)`，将所有溢出的冗余状态全部“坍缩”到 $f[0]$（即任务已完成的终止状态）上。
3. **极性初始化**：求最小代价时，全量初始化为 `INF`，仅令 `f[0] = 0`。

这个方法论不仅适用于背包，在带有随机性的期望 DP（例如：还需要打多少点伤害才能击杀 Boss，暴击伤害可能溢出）中，同样是极佳的处理方案。

### 4. 顶尖选手的编码流：心智负担的清零

面对这种题，想要写得快且一次 AC，编码过程必须做到高度解耦：

- **Phase 1: 环境极性设定 (Polarity Setup)**

  一旦看到“求最小代价”，立刻敲下 `vector f(m + 1, inf); f[0] = 0;`。完成这一步后，大脑退出“最大/最小”的思考纠结。

- **Phase 2: 模板展开 (Template Unrolling)**

  既然是完全背包，直接无脑写出两层嵌套的 `for` 循环。外层 $i$ 从 $0$ 到 $n-1$，内层 $j$ 从 $0$ 到 $m$（注意这里内层 $j$ 可以直接从 $0$ 开始，因为我们有坍缩机制，不需要像标准背包那样从 $v[i]$ 开始以防越界）。

- **Phase 3: 植入“坍缩补丁” (Applying the Collapse Patch)**

  在最内层的状态转移方程中，面对 `f[j - v[i]]` 这个表达式，打上 `max(0, ...)` 的补丁。

- **Phase 4: 锁定收尾 (Final Lock)**

  直接输出 `f[m]`。因为超额购买的最优解已经被坍缩机制折算进了 `f[m]` 内部，所以无需在最后写一个 `for` 循环去搜寻最小值。

### 5. 进阶视角探讨：最短路径等价性 (Shortest Path Equivalence)

既然你作为队伍的核心，常常需要站在更高的视角去审视问题，我们可以把这道题和**图论**打通。

所有的动态规划问题，本质上都是在有向无环图 (DAG) 上求最短或最长路径。

- 这道题可以看作：有 $0$ 到 $M$ 这 $M+1$ 个节点。
- 每种干草就是一种有向边，从节点 $j$ 指向节点 $j - v[i]$，边的权重是 $w[i]$。
- 你的目标是求从节点 $M$ 到节点 $0$ 的**最短路径**。
- 那个 `max(0, j - v[i])` 在图论里的意义是什么呢？它等价于：我们把所有小于 0 的节点，全部合并成了一个**“超级终点 (Super Sink)”**，也就是节点 0。任何跳出边界的边，都会被强行牵引到这个超级终点上。

建立这种“DP 状态等价于图论节点”的直觉，在后期遇到极其复杂的、带有环或者需要结合 SPFA/Dijkstra 去跑 DP 的神仙题时，能让你拥有瞬间看破红尘的洞察力。

接下来，你想不想挑战一下这两种模型的融合？比如，如果在状态转移中引入了“相互依赖”的环，导致普通的循环推导失效，只能借用最短路算法来解 DP（这就是经典的 **同余最短路** 模型，它本质上也是一个变异的完全背包），我们要不要拿一道经典的同余最短路题目来练练手？



## 完全背包

[P1616 疯狂的采药 - 洛谷](https://www.luogu.com.cn/problem/P1616)

在日常带领队伍做 DP 专题拔高时，从 01 背包跨越到完全背包，往往是打通队员“遍历方向”认知、建立起空间压缩直觉的关键一战。

如果说 01 背包是“谨慎的单次投资”，分组背包是“残酷的同组互斥”，那么完全背包（Unbounded Knapsack）探讨的就是**“无限制的边际收益”**。

我们继续以“标本解剖”的视角，把完全背包的特性揉碎了来看。

### 1. 算法的本质定位：状态的无限复用 (Infinite State Reuse)

完全背包的物理模型是：有 $N$ 种物品，每种物品的体积为 $w[i]$，价值为 $v[i]$，**每种物品都有无限件可用**。求在容量为 $V$ 的背包中能装入的最大价值。

它的数学本质是：**解除决策上限的多元一次不定方程的最优解问题**。在 01 背包中，第 $i$ 件物品的状态转移强依赖于“前 $i-1$ 件物品”的干净历史；而在完全背包中，第 $i$ 件物品的最优解，可以并且**必须**依赖于“当前第 $i$ 件物品已经放入过”的历史状态。

### 2. 核心推导与“反向直觉”的破局

**第一步：朴素的三维认知（多重背包的退化）**

如果不做任何思维优化，按照最暴力的思路，我们会给每种物品加一个枚举数量 $k$ 的循环（$k \times w[i] \le j$）：

$$dp[i][j] = \max_{k \ge 0} (dp[i-1][j - k \cdot w[i]] + k \cdot v[i])$$

这会让时间复杂度飙升到 $O(NV \sum \frac{V}{w_i})$。在 ICPC 赛场上，这种复杂度几乎一定是 TLE（超时）的。

**第二步：数学代换与降维打击**

仔细观察朴素方程，当我们要计算 $dp[i][j]$ 时，其实 $dp[i][j-w[i]]$ 这个状态已经把“选 1 件、选 2 件、选 $k-1$ 件”的情况全部计算并包裹进去了。

通过数学上的错位相减（或者状态机视角的等价替换），我们可以把那层恶心的 $k$ 循环彻底消灭，得到一个极为优美的二维方程：

$$dp[i][j] = \max(dp[i-1][j], dp[i][j-w[i]] + v[i])$$

**请死死盯住这个方程的右半部分！** 是 $dp[i][j-w[i]]$，而不是 01 背包中的 $dp[i-1][j-w[i]]$。这意味着：**你要想在容量 $j$ 时再放一件物品 $i$，你可以直接从“已经考虑过物品 $i$”的容量 $j-w[i]$ 转移过来。**

**第三步：空间压缩与“正序遍历”的魔法**

当我们把第一维的 $i$ 抹去，使用一维滚动数组时，完全背包迎来了它最迷人的代码形态：

```cpp
for(int i = 0; i < n; i ++) {
    for(int j = v[i]; j <= m; j ++) {
        f[j] = max(f[j], f[j - v[i]] + w[i]);
    }
}
```

- **自我问答：为什么 01 背包必须逆序，完全背包必须正序？**
  - 在 01 背包中，逆序是为了**保护上一层的旧状态不被污染**，确保每件物品只拿一次。
  - 而在完全背包中，正序遍历恰恰是**为了刻意“污染”当前层的状态**！当我们在算容量 $j$ 的时候，容量 $j-w[i]$ 刚刚被第 $i$ 件物品更新过。我们拿一个“已经装了物品 $i$”的状态，再强行往里面塞一个物品 $i$，完美契合了“无限拿取”的物理意义。这就是 DP 空间复用中最精彩的 One-liner（单行代码）魔法。

### 4. ICPC 金牌视角的降维打击：当常规 DP 失效时

作为志在夺金的选手，我们绝不能停留在 $O(NV)$ 的模板上。如果在区域赛中，出题人给出了这样的数据范围：

> $N \le 50$，但是背包容量 $V$ 高达 $10^9$。

这时候一维数组根本开不下（MLE），时间也绝对超（TLE）。完全背包模板直接报废。此时该如何破局？

这会引入完全背包的高阶形态：**贪心 + 阈值 DP (Greedy with DP Threshold)**。

- **破局直觉**：既然物品可以无限拿，当背包容量极大时，我们**最终一定会在性价比（$v[i]/w[i]$）最高的那件物品上疯狂投入**。
- **本质推导**：只有在背包剩余容量较小，不足以用性价比填补空隙时，我们才会考虑那些“性价比次之，但体积更灵活”的物品来拼凑边界。
- **算法流**：
  1. 找出性价比最高的物品 $MAX\_I$。
  2. 设定一个阈值（通常是 $\max(w[i]) \times \max(w[i])$ 级别）。
  3. 对于容量小于阈值的部分，跑普通的完全背包 DP。
  4. 对于超过阈值的部分，全部无脑除以 $w[MAX\_I]$，直接用数学公式贪心求解。

这种将“无穷大状态空间”在数学上截断，化归为“局部 DP + 全局贪心”的技巧，是极其经典的 ICPC 考点。

### 1. 题目归类：算法的本质定位

表面上看，这是一道**双序列动态规划 (Two-Sequence DP)**。

但它的底层灵魂，其实是 **状态机模拟 (State Machine Simulation) + 完全背包的降维思想**。

在这道题中，模式串 $t$ 的每一个字符（或者 `字符+*` 的组合）都可以看作是一个“状态节点”或“物品”。字符串 $s$ 就是我们要花费的“背包容量”。

- 普通字符 `a-z` 和 `.`：这就像是 **01 背包**，遇到一次，只能消耗 $s$ 中的一个字符，然后状态必须转移到下一个。
- 带有 `*` 的组合（如 `a*`）：这完全就是 **完全背包**！你可以选择不选（消耗 0 个），也可以无限次选择（只要 $s$ 的当前字符匹配），而且它的状态转移方程和完全背包一模一样。

### 2. 思维复盘：那段被注释的 `for` 循环是如何涅槃的？

假如我们在赛场上第一次遇到带 `*` 的匹配，脑海中会经历怎样的推演？

**第一步：定义基础状态**

很自然地想到，设 $f[i][j]$ 表示 $s$ 的前 $i$ 个字符和 $t$ 的前 $j$ 个字符是否匹配。

**第二步：遇到拦路虎 `\*`（朴素思维 = 多重背包）**

当 $t[j]$ 是 `*` 时，它和 $t[j-1]$ 绑定成了一个整体。这个整体可以匹配 0 个、1 个、2 个... $k$ 个 $s$ 中的字符。

就像你注释掉的代码那样，第一反应是写一个 $O(N)$ 的 `for` 循环，去枚举这个 `*` 到底吃掉了多少个字符：

```cpp
// 你的朴素思维：向后扫描，直到字符不匹配为止
for(int k = i + 1; k <= n; k ++) {
    if(t[j] == '.' || s[k - 1] == t[j]) ok |= self(self, k, j + 2);
    else break;
}
```

**第三步：顿悟时刻（状态重用 = 完全背包）**

写完 `for` 循环后，稍微一想：算 $f[i][j]$ 的时候我们枚举了匹配 1 个、2 个、3 个... 那在算 $f[i-1][j]$ 的时候，是不是也枚举了匹配 1 个、2 个...？这中间产生了大量的**重叠子问题**！

```cpp
if(t[j] == '.' || s[i] == t[j]) {
    ok |= self(self, i + 1, j);
}
```

这就回到了我们聊“完全背包”时的那个魔法：**我们不需要枚举拿多少件，我们只需要决定“这一次拿不拿”，并且拿完之后“依然停留在当前物品上”。**

- **不拿（匹配 0 次）**：废弃掉当前的 `a*` 组合，去看前面的模式。转移为 $f[i][j-2]$。
- **拿一次（匹配 $\ge 1$ 次）**：前提是 $s[i]$ 必须和 $t[j-1]$ 匹配。一旦匹配，我们消耗掉 $s[i]$，但是**保持模式串的状态 $j$ 不变**，也就是转移到 $f[i-1][j]$。

通过这种“状态滞留”，成功把原本需要 $O(N)$ 的内层循环，拍扁成了 $O(1)$ 的状态转移。这就是你代码里 `f[i][j] |= f[i - 1][j]` 的数学本质。

### 3. 通用解题方法论：处理复杂序列匹配的“降维打击”

以后在 ICPC 赛场上遇到陌生的、带有复杂规则的字符串/序列匹配题，套用这个分析流：

1. **词法分析（Tokenization）**：永远不要把字符串看成一个个孤立的字符。先扫描一遍模式串，把有连带关系的符号（比如本题的 `a*`，或者其他题的转义字符 `\`）打包成一个不可分割的 **Token（逻辑单元）**。
2. **尾部对齐原则**：序列 DP 永远是从“最后一步”开始思考。问自己：“要让 $s[1..i]$ 匹配 $t[1..j]$，最后这一个 Token 发生了什么？”
3. **消除不定长操作**：一旦发现状态转移中出现了“不定长”的操作（比如删掉任意多个、匹配任意多个），立刻警觉！尝试寻找一个比它“短一步”的相邻状态（通常是 $i-1$ 或 $j-1$），利用那个状态已经计算好的结果来进行 $O(1)$ 的递推。

### 4. 顶尖选手的编码流：把恶心的边界转化为哨兵

双序列 DP 最容易卡住、写出满篇 Bug 的地方就是**数组越界**和**空串初始化**。你的第三版代码（加 `#`）完美避开了这些坑。标准的编码流应该是这样的：

- **Phase 1: 穿上防弹衣（加哨兵）**

  拿到题，什么都不想，先敲下 `s = '#' + s; t = '#' + t;`。

  这一步的心理暗示是：从此以后，字符串的有效下标严格从 1 开始。$i=0$ 和 $j=0$ 统一代表“空串”。

- **Phase 2: 锚定原点（初始化）**

  `f[0][0] = 1;` 两个空串绝对匹配。

- **Phase 3: 确定遍历域（注意 $i=0$）**

  外层遍历被匹配串 $s$，内层遍历模式串 $t$。

  **重中之重**：$i$ 的循环必须从 $0$ 开始！因为空串 `s=""` 是可以匹配模式串 `t="a*b*"` 的。而 $j$ 可以从 $1$ 开始，因为非空字符串不可能匹配空模式串。

  `for(int i = 0; i <= n; i++) for(int j = 1; j <= m; j++)`

- **Phase 4: 以目标串为核心进行 Branching**

  在这个循环里，大脑不处理 $s$，只处理 $t$。判断当前的 token 是什么？

  - `if (t[j] == '*')` -> 这是一个完全背包 token。写出拿 0 次（依赖 $j-2$）和拿 1 次（依赖 $i-1$）的转移。
  - `else` -> 这是一个 01 背包 token。写出普通的对角线转移（依赖 $i-1$ 和 $j-1$）。

这就是为什么你的第三版代码看起来极其丝滑，没有任何恶心的 `if (i > 0 && j > 0)` 的嵌套。

### 5. 值得一提的进阶视角：自动机 (NFA) 视角

站在更高维度的计算机科学视角，带有 `*` 和 `.` 的正则表达式，在编译原理中被称为 **非确定性有限状态自动机 (NFA)**。

这道题的 DP 矩阵，本质上是在做 **NFA 的图遍历**。

- 矩阵的列 $j$ 就是 NFA 的节点。
- 矩阵的行 $i$ 就是我们在处理字符串 $s$ 时的“时间步”。
- $f[i][j]$ 为 `true`，就意味着“在消耗了 $s$ 的前 $i$ 个字符后，机器可能停留在状态节点 $j$ 上”。

很多更变态的 ICPC 字符串匹配题（比如结合了 KMP 的 DP，或者 Aho-Corasick 自动机上的 DP），其实都是这道题的扩展。当你把 DP 数组理解为“自动机状态随时间的演变图”时，你会发现所有字符串匹配算法（包括原生的正则引擎）其实都是一家人。

## 分组背包

[P1757 通天之分组背包 - 洛谷](https://www.luogu.com.cn/problem/P1757)

从 01 背包走到分组背包，我们其实跨越了一个非常关键的认知门槛：**从“独立决策”走向了“互斥决策”**。

在区域赛甚至更高维度的比赛中，纯粹的分组背包裸题已经很少见了。它更多时候是作为一种**“底层算子”**，被无缝镶嵌在树形 DP（树上背包）、状态压缩 DP 或是带有复杂业务逻辑的图论模型中。

我们以同样的“标本解剖”视角，来深度拆解分组背包的特性。

### 1. 算法的本质定位：互斥约束 (Mutual Exclusivity)

如果说 01 背包探讨的是“选与不选”的二元论，完全背包探讨的是“无限制贪婪”的极限，那么多重背包和分组背包探讨的就是**“受限的资源分配”**。

分组背包的数学本质是：给定 $N$ 个集合，每个集合内有若干元素。你必须从每个集合中选择**至多一个**（或恰好一个）元素，在总代价不超过 $V$ 的前提下最大化收益。

它的核心灵魂只有两个字：**互斥**。同一组内的物品，天生是“你死我活”的排他关系。

### 2. 状态定义与转移矩阵：维度的坍缩

按照最严谨的模型定义，设 $dp[i][j]$ 表示考虑前 $i$ 个分组，在使用容量不超过 $j$ 的情况下的最大价值。

对于第 $i$ 个分组，我们的决策空间不再是单纯的 0 和 1，而是扩展成了一个大小为 $|Group_i| + 1$ 的多叉分支：

- **不选**本组任何物品：继承 $dp[i-1][j]$。
- **选**本组的第 $k$ 个物品（假设其代价为 $w[k]$，价值为 $v[k]$）：转移自 $dp[i-1][j-w[k]] + v[k]$。

状态转移方程自然地被写为：

$$dp[i][j] = \max \left( dp[i-1][j], \max_{k \in Group_i} (dp[i-1][j-w[k]] + v[k]) \right)$$

### 3. 核心陷阱与思维复盘：循环嵌套的“拓扑序”

当我们将二维状态压缩为一维滚动数组 $dp[j]$ 时，分组背包会暴露出它最臭名昭著的陷阱——**循环的嵌套顺序**。这也是无数队伍在赛场上 Debug 数小时的元凶。

标准的一维空间优化代码必须是这样的拓扑序：

```cpp
for(int i = 0; i <= n; i ++) {
    for(int j = m; j >= 0; j --) {
        for(int k = 0; k < v[i].size(); k ++) {
            if(j - v[i][k] >= 0) {
                f[j] = max(f[j], f[j - v[i][k]] + w[i][k]);
            }
        }
    }
}
```

**自我问答排雷：为什么 `j` 的循环必须在 `k` 的外面？如果把 `k` 放外面会怎样？**

- **如果 `j` 在外，`k` 在内**：对于当前的容量 $j$，我们枚举组内的每一个物品 $k$。在更新 `dp[j]` 时，等号右边的 `dp[j - w[i][k]]` 绝对是**上一层（前 $i-1$ 组）\**的干净状态。这意味着，对于当前的容量 $j$，组内所有的物品都在公平地竞争这\**唯一一次**的更新机会，完美保证了“互斥性”。
- **如果 `k` 在外，`j` 在内**：这就变成了“先拿组内的第 1 个物品去跑一遍 01 背包，再拿组内的第 2 个物品去跑一遍 01 背包”。此时，第 2 个物品更新时，可能会用到第 1 个物品刚刚更新过的新状态！这就导致同一组内的物品被**重复选中**，分组背包瞬间退化成了“有多个物品的 01 背包”。

**顶尖选手的肌肉记忆：** 分组背包的 `for` 循环三要素是：**组（阶段） -> 容（状态） -> 物（决策）**。一旦遇到组内互斥，逆序容量循环必须包裹住物品遍历。

### 4. 进阶视角探讨：向 ICPC 金牌区的延伸 (泛化物品与树上背包)

理解了分组背包的互斥性，我们就能打通 ICPC 树形 DP 中最难的领域之一：**树上背包（Tree Knapsack）**。

在树上做背包时，合并一个节点 $U$ 和它的子节点 $V$ 的过程，**本质上就是一个极度复杂的分组背包**。

- 子节点 $V$ 分配到体积为 0, 1, 2... $K$ 的状态，可以看作是一个“泛化物品”。
- 这个泛化物品有多个互斥的变体：要么给 $V$ 分配 1 体积，要么分配 2 体积……你只能从中**选择一种体积分配方案**。
- 这就完美对应了分组背包中“同组物品多选一”的性质。

因此，树上背包合并时的三层循环：遍历当前节点已算出的体积、遍历子节点的体积、更新总容量，其底层代码结构和分组背包的三层循环如出一辙。

## 多重背包

[P1776 宝物筛选 - 洛谷](https://www.luogu.com.cn/problem/P1776)

这份代码绝对是教科书级别的多重背包实现！同时包含了**二进制拆分**和**单调队列优化**两种写法，这说明你不仅追求能 AC 题目，而且在刻意建立自己的代码模板库和思维体系。能够手撕单调队列优化背包，说明你的 DP 功底和对状态转移代数变形的理解已经完全具备了冲击区域赛甚至更高奖项的硬实力。

作为背包家族的“终极形态”，多重背包（Bounded Knapsack）将 01 背包的“选与不选”和完全背包的“无限拿取”结合在了一起，演变成了一个**带有截断性质的连续资源分配模型**。

我们直接跳过基础的 $O(N \cdot V \cdot C)$ 暴力思路，从你代码里的这两种金牌级优化方案入手，来一场深度的思维解剖。

### 1. 题目归类：算法的本质定位

多重背包的本质是：**有限状态空间的基底映射 (Basis Mapping) 与 同余状态的滑动窗口极值 (Sliding Window Extremum over Congruence Classes)**。

- **二进制优化**，本质上是代数上的“基底映射”。它用极其轻量级的思维，通过改变物品的物理形态，将复杂问题**降维**回了 01 背包。
- **单调队列优化**，本质上是数论与数据结构的结合。它没有改变物品，而是直接在 DP 的状态转移矩阵上动刀，通过“同余分组”，将转移方程变成了一个**严格的一维滑动窗口问题**。

------

### 2. 思维复盘：二进制优化的“基底构建”

如果赛场上我们要自己推导二进制优化，核心直觉是什么？

**第一步：发现重复打包的冗余**

如果有 1000 个同样的苹果，我们不可能在买的时候一个一个拿去试。现实生活中我们会怎么做？我们会把它们按特定的数量打包。

**第二步：寻找最完备的“打包基底”**

我们需要把 $C$ 个物品打包成若干个大包裹，使得组合这些大包裹，**能够精确凑出 $1$ 到 $C$ 之间的任何一个整数，且不能凑出超过 $C$ 的数**。

- 自然联想：计算机底层是怎么表示任何一个整数的？**二进制**！
- 你的代码 `while(cnt)` 完美实现了这一点：$1, 2, 4, 8, \dots, 2^k$。这些数可以完美组合出 $1$ 到 $2^{k+1}-1$ 的所有数。
- **边界截断**：如果 $C$ 不是全 1 的二进制数怎么办？那就把最后剩下来的余数 $R = C - (2^{k+1}-1)$ 单独打包。这样就保证了最高只能拼出 $C$。

**第三步：模板特异化**

经过 `v.emplace_back` 和 `w.emplace_back` 的重构，原问题彻底灰飞烟灭。大脑立刻切换回 01 背包的“逆序一维滚动数组”模板。

------

### 3. 思维复盘：单调队列优化的“同余分组与滑动窗口”

这是你未注释的那段代码，也是多重背包真正的巅峰形态。它将时间复杂度从 $O(N \cdot V \cdot \log C)$ 强行压到了极限的 $O(N \cdot V)$。这段代码如果没有彻底理解其代数本质，在赛场高压下极容易写出 Bug。

**第一步：写出原始转移方程并发现阻碍**

$$dp[j] = \max_{0 \le k \le C} (dp[j - k \cdot v] + k \cdot w)$$

这里有一个巨大的痛点：当 $j$ 递增时，$j - k \cdot v$ 跨度很大，前后两个状态的决策集合似乎没有交集，无法使用双指针或滑动窗口。

**第二步：数论视角的破局 —— 同余分组**

仔细观察 $j$ 和 $j - k \cdot v$ 的关系。它们除以体积 $v$ 的**余数一定相同**！

这意味着，整个 $1$ 到 $V$ 的容量空间，被体积 $v$ 完美切割成了 $v$ 条互不相交的“平行宇宙”（也就是你代码外层的 `for(int k = 0; k < min(m + 1, v[i]); k ++)`）。在同一个平行宇宙里，容量的变化是连续的跳跃：$k, k+v, k+2v \dots$

**第三步：代数分离与相对价值评估（核心难点）**

在同一个余数分组里，设当前容量为 $j$，我们要从历史容量 $q.back()$ 转移过来。

此时我们比较两个历史状态 $A$ 和 $B$，谁更优秀？

我们**不能仅仅比较它们原本的 $dp$ 值**！因为容量小的状态，虽然当前 $dp$ 值可能低，但它留给当前物品的剩余空间大，能装入更多的当前物品。

你代码中这行极其精妙：

```
dp[q.back()] + (j - q.back()) / v[i] * w[i] <= dp[j]
```

它的数学本质是：**将所有历史状态，统一折算到当前容量 $j$ 时的最终收益，再进行绝对大小的比拼**。如果一个历史状态不仅本来价值低，就算补齐了差价（装满当前物品）还是比不过别人，那就彻底把它从单调队列中踢出去（`q.pop_back()`）。

------

### 4. 通用解题方法论：复杂状态转移的降维法则

面对诸如多重背包这种带有“大范围、有上限的选择”的 DP 模型，通用的降维法则如下：

1. **观察转移步长**：如果转移步长是 1（连续），优先考虑**前缀和优化**。
2. **提取同余类**：如果转移步长是一个常数 $v$（跳跃连续），立刻将状态空间按模 $v$ 的余数进行**分组隔离**。
3. **提取单调性**：写出转移方程，尝试把变量拆分为 $f(当前状态) + g(历史状态)$。如果 $g(历史状态)$ 可以独立计算并比大小，且选择范围是一个定长区间（比如最多选 $C$ 个），直接上**单调队列**。
4. **退路准备**：如果方程无法拆分，或者推导卡壳，立刻降级使用**二进制拆分**。在绝大多数 ICPC 赛场上，多一个 $\log$ 的复杂度通常不会成为卡题的致命点，稳健 AC 才是王道。

------

### 5. 顶尖选手的编码流：单调队列的无痛实现

单调队列优化背包最怕的就是内外层变量名写混。日常训练中，你可以按照这样的心智模型来规范你的编码流：

- **Phase 1: 开启平行宇宙 (Congruence Grouping)**

  外层 `i` 遍历物品。内层直接开启一个 `ndp` 备用数组（避免当前层污染上一层，因为必须正序遍历）。

  紧接着开启余数枚举：`for (int k = 0; k < v[i]; k++)`。

- **Phase 2: 队列初始化与滑动扫描 (Sliding Window Setup)**

  在余数循环内部，清空/新建单调队列 `deque<int> q`。

  开启真实容量的扫描：`for (int j = k; j <= m; j += v[i])`。**大脑此时只关注一维直线上的滑动窗口。**

- **Phase 3: 队列维护三步曲 (The 3-Step Queue Rule)**

  这三步雷打不动：

  1. **掐头（维系窗口大小）**：如果队头状态距离当前 $j$ 超过了 $C \times v[i]$（即超过了物品数量上限），直接从队头弹出。
  2. **去尾（维系单调性）**：新状态 `j` 准备入队前，把队列尾部那些“即使加上补齐价值也比不过 `j`”的废物状态全部弹掉。
  3. **入队（接纳新状态）**：将 `j` 的下标推入队列。

- **Phase 4: 转移赋值**

  `ndp[j] = 计算队头元素补齐后的价值;`

按照这四个步骤，你可以像写 01 背包一样丝滑地写出极其复杂的单调队列优化。

你的代码已经将背包的底层逻辑挖掘得很深了。从单调队列优化的推导可以看出，你对**状态分离方程**有着很好的敏锐度。沿着这条线继续往 ICPC 金牌区走，下一个绕不开的终极 DP 话题就是**斜率优化 DP (Convex Hull Trick)** 或者 **四边形不等式优化 (Divide & Conquer DP)**，它们处理的是“转移代价不再是线性，而是二次方程或更复杂形式”的情况。
