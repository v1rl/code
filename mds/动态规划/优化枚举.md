### 2. 思维复盘：零视角推导与模板特异化

假设我们第一次见到这道题，如何一步步推导到你现在写的这份完美代码？

**第一步：捕获条件与朴素建模**

- **关键条件**：“最多 $k$ 笔交易”，“不能同时参与多笔”。
- **直觉翻译**：由于每天只能“买、卖、无操作”，且受限于之前的状态，这天然是一个图论中的 DAG（有向无环图）寻路问题，即状态机 DP。
- **朴素状态**：我需要知道“今天第几天 $i$”、“已经做了几次交易 $j$”、“当前手里有没有股票 $0/1$”。所以最初的模板一定是 $dp[i][j][0/1]$。
  - $dp[i][j][0] = \max(dp[i-1][j][0], dp[i-1][j-1][1] + a[i])$ // 卖出或保持空仓
  - $dp[i][j][1] = \max(dp[i-1][j][1], dp[i-1][j][0] - a[i])$ // 买入或保持持仓

**第二步：目标条件转换（你的代码所做的事情）**

- **瓶颈**：朴素状态机虽然能解，但思维负荷略重。我们尝试从“收益”本身去推导。

- **代数变形**：假设我们在第 $i$ 天完成第 $j$ 次交易（卖出），我们在哪一天买入最划算？假设在第 $p$ 天买入（$p < i$）。

  那么第 $j$ 次交易的利润是：$a[i] - a[p]$。

  加上前 $j-1$ 次的利润，总利润就是：$f[j-1][p] + a[i] - a[p]$。

- **分离变量（极其关键）**：

  将式子重组为：**$a[i] + (f[j-1][p] - a[p])$**

  对于当前的第 $i$ 天来说，$a[i]$ 是常数。我们要最大化总利润，就等价于在所有 $p < i$ 的历史中，寻找 **$f[j-1][p] - a[p]$ 的最大值**。

**第三步：算法定位与特异化**

- **引入数据结构/思想**：我们需要频繁查询历史某个指标的最大值。因为 $p$ 是随着 $i$ 单调递增的，这不需要线段树，只需要一个**前缀最大值变量**即可！
- 这正是你代码中 `pmx[j-1][i]` 的来源：它代表了“在第 $i$ 天之前，完成 $j-1$ 次交易并买入第 $j$ 支股票后，手中剩余的最大现金”。

**自我问答设计（针对此类题目）：**

1. *“当前状态的转移需要遍历历史状态吗？”* -> 需要，找历史最佳买入点。
2. *“转移方程中，能不能把包含当前阶段 $i$ 的变量，和历史阶段 $p$ 的变量拆开？”* -> 能拆成 $a[i]$ 和 $(f - a[p])$。
3. *“拆开后，历史变量的部分可以边遍历边维护吗？”* -> 可以，用前缀最值。

### 3. 通用解题方法论：降维打击法

针对这种“不知从何下手”的序列决策题，建议形成以下思考肌肉记忆，我称之为 **“朴素表达 -> 代数剥离 -> 降维维护”** 方法论：

1. **容忍低效，写出暴力 DP / 记忆化搜索**：

   遇到难题，先不要想最终的最优解。用最直白的方式定义状态。不要怕多开维度（比如开到三维、四维）。只要能把状态转移方程**严谨地用数学公式写出来**，第一步就算胜利。

2. **公式展开，执行“变量分离”**：

   死盯你的转移方程。如果方程形如 $dp[i] = \max(dp[p] + cost(p, i))$，尝试把 $cost(p, i)$ 拆解为纯 $p$ 的部分和纯 $i$ 的部分。

3. **寻找前缀/区间结构，进行数据结构降维**：

   - 如果纯 $p$ 的部分只与前缀有关 -> **前缀最值优化**（如本题）。
   - 如果 $p$ 有一个滑动窗口限制 -> **单调队列优化**。
   - 如果拆不开，存在 $p$ 和 $i$ 的乘积项（如 $a[i] \times b[p]$） -> **斜率优化 / 李超线段树**。
   - 如果 $p$ 涉及复杂的区间加减 -> **线段树优化 DP**。

### 2. 贪心算法剖析：精妙的“反悔贪心”机制

你最终 AC 的这段未注释代码，非常精简，但背后的逻辑其实比 DP 更难想透。这是一种带有**反悔机制 (Regret)** 的贪心。

**核心逻辑：**

- 变量 `cur` 记录的是**“当前这一轮交易的最低成本（含手续费）”**。初始值为无穷大。

- **逻辑分支 1：逢低买入 (更新底线)**

  `if (a[i] + c < cur)`：如果发现今天的价格加上手续费，比我手里记着的成本还要低，那毫无疑问，应该放弃之前的计划，改为今天买入。更新 `cur = a[i] + c`。

- **逻辑分支 2：逢高卖出并“保留反悔权利” (虚拟卖出)**

  `if (a[i] > cur)`：如果今天的价格高于成本，说明**有钱赚**！

  1. 先把利润落袋为安：`ans += a[i] - cur`。
  2. **神来之笔**：把成本更新为今天的价格 `cur = a[i]`（注意，**没有加手续费 `c`**）。

**为什么 `cur = a[i]` 是神来之笔？（反悔原理）**

这是贪心算法不出错的关键。当你在第 $i$ 天卖出赚了差价后，如果第 $i+1$ 天价格还在涨（即 `a[i+1] > a[i]`），根据逻辑分支 2，你会再次触发利润累加：`ans += a[i+1] - a[i]`。

这相当于：

```
(a[i] - 真实的初始成本) + (a[i+1] - a[i]) = a[i+1] - 真实的初始成本
```

这就**等价于**：你在第 $i$ 天其实**没有卖**，而是继续持有到了第 $i+1$ 天才卖！

不加手续费的 `cur = a[i]` 就是一张“反悔券”：它免除了连涨行情下重复收取的手续费，把多次连续的差价合并成了一次长线交易。

**分析与评价：**

- **优点**：极致的运行效率和代码长度，空间复杂度极简 $O(1)$，常数极小。
- **缺点**：**思维跳跃性大，难以扩展**。如果题目加上诸如“每次买卖有 1 天冷冻期”的条件，贪心策略就会瞬间崩塌，很难打上补丁，必须切回 DP。

### 1. 题目本质归类：相对大小 DP (Relative Order DP)

这道题在区域赛的考点树中，属于 **排列 DP (Permutation DP)** 的一个经典分支，我们通常称之为 **相对大小 DP**（或者叫**插入 DP / 离散化状态 DP**）。

假设我们在赛场上第一次读到这题，如何从零推导到你的最终代码？

**第一步：朴素建模与碰壁 (发现信息冗余)**

- **直觉**：这是一个排列组合问题。由于受前面选择的影响，常规思路是状压 DP：$f[i][mask]$ 表示前 $i$ 个数，使用的数字集合为 $mask$ 时的方案数。

**第二步：目标条件转换 (信息压缩压缩)**

- **关键提问**：为了判断第 $i$ 步能不能填某个数，我**真的需要**知道前面填了什么吗？
- **核心顿悟**：不需要！如果当前要求是 `D`（下降），我只要保证我选的数比上一个数小就行。至于剩下的未选数字具体是 $1, 4, 5$ 还是 $2, 8, 9$，没有任何区别——**它们都构成了 3 个可选项**。
- **状态重构**：将状态从“具体数值”压缩为“相对排名”。我们不记录上一个数是多少，而是记录**上一个数在当时剩下的数字中，排第几小**（也就是你的状态定义：比它小的还没选的数有 $j$ 个）。

**第三步：算法推导与特异化 QA**

- **Q: 转移方程怎么写？**
  - A: 假设上一步选的数有 $j$ 个更小的可用数。
  - 如果当前是 `D` (下降)，说明我必须在这 $j$ 个数里挑。假设我挑了第 $k$ 小的数 ($0 \le k < j$)，那么转移就是把所有的 $f[i-1][j]$ 累加给 $f[i][k]$。

- **Q: $O(N^3)$ 复杂度能过吗？**
  - A: $200^3 = 8 \times 10^6$，在有些机器上能擦边过，但极有可能 TLE。
- **Q: 如何针对本题进行优化？**
  - A: 观察转移方程，发现它是**对一段连续区间求和**。凡是区间求和的 DP，都可以通过**维护前缀和/后缀和**将转移降到 $O(1)$。这就导出了你最终的 $O(N^2)$ 代码。

1. **无脑扩张 $R$**：只要未来的区块加上当前的 `res` 不超标，无条件跳跃。

   `while(r < n && res + rmn[r] <= k) { res += rmn[r]; r = rpos[r]; }`

2. **结算答案并维护 $L$**：此时 $R$ 已经卡住了（再往右一步就会超标）。

   - 计算当前长度 `r - l` 挑战答案。
   - 准备让 $L$ 吐出一个元素：`res -= a[l]`。

3. **防崩溃补丁（极度重要）**：如果在某一步，$L$ 已经追上了 $R$（即 $r \le l$），这说明连 $a[l]$ 这单独一个元素都大于 $k$，区间为空。这时候 $L$ 要往右走，必须强制拉着 $R$ 一起走，否则会出现 $L > R$ 的幽灵区间：`if(r == l) r++; else res -= a[l];` （你在代码里写成 `if (r > l)` 分支，逻辑完美一致）。

掌握了这种分离“扩张尝试”和“吐出维护”的结构，任何复杂的双指针都不会再让你写出死循环。
