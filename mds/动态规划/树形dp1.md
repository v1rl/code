## 树形dp

你现在的这种思考方式，正是从“题海战术”向“建立算法体系”跨越的关键一步。能够把这 5 道题放在一起对比，并提取出**“向子树要信息”**和**“合并全集信息”**这两个核心动作，说明你已经触摸到了**树形 DP（特别是自底向上类型）的灵魂**。

结合你刚刚做过的这些题目，我们将这种类型的树形 DP 提炼为一套名为**“黑盒状态封装与正交转移”**的方法论。

### 一、 核心思想：把子树看作“黑盒”

在处理常规数组或字符串的 DP 时，我们习惯于思考“第 $i$ 个状态怎么从第 $i-1$ 个状态推导过来”。但在树形结构中，父节点面对的是两个分支（左右子树）。

树形 DP 的第一视角必须是**“后序遍历”**。当你站在父节点时，你必须假设左右子树已经完全处理完毕。此时，你不能再去关心子树内部是怎么运作的，你只能把它们当成两个**黑盒**。你的唯一任务是：**打开这两个黑盒提供的信息包，结合你自身的数据，重新打包成一个新的黑盒，递交给你的父节点。**

### 二、 拆解“全集信息”：为了消除后效性

你提到的“分析父树得到答案需要子树的哪些信息”，在学术上叫做**状态的无后效性设计**。

父节点在打包新黑盒时，决不能出现“哎呀，因为左子树内部某个节点的特殊情况，我不知道该怎么计算了”的窘境。为了避免这种情况，子树返回的“全集信息”必须具备**完备性**（即包含父节点决策所需的一切维度）。

回顾你做的题目，信息的扩维轨迹非常清晰：

1. **极简维度（求树的直径）**：父节点只需要知道最长链，所以子树返回的全集信息仅仅是一个数字：`深度`。
2. **物理量累加（发金币）**：父节点要知道流通量，所以全集信息变成了客观的统计量：`[节点数, 金币数]`。
3. **状态机裂变（打家劫舍 / 摄像头）**：父节点在决定“选与不选”或“装与不装”时，强烈依赖子节点的当前状态。所以全集信息变成了互斥的状态数组：`[选的最大收益, 不选的最大收益]` 或 `[自己装, 靠儿子, 等爹救]`。
4. **校验与降维（最大 BST）**：父节点不仅要算大小，还要判断合法性。所以全集信息必须包含边界，变成了：`[是否合法, 最小值, 最大值, 权值和/节点数]`。

**方法论提炼**：不要一开始就去凑转移方程。先用白话问自己：“如果我是父节点，为了计算我的状态，我**最少**得向左右儿子索要哪几个独立的属性？”这些属性就是你的全集信息。

### 三、 整合与返回：状态流与答案流的剥离

当你拿到了左右子树的“全集信息”，接下来就是“整合并返回”。在这个过程中，最容易犯的错误就是**把“全局答案”和“传递给父节点的状态”混为一谈**。

仔细观察这几道题的高级解法，它们都完美贯彻了**“旁路更新”**的思想：

- **状态流（Return 的东西）**：永远是忠实反映当前以 $x$ 为根的子树的**客观物理状态**。无论这个子树是不是 BST（BST 题），无论它有没有形成最长直径（直径题），它都必须如实向上汇报自己的“节点和、极值”或“深度”。
- **答案流（Ans 变量）**：是一个游离于状态流之外的全局变量。只有当前节点的状态触发了某种“合法条件”（比如确认是 BST，或者计算出了当前的跨根路径长度）时，才去挑战一下全局最优解。

### 四、 幺元设计：树的基石

最后，整合全集信息时，最底层的基础是空节点（Null/0 节点）。

树形 DP 的代码是否优雅，很大程度上取决于你给空节点赋予了什么“全集信息”。

- 在发金币里，空节点是 `[0, 0]`。
- 在摄像头里，空节点是 `[0, 0, INF]`（不需要救，已经安全，绝对不能放）。
- 在 BST 里，空节点是 `[true, INF, -INF, 0]`。

设计好空节点的信息，能让你在整合父节点信息时，彻底抛弃繁琐的 `if (left == null)` 特判。

### 一、 算法类型与问题本质

**算法归类**：**树形 DP（Tree DP）** / **后序遍历（Post-order Traversal）** / **自底向上的信息合并**。

**问题本质**：**全局属性的局部化校验与状态压缩**。

判断一棵树是否为 BST 是一个“全局”属性（要求左子树的**所有**节点小于根，右子树的**所有**节点大于根）。这道题的本质在于，我们不能每次都去遍历子树的所有节点来做校验，而是必须把“子树中所有节点”这个庞大的集合，**压缩（提炼）**成极简的边界信息（最大值 `mx` 和最小值 `mn`），从而将 $O(N)$ 的全局校验降维打击变成了 $O(1)$ 的局部状态转移。

### 二、 思维复盘：假如没有上帝视角，如何从零推导？

在赛场上遇到这道题，从读完题到得出最优解，大脑的思考链路应该是这样的：

**第一步：确立最暴力的基准解法（捕捉核心矛盾）**

- *直觉思考*：要想找到最大的 BST 子树，最直接的方法是遍历每个节点，然后写一个 `check(node)` 验证以它为根的树是不是 BST。如果是，统计节点个数，更新最大值。
- *发现瓶颈*：这样的最坏时间复杂度是 $O(N^2)$。在 ICPC 中，树的节点数通常在 $10^5$ 级别，必然 TLE。为什么慢？因为父节点在 `check` 时，把子节点已经 `check` 过的信息又重复检查了一遍。

**第二步：倒推状态需求（将条件做等价转换）**

- *核心提问*：如果我站在节点 $x$，我已经知道了它的左儿子 $l$ 和右儿子 $r$ 的情况，我**还需要什么**才能在 $O(1)$ 时间内知道 $x$ 是不是 BST？
- *陷阱排雷*：很多人第一反应是 $x > l$ 且 $x < r$。但立马要推翻自己——不对，BST 要求 $x$ 大于左子树的**最大值**，小于右子树的**最小值**。
- *锁定目标条件*：因此，要判断 $x$ 所在的树是不是 BST，我必须向左右子树索要 3 个信息：
  1. 你本身是 BST 吗？（`isbst`）
  2. 你这棵树的最大值是多少？（`mx`）
  3. 你这棵树的最小值是多少？（`mn`）
- *顺带维护*：题目要求最大的 BST 节点数，所以我还要索要第 4 个信息：你这棵树里最大的 BST size 是多少？（`siz`）

**第三步：敲定算法框架**

- 既然当前节点的状态完全依赖于子节点的状态，那么毫无疑问，遍历顺序必须是**后序遍历（先处理左右子树，再处理根节点）**。这自然而然地导向了树形 DP 或带返回值的 DFS 递归。

### 三、 通用解题方法论：信息需求倒推法

面对陌生的树上/图上/序列上的最优化或计数问题，脑中一片空白时，可以使用这套**“信息需求倒推法”**模板来破局：

1. **宏观假设**：假设你当前处在一个通用的中间状态 $i$（比如树上的任意一个节点，或序列的第 $i$ 个位置）。
2. **本源提问**：为了计算状态 $i$ 的最终答案，我**必须**知道关于子状态（孩子节点或前一个位置）的哪些信息？
3. **试错与扩充状态**：
   - 一开始只设最少的信息（比如只问：你是 BST 吗？）。
   - 推导转移方程时发现卡住了（发现只知道子树是 BST 不够，还需要比较大小）。
   - **缺什么补什么**：果断在 DP 状态中增加维度（把 `min` 和 `max` 加入状态）。
4. **边界收敛**：思考最底部的状态（叶子节点或空节点）该如何初始化，使其能够无缝融入你推导出的转移方程。

这种方法的核心在于：**不要试图从开头往后顺推，而是站在中间往后看，缺什么武器就去打造什么武器。**

### 一、 算法归类与问题本质

**算法归类**：**树形 DP（自底向上的状态合并）**。

**问题本质**：**合法状态的严格界定与全局最优解的旁路更新**。

这道题的本质依然是将 $O(N^2)$ 的暴力校验压缩为 $O(N)$ 的状态转移。但它比上一题多了一个关键的陷阱：**节点权值可能为负数**。这意味着一棵合法的 BST，其权值和可能小于 0，而题目允许“不选择任何节点（即空树，和为 0）”。因此，问题的核心不仅仅是找到 BST，而是在确认合法 BST 的瞬间，将它的局部总和作为一个候选答案去“旁路更新”全局的最大值。

### 二、 思维复盘：从零推导与条件转换

假设我们在赛场上第一次遇到这道题，如何通过自我问答推导出完美解法？

1. **捕捉关键与初步转换**：
   - **已知条件**：需要找 BST，且要求 BST 的节点值之和最大。
   - **提问**：判断一棵树是 BST 需要什么条件？
   - **回答**：左子树是 BST，右子树是 BST，且当前节点值严格大于左子树最大值，严格小于右子树最小值。
   - **推论**：因此我必须向子树索要三个信息：`is_bst`, `max_val`, `min_val`。
2. **锁定目标与状态扩充**：
   - **提问**：题目要求的是“最大和”，我怎么算当前 BST 的和？
   - **回答**：如果当前树是 BST，它的和等于 `左子树和 + 右子树和 + 当前节点值`。
   - **推论**：所以我还需要向子树索要第四个信息：`sum`（子树的权值和）。
3. **应对特异化（坑点排查）**：
   - **提问**：如果当前节点带有负权，或者某个子树的和是负数怎么办？
   - **回答**：只要它符合 BST 定义，不管和是正是负，它都是合法的状态，必须如实向上传递 `sum`。但是，在更新全局答案 `ans` 时，我们需要警惕。因为题目隐含空树也是 BST，所以全局最大值 `ans` 至少是 0。
   - **修改模板**：初始化 `ans = 0`，并且**只有在当前子树是合法 BST 时**，才用当前的 `sum` 去挑战 `ans`。

### 三、 通用解题方法论：状态正交化与旁路更新

在 ICPC 赛场上遇到完全陌生的树上/图上最优化问题，脑子一片空白时，可以按照这套**“状态正交化”**的思考流来剖析：

1. **状态正交拆解**：不要试图用一个变量表达所有信息。问自己：“为了决定当前节点的操作，我**最少且必须**知道子节点的哪几个完全独立的属性？”（在这题里就是：合法性、极左极右边界、权值和）。
2. **定义幺元（空状态）**：树形 DP 最容易在叶子节点的子节点（空节点）处写出一堆 `if` 特判。去思考空节点的这几个独立属性应该是什么，才能让状态转移方程不需要任何修改就能顺滑执行。（空节点：`is_bst = true`, `max = -INF`, `min = INF`, `sum = 0`）。
3. **区分“状态流”与“答案流”（关键）**：
   - **状态流**：是需要 `return` 给父节点的信息。
   - **答案流**：是题目最终要求的东西（比如全局最大和）。
   - 很多时候，某个节点虽然不是合法解（比如它不是 BST），但它必须给父节点返回状态（告知父节点“我不是 BST”），此时**不要把“答案流”混入“状态流”中**。

### 一、 算法归类与问题本质

**算法归类**：**树形 DP（树上状态机）**。在图论的经典模型中，这道题是**“树上最小支配集（Minimum Dominating Set on Trees）”**的变种。

**问题本质**：**相邻节点状态的互相约束与完备性划分**。

与“最大独立集”（如打家劫舍 III，父子不能同时选）不同，“最小支配集”的核心是**覆盖**。一个节点的合法性不仅取决于自身，还取决于父节点和子节点。这种“跨越层级”的依赖关系，迫使我们必须将单一的“选/不选”裂变为能够表达“覆盖来源”的多维状态。

### 二、 思维复盘：假如没有上帝视角，如何从零推导？

在赛场上，面对这道题，严谨的思维推导链条应该是这样的：

**第一步：确立核心矛盾（灵魂三问）**

- *提问*：任意一个节点 $x$，它想被监控覆盖，只有哪几种途径？
- *回答*：只有 3 种。自己装摄像头、被子节点覆盖、等待父节点来覆盖。

**第二步：基于约束推导 DP 状态（状态裂变）**

- *尝试朴素状态*：如果像打家劫舍一样，只设 `f[x][0]`(不装) 和 `f[x][1]`(装)，够用吗？
- *发现缺陷*：假设 $x$ 不装（`f[x][0]`），父节点在转移时，根本不知道 $x$ 此刻是“已经被子节点覆盖了”，还是“正嗷嗷待哺等父节点来覆盖”。如果是前者，父节点可以不装；如果是后者，父节点必须装。**信息丢失了。**
- *状态扩维（正交化）*：既然“不装”包含了两种截然不同的物理意义，我们就必须把它拆开。从而得到完备的 3 状态：
  1. `状态 2`：**自己装了**（自带覆盖，且能辐射上下）。
  2. `状态 1`：**自己没装，但被子节点覆盖了**（自己安全，但帮不了父节点）。
  3. `状态 0`：**自己没装，子节点也没装**（自己不安全，**强行要求**父节点必须装）。

**第三步：推导转移方程（语义翻译）**

将确立好的状态进行严格的逻辑翻译：

- 要想得到 $x$ 的 `状态 2`（自己装）：子节点装不装无所谓，只要子节点自身合法就行。所以取子节点 3 个状态的最小值相加，再加 1。
- 要想得到 $x$ 的 `状态 0`（等父节点救）：这意味着子节点绝对不能装摄像头（否则 $x$ 就成状态 1 了），且子节点必须已经安全。所以子节点只能是 `状态 1`。
- 要想得到 $x$ 的 `状态 1`（被子节点救）：意味着左或右至少有一个子节点装了摄像头（状态 2）。枚举三种情况：左装右不装、右装左不装、左右都装。

### 三、 通用解题方法论：状态正交划分与完备性检验

在 ICPC 中遇到需要向父子节点“借状态”或“还状态”的复杂树上/图上问题时，可以使用这套**“状态正交与完备化”**方法论：

1. **寻找判定源**：问自己：“决定当前节点合法的因素，除了它自己，最多能延伸到距离它多远的节点？”（本题是距离 1 的父子节点）。
2. **罗列物理处境（MECE 原则）**：按照“相互独立、完全穷尽”（MECE）的原则，用大白话列出当前节点所有可能的物理处境。比如：“我自己有”、“我没有但别人给了我”、“我没有且极度缺”。
3. **消除后效性**：检查你列出的状态，如果在向父节点汇报时，父节点依然需要问“你这个状态具体是因为左儿子还是右儿子导致的？”，那就说明状态不够，需要继续增加维度，直到父节点拿这个数字就能直接用为止。

> 树上前缀和与状态回溯问题
> 题目要求路径必须自顶向下（从父到子），这在树的形态上构成了一条条独立的“链”。
> 因此，问题的本质是将一维数组中求“连续子数组和等于目标值”的技巧，降维平移到了树的单条祖宗链上。
> 采用 DFS 向下遍历，同时携带并计算当前路径的前缀和。利用哈希表记录当前“祖宗链”上各个前缀和出现的次数，通过 $O(1)$ 查找 当前前缀和 - 目标值，即可快速统计出以当前节点为底端的合法路径数。
