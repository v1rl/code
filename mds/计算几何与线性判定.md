### 一、 题目归类：给知识库打标签

这道题的本质并不是单纯的“模拟”，在算法竞赛的知识体系中，它属于以下交叉领域：

1. **计算几何 (Computational Geometry)**
   - **核心子类**：点到线段的距离 (Distance from point to segment)。
   - **基础操作**：向量运算（加减、数乘）、点积（投影）、叉积（面积/方向）。
2. **函数最值问题 (Convex Optimization)**
   - **核心性质**：欧几里得距离函数 $f(t) = \sqrt{(at+b)^2 + (ct+d)^2}$ 是一个**下凸函数 (Convex Function)**。
   - **常用解法**：由于其凸性，可以使用**三分法 (Ternary Search)** 求解；或者利用几何性质直接推导解析解。
3. **物理模型简化 (Relative Motion)**
   - **核心思想**：相对运动。将“双动点问题”转化为“单动点相对静止参考系的问题”。

**金牌选手的标签库：**

> 遇到“两个物体匀速移动求最值” $\rightarrow$ 立即联想 相对运动 + 几何轨迹。
>
> 遇到“不同时刻停止/变速” $\rightarrow$ 立即联想 分段处理 (Piecewise)。

### 二、 思维复盘：从零推导的逻辑链

假设我们现在面对一道陌生的题目，脑中是一片空白的。顶尖选手是如何一步步捕捉信号并建模的？

#### Phase 1: 信号捕捉与降维 (Signal & Reduction)

- **信号**：题目有两个动点 $T$ 和 $A$。
- **直觉**：同时处理两个变量 $T(t)$ 和 $A(t)$ 极其痛苦，容易出错。
- **转化 (Trick)**：能不能定住一个？
  - 令 $P(t) = T(t) - A(t)$。
  - 问题瞬间降维：求向量 $P(t)$ 的模长 $|P(t)|$ 的最小值，也就是求动点 $P$ 到原点 $O(0,0)$ 的最小距离。
- **验证**：因为 $T$ 和 $A$ 都是匀速直线运动，它们的坐标关于时间 $t$ 是线性的。线性减线性还是线性。
  - 结论：相对位置 $P(t)$ 的轨迹一定也是**直线**（或线段）。

#### Phase 2: 寻找“奇点”与分段 (Singularities)

- **障碍**：题目说“到达终点后停止”。
- **分析**：
  - 如果两人都不停，相对轨迹是一条无限长的直线。
  - 如果一人停了，另一人还在走，相对速度向量 $\vec{v}_{rel} = \vec{v}_T - \vec{v}_A$ 会发生突变。
  - 例如：青木停了，$\vec{v}_A$ 变为 0，此时 $\vec{v}_{rel} = \vec{v}_T$。
- **建模**：这是一个**分段函数**问题。
  - 计算两人到达终点的时间 $t_T$ 和 $t_A$。
  - 设 $t_{min} = \min(t_T, t_A)$, $t_{max} = \max(t_T, t_A)$。
  - 时间轴被切分为：$[0, t_{min}]$ 和 $[t_{min}, t_{max}]$。
  - 至于 $t > t_{max}$，两人都停了，距离不变，不需要考虑。

#### Phase 3: 模型标准化 (Standardization)

- 现在问题已经被剥离了所有物理外衣，变成了纯几何问题：
  1. 算出 $t=0$ 时的相对点 $P_0$。
  2. 算出 $t=t_{min}$ 时的相对点 $P_1$。
  3. 算出 $t=t_{max}$ 时的相对点 $P_2$。
- **最终目标**：求原点 $O$ 到线段 $P_0P_1$ 的距离 $d_1$，和原点 $O$ 到线段 $P_1P_2$ 的距离 $d_2$。
- **答案**：$\min(d_1, d_2)$。

### 三、 通用解题方法论：面对陌生题目的“手术刀”

为了在遇到新题时不卡壳，你可以建立这套 **"S.R.M." (Simplify, Represent, Map)** 方法论：

#### 1. Simplify (变量削减)

- **问自己**：变量太多了吗？能不能通过**相对量**、**差分**、**固定一个变量**来减少自由度？
- *本题应用*：双动点 $\to$ 相对向量。
- *举一反三*：区间查询题 $\to$ 差分数组；树上路径题 $\to$ LCA转化为根路径差。

#### 2. Represent (数学表征)

- **问自己**：这个过程的数学性质是什么？是线性的？单调的？还是凸的？
- *本题应用*：匀速运动 $\to$ 线性方程 $\to$ 凸函数距离。
- *举一反三*：如果看到求最大值/最小值，先画个草图，如果是单峰函数 $\to$ 三分法/爬山法；如果是线性约束 $\to$ 线性规划/半平面交。

#### 3. Map (映射到已知模型)

- **问自己**：剥离背景后，这是模板库里的哪个板子？
- *本题应用*：剥离物理背景 $\to$ 计算几何模板中的 `dist_point_to_segment`。
- *关键习惯*：**不要在 `main` 函数里推导数学公式**。如果你发现自己在写复杂的 `sqrt` 和 `if-else`，停下来，问自己这是不是个标准几何操作？如果是，调用模板（或者现场写一个封装好的函数）。

### 四、 顶尖选手的编码流 (Coding Flow)

很多选手卡住是因为**“边写边想”**。金牌选手的编码过程是**“配置优先，逻辑后置”**。

针对本题，一个行云流水的编码流应该是这样的：

#### Step 1: 基础设施搭建 (Infrastructure)

**不要**上来就写 `cin >> tx >> ty`。先写几何结构体。这看似浪费时间，实则是在为大脑减负。

- 定义 `Point` 结构体。
- 重载 `operator-` (向量减法)。
- 重载 `operator*` (数乘，用于插值)。
- 定义 `len()` (模长)。
- *心理状态*：此时完全不动脑子，只是机械地敲击键盘热身。

#### Step 2: 核心工具函数 (Core Solver)

实现一个黑盒函数：`get_min_dist(Segment s)`。

- 这里你有两个选择：
  1. **三分法**：适合不想推导几何公式时，无脑写一个三分。
  2. **几何投影法**：点积判断垂足。
- **金牌习惯**：将其封装。`solve_dist(Point start, Point end)`。保证这个函数只要输入对，输出一定对。

#### Step 3: 数据归一化 (Normalization)

进入 `solve()` 主逻辑。

- 读入数据。
- 计算时间 $P$ 和 $Q$。
- **关键操作**：**不要写两套逻辑！** 如果 $P < Q$，直接 `swap` 两人的所有数据。始终保证 $P$ 是那个走得久的人（或者反之）。
- *收益*：代码量减少一半，Debug 难度降低 90%。

#### Step 4: 关键帧计算 (Keyframes)

利用向量插值计算关键点。

- `Point M = Ts + (Tg - Ts) * (q_time / p_time);` // 高桥在 t=q 时的位置
- 此时代码非常语义化，像读英语一样清晰。

#### Step 5: 组装与输出

- `d1 = dist_to_segment(Ts - As, M - Ag)`
- `d2 = dist_to_segment(M - Ag, Tg - Ag)`
- `cout << min(d1, d2)`

------

### 示例：编码流视角的代码结构

C++

```
// --- Step 1: 基础设施 (盲打部分) ---
struct Point {
    double x, y;
    Point operator-(const Point& b) const { return {x - b.x, y - b.y}; }
    Point operator+(const Point& b) const { return {x + b.x, y + b.y}; }
    Point operator*(double k) const { return {x * k, y * k}; }
    double len() const { return hypot(x, y); }
};

// --- Step 2: 核心工具 (黑盒) ---
// 原点到线段 AB 的距离
double dist_origin_to_segment(Point a, Point b) {
    // 既然你是练算法，这里推荐写三分，通用性最强且不容易写错公式
    auto f = [&](double k) {
        Point p = a + (b - a) * k;
        return p.len();
    };
    double l = 0, r = 1;
    for(int i=0; i<100; i++) {
        double m1 = l + (r-l)/3;
        double m2 = r - (r-l)/3;
        if(f(m1) < f(m2)) r = m2; else l = m1;
    }
    return f(l);
}

// --- Step 3 & 4: 业务逻辑 (清晰的语义) ---
void solve() {
    Point ts, tg, as, ag;
    // 读入略...
    
    double p = (ts - tg).len();
    double q = (as - ag).len();
    
    // Normalization: 强行让 P >= Q，少写一半代码
    if (p < q) {
        swap(ts, as); swap(tg, ag); swap(p, q);
    }
    
    // 边界处理：防止除0
    if (p < 1e-9) { cout << (ts - as).len() << endl; return; }
    
    // 计算中间态：t=q 时，高桥在哪？
    // 公式：起点 + 向量 * 比例
    Point tm = ts + (tg - ts) * (q / p);
    
    // 阶段1：相对起点 -> 相对中点
    double ans1 = dist_origin_to_segment(ts - as, tm - ag);
    
    // 阶段2：相对中点 -> 相对终点 (此时青木已在 ag 停住)
    double ans2 = dist_origin_to_segment(tm - ag, tg - ag);
    
    cout << min(ans1, ans2) << endl;
}
```

按照这个 **“基础设施 -> 黑盒工具 -> 数据归一 -> 语义化组装”** 的流程，你会发现代码是一气呵成的，不会出现写到一半不知道变量 `x1` 到底是高桥的 x 还是青木的 x 这种情况。
