## 并查集

既然你的目标是 ICPC 金牌，并且在前面的题目中已经能非常熟练地用 DSU 维护连通块极值和 size 等附加信息，那我们就跳过最基础的“如何合并”和“如何查找”，直接从 **XCPC 竞赛的实战视角**，聊聊并查集几个最核心的进阶性质和思维拓展。

在高级别竞赛中，并查集往往不再是一个单纯的连通性工具，而是作为一个**底层的动态结构**，用来支撑更复杂的算法。

### 1. 均摊复杂度的极限与舍弃

并查集最引以为傲的是它的时间复杂度。

- **极致的 $O(\alpha(n))$**：同时使用**路径压缩（Path Compression）**和**按秩/大小合并（Union by Rank/Size）**时，单次操作的均摊复杂度为反阿克曼函数 $O(\alpha(n))$，在宇宙可观测的物理尺度内 $\alpha(n) \le 4$，可以视作绝对的常数。
- **性质的舍弃（撤销操作）**：在很多高级树分治、线段树分治或莫队算法中，我们需要在图上做“时光倒流”（即**撤销**某次合并操作）。路径压缩会彻底破坏树的原有拓扑结构，导致无法撤销。因此，**可撤销并查集**必须**完全舍弃路径压缩**，仅保留按秩/大小合并。此时树的高度被严格保证在 $O(\log n)$，单次操作的复杂度退化为极其稳定的 $O(\log n)$。

### 2. 信息传递的偏序关系（带权并查集）

你刚才的代码维护的是“连通块整体的信息”（如极值、大小），这属于**集合属性**。并查集的另一个强大性质是它可以维护**节点之间的相对属性（边权）**。

- **性质**：如果在执行 `find(x)` 进行路径压缩的同时，更新节点 $x$ 到其直接父亲的距离/权重，我们就可以维护节点 $x$ 到所在连通块根节点 $root$ 的相对关系。
- **实战意义**：这被称为**带权并查集**。只要两个元素之间的关系满足**矢量加法（或者群的性质，比如异或和、模意义下的加法）**，就可以用带权并查集来维护。比如经典的“区间异或和查询”、“种类并查集（敌人的敌人是朋友）”、“模 3 意义下的食物链关系”。

### 3. “只增不减”的单调性与离线处理

- **性质**：标准的并查集只支持“加边（合并）”，不支持“删边（分裂）”。它是一个信息单调递增的过程。
- **实战转化**：这就解释了为什么前面那道题（最小化恶意软件传播）需要用到**正难则反**的 Trick。当你面临一个需要“从图中不断删点/删边，并随时查询连通块信息”的问题时，XCPC 的标准套路是：**将操作全部离线，反转时间轴**。把最后的状态作为初始状态，把“删边”变成“加边”，用并查集轻松维护。

### 4. 树形拓扑的延展（Kruskal 重构树）

刚才提到的点权转边权 Kruskal 式加边，其实只差最后一步就能推导出一个极其强大的图论数据结构。

- **性质**：在并查集合并 $x$ 和 $y$ 所在的两个连通块时，如果我们**不直接合并**，而是新建一个虚拟节点 $Z$，将 $Z$ 的点权设为当前处理的边权，然后让 $Z$ 作为 $x$ 和 $y$ 所在集合的父亲。
- **实战意义**：当所有边处理完后，并查集不仅告诉你它们连通，还在底层形成了一棵严格包含 $2n-1$ 个节点的二叉树——这就是 **Kruskal 重构树**。这棵树有一个极其华丽的性质：**原图中节点 $u$ 到节点 $v$ 的所有路径中，最大边权的最小值，就等于重构树上 $u$ 和 $v$ 的最近公共祖先 (LCA) 的点权。** 结合倍增算法，可以秒杀海量的“路径瓶颈极值”查询问题。

## 最小交换次数：并查集与连通块分解

[765. 情侣牵手 - 力扣（LeetCode）](https://leetcode.cn/problems/couples-holding-hands/description/)

给定一个长为 $2N$、包含 $0 \sim 2N-1$ 的数组，相邻两个位置视为一个座位。定义 $(0,1), (2,3) \dots$ 为目标配对集合。求：在允许任意交换两数位置的前提下，使所有目标配对均相邻的**最小交换次数**。

------

### 思路总结

本题的本质是**图的连通块分解**。

1. **建模转换：** 将每对目标配对（情侣）抽象为一个节点（编号为 `x / 2`）。如果两个不同配对的人被初始安排在了同一个座位上，就在这两个配对之间连一条边。
2. **核心结论：** 连好边后，图会被划分为若干个环（连通块）。对于一个包含了 $k$ 对情侣的错误连通块，只需 $k-1$ 次交换就能将他们全部理顺。
3. **公式推导：** 假设共有 $N$ 对情侣，形成了 $C$ 个连通块，那么**最小交换次数 = $\sum (k_i - 1) = N - C$** （即：总对数 - 连通块数量）。
4. **代码巧思：** 您的代码通过并查集（DSU）将座位上两人的组号 `a[i] / 2` 和 `a[i+1] / 2` 进行合并。代码中巧妙地将答案初始化为数组长度 $n$（即 $2N$），减去所有的 DSU 根节点数量（其中包含 $C$ 个实际连通块的根，以及 $N$ 个未被使用的多余节点的根），最终得到的结果恰好是 $2N - (C + N) = N - C$，逻辑非常简洁闭环。

## 分时段并查集：瞬时传递与无效合并的重置撤销

[2092. 找出知晓秘密的所有专家 - 力扣（LeetCode）](https://leetcode.cn/problems/find-all-people-with-secret/description/)

$N$ 个人，0 号和 `firstPerson` 初始获知秘密；给定一系列会议 `(u, v, time)`，秘密会在参会者间即时传递。求最终所有获知秘密的人。

------

### 二、 思路总结

本题的核心挑战在于**秘密传递的即时性**（同一时刻内秘密可以多级传递）和**时间的一致性**（秘密不能传回过去）。

1. **按时排序：** 将会议按时间 $t$ 升序排列，保证秘密按时间线向后流动。
2. **分批处理：** 将同一时刻 $t$ 的所有会议作为一个批次处理。在这一时刻内，所有参会者构成的连通关系是“瞬时”的。
3. **并查集动态维护：**
   - **合并：** 将同一时刻开会的人在并查集中 `merge`。
   - **判定与撤销（核心）：** 处理完该时刻的所有会议后，遍历该时刻的参会者。如果某人所在的连通块**没有**包含知秘者（即不与 0 号节点连通），说明他在该时刻并未获得秘密。
   - **重置状态：** 对于未获秘的人，必须将其在并查集中**重置**（`dsu.set(i, i)`），断开本次会议产生的无效连接，防止其错误地将“未知的秘密”带到下一个时间点的会议中。



## Kruskal 式点权加边

[2421. 好路径的数目 - 力扣（LeetCode）](https://leetcode.cn/problems/number-of-good-paths/description/)

**给定一棵点权树，求满足以下条件的路径总数：起点和终点点权相同，且路径上所有节点的点权均不大于该端点点权。** *(注：单个节点也算一条合法路径)*

------

### 核心 Trick 解析

这道题属于非常典型的“路径瓶颈”问题，它的核心跳跃在于**将“对路径的静态查询”转化为“按阈值动态加边”的过程**。具体的思维跳跃（Trick）可以拆解为以下三层：

1. **Trick 1：点权转边权（提取瓶颈）**

   题目限制了路径上的点权不能超过端点。如果我们要从节点 $u$ 走到节点 $v$，这条边能被走通的“门槛”就是 $\max(val[u], val[v])$。因此，我们可以直接将无权边的边权定义为两端点点权的最大值。这是图论中处理点权连通性问题的常用起手式。

2. **Trick 2：离线加边（Kruskal 思想）**

   既然路径的合法性由最大点权决定，我们可以采用类似 Kruskal 求最小生成树的思路：**将所有边按重新定义的“边权”从小到大排序，依次加入图中。** 这个跳跃极其关键——当你以这种顺序加边时，可以绝对保证：**当前连通块内的所有节点的点权，一定小于等于当前正在处理的边权。** 这样就彻底规避了“判断路径中间节点是否越界”的繁琐操作。

3. **Trick 3：并查集维护附加信息（乘法原理）**

   在从小到大加边的过程中，当一条边连接了两个连通块时，如果这两个连通块的“最大点权”相同（且等于当前正在处理的边权），那么这两个连通块内部拥有该最大点权的节点，就可以两两组合形成合法路径。因此，让并查集额外维护 `mx`（最大点权）和 `cnt`（最大点权出现次数），每次合并时执行 `ans += cnt[x] * cnt[y]` 即可。

## 正难则反 / 干净连通块的唯一感染源 (贡献法)

[928. 尽量减少恶意软件的传播 II - 力扣（LeetCode）](https://leetcode.cn/problems/minimize-malware-spread-ii/)

给定无向图和若干初始感染点（感染沿边蔓延）。求**彻底删除**（抹除该节点及其所有连边）哪一个初始感染点，能使得最终全图的感染节点总数最少。若收益相同，取索引最小的。

------

### 核心 Trick 解析

这道题是典型的“正难则反”与“贡献法”的结合。它的核心跳跃（Trick）在于**视角的翻转：从“模拟污染过程”转变为“评估安全区的拯救条件”**。具体可以拆解为以下两层：

1. **Trick 1：正难则反（先处理干净节点）**

   常规的线性思维是：枚举删除每个感染点 $\rightarrow$ 跑一遍 BFS/DSU 算感染总数 $\rightarrow$ 取最小值。这种正向模拟不仅复杂度偏高，而且存在大量重复计算。

   **跳跃点在于**：完全反过来，先把所有**未感染的“干净点”**用并查集缩点，把它们合并成若干个孤立的“安全连通块”。

2. **Trick 2：贡献法与唯一依赖（割点逻辑）**

   建好安全连通块后，再把感染点当成外围节点，观察它们与安全块的连接关系。一个安全块最终会不会被感染，取决于它的“感染源”数量：

   - 连接了 $\ge 2$ 个不同的感染点：绝对没救了，因为题目只能删一个点，删了一个还有另一个会感染它。

   - **连接了恰好 $1$ 个感染点：这是破局的关键！** 只要删掉这个唯一的感染源，这整个安全连通块就被“拯救”了。

     因此，原本复杂的全局模拟，被巧妙地转换成了局部贡献统计：遍历所有感染点，如果它是某个安全块的**唯一感染源**，就把该块的大小累加到这个感染点的“拯救贡献值”上。最后找出贡献值最大的感染点即可。
