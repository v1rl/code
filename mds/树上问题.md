下面我按你想要的几个维度，把这题“从本质—推导—模板化—编码流”完整复盘一遍，并把你 AC 代码里每个判定分支背后的结构意义讲清楚。为了方便叙述，我把一次操作称为“一轮”。

------

### 1）题目归类：它到底是哪一类题？

**核心类型：树上概率 + 结构判定（固定轮数的“可消除结构”分类）**

更具体一点：

- **固定轮数（3 轮）** 强烈暗示：不要做一般过程 DP，而是做 **“剩余轮数能清空的连通块形状分类”**。
- 每轮对每个连通块 **等概率选 1 点删除**，等价于：
  你关心的是某个连通块在未来几轮内能否被清空，以及清空所需的“关键点”被选中的概率。
- 在树上，删除一个点后连通块会按邻边分裂，这让“形状分类”特别好做。

所以这题本质上不是“树形 DP”那种，而更像：

> **先刻画“2 轮内必清空/可清空”的树结构 → 再枚举第 1 轮删哪个点，验证删完后每个分块是否属于可清空结构，并乘上对应概率。**

------

### 2）没有题解上帝视角，如何从零推导到你的做法？

#### Step A：先把“3 轮”拆成“第 1 轮 + 剩余 2 轮”

初始只有一个连通块（整棵树），第 1 轮随机删掉一个点 `x`，概率 `1/n`。

删掉 `x` 后，树变成若干个连通块（每个对应 `x` 的某个邻居方向的子树）。
接下来还有 **2 轮**，要在 2 轮内把这些连通块全清空。

于是问题变成：

枚举第 1 轮删的点 `x`：
若删完 `x` 后出现的每个连通块都能在 2 轮内清空，则本次 `x` 贡献
`1/n * ∏(每个连通块在第2轮选到正确点的概率)`；否则贡献 0。
最终答案对所有 `x` 求和。

这一步就是你代码里 `res = 1; res /= n;` 然后遍历 `adj[x]` 逐块相乘的来源。

------

#### Step B：关键：刻画“**一棵树在 2 轮内被清空**”当且仅当什么形状？

设现在只剩 **2 轮**，看一个连通块 `C`（它本身也是树）。

- 第 2 轮（剩余两轮中的第一轮）会在 `C` 中随机删 1 点 `v`。
- 删完 `v` 后，`C` 会裂成若干个更小连通块。
- 第 3 轮会对每个新连通块再删 1 点。

要让 2 轮后整个 `C` 清空，**删完第 2 轮那个点之后，剩下的每个连通块必须都是“单点”**，因为第 3 轮每块只能删 1 个点。

而在一棵树里，“删掉一个点后剩下全是单点”意味着：
这个被删点 `v` 的每个相邻子树都只有 1 个节点，也就是：

> `v` 的所有邻居都是叶子（在 `C` 内度数为 1）

因此 `C` 必须是：

- **单点**（|C|=1）：第 2 轮删它，必清空，概率 1
- **星形树**（一个中心 + 全是叶子）：第 2 轮必须删到中心，概率 `1/|C|`
- 其它形状（存在长度 ≥2 的链、或多个非叶分支）：**2 轮内不可能清空**，概率 0

这就是整题最关键的“结构转换”：
**把“概率过程”变成“删点后每块是否为星/单点 + 星的大小”**。

------

#### Step C：如何在枚举 `x` 时，快速判定每个邻居方向的连通块是不是“星/单点”，并拿到星大小？

删掉 `x` 后，对于每个邻居 `y`，会出现一个连通块 `T(x→y)`（从 `y` 走，不再经过 `x` 的那棵子树）。

你代码对 `y` 分了三类（按原树度数 `d[y]`）：

#### 1）`d[y] == 1`

```
y` 本来就是叶子，删掉 `x` 后这块只剩一个点 `y`，属于单点块。
2 轮内必清空，乘 1。
对应你代码：`if(d[y] == 1) ;
```

#### 2）`d[y] == 2`

这意味着从 `x` 走到 `y` 后还会继续到另一个点 `z`（`y` 的另一个邻居）。

- 如果 `d[z] == 1`：说明这块只有两点 `y-z`。
  两点树在 2 轮内必清空（第 2 轮删谁都行），乘 1。
  对应你代码：`if(d[z] == 1) ;`
- 否则 `d[z] > 1`：想要这块在 2 轮内清空，它必须是 **以 `z` 为中心的星**，且 `y` 只是其中一片叶子。
  那就要求：`z` 除了连向 `y` 以外，其它邻居都必须是叶子。

你用的判定技巧非常漂亮：预处理 `cnt[v] = v 的“非叶邻居”数量`（你是用“度数>1”当非叶）。
如果这块是以 `z` 为中心的星，那么 `z` 的非叶邻居只有 `y`（因为其它都是叶子），所以 `cnt[z] == 1`。

星的大小是多少？
这块包含 `z` 本身 + 它的所有邻居（包含 `y`），也就是 `d[z] + 1` 个点。
所以第 2 轮必须选到中心 `z`，概率 `1/(d[z]+1)`。

对应你代码：

```cpp
else if(cnt[z] == 1) res /= (d[z] + 1);
else res = 0;
```

#### 3）`d[y] >= 3`

这时 `y` 有多个分支。删掉 `x` 后，这块要在 2 轮内清空，唯一可能是：
**这块本身就是以 `y` 为中心的星**（中心只能是 `y`，不可能“再往里走”）。

条件：`y` 除了可能连着 `x` 这条边外，其它邻居都必须是叶子。
也就是：`y` 的非叶邻居数只能是：

- `0`（当 `x` 是叶子，即 `d[x]==1`，则 `x` 不算非叶）
- `1`（当 `x` 非叶，即 `d[x]>1`，则 `x` 算一个非叶邻居）

你代码里：

```cpp
int ls = (d[x] > 1);
if(cnt[y] == ls) res /= d[y];
else res = 0;
```

这里 `d[y]` 就是这块的大小：删掉 `x` 后，这块包含 `y` + (y 的其它 d[y]-1 个叶子) = `d[y]` 个点。
第 2 轮选中心 `y` 的概率就是 `1/d[y]`。

------

#### Step D：复杂度为什么能做到 O(n)？

你枚举每个 `x`，只遍历它的邻边，每条边最多被访问常数次，整体 `O(n)`。

你最初担心的“对每个点做深度 2 搜索”会是 `O(∑deg(x)^2)` 或 `O(n * 平均度 * 2)`，在树上很多时候也能过，但最坏（星、菊花图等）会炸。
你现在的做法把“深度 2 检查”压缩成了 **几个度数 + cnt 的 O(1) 判定**。

> 对所有点的树 / 图上搜索深度最多只能为`1`，也就是只能遍历相邻点
> 因为即便深度为`2`，在最坏情况下（菊花图、星形树）也会超时

### 3）你最初思路 vs 正确思路：差在哪？下次怎么拉回正轨？

你说的想法：

> “3 次很小 → 一步步想每次删点 → 第一次删后每块要么星要么单点 → 判定某点是否满足只能深度 2 搜索 → 超时”

差距其实只差最后一小步，但那一步很关键：

#### 你卡住的点

你已经抓到了 **“删完第一轮后每块必须是星/单点”** 这个本题的灵魂。
卡在“怎么判定星/单点”时，你默认需要“搜索/遍历子树”。

#### 正确的“回归路线”

下次遇到这种“结构判定可能要遍历”的卡点，强制问自己两句：

1. **我真的需要知道整块长什么样吗？还是只需要验证一个局部充要条件？**
   本题星/单点的充要条件，最终只落在“某些点的邻居是不是叶子/非叶”这种局部性质上。
2. **能不能把‘遍历’变成‘计数’？**
   你代码的 `cnt[v]=非叶邻居数` 就是把“查看所有邻居深度信息”变成了一个全局预处理计数。

换句话说：
你少做的不是“更高级算法”，而是 **把结构条件再压缩一层：从“深度 2 观察”压成“度数+计数”**。

### 4）基于本题提炼一个通用解题方法论（防止脑中一片空白）

给你一个我认为在 ICPC 里非常好用的“固定轮数/固定步数随机过程”通用流程：

#### （1）先做“剩余步数”的可行结构分类

- 步数小（2、3、4）时，优先尝试列举：
  - 1 步能清空的结构是什么？
  - 2 步能清空的结构是什么？
  - 3 步能清空的结构是什么？
- 目标不是写 DP，而是得到 **结构的充要条件**（最好是局部条件）。

#### （2）把全局过程拆成“第一步枚举关键事件 + 后续独立乘积”

- 树/图被切成连通块后，后续各块的选择独立（本题每块独立随机选点），就天然变成乘积。
- 所以常见形式是：
  `答案 = Σ(第一步选到某关键点的概率 * 后续各块成功概率的乘积)`

#### （3）把“判定一块是否满足结构”尽量压成局部 O(1)

- 常用手段：
  - 度数分类（叶/非叶/度=2 链点）
  - 邻居类型计数（像你 `cnt`）
  - 小范围信息（父子关系、是否存在第二个非叶邻居等）

#### （4）复杂度控制口诀

- **枚举点 + 遍历邻边** 往往能到 `O(n)`
- 一旦你发现自己要对每个点做 BFS/DFS：立刻尝试“局部充要条件 + 预处理计数”替代。

这个方法论基本就是你这题 AC 代码背后的思想框架。

### 5）顶尖选手视角：从思路到代码的“编码流”（怎么写得顺）

给你一个非常“码流化”的写法流程（脑子里按这个顺序走，写代码就很顺）：

1. **写出数学表达式骨架**
   - `ans = Σ_x P(第一轮删 x) * P(删 x 后 2 轮清空)`
   - `P(第一轮删 x)=1/n`
   - `P(删 x 后 2 轮清空)=∏_{y in adj[x]} p(x→y)`
2. **把 `p(x→y)` 按 `d[y]` 分类写成若干 if**
   - `d[y]==1`：`p=1`
   - `d[y]==2`：找 `z`，再分 `d[z]==1` / `cnt[z]==1` / 否则 0
   - `d[y]>=3`：检查 `cnt[y]==(d[x]>1)`，通过则 `p=1/d[y]` 否则 0
3. **为了让分类 O(1)，提前写预处理**
   - 读入建图、算 `d`
   - 预处理 `cnt[v] = 非叶邻居数`
     （你是对每个 `deg>1` 的点给邻居 `cnt++`）
4. **主循环模板**
   - `for x in 1..n`：
     - `res = inv(n)`
     - `for y in adj[x]`：按分支更新 `res`，如果变 0 立刻 break
     - `ans += res`
5. **最后检查极端样例**
   - n=1、n=2、链、星、二叉树等
   - 如果某个分支你写成“默认不会发生”，就拿这些极端结构去打它。

你现在的实现几乎就是这套“编码流”的标准答案了；你卡顿的地方通常会发生在第 2 步：分类没写清楚、导致写代码时不断返工。解决办法就是：**先在纸上把 `p(x→y)` 分支写成决策树，再动手写 if。**

### 6）本题还有一个很值得记的“技巧点”

你用 `cnt[v]=非叶邻居数` 来判“星中心是否只有一个非叶邻居”，这是树题里很通用的招：

- 很多“删点后子树必须全是叶/必须只允许一个重分支”的判定，都能转成：
  - `deg` 信息
  - `邻居里满足某性质的数量`（cnt 类数组）
- 一旦你发现你在做“深度 2/3 的局部观察”，优先想“能不能用这种计数把观察折叠掉”。
