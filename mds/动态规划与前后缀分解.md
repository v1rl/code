### 一、 题目归类：给算法贴上更本质的标签

这道题属于 **“DP 状态的逆操作/排除法”** 类问题，具体细分可以归纳为：

1. **大类**：背包 DP (Knapsack DP)。
2. **核心技巧**：**前后缀分解 (Prefix/Suffix Decomposition)**。
3. **抽象模型**：**“删去一个物品后的背包问题”**。

**为什么要做这个归类？**

很多数据结构题（如“除自身以外的数组乘积”）或者 DP 题（如“删去树上一个节点后的直径”），本质都是：**整体信息 = 前缀信息 $\oplus$ 后缀信息**。

当你下次看到“对于每一个 $i$，计算不包含 $i$ 的某种全局最优解”时，你的第一反应不应该是“暴力重算”，而应该是“**能不能用预处理的前缀和后缀拼接出来**”。

### 二、 思维复盘：从零推导的逻辑链条

假设我们现在面对这道题，脑子一片空白，如何一步步推导出 AC 解法？

#### 阶段 1：定义核心矛盾

**题目问**：每个物品是“必须选”、“由于选”、“绝不选”。

**转化条件**：

为了判断物品 $i$ 的属性，我们需要比较两个值：

1. $V_{max}$：全局最大价值。
2. $V_{\text{without } i}$：强制**不选** $i$ 能达到的最大价值。
3. $V_{\text{with } i}$：强制**选** $i$ 能达到的最大价值。

**判据**：

- **Must (A)**: $V_{\text{without } i} < V_{max}$ （没它不行）。
- **Never (C)**: $V_{\text{with } i} < V_{max}$ （有它不行）。*(注意：此时必然 $V_{\text{without } i} == V_{max}$)*
- **Optional (B)**: $V_{\text{without } i} == V_{max}$ 且 $V_{\text{with } i} == V_{max}$。

> 这一点类似组合数公式中的`C(n, m) = C(n - 1, m) + C(n - 1, m - 1)`

#### 阶段 2：识别瓶颈（你的 TLE 代码卡在这里）

你的 TLE 代码试图找出“所有构成 $V_{max}$ 的方案”并统计每个物品出现的次数。

- **思维陷阱**：方案数可能是指数级的。比如所有物品体积价值都一样，方案数就是组合数 $C(n, k)$。DFS 遍历方案必然超时。
- **修正方向**：我们不需要知道“有多少种方案”，只需要知道“**是否存在**一种方案”。这暗示我们应该继续留在 DP 的体系内，而不是跳出去做搜索。

> 本题题目只需要知道单点对整体的影响，而暴力操作直接求得了所有`dp`的路径，这显然有些小题大做了
> 考虑自己当前执行的操作，相比题目真正需要的目标操作，是否出现了过多的冗余

#### 阶段 3：独立性与拼凑（Breakthrough）

要算 $V_{\text{without } i}$，相当于在集合 $\{1, \dots, i-1\} \cup \{i+1, \dots, N\}$ 中做背包。

- **朴素做法**：对每个 $i$ 跑一次 $O(NM)$ 的 DP，总复杂度 $O(N^2M)$。$\rightarrow$ 超时。
- **观察结构**：
  - $\{1, \dots, i-1\}$ 是一个连续的前缀。
  - $\{i+1, \dots, N\}$ 是一个连续的后缀。
  - 背包 DP 具有**无后效性**：前 $i-1$ 个物品怎么凑出重量 $w$，和后 $i+1$ 个物品怎么凑，是完全独立的！

#### 阶段 4：空间换时间

既然独立，那我们可以预处理：

- $pre[i][w]$：前 $i$ 个物品凑出重量 $w$ 的最大价值。
- $suf[i][w]$：后 $i$ 个物品（从 $i$ 到 $n$）凑出重量 $w$ 的最大价值。

**关键的合并（Merge）**：

当我们把物品 $i$ 挖掉，剩下的空间 $M$ 怎么分配给前缀和后缀？

我们需要枚举前缀占用的空间 $j$，那么后缀只能占用 $\le M-j$ 的空间。

$$V_{\text{without } i} = \max_{0 \le j \le M} (pre[i-1][j] + suf[i+1][M-j])$$

*(注：这里假设 DP 状态定义为容量“恰好”或“不超过”，你的 AC 代码用的是“不超过”，这在合并时更方便)*

至此，复杂度降为 $O(NM)$ 预处理 + $O(N \times M)$ 查询 = $O(NM)$。AC！

### 三、 通用解法方法论：如何解决“陌生题”

针对 ICPC 金牌目标，当你遇到一道这就无法下手的题时，建议启动以下 **Checklist**：

1. **判定暴力复杂度**：

   - 如果只做一次是 $O(K)$，做 $N$ 次是 $O(NK)$，能不能优化 $NK$？
   - 通常意味着需要利用**重复子结构**。

2. **寻找“可拆分性” (Decomposability)**：

   - 题目是否询问关于“区间”、“剔除某元素”、“修改某元素”后的全局属性？
   - 如果是，考虑 **分治 (Divide & Conquer)** 或 **前后缀分解**。

   > 分治与前后缀分解[G - Range Knapsack Query](https://atcoder.jp/contests/abc426/tasks/abc426_g)

   - *口诀*：询问“去掉 $i$”，就是问“$Prefix(i-1) + Suffix(i+1)$”。

3. **状态合并 (State Merging)**：

   - 如果已知左半边的 DP 表和右半边的 DP 表，如何合并？
   - 背包问题 $\rightarrow$ $(\max, +)$ 卷积 (本题)。
   - 计数问题 $\rightarrow$ 多项式乘法 (FFT/NTT)。
   - 连通性问题 $\rightarrow$ 矩阵乘法。

4. **维度交换 (Dimension Swap)**：

   - 如果 $M$ 很大（如 $10^9$）但 $N$ 和 $V$ 很小？
   - 把价值作为 DP 的下标，重量作为 DP 的值。

**针对本题的各种变体训练：**

- 如果问：去掉物品 $i$ 后，凑出重量 $M$ 的方案数？ $\rightarrow$ 同样前后缀 DP，合并时是乘法原理。
- 如果问：必须选物品 $i$ 和 $j$，最大价值？ $\rightarrow$ 同样思路，中间挖空两块。

### 四、 顶尖选手的编码流 (Coding Flow)

很多时候思路有了，代码写得磕磕绊绊，主要是因为**“心智负担”过重**（同时在想逻辑、下标、边界、语法）。顶尖选手通常采用**模块化编码**来降低心智负担。

针对这道题，高手的编码节奏是这样的：

**Step 1: 骨架与数据结构 (Skeleton & Data)**

- 先写 IO 优化。
- 直接把 $N, M$ 这种大概率会溢出栈的 DP 数组开在全局 (`vector` 在堆上也可以，但全局数组写起来快且不易错)。
- **Decision**: 既然要前后缀，我需要两个 DP 数组 `pf` 和 `sf`。

```cpp
const int MAXN = 1005, MAXM = 50005; // 看着数据范围写
long long pf[MAXN][MAXM], sf[MAXN][MAXM]; // 全局开，省心
int v[MAXN], w[MAXN];
```

**Step 2: 核心引擎 (The DP Engine)**

- 不要想具体的逻辑，先把“填充 DP 表”这个机械任务完成。
- 先写前缀，复制，粘贴，修改为后缀。
- **Focus**: 此时只关注下标别越界。

```cpp
// 填前缀
for(int i=1; i<=n; i++) {
    for(int j=0; j<=m; j++) { // 0 到 m 全遍历，防止漏状态
        pf[i][j] = pf[i-1][j];
        if(j >= v[i]) pf[i][j] = max(pf[i][j], pf[i-1][j-v[i]] + w[i]);
    }
}
// 填后缀 (注意 i 从 n 到 1，状态来源是 i+1)
for(int i=n; i>=1; i--) {
    for(int j=0; j<=m; j++) {
        sf[i][j] = sf[i+1][j];
        if(j >= v[i]) sf[i][j] = max(sf[i][j], sf[i+1][j-v[i]] + w[i]);
    }
}
```

**Step 3: 业务逻辑 (The Logic)**

- 现在数据都有了，开始处理题目特定的分类逻辑。
- **Trick**: 将“合并前缀和后缀”这个操作抽象化。虽然不必写成函数，但脑子里要把它当成一个黑盒操作。
- 先算 $X_{out}$ (不选 $i$)，再算 $X_{in}$ (选 $i$)。

C++

```
long long max_val = pf[n][m]; // 全局最优

for(int i=1; i<=n; i++) {
    // 1. 计算不选 i 的最大值 (Merge pf[i-1] and sf[i+1])
    long long val_without = 0;
    for(int j=0; j<=m; j++) {
        val_without = max(val_without, pf[i-1][j] + sf[i+1][m-j]);
    }

    // 2. 计算必选 i 的最大值 (Base w[i] + Merge rest space)
    long long val_with = -1; // 默认不可能
    if (m >= v[i]) {
        long long rem = m - v[i];
        long long best_merge = 0;
        // 注意：这里很容易写错，要专注剩余空间 rem
        for(int j=0; j<=rem; j++) {
            best_merge = max(best_merge, pf[i-1][j] + sf[i+1][rem-j]);
        }
        val_with = best_merge + w[i];
    }

    // 3. 判定输出
    if (val_with == max_val && val_without < max_val) cout << "A";
    else if (val_with == max_val && val_without == max_val) cout << "B";
    else cout << "C";
}
```

**总结编码流的关键点：**

1. **分离计算与查询**：先把 DP 表填完，再做查询逻辑，不要混在一起写。
2. **对称性利用**：写完前缀逻辑后，后缀逻辑几乎是镜像的，利用这一点快速编码但要小心下标。
3. **变量命名清晰**：`val_with` 和 `val_without` 比 `res1`, `res2` 更能防止逻辑混淆。
