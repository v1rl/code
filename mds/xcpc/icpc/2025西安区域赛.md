## B 带边界约束的绝对众数插空判定

**题意简述**

给定一个仅包含 3 种字符的字符串。求找到一个**最短的区间** $[l, r]$，对该区间内的字符进行**重排**，使得最终整个字符串中**不存在相邻的相同字符**。如存在，输出区间左右端点及重排后的完整字符串。

------

**思维 Trick (关键跳跃)**

这道题的精髓在于**将“是否存在合法排列”这个复杂的组合构造问题，降维成了一个极其简单的代数不等式**。

1. **绝对众数的“插空法”代数化表示 (核心 Trick)**
   - 通常处理“不相邻排列”问题，直觉是优先排最多的字符（插空法）。但本题带了**外部边界约束**（重排区间的左右外侧字符也是固定的），分类讨论极其繁琐。
   - **跳跃点：** 我们只关心“数量最多的字符”是否会溢出。假设区间长为 $len$，某字符数量为 $cnt$，其余字符数量为 $len - cnt$（可作为隔板）。
   - 基础插空要求：隔板数 $\ge cnt - 1$，即 $(len - cnt) \ge cnt - 1 \implies 2 \times cnt - len \le 1$。
   - 引入边界：如果左/右边界的外部字符恰好也是该字符，相当于该字符不能贴边放，需要的隔板数量变多了。每有一个同色边界，公式的右侧（也就是需要的隔板数目）就 $+1$。
   - 最终化简为优雅的不等式：**$2 \times cnt - len + \text{同色左边界} + \text{同色右边界} \le 1$**。只要所有三种字符都满足此条件，就一定存在合法重排。
2. **基于“溢出无法自救”的单调性 (双指针前提)**
   - 找最短区间通常想二分，但本题不仅满足单调性，还可以用双指针进一步压榨时间到 $O(N)$。
   - **跳跃点：** 为什么 $l$ 右移时，$r$ 一定不回退？因为如果当前区间因为某种颜色“溢出”而不合法，此时右移 $l$（缩小区间）只会扔掉字符：
     - 如果扔掉的是非溢出字符，溢出字符的占比会更大，更不合法。
     - 如果扔掉的是溢出字符，由于产生了同色边界惩罚，该颜色的拥挤度并不会降低。
   - 因此，**缩小左边界绝对无法拯救一个不合法的区间**，必须向右扩展 $r$ 去吃进更多其他字符来稀释它。
3. **自相似的贪心构造 (Self-Reducibility)**
   - 既然有了 $O(1)$ 的状态校验函数，构造方案就不需要去手写复杂的按频率交替放置逻辑。
   - **跳跃点：** 从左到右逐位枚举，尝试放入 `'C'`, `'W'`, `'P'`。只要放入该字符后，**剩余的未填区间依然满足上述的“拥挤度校验函数”**，就直接敲定（贪心）。这用极其简洁的逻辑完成了构造。

------

**编码 Trick (优雅实现)**

你的代码中其实已经包含了一些很好的习惯，这里提炼并补充，形成可以复用的编码范式：

1. **字符到状态空间的映射 (Enum/Index Mapping)**

   - **技巧：** 不要在逻辑代码里写 `'C'`, `'W'`, `'P'`。像你的 `tonum` 函数和 `"CWP"[c]` 数组一样，立刻将它们映射为 `0, 1, 2`。
   - **收益：** 所有的状态校验、枚举全部可以用 `for(int c=0; c<3; c++)` 一个循环搞定，彻底消灭 `if-else` 分支带来的代码冗余和笔误风险。

2. **状态验证器的 Lambda 闭包提取**

   - **技巧：** 将最核心的数学校验逻辑单独抽离成 `check(l, r)` 和 `ansCheck(C, l, r)` 的 Lambda 表达式。
   - **收益：** 这使得主逻辑（寻找最短区间的双指针循环、构造答案的贪心循环）变得异常干净，思维负担极小。当需要在构造时进行前瞻预测时，直接调用闭包即可。

3. **基于回溯的“试填法”范式**

   - **技巧：** 在构造字符串时，采用 `操作 -> 校验 -> 回退` 的微型回溯。

     ```cpp
     cnt[c]--; // 1. 试探
     if(ansCheck(...)) { ans[i] = ...; break; } // 2. 校验成功则确立并跳出
     cnt[c]++; // 3. 校验失败则回退
     ```

   - **收益：** 这是一种极其通用的状态机构造法，只要判定定理正确，这套写法就是免死金牌，比手动去写“优先队列维护剩余数量最多”要好写、稳定得多。

## I 树上三点异或中心构造

**题目简述**

给定一个 $N \times N$ 的矩阵 $A$（$N \le 2000$），其中 $A_{u,v}$ 表示未知树上从 $u$ 到 $v$ 简单路径上所有节点编号的**异或和**。要求根据矩阵 $A$ 重构出这棵树，输出所有边。

------

**核心 Trick（思维跳跃点）**

这道题的突破口在于**树的拓扑性质**与异或运算（$x \oplus x = 0$）的完美结合。

**1. 三路径异或抵消找中心点（最关键的注意力跳跃）**

在树上任取三个点 $u, v, w$，它们两两之间的简单路径一定会交汇于唯一的一个中心点（或称分叉点）$c$。

如果你把 $A_{u,v} \oplus A_{v,w} \oplus A_{w,u}$ 算出来，会发生奇妙的抵消：

- 除了中心点 $c$ 之外，这三条路径覆盖的所有其他点，都恰好被包含在了**两条**路径中。
- 中心点 $c$ 是三条路径的唯一交点，被包含了 **3 次**。
- 根据异或性质，出现 2 次的点全被抵消为 0，出现 3 次的 $c$ 被保留。
- **结论**：任意三点 $u, v, w$ 的交汇中心点 $c = A_{u,v} \oplus A_{v,w} \oplus A_{w,u}$。这让你能在 $O(1)$ 时间内扒出一个树上拓扑结构！

**2. 强行定根提取偏序关系**

无根树很难直接还原边，转化为有根树就能利用偏序关系（祖先/后代）。

- 强行钦定节点 `1` 为根。
- 枚举剩下的任意一对点 $x$ 和 $y$，利用 Trick 1 计算 $1, x, y$ 的中心点 $c = A_{1,x} \oplus A_{1,y} \oplus A_{x,y}$。
- 如果算出来 $c = x$，说明 $x$ 刚好卡在 $1 \to y$ 的路径上，这就意味着 **$x$ 是 $y$ 的祖先**！

**3. 利用偏序关系与深度还原直接连边**

求出了全局的祖先关系矩阵 `p[i][j]` 后，怎么找确切的边？

- **求深度**：一个节点 $u$ 的深度 `dep[u]`，就等于它的祖先数量。
- **找父子**：如果 $x$ 是 $y$ 的祖先，且满足深度差 $dep[y] - dep[x] == 1$，那么中间没有其他节点了，$x$ 就是 $y$ 的直接父节点，两点之间必定有边。

## L 多边形不等式与最长边贪心剔除

------

**题意简述**

给定 $n$ 根木棒，对于每一个 $k \in [1, n]$，要求选出 $k$ 根木棒组成一个非退化凸多边形，并使得该多边形的周长最大。如果无法组成，则对该 $k$ 输出 0。

------

**核心 Trick (思维跳跃点)**

这道题的推导是层层递进的，最关键的跳跃在于**发现不合法状态的“向下兼容性”并直接舍弃**。

1. **几何转代数 (充要条件转化)：**

   一个 $k$ 边形合法的充要条件是：**最长边 $<$ 其余 $(k-1)$ 条边之和**。这使得我们可以将原数组排序，问题转化为在有序数组中挑数字。

2. **贪心连续段 (局部最优解)：**

   为了让周长最大，如果钦定选出的最长边是 $a_r$，那么剩下的 $k-1$ 条边显然应该贪心地选紧挨着 $a_r$ 左边的 $k-1$ 个数（即 $a_{r-k+1} \dots a_{r-1}$）。如果连这 $k-1$ 个**最大的候选边**加起来都无法大于 $a_r$，那么选任何其他的组合都必然失败。

3. **最长边永久剔除 (核心 Attention / 跳跃点)：**

   如果 $a_r$ 在选 $k$ 条边时失败了（即 $a_r \ge \sum_{i=r-k+1}^{r-1} a_i$），它能不能在选 $k-1$ 条边时成功？

   **绝对不可能。** 因为边数减少，可用的最大边之和只会更小，差距会被拉得更大。

   这就是本题最核心的 Trick：**一旦某个最长边 $a_r$ 满足不了当前的 $k$，它就永远满足不了更小的 $k$。它变成了一个“彻底的废品”，我们可以毫无顾忌地将其永久丢弃（$r \leftarrow r-1$）。**

------

**编码 Trick (优雅实现)**

你的代码中其实已经自然而然地使用了这些优秀的编码习惯：

1. **逆向双指针 (状态机的均摊 $O(N)$)：**

   没有为每一个 $k$ 去做二分或复杂的线段树查询，而是直接设置两个指针：目标边数 $k$ 从 $n$ 开始递减，可用最长边下标 $r$ 从 $n$ 开始递减。

   - 如果合法，说明对于当前的 $k$ 找到了最优解，保存答案并 `k--`。

   - 如果不合法，说明 $a_r$ 彻底废了，直接 `r--`。

     这种写法将原本可能长达 $\mathcal{O}(N^2)$ 的暴力判断，利用单调性均摊成了极致的 $\mathcal{O}(N)$ 扫描。

通过这种“降维打击”，整道题的核心逻辑被压缩进了一个非常简短的 `while` 循环里，这就是 ICPC 代码美感所在。
