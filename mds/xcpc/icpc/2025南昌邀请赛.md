## I 极大区间容斥

------

 **一句话题意**

给定一个长为 $n$ 的 01 串，允许进行**恰好一次**操作：选择一个恰好包含 $k$ 个 `1` 的子串，在保持该子串内 `1` 的数量仍为 $k$ 的前提下，**任意重排**该子串。求最终能产生多少种**不同**的 01 串。（结果对 998244353 取模）

------

**思路 Trick (注意力跳跃点)**

**1. 降维打击：从“任意合法区间”跳跃到“极大合法区间”**

如果枚举所有恰好包含 $k$ 个 `1` 的区间去计算组合数，会产生海量的重复，因为小区间能生成的状态，包含它的大区间也能生成。

**Trick:** 强制限定只研究**“极大合法区间”**（即向左向右延伸直到碰到下一个 `1` 为止的区间）。只有站在极大区间的视角，才能为后续的去重提供清晰的基准。

**2. 核心化简：一维连续区间容斥的极简性**

这是本题最精髓的数学跳跃。对于多个集合的容斥，通常是 $A + B + C - AB - BC - AC + ABC$。

**Trick:** 因为本题的极大区间是**在一维直线上连续滑动**的，一个合法状态如果横跨了 $m$ 个连续的极大区间，它在 $\sum$极大区间 时会被加 $m$ 次，在 $\sum$相邻交集 时会被减 $m-1$ 次，最终 $m - (m-1) = 1$，恰好不重不漏！

因此，多重容斥被完美坍缩成了两项：

$$\text{Ans} = \sum \binom{\text{极大区间长度}}{k} - \sum \binom{\text{相邻极大区间交集长度}}{k-1}$$

**3. 交集状态的物理意义：被“钉死”的边界**

为什么交集里是算 $k-1$ 的组合数？

**Trick:** 如果一个 01 串既能由极大区间 A 生成，也能由右侧相邻的极大区间 B 生成，说明它**在 A 和 B 不重合的边缘部分，必须与原串一模一样**。这相当于交集左右两端各有一个 `1` 被原位“钉死”了。总共 $k$ 个 `1`，外面钉死 1 个（相对于交集而言），所以只能在交集长度里自由排布剩下的 $k-1$ 个 `1`。

------

**编码 Trick (优雅实现)**

**1. 哨兵思想 (Sentinel Values)：消灭所有边界特判**

为了计算包含 $k$ 个 `1` 的极大区间长度，我们需要找这 $k$ 个 `1` 极左侧和极右侧的边界。如果在原串上处理，需要大量 `if (左边没有1了)` 或 `if (右边到头了)` 的特判。

**Trick:** 将所有 `1` 的下标存入数组 `p`，并在首尾强行塞入 `-1` 和 `n` 作为哨兵。

这样第 $i$ 个极大区间的长度公式 `p[i+k] - p[i-1] - 1` 在任何位置都完美通用，思维负担降为 0。

**2. 下标数组化：将“滑动窗口”转化为“索引距离”**

很多人的第一直觉是写个双指针 `l` 和 `r` 在字符串上维护 `cnt == k`。这不仅容易写错，而且求交集长度时会非常抓狂。

**Trick:** 遇到这种“统计特定字符数量”并涉及区间长度的题，直接提取该字符的 index 组成新数组 `p`。区间长度直接转化为极其简单的下标减法：`len = p[R] - p[L] - 1`，逻辑极其清晰。

**3. 防溢出减法取模**

这是 XCPC 选手的肌肉记忆，但在紧张的比赛中容易手抖漏掉。

**Trick:** 凡是涉及减法的取模，一律写成 `(a - b % mod + mod) % mod`。你的代码中 `(ans - C(len, k - 1) + mod) % mod` 就是极其标准的防御性编码。



## G 极限步数截断图 DP

------

**一句话题意**

给定 $n$ 点 $m$ 边的有向带权图（边权 $d \ge 2$）。给定初始值 $X$（$X \le 10^9$），每次经过边权为 $d$ 的边，$X$ 变为 $\lfloor X/d \rfloor$。多次独立查询，求从指定起点出发，使 $X$ 降为 0 至少需要经过几条边。

------

**思路 Trick (注意力跳跃点)**

**1. 运算转换：除法嵌套转为乘法累积**

直接模拟向下取整的除法，状态会因为初始值的不同而四分五裂。

**Trick:** 利用数学性质 $\lfloor \lfloor x / a \rfloor / b \rfloor = \lfloor x / (a \times b) \rfloor$。目标是从 $X$ 降到 0，等价于**寻找一条路径，使其边权连乘积严格大于 $X$**。问题瞬间转化为图上求最大乘积路径。

**2. 维度坍缩：利用阈值进行极限步数截断**

图存在环，求最长/最大路径极其容易陷入死循环。

**Trick:** 注意到题目限制边权 $d \ge 2$，而 $X \le 10^9$。由于 $2^{30} > 10^9$，这意味着**任何合法的路径最多走 30 步必定结束**。这就将一个可能无限的图论图，强行“截断”成了一个只有 30 层的有向无环图（DAG），从而可以使用动态规划轻松求解。

**3. 状态截断：防溢出技巧**

边权乘积在 30 步内会轻易爆 `long long` 甚至 `__int128`。

**Trick:** 既然最大的 $X$ 只有 $10^9$，对于任何大于 $10^9$ 的乘积，效果完全等价。状态转移时直接与 $10^9 + 10$ 取 $\min$，将有效状态卡在 `long long` 的安全范围内，避免高精度运算。

------

**编码 Trick (关于 TLE 与维度的秘密)**

你提到的“第一维度交换导致 TLE 变 AC”，这是一个极其经典的**计算机底层架构问题**，也是 XCPC 选手必须掌握的常识：**CPU 缓存命中率与空间局部性 (Spatial Locality)**。

在 C++ 中，`vector<vector<int>>` **并不是真正的二维连续数组**，而是一个包含多个指针的一维数组，每个指针再指向堆上的一块独立内存。

- **超时的写法（`vector<vector> f(n + 1, vector(31))`）：**

  你分配了 $2 \times 10^5$ 个碎片化的小数组，每个长度只有 31。

  在你的状态转移内层循环：

  `f[x][i] = max(f[x][i], f[y][i-1] * w)`

  由于图的边是无序的，`x` 和 `y` 的访问是极其随机的。CPU 在访问 `f[x]` 和 `f[y]` 时，指针要在 $2 \times 10^5$ 个不连续的内存块中反复横跳，导致 CPU Cache 疯狂 Miss，必须不断从慢速的内存条读取数据，产生巨大的性能损耗。此外，开辟几十万个微小的 vector 也会带来极大的内存分配开销。

- **AC 的写法（`vector<vector> f(31, vector(n + 1))`）：**

  你分配了 31 个大数组，每个长度为 $2 \times 10^5$。

  此时外层循环是步数 $i$。在处理第 $i$ 步时，你只在 `f[i]` 和 `f[i-1]` 这**两个连续的大内存块**上进行读写。尽管 `x` 和 `y` 依然随机跳跃，但由于整个大数组被缓存在 CPU Cache 中，命中率大幅提升，内存开销也从几十万次分配降到了仅仅 31 次。

**编码总结论：** 在设计 DP 状态时，**务必将“阶段（如时间、步数）”作为最外维，将“状态（如节点、坐标）”作为内维**。如果必须写成 `[节点][步数]` 的逻辑形式，应该使用 `vector<array<int, 31>> f(n + 1)`，这样既保证了物理内存的绝对连续，又符合逻辑直觉。



## D 三维正交投影转一维区间交

------

**一句话题意**

给定三维空间中的 $n$ 条线段。要求找到一个**垂直于任意坐标轴**（$x$、$y$ 或 $z$ 轴）的无限大平面，使其与**最多**的线段相交。求出这个最大相交的线段数量。

------

**思路 Trick (注意力跳跃点)**

**1. 物理降维打击：正交平面的充要条件（核心 Trick）**

面对三维几何，第一直觉往往是去推导复杂的空间直线与平面的相交公式。但本题的极其关键的限制条件是：**攻击平面垂直于坐标轴**。

**Trick:** 一个垂直于 $x$ 轴的平面（方程式为 $x = k$），它与一条三维线段相交的**充要条件**，仅仅是 $k$ 落在这条线段在 $x$ 轴的**一维投影区间** $[\min(x_1, x_2), \max(x_1, x_2)]$ 内。至于 $y$ 轴和 $z$ 轴的坐标伸展到哪里，**完全无关紧要**！

这一个极其惊艳的注意力跳跃，直接将吓人的三维空间几何题，彻底“拍扁”成了 3 个完全独立的一维线段求交问题。

**2. 扫描线算法 (Sweep-line)：一维区间最大重叠**

将问题转化为求一维坐标轴上最多有多少个区间重合后，如果暴力枚举端点复杂度是 $\mathcal{O}(n^2)$，会超时。

**Trick:** 使用**排序 + 最小堆**的经典扫描线套路。

- 将所有区间按左端点升序排序（保证我们从左到右扫描的时间线是正确的）。
- 用最小堆（`priority_queue<int, vector<int>, greater<int>>`）来维护当前所有跨越扫描线的区间的**右端点**。
- 每扫到一个新区间，就把堆顶所有小于当前左端点的值弹掉（因为扫描线已经越过了这些区间的右边界，它们不再重合）。
- 此时堆内元素的数量 $+ 1$（包含当前区间本身），就是当前点的最大重叠数。这个 Trick 稳定将时间复杂度压榨至 $\mathcal{O}(n \log n)$。

------

**编码 Trick (优雅实现)**

**区间规范化：无脑 `min`/`max` 屏蔽方向性**

题目给出的线段端点并没有保证坐标大小的顺序（比如给定的两点可能是从 $x=5$ 连向 $x=2$）。如果在处理区间时不注意，极其容易存入 `[5, 2]` 这样非法的逆向区间，导致扫描线算法排序和进出堆的逻辑彻底崩溃。

**Trick:** 在读入端点数据存入 `range` 数组时，**不假思索地直接套用 `min` 和 `max`**（如 `min(a[0], b[0]), max(a[0], b[0])`）。这一个小小的编码习惯，可以在根源上抹平向量的方向性，确保后续处理的必定是合法的数学区间 `[L, R]`（其中 $L \le R$），彻底杜绝边界 Bug。



## F 系数定号贪心（或：全局贡献拆解）

------

**一句话题意**

给定长度为 $n$ 的数列 $r$ 和 $c$。已知 $r_i \in [L, R]$，其中 $k$ 个位置的 $r_i$ 是强行固定的。数列 $c$ 满足递推式 $c_i = p \times c_{i-1} + (1-p) \times r_{i-1}$（$r_0, c_0, L, R, p$ 均为给定常数，且 $0 < p < 1$）。

求在合法范围内自由填入未固定的 $r_i$，使得总和 $\sum_{i=1}^n(c_i - r_i)$ 达到最大。

------

**思路 Trick (注意力跳跃点)**

**1. 障眼法：局部的“鱼与熊掌”**

直觉上，我们要求 $\max \sum(c_i - r_i)$。如果某一天 $r_i$ 取很小的值，当天的 $-r_i$ 会变大（有利）；但根据递推式，这会导致它传递给明天的 $c_{i+1}$ 变小，进而导致未来的所有 $c$ 都变小（有弊）。这看起来是一个极其复杂的具有后效性的状态转移方程，很容易引诱选手去想 DP。

**2. 核心 Trick：算总账（拆解单个变量的全局贡献）**

在 XCPC 的数学推导题中，当递推式是**线性**的时候，最强大的破局点就是**分离变量**。我们不看每一天，而是单独揪出一个自由的变量 $r_j$（$j \ge 1$），看看它对最终总目标 $\sum_{i=1}^n(c_i - r_i)$ 到底产生了多少总净贡献。

- **当前天的直接贡献：** 它作为 $-r_j$ 出现，系数是 $-1$。

- **对未来的连锁贡献：**

  - 它让 $c_{j+1}$ 增加了 $(1-p) \times r_j$

  - 它让 $c_{j+2}$ 增加了 $p(1-p) \times r_j$

  - 它让 $c_{j+3}$ 增加了 $p^2(1-p) \times r_j$

  - ...

  - 它对之后所有 $c$ 产生的增量系数求和，这是一个公比为 $p$ 的等比数列：

    $$S = (1-p) + p(1-p) + p^2(1-p) + \dots + p^{n-j-1}(1-p)$$

    利用等比数列求和公式（或者提取 $1-p$ 直接消项），算得 $S = 1 - p^{n-j}$。

**3. 注意力跳跃：系数永负**

将 $r_j$ 的“直接贡献系数”加上“对未来的连锁贡献系数”：

$$\text{总系数} = -1 + (1 - p^{n-j}) = -p^{n-j}$$

因为题目保证 $0 < p < 1$，且 $n-j > 0$，所以 **$-p^{n-j}$ 永远是一个严格的负数！**

**Trick 结论：**

真相大白。不管未来有多长，不管 $p$ 是多少，任何一个非固定的 $r_j$，它每增加 1，都会导致最终的总和确确实实地减少 $p^{n-j}$。因此，为了让总和最大，**没有任何权衡，所有未固定的 $r_j$ 必须无脑取到其给定的最小下界 $L$。**

推导出这一步后，题目就变成了一个 $\mathcal{O}(N)$ 的单纯模拟题。



## M 守恒反转消元

------

**一句话题意**

$n$ 枚硬币初始朝上，对手暗中将其中 $k$ 枚翻面（位置未知）。你需要为每枚硬币分配归属（一号堆或二号堆）及状态（翻转或不翻），使得最终**两堆朝上的硬币数量严格相等**。

------

**思路 Trick (注意力跳跃点)**

**1. 局部未知与全局守恒的转换**

对手翻转了 $k$ 枚硬币，具体是哪 $k$ 枚我们根本不知道。

**Trick:** 既然总共翻了 $k$ 个，那我们就**强行截取 $k$ 枚硬币**放入一号堆（不翻转，操作 1）。

假设这 $k$ 枚硬币中，恰好有 $x$ 枚是被对手翻暗的。那么一号堆里现在**朝上的硬币数量就是 $k - x$**。

我们不知道 $x$ 是多少，但根据**全局数量守恒**，既然全场总共有 $k$ 枚暗币，一号堆占了 $x$ 枚，那么剩下的 $n - k$ 枚硬币里，**必定包含恰好 $k - x$ 枚暗币！**

**2. 状态反转实现“变废为宝”（核心消元）**

现在我们手里剩下一堆硬币，里面有 $k - x$ 枚是暗的，其余是明的。而我们的目标是让二号堆产生 $k - x$ 枚明币。

**Trick:** 怎么把 $k - x$ 个暗的变成明的？**全部翻转！**

我们将剩下的 $n - k$ 枚硬币全部放入二号堆并翻转（操作 4）。

这样一来，原本朝下的 $k - x$ 枚硬币瞬间面朝上了，而原本朝上的全被盖了下去。

此时，二号堆朝上的数量精准地等于 $k - x$。

一号堆朝上 $k - x$，二号堆朝上 $k - x$。未知变量 $x$ 被完美消元，双方强行平局！



## K 相对参照系与锚点枚举

------

**一句话题意**

有 **n** 尊雕像，初始分别朝向 4 个方向（0上, 1右, 2下, 3左）。

操作一：按某雕像鼻子，使**其他**所有雕像顺时针转 90°；

操作二：按自己鼻子，使**所有**雕像顺时针转 90°。

求让所有雕像都朝向 0 的最少总操作次数。

------

**思路 Trick (注意力跳跃点)**

**1. 物理参照系翻转：从“带动他人”变为“自我倒退”**

原题的操作一“按 $i$ 导致其他 $n-1$ 个雕像 +1”在逻辑上极其笨重。

**Trick:** 切换参照系！把操作一等价转换为：**发生了一次全局顺时针旋转（所有人 +1），但被按的雕像 $i$ 停在原地（相对大部队相当于 -1，逆时针转了 90°）**。

我们要让所有雕像朝向统一，本质上就是不断按压那些“超前”的雕像，让它们相对大部队“倒退”，直到与大部队处于同一频道。

**2. 核心贪心：绝对存在“0操作”的锚点雕像**

如果我们把每个雕像都按了至少 1 次，总共花费了 **n** 次操作，其效果是每个雕像都相对倒退了 1 次，大家相对位置根本没变，仅仅是全局顺时针转了 **n-1** 次。这显然不如直接用操作二按自己鼻子 **n-1** 次划算。

**Trick:** 最优解中，**必定存在至少一尊雕像，我们一次鼻子都没按过它！** 既然它一次都没被按过，它就可以作为“全局参照锚点”。而题目中初始朝向只有 4 种，我们只需要 $\mathcal{O}(1)$ 地枚举这 4 种朝向 **j**（0, 1, 2, 3），假设朝向为 **j** 的雕像就是我们碰都不碰的那个锚点。

**3. 解耦计算：先相对对齐，再全局归零**

一旦锚点 **j** 确定，整个问题瞬间解耦成两步极简运算：

- **第一步（内部对齐）：** 初始朝向为 **x** 的雕像为了和锚点 **j** 对齐，必须通过按鼻子来倒退。按压次数正好是差值 `(x - j + 4) % 4`。遍历数组求和，得到总对齐代价 **S**。

- **第二步（全局归零）：** 在按压这 **S** 次的过程中，锚点雕像（没被按过）其实被动跟着全局顺时针转了 **S** 次。所以对齐后，全体的统一朝向变成了 `(j + S) % 4`。此时再用操作二，将整体拨回 0，代价是 `(4 - (j + S) % 4) % 4`。

  两者相加取最小值，完美将题意压缩成了 $\mathcal{O}(n)$ 的遍历运算！

------

**编码 Trick (优雅实现)**

在你的代码中，有一个非常棒的常量数组运用：

```
vector<int> pos(4, -1);
```

你没有去纠结具体是“哪个”雕像作为锚点，而是只要初始数组里存在某个朝向 **j**（即 `pos[j] != -1`），就直接把它作为锚点的代表。这种**提取特征值代替具体对象**的编码习惯，极大地降低了维护下标的思维负担。



## C 顺序抽牌转无放回分配

------

**题意极简总结**

- **初始状态**：一副 52 张的扑克牌（1-13 各 4 张）。已知己方手牌 5 张，牌库剩余 47 张未知牌。
- **操作与结算**：对于每一个 $k \in [1, 5]$，己方可以策略性地弃掉 $k$ 张牌，然后从牌库中抽 $k$ 张补充；同时对手从牌库中抽 5 张。
- **胜负判定**：若己方最终 5 张牌点数之和 > 对方 5 张牌点数之和，己方得 $k$ 分；小于扣 $k$ 分；平局得 0 分。
- **目标**：求出己方在采用**最优弃牌策略**下，每个 $k$ 对应的期望得分对 998244353 取模的结果。

------

**思路 Trick (关键跳跃)**

这道题的区分度在于两个思维上的“跃迁”：

**1. 贪心策略的必然性（消除决策树）**

题目提到“最优决策”，这往往让人联想到需要在 DP 中做 Max 转移。但这里的关键跳跃是：在补牌完全随机的情况下，要最大化胜率，唯一的最优解就是**固定扔掉手里面积最小的 $k$ 张牌**。

- **Trick 价值**：这一步直接把“动态博弈/决策”降维成了“静态统计”。对于固定的 $k$，己方保留的 $5-k$ 张底牌及其点数和 $S_{kept}$ 是常数，彻底与后续的 DP 过程解耦。

**2. 概率模型的同构映射（时序转静态）**

直观的题意是“对手先抽 5 张，己方再抽 $k$ 张”，这是一个带有先后时序的条件概率过程。

- **Trick 价值**：由于未知的 47 张牌在暗牌状态下概率完全等价，这个“先后抽取”的过程，**等价于在 47 张牌中同时选定 5 张涂成红色（给对手），选定 $k$ 张涂成蓝色（给自己）**。
- 将复杂的概率分步树，转化为一个经典的**多维 0-1 背包方案数统计**。期望的计算直接变成了 $\frac{\text{特定点数差的组合方案数}}{\text{总组合方案数}}$。

------

**编码 Trick (优雅实现)**

你的 AC 代码在实现上已经具备了很好的竞技编程素养，这里总结其中能大幅降低思维负担和 Debug 成本的编码习惯：

**1. 降维滚动与逆序遍历**

将原本需要 5 维的 DP 状态 $f[i][k][x][j][y]$（前 $i$ 张牌...）优化为 4 维 $f[k][x][j][y]$。

- **编码优雅点**：外层循环遍历 47 张牌，内层关于选牌数量（$k$ 和 $j$）以及点数（$x$ 和 $y$）的遍历**全部采用逆序**。这完美契合 0-1 背包“每件物品只用一次”的性质，免去了繁琐的 `memset` 拷贝上一层状态的操作，代码极其紧凑。

**2. 用 DP 数组的和作为自洽分母**

在计算期望的分母（总方案数）时，没有去手写公式算 $\binom{47}{5} \times \binom{42}{k}$。

- **编码优雅点**：直接使用 `bs = (bs + f[k][x][5][y]) % mod` 将 DP 跑出来的所有合法状态求和作为分母。这是一种极佳的**防御性编程**习惯。它保证了分子和分母绝对处于**同一个样本空间**内。即使你的 DP 转移状态写得有细微瑕疵，分子分母同口径统计也能大概率规避掉由于组合数学公式推导错误带来的致命 Bug。

**3. 防御性取模函数的封装**

- **编码优雅点**：使用了按引用传递的 `chadd(int &x, int y)` 和 `chmul`，并在内部直接处理了 `< 0` 的情况。在处理期望题中“扣除 $k$ 分”产生负数时，避免了在核心的六重循环内出现 `ans = ((ans + ...) % mod + mod) % mod` 这种极易少写一个括号或取模导致 WA 且极难排查的“面条代码”。核心逻辑显得非常干净。



## E 区间频次同余映射 (Zobrist哈希+莫队)

------

**一句话题意**

给定一个长度为 $n$ 的字符串 $s$。回答 $q$ 次询问，每次询问给定区间 $[x, y]$，求该区间内有多少个子串 $[l, r]$ 满足：子串中每种字符的出现次数均是 $k$ 的倍数。

------

**思维 Trick (题意转换与降维)**

- **Trick 1：区间频次倍数 $\rightarrow$ 前缀和同余状态匹配**

  “每种字符出现次数为 $k$ 的倍数”直接统计很难。将其转化为前缀和思想：对于任意字符 $\alpha$，区间 $[l, r]$ 满足条件，当且仅当 $c_{r, \alpha} \equiv c_{l-1, \alpha} \pmod k$。

  这一步将“验证一个区间”的判定问题，极其巧妙地降维成了“在 $[x-1, y]$ 区间内寻找**状态完全相同**的端点对”的计数问题。

- **Trick 2：高维状态哈希降维 (Zobrist Hashing)**

  前缀状态是一个长度为 26 的向量 $(c_a \bmod k, \dots, c_z \bmod k)$，直接比较复杂度为 $O(\Sigma)$。

  引入随机权值：为 26 个字母预先生成不可预测的 64 位随机数 $v[\alpha]$。将多维向量的映射压缩为一个 64 位的哈希值 $\sum (c_{i, \alpha} \bmod k) \times v[\alpha]$。这使得复杂状态的比对变成了极低碰撞率的 $O(1)$ 整数判等，完美契合离线查询的需求。

------

**编码 Trick (降智与常数优化)**

- **Trick 1：加减法 $O(1)$ 模拟取模状态转移**

  在莫队或前缀和递推的 `add/del` 中，如果不假思索地写 `cnt = (cnt + 1) % k`，会导致哈希值的重算非常麻烦。

  你的代码中使用了极优雅的差分思维：常规情况下增加 $v[ch]$；当 `cnt[ch] == k` 时，直接减去 $v[ch] \times (k-1)$，相当于瞬间将该字符的哈希贡献清零。这种做法避免了取模运算，且正反操作完全对称，大大降低了莫队转移函数的编码难度。

- **Trick 2：莫队指针移动的“防越界”标准范式**

  莫队的四个 `while` 循环顺序如果写错，极易在中间状态产生 $l > r$ 的幽灵区间导致 RE 或错解。

  牢记**“先扩大，再缩小”**的四句真言范式：

  1. `while(l > x) add(--l);` (左端扩大)

  2. `while(r < y) add(++r);` (右端扩大)

  3. `while(l < x) del(l++);` (左端缩小)

  4. `while(r > y) del(r--);` (右端缩小)

     这种写法可以免疫一切区间端点交错的 edge case。

- **Trick 3：预处理离散化彻底绞杀 `unordered_map` 的常数**

  很多选手在面对 64 位哈希值统计频次时，会下意识使用 `unordered_map`，这在 $O(n \sqrt{q})$ 的高频调用下必然 TLE。

  你的代码在莫队运行前，将所有可能出现的前缀哈希值提取出来，通过 `sort` + `unique` + `lower_bound` 离散化为紧凑的秩 `rk[i]`。莫队内部只操作原生的定长数组 `num`，将哈希表的巨大常数降维打击到极致的真正 $O(1)$，这是非常成熟的竞赛代码习惯。

## 【ODT+权值线段树】等差区间覆盖转值域区间加减

**题意简述**

维护一个长度为 `n` 的数组，支持以下操作：

1. **区间修改**：将下标区间 `[l, r]` 的元素替换为首项为 `d`、公差为 1 的等差数列（即 `d, d+1, ..., d+r-l`）。

2. **全局查询**：每次修改后，求数组中出现次数最多的元素值。若有多个，输出值最小的那个。

   （数据范围：`n`, `m` 均在 200,000 级别）

------

**思维 Trick (核心转化)**

**【下标连续段 映射 值域区间加减】**

常规的区间覆盖通常是把 `[l, r]` 全部变成同一个数 `v`（值域单点 `v` 的频次增加 `r-l+1`）。

但这道题的跳跃点在于：用公差为 1 的等差数列覆盖 `[l, r]`，意味着在此操作下，**该区间内产生的每一个新值都恰好只出现一次**。

由此产生极其优雅的域转换：

- **销毁旧状态**：在下标域抹除一段旧的等差数列区间 `[cl, cr]`（首项为 `cv`），等同于在**值域**上将区间 `[cv, cv + cr - cl]` 内的所有数字的频次统一 `-1`。
- **建立新状态**：在下标域插入新的等差数列 `[l, r]`（首项为 `d`），等同于在**值域**上将区间 `[d, d + r - l]` 内的所有数字的频次统一 `+1`。

这一步 Trick 直接将题目解构为两个正交的经典模型：**ODT 维护下标域连续段 + 权值线段树维护全局最值**。

------

**编码 Trick (降智技巧)**

在你的 AC 代码中，有几个极其优秀的工程化和防踩坑实现，非常值得固化到你的个人模板库中：

1. **天然的 Tie-breaker（完美利用线段树拓扑结构）**

   题目要求“频次相同取编号最小”。在你的线段树 `pushup` 函数中，仅仅通过 `if(l.mx >= r.mx)` 这一行就解决了。因为线段树的左子树天然代表更小的值域编号，优先取左子树即可自动满足编号最小的条件，完全避免了去手写繁琐的 `pair<int, int>` 重载比较逻辑。

2. **ODT 迭代器防失效圣经**

   在 `assign`（或 `workOdt`）操作中，严格贯彻了 **先 `split(r + 1)`，后 `split(l)`** 的顺序。这在比赛高压环境下是避免莫名其妙 RE 的死规矩。

3. **值域上限前瞻与静态化**

   没有因为值域可能会增加而去写复杂的动态开点线段树。而是精准预判出极值上限不会超过 `2n`，开局直接用静态数组 `vector<Info> a(n * 2 + 1)` 一次性铺满，大幅降低了编码复杂度和 Debug 成本。
