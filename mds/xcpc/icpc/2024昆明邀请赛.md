## E 差分不变性 + 前缀 GCD 跳变点降维

**1. 一句话简明题意 (Abstract)**

给定长度为 `n` 的正整数数组和一个非负整数 `k`。你可以至多选择一个连续子段，将该子段内的所有元素加上 `k`。求操作后，整个数组可能的最大公约数（GCD）。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题的思维跨度很大，核心在于将**动态的区间修改**转化为**静态的属性查询**，并利用**单调性**进行降维打击：

- **Trick 1：更相减损术剥离动态变量（差分不变性）**
  - **线索**：对区间加上 `k` 后，区间内所有的数都变了，直接求 GCD 毫无头绪。
  - **跳跃**：利用经典性质 $\gcd(x, y) = \gcd(x, |x - y|)$。对于修改后的区间 $[a_l+k, \dots, a_r+k]$，它的 GCD 等价于 $\gcd(a_r+k, |a_l-a_{l+1}|, \dots, |a_{r-1}-a_r|)$。
  - **本质**：极其巧妙地发现**区间整体加法不会改变相邻元素的差值**。这样就把一个动态区间拆成了“静态的差分 GCD”和一个“动态的基底（$a_r+k$）”，将原数组解构为四个互不干扰的独立部分：前缀、后缀、区间差分、基底。
- **Trick 2：利用跳变点降维搜索空间（单调性贪心）**
  - **线索**：有了四个独立部分，暴力枚举左右端点 `l` 和 `r` 的复杂度是 $O(n^2)$，依然会 TLE。
  - **跳跃**：固定右端点 `r`，观察左端点 `l` 左移的过程。前缀的长度变短，前缀 GCD 必然**单调递增**（或不变）；同时区间差分涵盖的范围变大，差分 GCD 必然**单调递减**（或不变）。
  - **本质**：因为前缀 GCD 只有在无法整除时才会减小，整个数组的前缀 GCD 最多只会发生 $\log(\max A)$ 次“跳变”。在每一段前缀 GCD 相同的值域里，我们自然希望中间的差分 GCD 越大越好，所以只需要枚举那些**前缀 GCD 刚好发生跳变的位置**作为 `l` 即可。这把 $O(n^2)$ 直接降维到了 $O(n \log(\max A))$。

------

**3. 编码 Trick (优雅实现)**

- **编码 Trick：多开数组长度以容纳数学边界 $\gcd(x, 0) = x$**
  - **痛点**：在计算后缀、前缀或枚举到边界（如 `r = n`）时，很容易因为访问 `suf[n+1]` 导致数组越界，或者需要写一堆 `if (r == n)` 的特判逻辑。
  - **优雅解法**：在你的 AC 代码中，直接将 `suf` 开到了 `n+2`，默认初始化为 0。在数学上，任何正整数与 0 的最大公约数都是它本身（$\gcd(X, 0) = X$）。这样当越界访问到 `suf[n+1]` 时，它取出的 0 参与 GCD 运算相当于**自动隐身**，完全符合“不存在后缀”的物理意义，省去了大量臃肿的特判代码。

------

**4. 破局“贪心打补丁”泥潭的建议**

在 XCPC 中，“凭直觉写贪心 -> 遇到反例 WA -> 加 if 补丁 -> 遇到新反例再 WA” 是最消耗精力和心态的死亡循环。要快速回归正确思路，建议养成以下习惯：

1. **设立“事不过三”的止损线**：如果你的代码里出现了超过 3 个为了过某个特殊 case 而硬加的 `if-else` 分支，**立即把手离开键盘**。这说明你的贪心模型在根基上就是错的，补丁永远打不完。
2. **寻找“不变量” (Invariants)**：贪心失效通常是因为状态空间互相耦合（比如这题的区间加法改变了所有元素）。退一步想，操作中**什么属性是绝对不变的**？（例如：和不变、差不变、奇偶性不变、异或和不变）。从不变量重新切入，往往能看到题目的本质。
3. **逆向思考（答案验证法）**：不要去想“我该怎么操作出最大的 GCD”，而是假设“如果答案是 $g$，原数组必须满足什么条件？”。把构造问题转化为判定问题，思路会清晰很多。
4. **手玩极端反例**：不要只用样例测试。在纸上画一个全 0 数组、全同数组、完全互质数组、或者递增/递减数组。如果你的“打补丁贪心”在脑内跑这些极端数据时逻辑极其别扭，大概率大方向错了。

## J **二元字典序状态最短路**

------

**题意精简**

给定一张带颜色和边权的无向图，以及 $k$ 张按固定顺序排列的车票（每张票指定了颜色和最大容量）。

**规则**：一张票只能走颜色匹配且总长不超过其容量的连续路径。车票必须按序号递增使用（允许跳过部分车票）。

**目标**：从起点 $1$ 出发，求这 $k$ 张票按序遍历后，能够到达的所有节点集合。

------

**思维 Trick (关键跳跃)**

1. **状态空间的二元重定义 (Distance Redefinition)**

   传统最短路维护的是标量（总路径最短），而本题为了给后续移动保留最大的选择空间，核心贪心原则是：“到达某点时，用的票越早越好；如果用的票一样早，这张票剩下的容量越多越好”。

   **跳跃点**：将距离定义重构为二元组 `(当前使用的车票序号 r, 这张票已使用的容量 d)`，并利用**字典序比较**。这完美契合了 Dijkstra 算法中“无后效性”和“贪心拓展”的前提。

2. **转移决策的极致解耦 (Greedy Decision Split)**

   面对“按序使用且可跳过”的限制，最容易陷入的误区是去模拟“跳过”这个动作（即在原地点耗费状态去换票）。

   **跳跃点**：在拓展边时，将决策严格二分为两类：

   - **白嫖当前票**：颜色对得上且容量够，直接转移状态 `(r, d + w)`。
   - **新开一张票**：无视当前票，直接贪心地去“未来”找**最早的一张**颜色匹配且总容量 $\ge w$ 的票。将寻找下一张票的动作与图上转移合并，状态直接跃迁为 `(next_r, w)`。

------

 **编码 Trick (工业级实现习惯)**

你这份 AC 代码的结构非常漂亮，展现了极强的工程化思维。以下是值得固化进你的“编码流 (Coding Stream)”中的优秀习惯：

1. **数据结构与状态机逻辑的绝对解耦**

   将“寻找下一张合法车票”这一复杂动作，完全封装进了 `get(curC, curR, w)` 这个 Lambda 函数中。Dijkstra 的主循环体中没有任何关于 ST 表或二分查找的细节暴露。这种黑盒化处理，让主程序的拓扑流极其干净，排错时能迅速定位是图论逻辑错了，还是数据结构写挂了。

2. **`std::array` 替代自定义 `struct` 的妙用**

   使用 `typedef array<i64, 3> AL3;` 并结合 `priority_queue<AL3, vector<AL3>, greater<AL3>>`。

   `std::array` 和 `std::tuple` 原生自带严格的字典序比较逻辑。这省去了定义 `struct State` 并手写重载 `operator>` 的繁琐，不仅代码骨感，而且极大地降低了手残写错比较逻辑的风险。

3. **静态区间最值优选 Sparse Table 而非线段树**

   在处理寻找 `next_r` 的逻辑中，由于车票数据是完全静态的，选用 ST 表配合二分（`queryMax` 做到 $O(1)$），使得单次寻找的复杂度严格控制在极小常数的 $O(\log k)$。如果这里随手写一个线段树，在 $5 \times 10^5$ 的数据规模下，极大概率会被常数卡死（TLE）。

4. **Lambda 表达式捕获与内存隔离**

   在 `solve` 函数内部使用 `auto init = [&](...)` 和 `auto queryMax = [&](...)`。这避免了多组测试数据下全局变量污染和 `memset` 带来的巨额开销，内存随用随开，随函数结束自动释放，是处理 ICPC 级别多测题目的标准安全姿势。



## L **二维偏序 LIS 与扫描线动态贡献**

------

**题意精简**

在二维网格中，从起点 $(0,0)$ 出发走到矩阵 $(p,q)$ 范围内的任意整点。原本只能向上或向右走，现有 $n$ 条给定的斜向捷径（走 1 步等价于向右和向上各走 1 步）。求走到整个 $(p,q)$ 范围内所有目标点所需的**最短曼哈顿距离之和**。

------

**思维 Trick (关键跳跃)**

这道题是冲刺 ICPC 金牌非常典型的“思维大一统”题目，它的推导链条环环相扣：

1. **路径代价的“补集转化” (Distance Redefinition)**

   正向求每个点的最短距离极其复杂。**跳跃点**：将曼哈顿距离 $x+y$ 视为基础开销，每走一条斜线等价于“省下 1 步”。问题瞬间转化为：求所有点能够**最多利用的捷径数量之和**。

   公式化为：

   $$\sum \sum f(x,y) = Base - \sum_{x=0}^p \sum_{y=0}^q k_{max}(x,y)$$

2. **二维偏序的“降维打击” (2D Poset to 1D LIS)**

   由于捷径必须严格向右上方走才能连续使用，这构成了一个典型的二维偏序关系：$X_{end1} \le X_{start2}$ 且 $Y_{end1} \le Y_{start2}$。

   **跳跃点**：通过巧妙的排序规则（按 $X$ 升序为第一关键字，**$Y$ 降序**为第二关键字），直接把 $X$ 维度的拓扑序强制安排明白。此时，寻找最多能走多少条捷径，完美等价于在剩余的 $Y$ 维度上求**最长上升子序列 (LIS)**。$Y$ 降序的绝妙之处在于，它从物理规则上锁死了同一个 $X$ 坐标下最多只会被 LIS 选中一条捷径。

3. **从单点查询到“面积积分” (Dynamic Area Contribution)**

   普通的 LIS 只求一个最大长度，但本题要对 $10^{12}$ 级别数量级的网格点求和。

   **跳跃点**：LIS 的单调数组 `f[len]` 本质上维护的是一个“门槛”——即走 `len` 条捷径的最小 $Y$ 坐标。只要目标点的 $y \ge f[len]$，它就能享受到这 `len` 步的节省。因此，每个 `f[len]` 覆盖的有效目标点数量就是 $q - f[len] + 1$。这把复杂的二维前缀和，降维成了对 LIS 数组维护的一个动态标量 `cur`。

------

**编码 Trick (工业级实现习惯)**

你这份 AC 代码不仅逻辑严密，最出彩的地方在于**复杂度的极致控制**。这是一份极其优雅的“事件驱动型”板子：

1. **事件驱动的“跳跃式”扫描线 (Event-Driven Sweeping)**

   很多选手在推到最后一步时，会习惯性地写一个 `for(int X = 0; X <= p; X++)` 去按步推进扫描线。但你的代码：

   ```cpp
   res += 1ll * (cx - lst) * cur;
   lst = cx;
   ```

   直接利用相邻事件的横坐标差值 `(cx - lst)` 批量计算空闲区间的贡献。这把时间复杂度中的 $O(P)$ 项彻底抹去，使得算法的运行时间**完全独立于棋盘大小**，达到了严格的 $O(N \log N)$ 级别。这是极其高超的编码直觉。

2. **$O(1)$ 的差分式全局维护**

   在 LIS 数组更新时，没有选择重算整个数组的贡献，而是通过：

   ```cpp
   cur -= (q - f[pos] + 1); // 剥离旧贡献
   f[pos] = y;              // 状态跃迁
   cur += (q - f[pos] + 1); // 注入新贡献
   ```

   像维护线段树懒标记一样，用差分的思想完成了 $O(1)$ 的状态平滑过渡。

3. **`Two-Pointers` 批量处理无序同构态**

   使用了 `while(j < a.size() && a[j][0] == a[i][0])` 的双指针结构。在扫描线遇到多个相同 $X$ 坐标的事件时，将它们在同一个大轮次中消化掉。这避免了将它们拆解成不同时序导致 LIS 逻辑在同列内部产生自相矛盾的转移，是保证排序 Trick 2 绝对成立的最后一道防线。

------

这类“二维偏序转 LIS 并动态维护贡献”的模型在解决带约束的路径规划、覆盖问题时非常核心。沿着这个思维模型延展一下，如果这道题升阶为三维空间（增加一个 $Z$ 轴，整点到整点），你觉得除了排序降维外，还需要引入什么高级数据结构来替代这个一维的 `f` 数组？



## C 正难则反 / 间隙二分匹配

------

**极简题意**

- **环境**：$10^9 \times 10^9$ 棋盘，$n$ 个城堡，$m$ 个障碍物。
- **规则**：同行或同列且中间无任何实体（城堡/障碍）的两个城堡会互相攻击。
- **目标**：**移除** $k$ 个障碍物（等价于**保留** $m - k$ 个），使得互相攻击的城堡对数**最小化**（等价于被阻断的攻击对数**最大化**）。输出最小对数及移除方案。

------

**思维 Trick (Logic Tricks)**

这道题的思维跳跃非常经典，主要集中在以下三点：

1. **正难则反 (Reverse Thinking) & 贡献转化**
   - *Trick:* 移除障碍物会导致原本安全的城堡互相攻击，局面会“变乱”，极难维护。将其转化为“手里有 $m-k$ 个名额，如何放置障碍物能阻断最多的攻击”。
2. **虚实转化：把“空隙 (Gap)”抽象为资源**
   - *Trick:* 互相攻击的永远是**相邻**的城堡。因此，两个相邻城堡之间的线段可以视作一个“坑位（Gap）”。
   - 一个障碍物落在行 Gap 上，产生 1 点收益；落在列 Gap 上，产生 1 点收益；落在行、列 Gap 的**交点**上，产生 2 点收益（双效障碍物）。
3. **边际收益去重 $\rightarrow$ 二分图最大匹配**
   - *Trick:* 两个双效障碍物如果落在同一条 Gap 上，第二个障碍物的收益就会从 2 降级为 1 或 0（因为这条攻击线已经被阻断过了）。
   - *模型构建:* 为了让收益为 2 的障碍物最多且不冲突，我们将**行 Gap 作为左部点，列 Gap 作为右部点，交点上的障碍物作为边**。跑二分图最大匹配，匹配上的边就是绝对不冲突的、价值为 2 的障碍物集合。剩下的名额再贪心分配给单效或无效障碍物即可。

------

**编码 Trick (Coding Tricks)**

你的代码中已经包含了非常优秀的编码习惯，这里为你总结并强化这些能大幅降低思维复杂度的 Coding Tricks：

1. **统一实体扫描，抛弃繁琐的分支 (Unified Entity Sorting)**
   - *编码技巧:* 将城堡和障碍物封装进同一个 `Node` 结构体中，增加一个 `bool iscb` 标识。按坐标排序后，遍历时只需看 `iscb` 即可划分 Gap，障碍物会自动“掉进”它所属的 Gap 中。这彻底避免了写恶心的双指针去匹配坐标。
2. **高阶函数复用：Lambda 处理对称逻辑 (DRY with Lambdas)**
   - *编码技巧:* 行列的处理逻辑是**完全对称**的。你写了一个 `auto work = [&](auto &mp, int &cnt)` 的 Lambda 函数。这不仅让代码行数减半，更重要的是**排除了“复制粘贴后忘记改变量名 (x 变 y)”的低级 Debug 噩梦**。
3. **网络流回溯方案神技：返回边索引 (Edge Index Tracking)**
   - *编码技巧:* 在 Dinic 的 `addEdge` 中加入 `return e.size() - 2;`。很多选手跑完最大流后，为了找回具体是哪些边构成了匹配，会重新遍历整个图。而你提前存下了 `eid`，跑完后直接 $\mathcal{O}(1)$ 判断 `mf.e[eid].cap == 0`。这是写网络流输出方案最优雅、最不易错的 Trick。
4. **贪心降级的优雅实现 (Cascading Greedy)**
   - *编码技巧:* 在最后分配 `keep` 名额时，严格采用了“三级瀑布流”写法：
     - 第一级：优先满足网络流匹配成功的边（双效）。
     - 第二级：遍历还没选的，只要它的行或列 Gap 还没被覆盖，就选它（单效）。
     - 第三级：盲选剩下的兜底（无效）。
     - 每一级都带着 `if(!keep) break;`。逻辑极度清晰，不需要复杂的权值排序。



## H 局部最值域切分与互相关 NTT 计数

**极简题意**

给定长度为 $n$ 的数组，统计有多少个连续子数组满足**其内部最大值恰好出现了 $k$ 次**。对所有 $k \in [1, n]$ 算出答案 $c_k$，输出其对应的 Hash 值 $\sum_{i=1}^n (i \times c_i^2) \pmod{998244353}$。

------

**思维 Trick (核心跳跃点)**

- **Trick 1：从“枚举区间”到“枚举最值域” (贡献法转换)**
  - **跳跃点：** 放弃思考“某个子数组的最大值是谁”，转而思考“**某个元素在多大的范围内能绝对称霸**”。只要子数组不跨越比它大的元素，它就是最大值。
- **Trick 2：极值孤岛的组合切分 (Chunking)**
  - **跳跃点：** 若某区间 $(L, R)$ 内的最大值为 $V$，且 $V$ 出现了 $N$ 次，这 $N$ 个位置会把空隙切分成 $N+1$ 段。选出包含 $k$ 个 $V$ 的子数组，本质上是在挑左右端点。设第 $i$ 段可选端点数为 $a_i$，则包含 $k$ 个最大值的选法为 $\sum_{i} a_i \times a_{i+k}$。
- **Trick 3：滑动乘积转多项式卷积 (Cross-correlation 互相关)**
  - **跳跃点：** 形如 $\sum_{i} a_i \times a_{i+k}$ 的式子是“互相关”而不是标准卷积。**神仙操作在于翻转**：令 $b_i = a_{N-i}$，则式子变为 $\sum_{i} a_i \times b_{N-i-k}$，完美对应多项式乘法中 $x^{N-k}$ 的系数。直接上 NTT 降维打击。

------

**编码 Trick (优雅实现习惯)**

- **Trick 1：单调栈弹 `>=` 求严格超集**
  - **亮点：** 代码中 `while(!stk.empty() && a[stk.back()] <= a[i])`，不管是找左边界还是右边界，都坚持弹出小于**等于**的元素。这样找出来的 $L$ 和 $R$ 是**严格大于**当前元素的边界。这让连续相同的最大值（如 `3, 3, 3`）能完美共享同一个 $(L, R)$ 边界，避免了复杂的去重逻辑。
- **Trick 2：`map` 结合 `while` 的连通块提取**
  - **亮点：** 直接按权值用 `map` 分组后，通过 `while(j < M && L[pos[j]] == l && R[pos[j]] == r)` 双指针扫描。这行代码极度优雅地把原本需要并查集或复杂区间合并才能搞定的“相同最值连通块”直接抠了出来，思维负担降到最低。
- **Trick 3：翻转自乘的极简封装**
  - **亮点：** `vector b = a; reverse(b.begin(), b.end()); NTT(a, b);` 配合高度封装的 `NTT` 模板，让主逻辑完全不被底层数学运算的细节污染，像高精度乘法一样调用。
