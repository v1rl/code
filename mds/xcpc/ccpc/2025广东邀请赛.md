## K-间隔去重 DP

**一句话题意**

给定长度为 $n$ 的字符串 $S$ 和整数 $k$，求 **相邻字符在原串中下标差严格大于 $k$** 的**本质不同**子序列的总数。

------

**核心 Trick 总结**

这道题其实是经典问题“本质不同子序列（Distinct Subsequences）”的进阶版。它的核心思维跳跃（Trick）可以拆分为以下两点：

**Trick 1：将“距离约束”转化为“合法前缀总数”的延迟转移**

- **常规思维**：找合法的子序列，需要枚举上一个字符的位置。
- **跳跃点**：因为要求距离 $> k$，当前字符 $S[i]$ 能够拼接的前驱集合，刚好就是**前 $i-k-1$ 个字符所构成的所有合法子序列**。因此，我们根本不需要关心前面具体是什么字符，只需要维护一个数组 `f[x]` 表示“前 $x$ 个字符构成的合法序列总数”。每次转移时，直接取 `f[i-k-1]` 即可做到 $O(1)$ 转移。

**Trick 2：利用“包含关系”实现极简的整体去重（最核心的跳跃）**

- **常规思维**：去重要用哈希表或者复杂的容斥原理，把重复的序列一个个揪出来减掉。
- **跳跃点**：**当前字符带来的“净增量” = 假装没重复产生的新组合数 - 历史重复组合数。**
  - 假设字符 `ch` 上次出现在位置 $j$。由于 $j < i$，那么 $j$ 能拼接的合法前缀集合 $[1, j-k-1]$，**必然是**当前 $i$ 能拼接的合法前缀集合 $[1, i-k-1]$ 的**子集**！
  - 这意味着，之前在位置 $j$ 生成的所有以 `ch` 结尾的序列，在当前位置 $i$ **一定会被原封不动地再生成一次**。
  - 所以，我们只需要记录上次生成了多少个（即你的 `dp[ch]`），然后用这次算出的总数 `num`，直接无脑减去 `dp[ch]`，剩下的就绝对是纯净的、不重复的新增序列！这也是为什么你的未注释代码比注释掉的代码更加精炼的原因。

## I 视角逆转 + 压位 SOS DP

**题意极简提取**

给定 $n$ 个长度为 $3$ 的序列（每个序列内的三个数字各不相同）。

求合法的四元组 $(i, j, k, id)$ 数量，满足：

$i, j, k$ 分别提供一个**不同位置**的元素，这三个元素恰好能完全拼出第 $id$ 个序列，且 $id \in \{i, j, k\}$。

------

**思路 Trick (The "Aha!" Moments)**

**1. 视角逆转（Target Fixing）：化“无头苍蝇式组合”为“定向填坑”**

- **痛点：** 正向枚举 $i, j, k$ 去凑一个未知的目标队伍名，不仅复杂度爆炸，而且无法保证凑出来的人名真实存在。
- **跃迁：** 既然队伍名必定属于 $i, j, k$ 中的某一个人（假设是 $i$），那就**反客为主，直接枚举目标队伍名 $id = i$**。此时 $i$ 必定提供自己名字中的一个字，那么剩下的 $j$ 和 $k$ 只需要去填补 $i$ 名字里剩下的**任意两个不同位置**即可。问题瞬间从“三元素匹配”降维成了“二元素覆盖”。

**2. 查超集避实就虚，反演求精准态 (Superset to Exact-set)**

- **痛点：** 在哈希表中直接查询“恰好只在某些位置匹配”的人数（精确集）是极度困难的，因为要排除其他位置的干扰。
- **跃迁：** 避开精确匹配，去查**“至少在这些位置匹配”**的人数（超集）。只要把关心的位置提取出来，不在乎的位置直接无视，丢进 Map 统计即可。拿到超集 $g$ 数组后，利用容斥原理（或 SOS DP）将其向下反演，完美洗出精确的 $f$ 数组。这种“先放宽条件统计，再用数学工具去重”是计数类问题的核心技巧。

**3. 正难则反：合法组合数 = 总组合数 - 绝对互斥组合数**

- **痛点：** 算出精确的 $f$ 数组后，正向枚举合法的队友组合 $(j, k)$ 需要讨论的配对情况依然有点多。
- **跃迁：** 直接计算总的候选人两两组合数 $C(sum, 2)$，然后减去**“绝对不可能凑出两个不同位置”**的组合。什么时候绝不可能？只有当两个人恰好只匹配了相同的唯一位置时（即都在 $f[1], f[2], f[4]$ 的内部相互组合）。

------

**编码 Trick (Coding Elegance)**

**1. `i64` 极速压位 (Bitmask Packing)**

- 序列里的数字高达 $10^6$，但在二进制下 $2^{20} > 10^6$。你精准地利用了这一点，将三个数字分别左移 `0, 20, 40` 位，用按位或 `|=` 压缩进一个 64 位整数中。
- **优雅之处：** 彻底抛弃了手写字符串哈希或是臃肿的 `std::tuple<int, int, int>` 带来的常数灾难。用原生的整数比较和哈希替换了复杂的结构体，是极致的底层常数优化。

**2. 标准化 SOS DP 模板替代手写容斥**

- 当状态量只有 3 位时（共 8 个状态），很多人会选择手写一堆 `if-else` 来做容斥加减。但你使用了标准的 3 层 `for` 循环高维前缀差分模板。
- **优雅之处：** 极大降低了思维负荷和“手误”导致 WA 的概率。并且这个模板具有完美的扩展性，日后遇到需要状态压缩到 15 位甚至 20 位的类似套路题，这套写法可以直接照搬，而手写容斥则会直接瘫痪。



------

### 常数战神 —— `vector` + `sort` + 离散化去重 (最稳健)

无论你是用 `__int128` 还是 `std::array<int, 4>`，一旦塞进 `map` 或者 `unordered_map`，在 $N = 10^5$ 且常数较大的情况下，都有极大概率被卡 TLE 或 MLE（哈希冲突 / 红黑树指针开销过大）。

对于金牌选手来说，最稳健的做法是**彻底抛弃 Map**。

**编码 Trick：**

把不在 mask 里的位置直接填 `0`，打包成一个 `std::array` 或者 `tuple`，全部丢进 `vector` 里排序。排序后相同的状态必定相邻，利用 `std::lower_bound` 和 `std::upper_bound` 的差值就是 `cnt`。

```cpp
// mask 掩码处理，不在状态内的位置直接赋 0
array<int, 4> get(const array<int, 4>& a, int mask) {
    array<int, 4> res = {0, 0, 0, 0};
    if(mask & 1) res[0] = a[0];
    if(mask & 2) res[1] = a[1];
    if(mask & 4) res[2] = a[2];
    if(mask & 8) res[3] = a[3];
    return res;
}

// 预处理部分：
vector<array<int, 4>> states[16];
for(int i = 0; i < n; i++) {
    for(int mask = 1; mask < 16; mask++) {
        states[mask].push_back(get(a[i], mask));
    }
}

// 查询前必须排序
for(int mask = 1; mask < 16; mask++) {
    sort(states[mask].begin(), states[mask].end());
}

// 查询时，替代 cnt[mask][get(a[i], mask)] 的写法：
auto target = get(a[i], mask);
auto it1 = lower_bound(states[mask].begin(), states[mask].end(), target);
auto it2 = upper_bound(states[mask].begin(), states[mask].end(), target);
int count = it2 - it1; // 这就是查出来的出现次数
```

这种做法虽然写起来多几行，但**没有任何哈希冲突的可能，内存连续，Cache 命中率极高**，是防卡常的终极手段。

**如果发现会卡常，或者维度上升到 5 维、6 维（甚至上百维的字符串匹配）**，果断切换到**方案二的 `vector` + `sort` 体系**。



## A 状压枚举列状态 + 行等效分组二分 (贡献单调性)

**1. 一句话简明题意 (Abstract)**

给定一个 $n \times m$ 的 01 矩阵（$n \le 10^6, m \le 10$）和两个常数 $A, B$。你可以任意次翻转某一行或某一列（0 变 1，1 变 0）。

**得分规则**：最终矩阵中，若第 $i$ 行第 $j$ 列的元素为 1，则贡献 $A \cdot i + B \cdot j$ 的得分（为 0 则不贡献）。

**目标**：求经过若干次操作后，矩阵能获得的最大总得分。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题的思考路径可以拆解为以下几个关键的“注意力”跳跃：

- **Trick 1：极小数据范围暗示的“状态枚举” (变量剥离)**

  - **线索**：$m \le 10$ 极小，而 $n \le 10^6$ 很大。
  - **跳跃**：行和列的操作看似相互纠缠，但只要**固定列的操作**（总共只有 $2^m \le 1024$ 种可能），每一行的最优决策就变得完全独立了！这是解构二维矩阵操作的经典起手式。

- **Trick 2：同一状态的“打包处理” (等效归类)**

  - **线索**：如果直接在确定的列状态下，暴力遍历 $10^6$ 行去计算翻与不翻的贡献，单次检查是 $O(N)$，总复杂度 $O(N \cdot 2^m) \approx 10^9$，大概率 TLE。
  - **跳跃**：既然 $m$ 只有 10，那么初始时行的状态最多只有 $2^m$ 种。我们可以把**初始状态相同的行“打包”在一起**。用 `pos[st]` 记录初始状态为 `st` 的所有行号 $i$。

- **Trick 3：贡献差值的“单调性”与二分边界 (代数变形 -> 线性判定)**

  - **线索**：对于状态同为 `st` 的一堆行，它们在确定的列翻转状态下，变幻后的状态 `cur` 也是相同的。决定某一行 $i$ “翻”与“不翻”的条件是什么？

  - **跳跃**：写出翻与不翻的得分代数式作差。

    设该行 1 的数量为 $cnt$，1 所在列下标和为 $W$。

    不翻的得分：$i \cdot A \cdot cnt + B \cdot W$

    翻转的得分：$i \cdot A \cdot (m - cnt) + B \cdot (\frac{m(m+1)}{2} - W)$

    **作差判定**：令 $\Delta = \text{不翻} - \text{翻转}$。化简后你会发现，$\Delta > 0$ 的条件可以整理成形如 $i \cdot C_1 > C_2$ 的不等式（其中 $C_1, C_2$ 是只与当前行状态和 $A, B$ 有关的常数，与 $i$ 无关！）。

  - **绝杀**：因为我们是按顺序把行号 $i$ 压入 `pos[st]` 的，所以 `pos[st]` 内的 $i$ 是**天然严格递增**的。这意味着 $i \cdot C_1$ 具有严格的单调性！我们根本不需要遍历每个 $i$，只需要通过**二分查找**（$O(\log N)$）找到那个 $i \cdot C_1 = C_2$ 的分界点，分界点一侧全是不翻转，另一侧全是翻转，配合**前缀和** $O(1)$ 就能算出这一组行的总贡献。

------

祝你后面的训练顺利！这种将代数式化简后发现单调性，再用二分+前缀和替代暴力遍历的套路，在 XCPC 的推公式贪心题里非常普遍，吃透这一道，能顶很多道。如果有其他难啃的 Trick，随时发来一起拆解！



## G 独立代价贪心 / 潜在罚时增量排序

**1. 一句话简明题意 (Abstract)**

已知我方队伍最终的过题数 $a$ 和罚时 $b$。

给定另一支队伍全场的提交记录（包含通过 `ac`、拒绝 `rj` 以及封榜后的未知状态 `pd`）。

**目标**：在假设该队伍所有 `pd` 状态的提交都有可能通过的情况下，求他们**最少**需要通过多少个 `pd` 题目，才能在排名上严格超越我方（过题数大于 $a$，或过题数等于 $a$ 且总罚时严格小于 $b$）。如果全过也无法超越，则输出 `-1`。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题虽然披着“模拟/规则计算”的外衣，但核心是一个**贪心**。它的思维跳跃点在于对“最有利情况”的剥离：

- **Trick 1：最优解的“首次命中”原则 (状态收敛)**
  - **线索**：题目中一个队伍对同一道题可能在封榜后有多次 `pd` 提交。
  - **跳跃**：既然我们要判断他们“有没有可能”超越我们，就必须假设他们处于**最理想的平行宇宙**。对于任何一道处于 `pd` 状态的题目，如果它最终 AC 了，对该队伍最有利的情况一定是**在封榜后的第一次提交就 AC 了**。因为这样罚时最少（不用加后续提交产生的时间和 20 分钟惩罚）。所以在处理输入时，遇到第一个 `pd` 直接截断并计算潜在罚时即可，后续的 `pd` 统统无视。你的代码中 `time[ch] == -1` 完美处理了这一点。
- **Trick 2：题目间罚时的“独立性”与贪心 (代价排序)**
  - **线索**：需要求“最少”通过的题目数。
  - **跳跃**：每道题是否通过，对其他题目的罚时**毫无影响**。每道 `pd` 的题目如果 AC，都会带来一个固定的额外罚时增量（首次 `pd` 的时间 $t$ + 之前 `rj` 的次数 $\times 20$）。为了用最少的过题数超越我方，且在同等过题数下罚时尽量小，最优策略就是**优先挑罚时增量最小的题目 AC**。
  - **总结**：将所有 `pd` 题目的潜在罚时增量算出，放入数组 `ano` 中从小到大排序，然后从前往后贪心累加，直到排名超越我方为止。

## 贪心周期性：最大面值商余拆解与前缀和映射

**1. 一句话简明题意 (Abstract)**

给定一个包含 n 种面值的货币系统（最小面值 $A_1 = 1$，最大面值为 $A_n$），每次找零遵循“贪心”原则（优先使用最大面值）。有 q 次询问，每次询问给出一个正整数 m，求**有多少个正整数 x** 恰好需要 m 张纸币才能凑出。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题的核心难点在于 x 的范围是无限的，直接算显然不可能。它的思维跳跃点在于利用贪心策略的**周期性**，将无限域的计数问题映射到有限的剩余系中。

- **Trick 1：贪心策略的周期性（商余拆解）**

  - **线索**：题目强制要求“尽量先拿大面额”，这意味着对于任意极大的数，我们必然会疯狂使用最大面值 $A_n$。

  - **跳跃**：对于任意数 x，我们可以将其对最大面值 $A_n$ 做带余除法：

    $$x = q \cdot A_n + r \quad (0 \le r < A_n)$$

    由于贪心原则，凑出 x 的纸币总数必然等于“最大面值的张数”加上“凑出余数需要的张数”，即：

    $$f(x) = q + f(r)$$

- **Trick 2：方程解的唯一映射（核心跳跃）**

  - **线索**：题目要求满足 $f(x) = m$ 的正整数 x 的个数。

  - **跳跃**：将上式代入得 $q + f(r) = m$，即 $q = m - f(r)$。

    因为 x 是正整数，所以商 q 必须 $\ge 0$。这就意味着，**只要 $f(r) \le m$，就能唯一确定一个合法的商 q**（进而唯一确定一个 x）。

    因此，“有多少个 x 满足条件”等价于“在 $[0, A_n - 1]$ 这 $A_n$ 个余数中，**有多少个余数 r 满足 $f(r) \le m$**”。

    这直接把问题转变成了对 $[0, A_n - 1]$ 范围内求贪心步数的前缀和！

- **Trick 3：定义域的巧妙平移（代码实现细节）**

  - **线索**：理论上我们要统计的是 $r \in [0, A_n - 1]$ 的情况，其中 $f(0) = 0$。
  - **跳跃**：在你的代码中，循环是 `i` 从 1 到 $A_n$。为什么这样写也对？因为当查询 $m \ge 1$ 时，$f(0) = 0 \le m$ 永远成立，贡献了 1 个答案；而 $f(A_n) = 1 \le m$ 也永远成立，同样贡献了 1 个答案。用 $i = A_n$ 替换 $r = 0$，对最终的前缀和计数没有任何影响，反而让数组初始化和循环变得极其优雅。



## H 本质不同子序列去重 DP + 间隔限制 (时空延迟)

**1. 一句话简明题意 (Abstract)**

给定一个长度为 $n$ 的小写字母字符串 $S$ 和一个非负整数 $k$。求 $S$ 中满足**相邻字符在原串中下标差严格大于 $k$** 的**本质不同**非空子序列的数量，结果对 998244353 取模。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题的思考路径可以拆解为以下几个关键的“注意力”跳跃：

- **Trick 1：经典“本质不同子序列”的容斥去重套路 (De-duplication DP)**
  - **线索**：题目要求求“本质不同”的子序列。如果直接无脑转移，会把形如 `a...a` 产生重复的子序列多算进去。
  - **跳跃**：为了不重不漏，当我们在考虑加入第 $i$ 个字符（设为 `ch`）时，我们应该用“当前合法的可拼接前缀总数”减去“**之前已经计算过的以 `ch` 结尾的子序列数量**”。在你的代码中，`dp[ch]` 完美扮演了这个“历史记录”的角色，每次遇到字符 `ch`，直接减掉旧的 `dp[ch]` 即可完成去重。
- **Trick 2：距离限制转化为 DP 状态的“时空延迟” (State Shift)**
  - **线索**：相邻字符间隔至少为 $k$（即下标差 $> k$）。
  - **跳跃**：如果没有这个限制，第 $i$ 个字符可以接到前 $i-1$ 个字符形成的所有合法子序列后面。有了限制之后，第 $i$ 个字符**只能接到前 $i-k-1$ 个字符形成的合法子序列后面**。因此，我们在计算新增子序列时，用来拼接的“前缀池”从 $f[i-1]$ 变成了 $f[\max(0, i-k-1)]$。
- **Trick 3：用前缀总和数组 $f[i]$ 降维打击 (O(1) 转移)**
  - **线索**：如果每次都去遍历合法的结尾字符，复杂度会退化。
  - **跳跃**：直接定义 $f[i]$ 为“前 $i$ 个字符中本质不同的子序列总数”。那么对于第 $i$ 个字符，新增的合法序列数就是 `接上前缀池 (f[i-k-1] + 1) - 重复的 (dp[ch])`。将其累加到 $f[i-1]$ 上即可得到 $f[i]$，将状态转移压到了极致的 $O(1)$。





## J 带阈值的多源 Dijkstra (时间轴事件模拟)

**1. 一句话简明题意 (Abstract)**

在一个有向图中，每个节点（题目）需要累积接收到 $a_i$ 点能量才能解锁。

节点解锁后，会沿着所有出边向相邻节点发送 1 点能量，经过该边的时间花费为 $w$。

此外，还有若干“强制刷新器”，会在特定的时间 $t$ 直接将指定节点的能量需求降为 0（即强制解锁）。

**目标**：求所有节点的最早解锁时间（若永远无法解锁则输出 -1）。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题如果硬去模拟能量的流动会极其复杂，其核心跳跃在于**对优先队列（Dijkstra）状态定义的升维**：

- **Trick 1：将优先队列升级为“时间轴事件模拟器” (Event-Driven Simulation)**
  - **线索**：图中有边权的延迟（$w$），又有特定时间点触发的绝对事件（$t_j$）。
  - **跳跃**：不要把优先队列仅仅当成维护“节点最短距离”的工具，把它当成一条**全局时间轴**。队列里的元素不再是单纯的节点，而是“事件”。无论是初始 $a_i = 0$ 的解锁、强制刷新器的触发，还是 1 点能量的到达，都统一视为时间轴上的 Event `{time, node, type}`。每次取出时间最早的事件执行，天然保证了时间演进的正确性。
- **Trick 2：Dijkstra 与 Kahn（拓扑排序）逻辑的无缝融合 (Threshold Check)**
  - **线索**：普通 Dijkstra 中，节点第一次被访问就确定了最短路。但这里节点需要被访问（接收能量）多次才能解锁。
  - **跳跃**：借鉴拓扑排序中维护入度（in-degree）的思想，把 $a_i$ 看作一种“动态入度”。对于“能量到达”事件（`op = 0`），只有当 `--v[x] == 0` 时，才真正触发解锁操作并将距离固定。这种在最短路松弛过程中加入“阈值判定”的操作，是解决带依赖条件的图论题的经典套路。
- **Trick 3：统一逻辑与惰性删除 (Lazy Pruning)**
  - **线索**：强制解锁事件可能在节点已经被能量正常解锁后才发生；或者多次能量到达超出了 $a_i$ 的需求。
  - **跳跃**：不需要去动态修改或撤销队列里的事件。只需要一个 `ans[x] != -1`（是否已解锁）的标记。如果弹出的事件对应的节点已经解过锁了，直接 `continue` 扔掉。这种惰性删除极大简化了代码逻辑。

## K 折返映射：Manacher + Z函数 与 并查集区间染色

**1. 一句话简明题意 (Abstract)**

给定一个目标串 $S$，你需要用一台“回文打字机”生成它。打字机的工作原理是：给定一个长度为 $L$ 的模板串 $T$，它会按照下标 $1, 2, \dots, L, L-1, \dots, 2, 1, 2 \dots$ 的折返轨迹不断循环打印字符。

**目标**：对于目标串 $S$ 的每一个前缀 $S[1 \dots i]$，求出能打印出该前缀的**最短**模板串 $T$ 的长度。最后输出所有前缀答案的特定异或和。

------

**2. 核心 Trick 与思维跳跃点 (Key Tricks)**

这道题的核心难点在于将“打字机来回移动”这个动态物理过程，静态化为字符串的匹配特征。

- **Trick 1：物理过程向字符串性质的映射 (折返运动 = 回文 + 周期)**
  - **线索**：打字机的指针先正向走到尾，再反向走到底，再正向走……
  - **跳跃**：当模板串长度为 $L$ 时，它打出的前 $2L-1$ 个字符必然是一个**以 $L$（0-indexed 下标为 $L-1$）为中心的回文串**。并且，当它退回起点后再次正向移动时，相当于从下标 $2L-2$ 开始，**完美复读**了整个串的前缀。
- **Trick 2：拆解匹配条件 (Manacher + Z 函数的精准投喂)**
  - **线索**：既然生成的串由“中心回文”和“前缀复读”两部分拼接而成，我们只需验证 $S$ 是否满足这两个性质。
  - **跳跃**：
    1. **第一阶段（折返）**：用 **Manacher 算法** 预处理原串。对于枚举的模板长度 $L$，检查以 $L-1$ 为中心的回文半径，这决定了打字机第一趟能完美打印出 $S$ 的多长一段。
    2. **第二阶段（复读）**：如果回文半径足够大，说明指针成功“触底反弹”（即回文覆盖到了下标 0）。此时，从下标 $2L-2$ 开始的后续字符串，必须匹配 $S$ 的前缀。这正是 **Z 函数 (扩展 KMP)** 的拿手好戏！通过 `z[2L-2]`，可以 $O(1)$ 知道复读阶段能向右延伸多远。

------

**3. 编码 Trick (优雅实现)**

- **编码 Trick：并查集实现均摊 $O(1)$ 区间染色 (DSU for Range Assignment)**

  - **痛点**：我们要为每一个前缀求“最短”的 $L$。最直观的想法是：从小到大枚举 $L$，如果长度 $L$ 能匹配到区间 $[l, r]$，就把这个区间里还没填答案的位置都填上 $L$。但直接暴力填（双重循环）最坏复杂度是 $O(n^2)$，会 TLE。

  - **优雅解法**：在你的代码中，使用了**并查集 (DSU)** 来跳过已经填过答案的位置。

    `dsu.merge(i + 1, i)` 这行代码是精髓：每次给位置 `i` 赋值后，就把 `i` 指向 `i + 1`。下一次再遍历到 `i` 时，`find(i)` 会直接将指针瞬移到下一个**未被染色**的位置。

    这种“单向区间覆盖”的写法，将多次区间赋值的均摊复杂度降维打击到了 $O(n\alpha(n))$，不仅不需要写笨重的线段树，而且常数极小，是高阶选手必备的代码套路。
