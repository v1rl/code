## NTT

在 ICPC 冲金的训练路径上，NTT（快速数论变换）不仅是一个用于加速多项式乘法的黑盒模板，更是一套极其强大的代数工具。建立起对 NTT 的“直觉”，能够帮你把很多看似 $O(n^2)$ 的 DP 或枚举过程，降维打击到 $O(n \log n)$。

按照你注重的**方法论和 Trick 总结**习惯，我们把 NTT 的核心性质和在算竞中的高频应用场景梳理一下。

### 核心性质：为什么是 NTT 而不是 FFT？

NTT 本质上是在有限域（模 $p$ 意义下）进行的离散傅里叶变换。它完美继承了 FFT 的分治结构 ，但替换了底层的数学基底。

- **原根替代单位根：** FFT 依赖复平面上的单位根 $e^{2\pi i/n}$，而 NTT 利用了数论中的原根 $g$。在模 $p$ 意义下，$g^{(p-1)/n} \pmod{p}$ 具有和复数单位根完全相同的周期性和对称性。
- **绝对精度：** 这是算竞中最关键的性质。FFT 存在不可避免的浮点数精度误差，当答案极差较大或需要精确取模时容易 WA。NTT 在整数域上运行，结果绝对精确。
- **对模数的严苛要求：** NTT 要求模数 $p$ 必须能写成 $p = c \cdot 2^k + 1$ 的形式（且 $k$ 足够大，通常要大于序列总长度的对数）。算竞中最经典的 998244353 就是为了 NTT 而生的（$998244353 = 119 \times 2^{23} + 1$），原根为 3。如果题目给了奇怪的模数（比如 $10^9+7$），就需要用到任意模数 NTT（MTT）或拆系数 FFT。

### 算竞中的高频应用场景与 Tricks

从方法论的角度来看，只要你能把问题转化为 **“下标相加（或相减）为定值，且各项相乘求和”** 的形式，就可以掏出 NTT。

#### 1. 错位累加与序列匹配 (Cross-correlation)

这是将非标准数学问题代数化的经典 Trick。

- **特征：** 题目要求处理两个序列的“相对滑动”或“位移”，计算形如 $\sum A_i \cdot B_{i+k}$ 的贡献。
- **核心 Trick（翻转构造法）：** 正常的多项式乘法处理的是 $i + j = k$。面对 $j - i = k$（下标差为定值），核心思路是**翻转其中一个序列**。令 $A'_i = A_{n-1-i}$，则 $A'_i \cdot B_j$ 对应的下标和为 $n - 1 - i + j = n - 1 + k$（定值）。通过一次卷积，就能在 $O(n \log n)$ 内求出所有位移 $k$ 下的答案。这在处理带有通配符的字符串匹配、循环位移的最优解等问题中是绝杀。

#### 2. 生成函数 (Generating Functions, OGF/EGF)

这是 ICPC 金牌区最核心的计数推导工具。NTT 是实现生成函数运算的底层引擎。

- **特征：** 复杂的组合计数问题，背包方案数统计，或者多对象分配问题。
- **核心 Trick：** 将“选或不选”、“选几个”等逻辑，映射为多项式的系数和指数。例如，有体积为 1, 2, 3 的物品各无限个，求装满体积 $V$ 的方案数。写出其生成函数 $\frac{1}{1-x} \cdot \frac{1}{1-x^2} \cdot \frac{1}{1-x^3}$，然后利用多项式求逆或多项式 Exp 结合 NTT 快速求解。

#### 3. 分治 NTT (CDQ + NTT)

用来优化转移方程中含有卷积形式的动态规划。

- **特征：** 转移方程形如 $f_i = \sum_{j=1}^{i-1} f_j \cdot g_{i-j}$。这本身是一个自相关的卷积，但因为 $f_i$ 依赖于前面的 $f_j$，不能直接一次 NTT 跑完。
- **核心 Trick：** 采用类似 CDQ 分治的思想。递归求解左半部分 $[L, mid]$，然后把左半部分对右半部分 $(mid, R]$ 的贡献提取出来：将左半部分的 $f$ 和全局的 $g$ 进行一次 NTT 卷积，把结果累加到右半部分的 $f$ 上，最后递归处理右半部分。复杂度为 $O(n \log^2 n)$。

#### 4. 多项式全家桶 (Polynomial Family)

基于基础的 NTT 乘法，算竞中延申出了一整套多项式操作，被称为“多项式全家桶”。

- **多项式求逆 (Inv)：** 解决 $F(x) \cdot G(x) \equiv 1 \pmod{x^n}$。
- **多项式对数/指数 (Ln/Exp)：** 用于处理无标号/有标号对象的连通块与任意图转化问题（经典的 Euler 变换）。
- **多项式多点求值/快速插值：** 突破拉格朗日插值 $O(n^2)$ 的瓶颈。

### 补充视角 (Bonus)：错位累加与序列匹配通用思路

> 得到目标项（右移次数）关于实际卷积项（下标和）的数学公式

针对 FFT/NTT 处理“循环位移与同余差量”的题型，这里为你提炼一套**“闭眼推公式”的四步心法**。

无论题目怎么变形（翻转前缀、长度不等、多串匹配），只要记住这套推导流，下标映射绝对不会再错：

#### 核心心法：等式列准，移项取模

**第一步：写出原始目标的差量表达式**

明确题目要求的“代价”或“得分”在不卷的时候怎么算。

设位移量为 $d$，我们需要批量求算：

$$\sum A[i] \times B[(i+d) \bmod n]$$

> **痛点：** 这里的下标组合是 $(i+d)$ 和 $i$，两者的“差”是常量 $d$。但多项式乘法（卷积）只能处理下标“和”为常量的情况。

**第二步：翻转其中一维（变差常量为和常量）**

将数组 $A$ 翻转得到 $A'$。

牢记翻转后的下标定义：原数组的 $A[i]$，变成了新数组的 $A'[n-1-i]$。

**第三步：列出绝对下标等式（最关键一步）**

多项式 $A'$ 和 $B$ 卷积得到结果数组 $C$。结果数组 $C[k]$ 中的下标 $k$，必然等于参与相乘的两个元素的**新下标之和**。

直接把式子列出来：

$$k = (A'\text{的下标}) + (B\text{的下标})$$

$$k = (n - 1 - i) + (i + d)$$

化简得：

$$k = n - 1 + d$$

**第四步：移项推导目标变量，最后套上取模**

我们是遍历 $k$ 并得到对应的目标值 $d$ 以统计结果，所以应该得到目标值 $d$ 关于卷积项 $k$ 的同余式，直接对上一步的等式移项：

$$d = k - n + 1$$

因为题目是长度为 $n$ 的**循环位移**，所以直接在等式右边套上 $\bmod n$（注意 C++ 中负数取模的坑，加上 $n$ 保证正数）：

$$d \equiv (k - n + 1 + n) \pmod n \equiv (k + 1) \pmod n$$

#### 精炼总结与编码规范

以后遇到任何卷积求位移差量的题，不要凭空想象，直接在草稿纸上写下这两行：

1. **设卷积项：** $k = (L - 1 - i) + (i + d)$  *(其中 $L$ 为被翻转数组的长度)*
2. **解目标值：** $d = (k - L + 1) \bmod n$

**映射到代码的万能写法：**

```cpp
// i 代表卷积结果 C 的下标
for(int i = 0; i < C.size(); i++) {
    // 1. 严格按照草稿纸推导的公式算出对应的真实含义 d
    int d = (i - L + 1) % n; 
    if(d < 0) d += n; // C++ 负数取模修正
    
    // 2. 将 C[i] 的贡献精准累加到 d 对应的答案桶中
    ans[d] += C[i]; 
}
```
