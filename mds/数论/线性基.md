线性基只对非负数进行讨论

一堆数字能得到的非0异或和的结果，能被元素个数尽量少的集合，不多不少的全部得到，那么就说，这个元素个数尽量少的集合，是这一堆数字的异或空间线性基

注意，只关心去重的非0异或和能否全部得到，并不关心每种异或和的个数

如下几个结论是构建异或空间线性基的基础

一堆数字中，任意的a和b，用`a^b`的结果替代a、b中的一个数字，不会影响非0异或和的组成

一堆数字中，任意的a和b，如果有`a^b=0`，那么舍弃掉a、b其中一个数字，不会影响非0异或和的组成

一堆数字能否异或出0，在求出异或空间线性基之后，需要被单独标记

### 一、 线性基的“灵魂”是什么？（基本概念与应用场景）

**1. 一句话本质：**

线性基是一组原数组的**“浓缩精华”**。在异或（XOR）运算下，原数组能组合出的所有结果，这组“精华”都能组合出来；原数组组合不出来的，它也组合不出来。并且，这组“精华”是**最精简**的，没有任何多余的元素。

**2. 借用线性代数的视角（为什么叫“线性”基）：**

你可以把每一个数字看作一个长度为 62 的由 0 和 1 组成的向量。异或运算，其实就是**模 2 意义下的向量加法**（GF(2) 上的线性空间）。

- 原数组就是一个向量组。
- 线性基就是这个向量组的**极大线性无关组**。
- 你代码中的 `sz` 就是这个矩阵的**秩（Rank）**。
- 你代码里的 `zero = true` 就意味着原向量组中存在线性相关的向量（即存在子集异或和为 0）。

**3. 什么时候用它？（雷达触发词）**

当你读题时，看到以下关键词的组合，雷达就应该立刻响起来：

- **“给定一个数组 / 集合”**

### 二、 赛场默写指南：代码核心逻辑拆解

不要死记硬背代码，去理解**“它在贪心地维护一个什么样的结构”**。

你的 `b` 数组（`b[0]` 到 `b[62]`）是一个阶梯状的结构：**`b[i]` 保存的是“最高位为 $i$ 的那个基向量”。**

#### 1. 核心构建逻辑：`insert(x)`

**考场思维：** 来了一个新数字 $x$，我要把它“塞”进基里。怎么塞？从最高位开始，尽力用已有的基把它“消”掉。

- **找最高位 1：** 遍历第 $i$ 位（从高到低）。如果 $x$ 的第 $i$ 位是 1。
- **坑被占了？消它：** 如果 `b[i]` 已经有值了，说明在这之前已经有一个最高位是 $i$ 的数了。因为一山不容二虎，我们用 `x ^= b[i]` 把 $x$ 的第 $i$ 位变成 0，继续往下找坑。
- **坑是空的？占位：** 如果 `b[i]` 是空的，太好了，$x$ 现在就是这个维度的代表！`b[i] = x; return true;`
- **消成了 0？** 如果最后 $x$ 被彻底消成了 0，说明这个 $x$ 是个“废人”，它能被前面已经存在的基组合出来。此时 `zero = true`，记录一下原数组可以异或出 0。

#### 2. 最大值查询逻辑：`queryMax(x)`

**考场思维：** 贪心！从高位到低位看，能变大就变大。

- 对于第 $i$ 位，如果当前的答案 $x$ 在这一位是 0，并且存在 `b[i]`，那么异或上 `b[i]` 一定能让答案的第 $i$ 位变成 1，整体数值一定变大（因为 `b[i]` 最高位就是 $i$，异或它不会影响 $x$ 更高的位）。
- 所以条件是：`if (b[i] && ~x >> i & 1)` （如果基存在，且当前 $x$ 的这一位是 0），则 `x ^= b[i]`。

#### 3. 彻底重构基底：`gauss()` （为了求第 $k$ 大）

单纯的 `insert` 构建出来的基，只保证 `b[i]` 的最高位是 $i$，但不保证它的更低位没有 1。

**考场思维：** 为了求第 $k$ 大，我们需要把基变成**简化阶梯矩阵**。也就是说，如果 $b[i]$ 存在，那么整个基底里，只有 $b[i]$ 的第 $i$ 位是 1，其他基元素的第 $i$ 位必须是 0。

- 你代码里的 `gauss()` 就是在做“高斯-约旦消元”：用低位的基 `b[i]` 去消高位的基 `b[j]` 的第 $i$ 位。
- 消元后提取出的非零元素放进 `g` 数组里。`g` 数组里面的基，完全相互独立。

#### 4. 第 $k$ 大查询逻辑：`kth(k)`

**考场思维：** 二进制拆分。

有了 `gauss()` 之后，`g` 数组从小到大排列。你会发现一个极其优美的性质：**想求这组基能组合出的第 $k$ 小的数，只需要把 $k$ 进行二进制拆分。**

- 如果 $k$ 的二进制第 0 位是 1，就异或上 `g[0]`；

- 如果 $k$ 的二进制第 1 位是 1，就异或上 `g[1]`；

- 这就把一个复杂的组合问题，降维打击成了一个十进制转二进制的问题！

  *(注：如果有 `zero = true`，意味着能组合出 0，通常题目会把 0 算作第 1 小，所以你需要像代码里那样 `k -= zero`)*。

### 三、 总结：如何把知识“内化”

现在，请闭上眼睛想象一个漏斗，或者一个阶梯状的滤网：

1. 一堆乱七八糟的数字（原数组）像水一样倒进滤网。
2. 每一层滤网（第 $i$ 位）只能截留一个最高位为 $i$ 的数字（`b[i]`）。
3. 如果一个数字在某层被截留了，它就成为了基底的一部分。
4. 如果一个数字一路流到了底端，变成了 0，说明它没有提供任何新的二进制维度（线性相关）。
5. 最后留在滤网上的这些数字，就是线性基。

通过这种“消元”和“填坑”的意象，你在考场上写 `insert` 和查询时，就不会纠结，而是直接顺着水流的逻辑写下来

### 五、 进阶拓展：拟阵（Matroid）

由于你的目标是 ICPC 金牌，我非常建议你借此题了解一下**拟阵（Matroid）**的概念。

贪心算法通常缺乏坚实的理论保证，经常会被直觉欺骗。但在拟阵模型中，贪心是绝对正确的。

拟阵是一个满足特定公理（遗传性、交换性）的独立集系统。

- 无向图的森林（生成树的子集）构成一个拟阵。所以 Kruskal 算法求最小生成树（按边权排序贪心）是正确的。
- 线性空间中的线性无关向量组构成一个拟阵。所以这道题按价值排序贪心是正确的。
- 带期限和惩罚的单位时间任务调度问题，其合法调度集合也构成一个拟阵。

当你掌握了拟阵，赛场上遇到求“最大权/最小权子集”且限制条件具有“遗传性”（即子集的子集也合法）时，你就能瞬间判定是否可以大胆使用贪心，而无需花费大量时间去证明。

### 1. 题目归类与本质探讨

- **算法归类：** 线性代数 / 异或线性基（Linear Basis） / 构造（Constructive）。

### 2. 零视角思维复盘：如何从零推导

假设我们在赛场上，面对这道题，大脑应该如何一步步运转：

#### Step 2.1：捕获与转化关键条件

- **原始条件：** $c_i \in \{a_i, b_i\}$，且 $c_1 \oplus c_2 \oplus \dots \oplus c_n = 0$。
- **直觉阻碍：** 每个位置有两个选项，总共有 $2^n$ 种可能，直接搜索显然会 TLE（$n \le 10^5$）。
- **破局点（关键转换）：** 面对二选一，最有效的思维套路是**“确定一个基准线（Baseline）”**。
  - 假设我“摆烂”，所有位置全选 $a_i$。此时我会得到一个初始异或和：$Sum = a_1 \oplus a_2 \oplus \dots \oplus a_n$。
  - 如果我改变主意，在第 $i$ 个位置不选 $a_i$ 改选 $b_i$，对整体异或和的影响是什么？是多异或了一个 $a_i \oplus b_i$。
- **目标重塑：** 题目变成了：给定一个初始值 $Sum$ 和一个可选的“修改代价”数组 $val[i] = a_i \oplus b_i$。我们需要从 $val$ 中选出一个子集，使得这个子集的异或和恰好等于 $Sum$。即求方程 $x_1 \cdot val_1 \oplus x_2 \cdot val_2 \dots \oplus x_n \cdot val_n = Sum$ 的解（$x_i \in \{0, 1\}$）。

#### Step 2.2：映射算法

- “从一个集合中挑出一个子集，使其异或和等于某个特定值” —— 看到这个句式，你的大脑应该条件反射般地弹出：**线性基（XOR Basis）**。

#### Step 2.3：模板特异化改造（自我问答）

在套用线性基模板前，必须进行以下思考：

- **问：** 标准线性基能解决什么问题？

- **答：** 能判断一个数能否由集合中的元素异或得到（返回 true/false），或者求最大/最小异或和。

- **问：** 本题还需要什么？

- **答：** 题目不仅要求判断能否凑出 $Sum$，还要求**输出具体的构造方案**（具体选了哪些 $b_i$）。

- **问：** 如何改造模板来记录“路径”？

- **答：** 我需要知道线性基里的每一个基向量 $b[i]$ 是由原数组里的哪些元素异或来的。因为值域是 $2^{30}$，线性基最多只有 30 个有效向量。我完全可以用一个 `int`（最多 30 位）来作为状态压缩的 mask，记录当前的基向量是由哪些“成功插入线性基的原元素”组合而成的。

### 3. 通用解题方法论：基准线偏移法 (Baseline Shift)

从这道题可以提炼出一个极其强大的通用方法论，专门用于对付**“存在多个二选一（或多选一）状态，求最终满足某限制”**的构造题或最优化题。

**方法论步骤：**

1. **强行固定基准（Fix the Baseline）：** 不要试图同时思考所有的变化。强制假定所有选择都取“选项 A”（或者取最小值、取 0）。
2. **计算初始偏差（Calculate Initial State）：** 在全选 A 的情况下，算出当前的总状态值（本题中是 $Sum = \bigoplus a_i$）。
3. **定义“切换代价/增量”（Define the Delta）：** 思考如果我把某一个位置从“选项 A”切换到“选项 B”，状态会发生什么相对变化（本题中是增量 $a_i \oplus b_i$）。
4. **将原问题转化为“挑选增量”问题（Reduce to Subset Selection）：** 现在的目标变成了：如何从一堆“增量”中挑选一部分，来抵消掉初始状态与目标状态之间的差距。

以后遇到如图论中的边权二选一、网格图染色等问题，只要觉得状态爆炸，立刻尝试这套**“基准线+增量”**的思维模型。

### 4. 顶尖选手视角的编码流 (Coding Flow)

   - 读入 $a$ 和 $b$。
   - 求出基准线 $Sum = \bigoplus a_i$。
   - 计算出所有的增量 $val_i = a_i \oplus b_i$。
   - *心理活动：核心逻辑就这么简单，剩下的都交给那个叫 `LinearBasis` 的黑盒。*
2. **设计黑盒接口 (API Design):**
   - 我需要一个 `insert(val, index)`：不仅插入值，还得把它的原始下标传进去，因为最后要输出构造。
   - 我需要一个 `has(target, &a, &b)`：不仅判断能不能凑出 target，如果能，它得负责把 $a$ 数组里对应的元素直接替换成 $b$ 数组的元素。
3. **实现黑盒内部 (Implementing the Black-box - 这是最容易卡壳的地方):**
   - *难点专注：* 怎么记录组合方案？
   - *理清依赖：* 用 `id` 数组按顺序存入成功构成基的原始下标。此时 `id` 的大小 `sz` 就是当前向量空间的秩（最多 30）。
   - *状态追踪：* 新建一个 `set[31]` 数组。`set[i]` 是一个 bitmask，它的第 `k` 位为 1，代表基向量 `b[i]` 的组成成分中包含了 `id[k]` 所对应的那个增量。
   - *插入时的维护：* 当待插入元素 $x$ 遇到 $b[i]$ 被消去时 ($x \oplus= b[i]$)，它的组成成分也要合并 ($s \oplus= set[i]$)。如果最终找到了空位填入，这个元素的最终成分就是 $s$ 加上它自己。
4. **收尾与重构 (Reconstruction):**
   - 在 `has` 函数中，如果 target 能被消成 0，说明有解。消的过程中累积了一个 mask $s$。遍历这个 mask 的每一位，如果第 $i$ 位是 1，就把原始数组 $a[id[i]]$ 替换为 $b[id[i]]$。
   - 输出修改后的 $a$ 数组。

按照这种“主逻辑 -> 接口定义 -> 核心数据结构填空”的顺序写代码，你的大脑同一时间只需要思考一件事，代码就会如行云流水般写出，极少出现写到一半发现变量传错了的情况。

### 5. 补充视角：高阶代数观点

虽然用“异或组合”来理解很直观，但在 ICPC 训练中，建议你慢慢建立起**“线性空间”**的直觉。

你可以把 $a_i \oplus b_i$ 看作是 $GF(2)$ 上的一个高维向量。你把 $n$ 个向量扔进去，求出的线性基的秩（也就是你代码中的 `sz`）就是这个向量空间的最大维度 $R$。

如果 $n = 10^5$，但 $R \le 30$，这意味着什么？意味着这 $10^5$ 个选择中，存在着极度庞大的**“零空间”**（自由度为 $2^{n-R}$）。也就是说，如果这道题保证有解，那么合法的构造方案实际上是海量的，但线性基算法能够用最确定、最小代价的方式帮你找到其中字典序（或者说基于基向量顺序）最自然的一个特解。这种“化海量可能为确定性基底”的思想，是解决很多高级数学构造题的终极武器。

你觉得这种将问题转化为“基准线+增量”的思维方式，是否能套用在你近期做过的其他非数据结构题目上？如果有相关的题目，我们也可以拿出来对比拆解一下。
