下面我用**竞赛常用写法**把“逆 zeta 变换（超集差分 / 上确界差分）”的流程讲清楚：它对应的是 **Superset Zeta Transform 的逆运算**（也常叫 *mobius on superset*）。

------

### 1. 你要逆的到底是哪一种 zeta（先对齐定义）

## ✅ A. 超集 zeta（Superset Zeta）

定义为：$F[S]=\sum_{T\supseteq S} f[T]$

也就是：**把所有包含 (S) 的超集 (T) 的值加起来**。

那么它的逆变换（超集差分 / 超集 Möbius）就是：$f[S]=\sum_{T\supseteq S} (-1)^{|T|-|S|} F[T]$

而竞赛里最常用的实现方式是：**对每一位做“差分”**。

------

### 2. 逆超集 zeta 的核心流程（超集差分）

假设数组 `a[mask]` 里存的是 (F[mask])，你要把它变回 (f[mask])。

## ✅ 逆变换算法（in-place）

```cpp
// dp[mask] 初始存储的是 F(mask)
// 变换结束后，dp[mask] 将存储 f(mask)

for (int i = 0; i < n; i++) {
    for (int mask = 0; mask < (1 << n); mask++) {
        if ((mask & (1 << i)) == 0) {
            dp[mask] -= dp[mask | (1 << i)];
        }
    }
}
```

### 这段代码在干嘛？

对每一位 `i`：

- 如果 `mask` 的第 `i` 位是 `0`
- 那么它的超集里有一类是 `mask | (1<<i)`（即把这位补成 1）
- **我们把这部分贡献减掉，就完成“反卷积式”恢复**

这就叫：**超集差分**。

------

### 3. 为什么这样做就是逆运算（直觉解释）

超集 zeta 的正向过程会把 `f[mask | (1<<i)]` 的贡献加到 `F[mask]` 里（因为它是超集）。

因此逆过程就要把它减回来：$f[mask] = F[mask] - F[mask \cup {i}]$

然后对所有 bit 逐层展开，就完整恢复。

------

### 4. 完整对照：正向 vs 逆向（超集版）

#### ✅ 正向 Superset Zeta

$F[S]=\sum_{T\supseteq S} f[T]$

```cpp
for (int i = 0; i < n; i++) {
    for (int mask = 0; mask < (1 << n); mask++) {
        if ((mask & (1 << i)) == 0) {
            dp[mask] += dp[mask | (1 << i)];
        }
    }
}
```

#### ✅ 逆向 Superset Möbius（超集差分）

```cpp
for (int i = 0; i < n; i++) {
    for (int mask = 0; mask < (1 << n); mask++) {
        if ((mask & (1 << i)) == 0) {
            dp[mask] -= dp[mask | (1 << i)];
        }
    }
}
```

### 5. 举个小例子（n=2）感受一下（是怎么一次性减去多个超集且不重不漏的）

`mask` 从 `00, 01, 10, 11`。

假设原始 `f` 为：

$f[00]=a,\ f[01]=b,\ f[10]=c,\ f[11]=d$

做超集 zeta 得：

- $F[11]=d$
- $F[10]=c+d$
- $F[01]=b+d$
- $F[00]=a+b+c+d$

现在逆变换：

- 先处理 bit0（最低位）：
  - `00` 减 `01`：
    $F[00] \leftarrow (a+b+c+d)-(b+d)=a+c$
  - `10` 减 `11`：
    $F[10] \leftarrow (c+d)-d=c$
- 再处理 bit1：
  - `00` 减 `10`：
    $F[00] \leftarrow (a+c)-c=a$
  - `01` 减 `11`：
    $F[01] \leftarrow (b+d)-d=b$

结果恢复为 (a,b,c,d)，完全正确

------

### 6. 一句话总结流程

逆超集 zeta：对每一位 `i`，枚举所有该位为 0 的 `mask`，执行$a[mask] \mathrel{-}= a[mask \cup {i}]$
最终把 F() 还原成 f()。

### 1. 题目归类：给它贴个标签

这道题的本质可以归纳为：**小数据范围 ($N \le 20$) 下的集合计数问题**。

具体的标签体系如下：

- **一级分类**：组合数学 (Combinatorics) / 数论 (Number Theory)
- **核心考点**：
  - **容斥原理 (Inclusion-Exclusion Principle, IEP)**：特别是“恰好 $k$ 个”与“至少 $k$ 个”之间的转换。
  - **LCM 的性质**：集合内所有数的倍数 $\iff$ 集合 LCM 的倍数。
  - **高维差分 / 逆 Zeta 变换 (Inverse Zeta Transform)**：处理集合包含关系的 $O(N \cdot 2^N)$ 算法。
- **关键特征**：
  - $N \le 20$ $\rightarrow$ 暗示 $O(2^N)$ 或 $O(N \cdot 2^N)$ 复杂度。
  - $Y, A_i \le 10^{18}$ $\rightarrow$ 暗示不能遍历 $Y$，必须用 $\lfloor Y/X \rfloor$ 公式，且中间过程需要 `__int128`。
  - 关键词：“恰好 $M$ 个” (Exactly M)。

### 2. 思维复盘：从零推导的逻辑链

假设我们还在比赛中，拿到这道题，应该如何一步步思考？

#### 阶段一：捕获关键信息与约束

1. **看数据范围**：$N \le 20$。这是最强烈的信号，说明我们可以枚举所有蝉的子集（状态压缩）。
2. **看目标**：求 $[1, Y]$ 中“恰好”有 $M$ 种蝉爆发的年份数。
3. **看单一条件**：如果只有一种蝉 $i$，它在 $A_i, 2A_i, \dots$ 年爆发。
4. **看组合条件**：如果一个集合 $S$ 中的蝉**同时**爆发，那么该年份必须是 $\text{LCM}(\{A_i \mid i \in S\})$ 的倍数。

#### 阶段二：建立初步模型（你的卡点）

我们很容易算出一个集合 $S$ **全都**爆发的年份数量：

$$F(S) = \lfloor \frac{Y}{\text{LCM}(S)} \rfloor$$

**卡点分析**：这里的 $F(S)$ 并不代表“恰好只有 $S$ 爆发”，它代表的是“**至少** $S$ 爆发”（即 $S$ 的所有超集的情况都包含在内）。

#### 阶段三：转化目标（突破口）

我们无法直接算出“恰好”。在组合数学中，**“恰好 (Exactly)” 很难算，但“至少 (At least / Superset)” 很好算**。

这立刻指向了**容斥原理**。

我们有两种路径处理这个转化：

- **路径 A：代数推导（你的第一种解法）**
  - 我们想要的是“恰好有 $M$ 个”。
  - 如果我们统计所有大小 $\ge M$ 的集合 $S$ 的 $F(S)$。
  - 对于一个真实恰好有 $k$ 种蝉爆发的年份（$k \ge M$），它会被多少个 $S$ 统计到？
  - 它会被这 $k$ 种蝉里任意选 $M$ 个组成的集合统计到，也就是被统计了 $\binom{k}{M}$ 次，而不是 1 次。
  - 所以需要乘上容斥系数 $c_k$，使得 $\sum c_i \binom{k}{i} = [k=M]$。
  - 这导出了系数 $(-1)^{|S|-M} \binom{|S|}{M}$。
- **路径 B：结构化反演（你的第二种解法）**
  - 定义 $g(S)$ 为“**恰好**只有集合 $S$ 爆发的年份数”。
  - 定义 $f(S)$ 为“**至少**集合 $S$ 爆发的年份数”（即我们算出的 $\lfloor Y/\text{LCM} \rfloor$）。
  - 显然 $f(S) = \sum_{S \subseteq T} g(T)$。
  - 这是一个标准的“超集和”形式。我们要从 $f$ 求 $g$，就是**高维差分（FZT 的逆变换）**。
  - 对每一维做减法：`dp[mask] -= dp[mask | (1<<i)]`。

#### 阶段四：细节检查

- $Y$ 很大，LCM 计算过程会爆 `long long` $\rightarrow$ 必须用 `__int128`。
- LCM 超过 $Y$ 后，贡献为 0 $\rightarrow$ 剪枝。

### 3. 你的卡点分析与修正

**你的初次想法：**

> "根据题目我想到了在给定一个集合S的情况下，符合条件的数的个数为上界k除以集合S所有数的lcm，但是这样的计算方式并不能确保个数恰好为集合S的大小，于是不会了"

**分析：**

你的直觉非常准，第一步是对的。你卡在“**性质不纯**”上——算出来的东西包含杂质（包含了超集的情况）。

**如何修正思考路径（金牌思维）：**

当你发现直接算出的结果包含了“杂质”（多余的情况）时，不要放弃这个计算方式，而是问自己两个问题：

1. **这种“含杂质”的结果是否容易计算？** $\rightarrow$ 是的，$\lfloor Y/\text{LCM} \rfloor$ 极其好算。
2. **“杂质”是否有规律？** $\rightarrow$ 是的，算 $S$ 时，包含了所有 $S \subseteq T$ 的 $T$。

**结论：**

如果“含杂质”的好算，且杂质结构清晰（倍数关系、集合包含关系），那么**必然**可以通过**容斥原理**或**莫比乌斯反演**将“纯净”的结果还原出来。

下次遇到“我想算 A，但我只能算出 A+B+C...”，立刻想**容斥**。

### 4. 通用解题方法论（ICPC 模式）

遇到陌生题目，请按以下四步走：

#### Step 1: 扫描与降维 (Scan & Reduce)

- **看规模**：$N$ 是多少？
  - $N \le 20 \rightarrow$ 状压 / 容斥。
  - $N \le 40 \rightarrow$ 折半搜索 (Meet-in-the-middle)。
  - $N \le 100 \rightarrow$ $O(N^3)$ 区间 DP / 匹配 / 网络流。
  - $N \ge 10^5 \rightarrow$ 线段树 / 贪心 / 数学公式。
- **看关键字**：
  - "Exactly" (恰好) $\rightarrow$ 容斥 / 反演。
  - "Divisible / Multiple" (整除/倍数) $\rightarrow$ 数论 (GCD/LCM/约数)。

#### Step 2: 寻找“好算的状态” (Easy State)

- 不要强求直接算出答案。
- 问自己：什么最好算？
  - “恰好 $X$” 难算 $\rightarrow$ “至少 $X$” 或 “$X$ 的倍数” 好算吗？
  - “最大值最小” 难算 $\rightarrow$ 二分答案后“判断可行性”好算吗？
- 在本题中，发现“指定集合公倍数”是最好算的。

#### Step 3: 建立桥梁 (Bridge)

- 用数学工具连接“好算的状态”和“答案”。
  - 包含关系 $\rightarrow$ 容斥原理 / SOS DP。
  - 整除关系 $\rightarrow$ 莫比乌斯反演。
  - 递推关系 $\rightarrow$ 矩阵快速幂 / DP。

#### Step 4: 复杂度与边界 (Check)

- 计算量是否炸？(比如 $2^{20} \approx 10^6$，很安全)。
- 数据是否炸？(LCM 爆炸 $\rightarrow$ `__int128` 或 `double` 预判)。

### 5. 顶尖选手的编码流描述

脑子里有了 **FZT/SOS DP** 的思路后，如何行云流水地写出代码？

1. **宏观布局 (Template & Typedefs)**
   - *心里默念*：这题要算 LCM，绝对爆 `long long`。立刻写 `using i128 = __int128_t;`。
   - *输入*：读 $N, M, K$。读数组 $A$。
2. **核心逻辑 - 第一阶段：预处理 (Init)**
   - *心里默念*：我要先算出所有 $f(mask)$。开一个 $2^N$ 的 `dp` 数组。
   - *循环*：`for mask` 从 1 到 `(1<<N)-1`。
   - *计算 LCM*：这里是核心坑点。
     - 初始化 `lcm = 1`。
     - 遍历 $N$ 位，如果是 1，则更新。
     - **肌肉记忆**：更新 LCM 前必须判溢出！`lcm / gcd * a[i] > K` ? `break` : `update`。
     - 如果有溢出标志，`dp[mask] = 0`；否则 `dp[mask] = K / lcm`。
3. **核心逻辑 - 第二阶段：变换 (Transform)**
   - *心里默念*：现在 `dp` 存的是“至少”，我要变成“恰好”。这是超集反演（从大往小减）。
   - *SOS 模板*：
     - 外层 `i` 从 0 到 $N-1$ (枚举位)。
     - 内层 `mask` 从 0 到 `(1<<N)-1`。
     - 判断：`if (!(mask >> i & 1))` —— 如果这一位是 0，说明 `mask` 是 `mask | (1<<i)` 的子集。
     - 操作：`dp[mask] -= dp[mask | (1<<i)]` (把包含 $i$ 的超集情况减掉)。
4. **收尾 (Output)**
   - 遍历 `mask`，如果 `popcount(mask) == m`，加到 `ans`。
   - 输出。

### 6. 为什么逆zeta变换比容斥好？

虽然两者在数学本质上是一样的（都是容斥原理），但逆 Zeta 变换的做法有以下特点：

1. **结构化与通用性**：它求出了**所有**集合 $S$ 的“恰好”发生次数。如果你不仅想问“恰好 $M$ 种”，还想问“恰好 $M+1$ 种”或者“恰好是子集 $S$”，这个 $dp$ 数组能直接回答所有问题，而不需要重新跑 DFS。
2. **避免组合数系数**：DFS 方法需要手动推导 $(-1)^{k-M} \binom{k}{M}$ 这样的系数。而逆 Zeta 变换通过 $N$ 轮简单的减法操作，自动完成了系数的分配。
3. **代码实现模板化**：高维前缀和/差分是非常标准的几行代码，不易出错。
