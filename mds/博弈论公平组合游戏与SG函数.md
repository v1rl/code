### 1. 适用范围：公平组合游戏 (ICG)

SG 函数理论只适用于满足以下条件的 **公平组合游戏 (Impartial Combinatorial Games)**：

1. **两名玩家**交替行动。
2. **完全信息**：游戏状态对双方完全透明。
3. **无运气成分**：没有掷骰子等随机因素。
4. **有限步数**：游戏一定会在有限步内结束（无平局）。
5. **公平性（最关键）**：**一个局面的合法走法只取决于局面本身，与当前是谁在行动无关。**（这一点将它与象棋、围棋区分开，象棋中红方不能动黑子，所以象棋不是公平组合游戏）。
6. **Normal Play 规则**：**无法行动者判负**（即最后走一步的人赢）。

------

### 2. 核心状态：必胜态 (N) 与 必败态 (P)

我们可以把游戏看作一个 **有向无环图 (DAG)**，每个节点是一个局面，边代表合法的操作。

- **P-position (Previous player winning)**：**必败态**。即轮到我走这个局面，我输了（或者说，刚才走完的人赢了）。
- **N-position (Next player winning)**：**必胜态**。即轮到我走这个局面，我有办法最后赢。

**三条基本公理（状态转移方程）：**

1. **终局是 P**：如果你无法进行任何移动（出度为 0），你就输了。
2. **能走到 P 就是 N**：如果当前局面 $u$ 存在至少一条边指向必败态 $v$（$u \to v$），那么我只要把局面变成 $v$ 扔给对手，对手就输了。所以我处于必胜态。
3. **只能走到 N 就是 P**：如果当前局面 $u$ 的所有后继节点 $v_1, v_2...$ 全都是必胜态，无论我怎么走，对手接到的都是必胜态，对手一定会赢。所以我处于必败态。

------

### 3. 数学工具：MEX 运算

为了将上述逻辑量化，我们需要引入 MEX。

定义：$\text{MEX}(S)$ （Minimum Excluded value）是指不属于集合 $S$ 的最小非负整数。

**例子**：

- $\text{MEX}(\{1, 2, 3\}) = 0$
- $\text{MEX}(\{0, 1, 3\}) = 2$
- $\text{MEX}(\{0, 1, 2, 3\}) = 4$
- $\text{MEX}(\{\}) = 0$ （空集的 MEX 为 0）

------

> 当前存在多个独立的子游戏（xor）
>
> 当前状态可以转移至哪些状态（mex）
>
> 当前转移得到的状态使游戏发生分裂（变成了 Sum of Games），游戏分裂成了 $k$ 个互不干扰、独立进行的子游戏（xor）

### 4. 核心量化：SG 函数

**Sprague-Grundy (SG) 函数** 是对每个游戏状态的一个评估值。对于状态 $u$，其 SG 值定义为：

$$SG(u) = \text{mex}( \{ SG(v) \mid u \to v \} )$$

即：**当前状态的 SG 值 = 所有能到达的后继状态的 SG 值集合的 MEX。**

**SG 函数的物理意义：**

- **$SG(u) = 0$**：对应 **必败态 (P-pos)**。
  - 解释：因为 $MEX=0$，说明后继集合里没有 $0$。也就是说，无论怎么走，后继状态的 SG 值都非 0（都是必胜态）。这也符合公理 3。
- **$SG(u) \neq 0$**：对应 **必胜态 (N-pos)**。
  - 解释：因为 $MEX=k (k>0)$，说明后继集合里一定包含 $0$。也就是说，存在一种走法能走到 $SG=0$（必败态）的局面。这也符合公理 2。

**实际上，SG 值不仅仅告诉你是赢是输，它还量化了局面的“能量”。如果 $SG(u) = k$，这意味着局面 $u$ 等价于一个大小为 $k$ 的 Nim 堆。**

------

这是一个非常深刻的问题。对于初学者来说，知道“0是输，非0是赢”确实足够应付判断胜负的题目（Output "Yes"/"No"）。

但对于 ACM 金牌选手或致力于解决复杂博弈题的选手来说，**“$SG(u)=k$ 等价于大小为 $k$ 的 Nim 堆”** 这一认知是解决 **组合游戏** 和 **输出具体方案** 的核心。

它的核心作用体现在以下三点：

### 4.1 解决“多游戏组合”问题（最重要）

这是 SG 函数存在的根本意义。

假设你面对这样一个局面：

- **左边**：是一个刚才那样的 $N \times M$ 的剪纸游戏。
- **右边**：是一个有 $K$ 个石子的 Nim 堆。
- **规则**：每次操作，你可以选择在“剪纸游戏”里切一刀，**或者**在“Nim 堆”里拿走一些石子。

如果你只知道剪纸局面的状态是“必胜”还是“必败”，你是无法解决这个组合问题的。

- 剪纸（必胜） + Nim堆（必胜） = **结果未知**。
  - 例子 A：剪纸 $SG=3$（必胜），Nim堆 $3$ 个（必胜）。总 $SG = 3 \oplus 3 = 0$ $\to$ **必败**。
  - 例子 B：剪纸 $SG=3$（必胜），Nim堆 $2$ 个（必胜）。总 $SG = 3 \oplus 2 = 1$ $\to$ **必胜**。

结论：

只有知道了具体的 SG 值，你才能利用 Sprague-Grundy 定理 将这一堆完全不同的游戏（剪纸、取石子、图上游走）统一抽象成一堆数字：

$$Total\_SG = SG(\text{剪纸}) \oplus SG(\text{Nim}) \oplus SG(\text{其他独立游戏})$$

SG 值实现了不同游戏之间的“货币兑换”，让它们可以被加减计算。

------

### 4.2 构造具体的必胜策略（Output the Move）

很多题目不仅问你“能不能赢”，还会问“**第一步怎么走才能赢**”。

假设当前局面由三个独立游戏组成，SG 值分别是 $a, b, c$，且 $S = a \oplus b \oplus c \neq 0$（必胜）。

你想在第一个游戏（SG值为 $a$）中走一步，将其变为 $a'$，使得留给对手的局面异或和为 0。

即要求：

$$a' \oplus b \oplus c = 0$$

利用异或性质：

$$a' = b \oplus c$$

或者更通用的公式（两边同时异或 $S$）：

$$a' = a \oplus S$$

这时候 $SG(u)=k$ 的“量化”作用就出来了：

你需要遍历第一个游戏当前状态的所有后继状态 $v$，找到那个 $SG(v)$ 恰好等于 $a \oplus S$ 的状态，那就是你的必胜走法。

如果没有具体的数值，你只知道“必胜/必败”，你根本无法算出你需要把当前局面变成“多强”的局面才能平衡掉其他的游戏。

------

### 4.3 树上/图上删边游戏的“等效替代”

在处理 **Green Hackenbush**（树上删边博弈）或这类结构化博弈题时，这一性质能极大简化思维。

例子：

给定一棵树，根节点上长出 5 个分支（子树 $T_1, \dots, T_5$）。

规则：每次选一条边删掉，该边及其子树被移除（或者其他规则）。

由于各分支互不干扰，我们可以把根节点的 5 个分支看作 5 个独立的 Nim 堆。

- 我们计算出子树 $T_1$ 的 $SG$ 值是 5。
- **思维模型**：此时你可以直接把 $T_1$ 这棵复杂的子树，在脑海中**删掉**，替换成**一根连着 5 个单位长度的棍子**，或者**一堆 5 个石子**。
- 这使得递归处理变得极其自然：父节点的 SG 值就是所有子节点 SG 值的某种组合（通常是异或，具体视规则而定）。

------

> 只要得到了该局面的SG值，就可以将该局面想象成有SG函数个石子的石子堆，于是将任何公平组合游戏全部转化为Nim游戏

### 4.4总结

- **只定性（Win/Lose）**：只能解决单一游戏，且无法处理“必胜 + 必胜 = ？”的情况。
- **定量（SG Value）**：
  1. **统一单位**：将剪纸、棋盘、硬币等任何公平组合游戏全部转化为“Nim 石子堆”。
  2. **可加性**：利用异或运算处理多局并列的情况。
  3. **可逆性**：利用异或运算反推必胜步法。

**一句话：SG 值把“博弈推理”变成了“算术运算”。**

>

> 当前转移得到的状态使游戏发生分裂（变成了 Sum of Games），游戏分裂成了 $k$ 个互不干扰、独立进行的子游戏（xor）

### 5. 组合游戏的合并：SG 定理

这是解决复杂问题的终极武器。

如果一个大游戏 $G$ 可以被分解为 $k$ 个互不干扰、独立进行的子游戏 $G_1, G_2, \dots, G_k$（例如刚才的剪纸游戏，剪一刀后变成两张独立的纸），那么：

$$SG(G) = SG(G_1) \oplus SG(G_2) \oplus \dots \oplus SG(G_k)$$

**总局面的 SG 值 = 各个子游戏 SG 值的异或和 (XOR)。**

**判定准则**：

- 如果 $SG(G) \neq 0$：先手必胜。
- 如果 $SG(G) = 0$：先手必败。

------

### 6. 实战总结：做题三部曲

当你遇到一道博弈论题目（看似公平组合游戏）：

1. **判断能否分解**：
   - 如果不涉及游戏分裂（如单纯的一堆石子拿取），直接把整个过程看作一个 DAG，用记忆化搜索求 $SG(x) = \text{mex}(\dots)$。
   - 如果一步操作后游戏分裂成多个子游戏（如剪纸、一排硬币拿走中间变两段），则应用 SG 定理。
2. **编写 `sg(state)` 函数**：
   - **递归基**：也就是终局（如 $SG(0)=0$ 或 $SG(1,1)=0$）。
   - **枚举后继**：遍历所有合法的操作。
   - **计算后继 SG**：
     - 若分裂，则为 `sg(sub1) ^ sg(sub2)`。
     - 若不分裂，则为 `sg(next_state)`。
   - **求 MEX**：用一个哈希表或布尔数组标记所有出现的后继 SG 值，找到最小没出现的。
   - **记忆化**：别忘了保存结果。
3. **判定**：
   - 计算初始局面的 SG 值，非 0 输出 "Yes/Win"，为 0 输出 "No/Lose"。

### 举个最简单的例子：Nim 游戏

有 $n$ 堆石子，第 $i$ 堆有 $a_i$ 个。

- 对于单堆石子 $x$ 个，每次能拿 $1 \sim x$ 个。
  - $x \to 0, 1, \dots, x-1$。
  - $SG(0) = 0$
  - $SG(1) = \text{mex}(\{SG(0)\}) = \text{mex}(\{0\}) = 1$
  - $SG(2) = \text{mex}(\{SG(0), SG(1)\}) = \text{mex}(\{0, 1\}) = 2$
  - ... 归纳可得 **$SG(x) = x$**。
- 根据 SG 定理，总局面 $SG = a_1 \oplus a_2 \oplus \dots \oplus a_n$。
- 这就是为什么 Nim 游戏的结论是异或和不为 0 则必胜。

### 7.P/N分析

如果一个游戏**本身不再分裂**（即每次操作后状态仍然是单一的，不会变成“子游戏A + 子游戏B”），且你面对的**只有这一个独立的游戏**（不是多个游戏的和），那么求 SG 值（0, 1, 2, 3...）完全可以退化为求 **必胜/必败态（1/0 或 True/False）**。

这种方法在博弈论中有一个专门的术语，叫做 **P/N 分析（P-position / N-position Analysis）**。

------

### 7.1 简化的逻辑原理

在不需要处理“游戏加法”（XOR）的时候，我们确实只关心两个状态：赢、输。

SG 函数的定义是 `mex`，而 P/N 分析的定义是逻辑运算：

- **SG (完整版)**：
  - $SG(u) = \text{mex}(\{SG(v_1), SG(v_2), \dots\})$
  - 关注具体数值。
- **P/N (简化版)**：
  - **定义**：用 `Win` (必胜) 和 `Lose` (必败) 标记状态。
  - **规则 1**：如果一个状态能走到任何一个 `Lose` 状态，那它就是 `Win`。（只要有一种走法能坑死对手，我就赢了）
    - 对应 SG：只要后继集合里有 0，MEX 一定非 0。
  - **规则 2**：如果一个状态只能走到 `Win` 状态，那它就是 `Lose`。（无论怎么走对手都必胜，我就死定了）
    - 对应 SG：后继集合里全是非 0 值，MEX 一定是 0。

**代码上的简化对比：**

**SG 写法（通用，维护具体值）：**

C++

```
int sg(int u) {
    if (memo[u] != -1) return memo[u];
    unordered_set<int> S;
    for (int v : adj[u]) S.insert(sg(v)); // 递归求子状态 SG
    int res = 0;
    while (S.count(res)) res++; // 求 MEX
    return memo[u] = res;
}
```

**P/N 写法（简化，只维护胜负）：**

C++

```
bool can_win(int u) {
    if (memo[u] != -1) return memo[u];
    for (int v : adj[u]) {
        if (!can_win(v)) { // 如果能让对手必败
            return memo[u] = true; // 那我就必胜
        }
    }
    // 如果所有路都导致对手必胜，那我就必败
    return memo[u] = false;
}
```

*注：这种写法就是经典的 Minimax 或者 与或树（AND-OR Tree）搜索。*

------

### 7.2 这个简化的“大坑”（必须警惕！）

你说：“在确定当前游戏不会被划分成多个相互独立的子游戏的情况下”。

这句话有一个极易混淆的陷阱。

**“游戏不分裂” $\neq$ “只有一个游戏”**

请务必区分以下两种情况：

#### 情况 A：真正的单局游戏（可以使用简化）

- **场景**：棋盘上只有一个马，两人轮流走日字，谁无路可走谁输。
- **特点**：操作不分裂，且全场只有一个棋子。
- **结论**：**可以使用简化。** 不需要算 $SG=3$ 还是 $SG=4$，只需要知道 `can_win` 即可。

#### 情况 B：尼姆博弈（不能使用简化）

- **场景**：有 3 堆石子，数量分别为 3, 4, 5。每次只能选一堆拿。
- **特点**：
  - 你的操作是“从一堆里拿掉几个”，这个操作**并没有**把这一堆分裂成两堆（依然保持一堆）。符合你说的“不会被划分”。
  - 但是！整个局面是由 **3 个独立的堆** 组成的。
- **结论**：**绝对不能简化！**
  - 如果你只用 P/N 分析：堆 A (3) 是必胜，堆 B (4) 是必胜，堆 C (5) 是必胜。
  - 你想判断总局势：必胜 + 必胜 + 必胜 = ？？？（布尔逻辑无法处理）
  - 你必须用 SG 值：$3 \oplus 4 \oplus 5 = 2 \neq 0$，所以先手必胜。

核心判定标准：

只要题目中出现了**“多组”、“多个棋子”、“多个图”，哪怕单个操作不发生分裂，你也必须算具体的 SG 值来进行异或运算。只有当题目是完全孤立的一个局**时，才能退化为 P/N 分析。

------

### 7.3 什么时候推荐用简化版？

在满足“孤立单局”的前提下，简化版有巨大的优势：

1. **效率更高**：
   - SG 函数必须算出所有子状态的值才能求 MEX。
   - P/N 分析带有**剪枝**效果：`if (!can_win(v)) return true;`。一旦找到一条必胜路，剩下的子状态就**不用递归了**！这在状态分支很多时（如棋盘类），速度可能有数量级的提升。
2. **处理“有环”图**：
   - 标准的 SG 函数在有环图（Draw）上定义很麻烦（可能死循环或需要更复杂的处理）。
   - P/N 分析可以通过标记 `vis` 正在访问的节点来处理平局（Draw）的情况：
     - 能走到必败 $\to$ 必胜。
     - 只能走到必胜 $\to$ 必败。
     - 只能走到必胜或平局 $\to$ 平局。

### 总结

| **特性**     | **SG 函数 (MEX)**                         | **P/N 分析 (Boolean)**                     |
| ------------ | ----------------------------------------- | ------------------------------------------ |
| **返回值**   | 0, 1, 2, 3... (能量值)                    | True/False (胜/负)                         |
| **适用范围** | 通用，特别是**多游戏组合** (Sum of Games) | 仅限**单一孤立游戏**                       |
| **计算逻辑** | 必须遍历所有后继，求 MEX                  | 找到一个必败后继即可返回 True (**可剪枝**) |
| **实战建议** | 题目涉及多堆石子、多个棋子时**必须用**    | 题目只有一个棋子在图上跑时**推荐用**       |

所以，你的直觉是对的。在 AcWing 219 剪纸游戏这道题中，因为剪纸会**分裂**成两个子游戏（变成了 Sum of Games），所以**必须用 SG 值**，不能简化。但如果是“象棋马走日”那种不分裂的单子游戏，你的简化思路就是最优解。
