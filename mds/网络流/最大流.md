最大流（特别是你这里使用的 **Dinic 算法**）确实是算法竞赛和机试中最容易“背了忘，忘了背”的模板之一。这是因为它涉及多个相互关联的组件（图的结构、BFS分层、DFS增广、反向边、当前弧优化），死记硬背非常痛苦。

为了帮你真正记住它，我们需要把死板的代码转化为**逻辑链条**。Dinic 算法的核心骨架其实只有八个字：**BFS 分层，DFS 找流。**

以下我为你梳理的思路和记忆锚点（Memory Points）：

------

### 1. 核心数据结构：成对建图 (The Graph)

**记忆点：边存实体，点存索引，`0/1` 成对。**

- **为什么要 `vector<Edge> e` 和 `vector<int> g` 分开？**

  最大流的核心操作是“正向减容量，反向加容量”。如果只用邻接表存目标节点和容量，找反向边会极其困难。所以把所有边放在一个大池子 `e` 里，而 `g[x]` 只存第几条边。

- **技巧（死记住）：** 正向边和反向边必须**成对连续插入**。

  - 正向边索引如果是偶数 `i`，反向边就是 `i + 1`。
  - 通过位运算异或 `i ^ 1`（例如 `0^1=1`, `1^1=0`, `2^1=3`, `3^1=2`），可以极快地通过一条边找到它的反向边！
  - **代码体现：** `addEdge` 中连续 `emplace_back`，正向容量为 `cap`，反向（后悔药）初始容量必须为 `0`。

### 2. BFS：建立层次图 (The Layering)

**记忆点：只走有残量的边，不走回头路。**

- **目标：** 算出起点到每个点的最短距离数组 `d`。为了防止 DFS 瞎转圈（比如在环里死循环），我们强制要求水流只能往“下一层”流。
- **入队条件（关键判断）：**
  1. `cap > 0` （这条边得还能通水，没满）。
  2. `d[y] == -1` （这个点没被访问过）。
- **结束标志：** 如果遍历到了终点 `edd`，直接 `return true`（找到了一条路）。如果队列空了还没找到，说明图已经断了，`return false`。

### 3. DFS：榨干水流 (The Core Logic)

这是最容易写错的地方。把你当成一个水管的节点 `x`，你的任务是把上游冲过来的水（`limit`）尽可能多地分发给下游。

**记忆点：当前弧优化是灵魂，`flow` 记录剩余可用量。**

- **参数含义：**
  - `limit`：上游最多能给你多少水。
  - `flow`：你手里现在还剩下多少水**没有分发出去**（初始等于 `limit`）。
- **当前弧优化（必背细节）：** `for(int &i = cur[x]; ...)`。
  - 注意这里的 `&` 符号！这是 Dinic 能达到 $O(V^2E)$ 复杂度的关键。
  - 如果一条边已经被榨干了，下次再来到点 `x` 时，就没必要再检查它了。引用 `&i` 可以使得 `cur[x]` 跟着循环变量 `i` 一起递增，下次直接从没榨干的边开始找。
- **分发条件：** `d[y] == d[x] + 1`（必须是下一层）且 `cap > 0`（管道没满）。
- **递归与状态更新（核心 4 步）：**
  1. **传给下游：** `T t = dfs(y, edd, min(flow, cap));`（给下游的水，不能超过我自己剩下的，也不能超过管道容量）。
  2. **更新管道：** `e[j].cap -= t; e[j ^ 1].cap += t;`（正向减，反向加）。
  3. **更新自身：** `flow -= t;`（我手里剩下的水变少了）。
  4. **提前榨干：** `if(flow == 0) return limit;`（如果我手里的水全部分发完了，直接下班，不用看后面的管子了）。
- **返回值：** `limit - flow`。上游给我 `limit`，我剩了 `flow` 没发出去（死胡同退回来了），说明我**真正成功流出去**的水是 `limit - flow`。

### 4. 主控函数：Flow

**记忆点：只要还能分层，就疯狂 DFS。**

- **外层循环：** `while(bfs(stt, edd))`。只要还能建出层次图（说明还能走到终点）。
- **每次循环必做：**
  1. 重置当前弧：`cur.assign(n + 1, 0);`（因为图的层次变了，边又要重新遍历）。
  2. 无限量灌水：`flow += dfs(stt, edd, INF);`。

------

**考前/敲键盘前的大脑检查清单 (Mental Checklist)**

每次写最大流之前，脑子里默念这五个问题：

1. `vector<Edge>` 和 `vector<int>` 建图，`cap` 和 `0` 成对了没？
2. BFS 里有没有检查 `cap > 0` ？
3. DFS 的 for 循环里写没写 **引用符 `&i`**（当前弧优化）？
4. DFS 递归往下传的流量是不是 `min(flow, cap)`？
5. DFS 里有没有写反向边加残量 `e[j^1].cap += t`？

理解了每一层的作用，你就不需要像背课文一样去背它了，顺着水流的逻辑自然就能敲出来。
